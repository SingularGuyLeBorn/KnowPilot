# Session 2026-06-27 — KnowPilot 项目启动

## 任务: 从零新建 KnowPilot 项目 (L1-T01 工程初始化)

### 问题发现
用户要求将 MetaBlog 项目从 Vue/VitePress 完全迁移到 Next.js + React，新项目命名 KnowPilot，路径 `D:\ALL IN AI\KnowPilot`。

### 根因分析
MetaBlog 现有架构为 Vue 3 + VitePress + Express BFF，用户对 VitePress 体验不满意，希望重新开始。

### 方案选择
经过多轮讨论确定技术选型:
- **tRPC** (vs OpenAPI): 选 tRPC，端到端类型安全，单人全栈开发效率更高
- **Milkdown** (vs Vditor vs Tiptap): 选 Milkdown，Markdown 原生 + React 原生 + 定制性好
- **PostgreSQL + Prisma**: 从一开始就用，不走 sql.js
- **视觉风格**: 直接复刻 MetaBlog 的莫兰迪色系 Star River Design System
- **单用户模式**: 不需要登录态
- **任务分配**: 按层级分，层内按模块并行

### 实现过程
1. 创建根 monorepo 骨架 (package.json, pnpm-workspace.yaml, tsconfig.base.json)
2. 创建 `packages/shared` — Zod schema 定义 (Post CRUD 输入验证)
3. 创建 `apps/server` — Express + tRPC + Prisma
   - Prisma schema 定义 Post 模型
   - tRPC 初始化 (trpc.ts, context.ts)
   - Post router 实现完整 CRUD (list, getBySlug, getById, create, update, delete, search)
   - seed 脚本 (3 篇示例文章)
4. 创建 `apps/web` — Next.js 15 (create-next-app 初始化中)
