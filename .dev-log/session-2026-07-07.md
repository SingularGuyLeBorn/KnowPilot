- `apps/server/src/infra/mockMcpRegistry.ts`：filesystem / fetch / web_search 等 Mock MCP 工具。

- 接入点：`llmClient.ts` / `mcpClient.ts` / `nativeTools.ts` 三处按 `MOCK_LLM` / `MOCK_MCP` / `MOCK_NATIVE_TOOLS` 环境变量分支。

### 配置与脚本
- `apps/web/playwright.config.mock.ts`：3003/3011 独立 webServer，`cross-env` 解决 Windows 环境变量传递。
- `apps/web/package.json` / `apps/server/package.json`：新增 `dev:mock` / `build:mock` / `start:mock` / `test:e2e:mock` / `test:e2e:mock:headed`。

- `.gitignore` 加 `playwright-report-mock/` / `test-results-mock/` / `content/posts/e2e-trash-*.md`。

### E2E 用例
- `chat-mock.spec.ts`：问候 + 触发 web_search 工具并显示 pill/hint。
- `chat-thinking-mock.spec.ts`：思考时间线不重复。
- `chat-tool-error-mock.spec.ts`：工具失败红色 pill。
- `post-trash.spec.ts`：文章回收站删除/恢复（try/finally 强制清理 e2e-trash-* 残留）。
- `ui-components.spec.ts`：通用组件冒烟。
- `helpers/mockChatFixture.ts`：Mock Chat 断言封装，`waitForStreamingComplete` 改等 `assistant-message-bubble`（适配极快流式）。

### agentStream 架构精修
- Probe 短路：probe 返回无 toolCalls + 有 content + 无 reasoningContent 时直接发 token 返回，跳过冗余 `chatCompletionStream`。
- `maxToolRounds` 耗尽后若末轮执行了工具，补一次 `chatCompletionStream`（不带 tools）合成最终答案。
- `chatHistory.ts` 重构 `buildLlmMessagesFromHistory`：把扁平存储的 `assistant(content=final + toolCalls=[all])` 按 tool_call 拆成 N 条 `assistant(content=null, tool_calls=[single])` + `tool`，最终答案独立发出，符合 OpenAI 格式。**只适配新的扁平格式，不兼容旧格式**（避免代码臃肿）。

### Mock 模式护栏
- `index.ts` 启动时检查三开关，若部分启用则 warn「假 LLM + 真工具触网」混合态。

## 二、整改计划（REMEDIATION_PLAN.md，5 批全部完成）

完整文档见 `docs/development/REMEDIATION_PLAN.md`。摘要：

### 第 1 批 P0 — 严重安全/数据（commit fcbc08a）
- **P0-1 凭据 API 明文返回**：`CredentialService.formatEntity` 改返回遮蔽 `valuePreview`，永不返回明文；`credential.getById/list` 的 `aiReadable` 改 false，堵 Agent 反射拖密钥。
- **P0-2 无 master key 明文落库**：`NODE_ENV=production` 且无 `CREDENTIAL_MASTER_KEY` 时拒启动；新增 `assertCredentialEncryptionAvailable` 启动护栏。
- **P0-3 Git repo 路径绕过沙箱**：抽 `infra/safePath.ts`（`resolveSafePath` 拒绝绝对路径与 `..`，`assertPathWithinProjectRoot` 校验）；`nativeTools.resolveRepoPath` repoId 分支 + `GitService` 注册/解析全部走校验。
- **P0-4 ai.invoke 反射面过大**：`git.commit` / `git.pull` 入审批（与 `git.push` 同档）；`log.clearAll` / `session.delete` / `workspace.create` 的 `aiReadable` 改 false。
- **P0-5 sync cleanup 误删**：`sync-posts` cleanup 硬删改软删（设 `deletedAt`，进回收站可恢复）；空 activeSlugs 防御跳过；磁盘存在性检查（解析失败的文件不删）；upsert 不再覆盖 `deletedAt`；其余 6 个 syncer 统一加空目录防御。

### 第 2 批 P1-1~P1-6 + P1-10（commit 74714ce）
- **P1-1 静态资源无鉴权**：`/uploads`、`/api/posts/assets` 在 `AUTH_MODE=password` 时走 `verifyAuthHeader`；`AUTH_TOKEN` 回退 `AUTH_PASSWORD` 时 warn。
- **P1-2 TriggerEngine 阻塞 + 重入**：per-trigger 互斥 Map，同一触发器执行中跳过并发触发；多触发器不 await 并行，不阻塞 emit。
- **P1-3 TaskScheduler 叠跑**：cron 触发前查 DB `status === running` 则跳过。
- **P1-4 Task sync 覆盖运行时状态**：`sync-tasks` upsert 不再写 `status`，避免 running 被重置。
- **P1-5 Credential 缓存 + scope 子串**：CRUD 后 `clearCredentialCache`；`listCredentialsByScope` 改精确匹配（拉全量内存过滤），`"llm"` 不再误命中 `"myllm"`。
- **P1-6 错误处理**：`formatCredential` / `formatEntity` 的 `JSON.parse` 加 try/catch（`safeJsonParse`）。
- **P1-10 Trigger 日志脱敏**：`sanitizePayloadForLog` 遮蔽 password/secret/token/api_key/authorization/value 字段，长字符串截断。

### 第 3 批 P1-7 / P1-8 / P1-11（commit b4e8d9d）
- **P1-7 性能**：`PostService.list` 不返回 content；`SessionService.getById` messages 加 `take:500`；`rebuildFtsIndex` 改 `$transaction` 批量；`mapFtsHits` 消 N+1（批量 findMany + Map）。
- **P1-8 Prisma 外键**：`ChatSession.agentId` / `Run.agentId` / `Run.sessionId` 加外键 + `onDelete: SetNull`，删 Agent/Session 不再留孤儿。`prisma db push` + `db:sync` 重建 FTS。
- **P1-11 assertUnique 竞态**：`BaseService.create/update` 捕获 P2002 转友好 CONFLICT failure。

### 第 4 批 P2-2 / P2-4 / P2-7（commit 8a9c9f1）
- **P2-2 死代码**：移除 `internalProcedure` 未用 import；删 `runSync` 废弃函数。
- **P2-4 分页**：`listLogsSchema` pageSize max 200 → 100，与其余实体统一。
- **P2-7 Post 软删不触发事件**：`PostService.delete` 软删后显式 emit `post.deleted`，TriggerEngine 等监听器可联动。

### 暂缓项（已在计划文档标注理由）
- P1-9 Skill `node:vm` 沙箱逃逸面 → 留待 L6+（替换 isolated-vm 是较大改造）。
- P2-1 `nativeTools.ts` 过大 → 符合单文件收拢原则，拆分需先对齐规范。
- P2-3 Service 层 `any` → Prisma delegate 泛化代价，全量收紧工作量过大。

- P2-6 `recoverStaleAsyncJobs` 仅启动跑 → 加周期巡检需谨慎，留待 L6+。
