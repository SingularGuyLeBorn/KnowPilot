feat(swarm): HeartbeatEngine + 监控统计 + 邮件/API Key 工具

HeartbeatEngine（心跳机制 #15）：
- 基于 node-cron，启动时加载所有 heartbeat.enabled 的 Agent
- 触发时：预算检查 → 找/建主 session → 注入 source=system 心跳消息 →
  通过 orchestrator 启动 Agent 执行（并发控制 #13）
- 更新 lastRunAt/lastRunStatus/consecutiveFailures
- 连续失败 3 次 → 日志预警（邮件通知待 Phase 5 SMTP 配置）
- 60s 全量刷新注册（Agent 配置变更后生效）
- dormant/deleted Agent 不触发

监控统计（#25/#46）：
- analytics.swarmStats 路由：按 Agent 分组聚合 Run 表
  对话轮数/工具执行数/成功率/平均耗时/token 消耗
- 支持 days 范围过滤 + 单 Agent 查询

邮件通知工具（#47）：
- send_email native tool：支持 smtp / agentemail 两种 provider
- EMAIL_PROVIDER/EMAIL_TO/EMAIL_SMTP_* 环境变量配置
- config 加 emailProvider 字段

免费 API Key 工具（#48）：
- free_api_keys_list：列出 Credential 表中 source=free 的 key
- free_api_keys_fetch：轮询分配最久未使用的 key + 标记 lastUsedAt
- 不返回 key value 给 list（安全），fetch 返回 value 供 Agent 使用
