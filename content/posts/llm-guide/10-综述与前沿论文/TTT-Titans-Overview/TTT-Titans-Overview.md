# TTT(Test-Time Training)与 Titans: 2025-2026 年最受关注的新架构

## 2. TTT: 把隐藏状态变成模型

### 2.1 核心思想

2024 年 7 月，斯坦福、UCSD、UC Berkeley 和 Meta 的研究人员提出了 TTT 层. 核心想法非常直接: 

传统 RNN 的隐藏状态是一个固定大小的向量 $h_t$，更新规则是 $h_t = f(h_{t-1}, x_t)$. 这个向量容量有限，长上下文下信息会被压缩丢失. TTT 将隐藏状态替换为一个**微型机器学习模型**的权重 $W_t$，更新规则变为: 

$$
W_t = W_{t-1} - \eta \nabla \ell(W_{t-1}; x_t) \tag{1}
$$

也就是说，每处理一个 token，都对隐藏状态模型做一步梯度下降，以自监督方式"学习"这个 token 的信息. 输出则通过这个更新后的模型计算: 

$$
z_t = f_{W_t}(x_t) \tag{2}
$$
这个设计的巧妙之处在于: **隐藏状态的容量不再受限于固定维度，而是由模型 $f$ 的表达能力决定. ** 线性模型(TTT-Linear)的权重维度是 $d \times d$，MLP(TTT-MLP)则有更大的参数量. 

### 2.2 两种变体

论文提出了两种具体实现: 

**TTT-Linear**: 隐藏状态 $f$ 是一个线性模型 $f_W(x) = Wx$. 参数 $W$ 是 $d \times d$ 矩阵. 每次更新通过自监督重构损失做一步梯度下降. 

**TTT-MLP**: 隐藏状态 $f$ 是一个两层 MLP(隐藏维度 $4d$，GELU 激活)，带层归一化和残差连接. 表达能力更强，但计算成本更高. 

### 2.3 关键实验结果

**短上下文(Pile 8K)** : TTT-Linear 和 TTT-MLP 均明显优于 Mamba. 随着上下文变长，TTT 相对于 Mamba 的优势更大. 

**长上下文(Books 32K)** : TTT-MLP 表现最佳，Mamba 在 16K 后困惑度不再下降——RNN 的固定隐藏状态确实成为了长上下文的瓶颈. 

**实际运行时间**: 在 8K 上下文下，TTT-Linear 比 Transformer 更快，与 Mamba 相当. TTT 的对偶形式(dual form)通过矩阵乘法并行化了原本需要串行计算的梯度步骤，实现了 5 倍以上的 wall-clock 加速. 

### 2.4 理论意义: 对偶性

论文的一个重要理论贡献是证明了: **TTT-Linear + batch GD 等价于线性注意力(Linear Attention)** . 这意味着 TTT 统一了 RNN、线性注意力和参数化记忆三种范式——它们只是同一个框架的不同实例化. 

## 3. Titans: 神经长期记忆

### 3.1 从惊奇到记忆

2025 年 1 月，Google 提出了 Titans 架构. TTT 关注的是"如何把隐藏状态做得更有表达力"，Titans 关注的是"如何设计一个真正的长期记忆系统". 

Titans 的核心概念是**惊奇度量(Surprise)** ——模型应该记住那些令人"惊奇"的信息，而遗忘那些已经预料到的信息. 惊奇度量的计算基于记忆模型的梯度: 

$$
\text{Surprise}(x_t) = \|\nabla \ell(W_{t-1}; x_t)\| \tag{3}
$$

但这个朴素定义有一个问题: 一个事件可能只在第一次出现时令人惊奇，后续的"回味"却不被捕捉. 为此，Titans 提出**带动量的惊奇度量**——将惊奇分解为"瞬时惊奇"和"过去惊奇的动量"，类似带动量的梯度下降. 

### 3.2 深度神经记忆

Titans 的长期记忆模块是一个多层 MLP，通过元学习方式训练: 

- **内循环**: 在测试时，根据当前 token 的惊奇度量，对记忆 MLP 做一步梯度更新
- **外循环**: 在整个预训练过程中，学习记忆 MLP 的初始化参数和更新规则

目标是让模型学会"在测试时如何记忆"——不是记住具体的数据模式，而是学会一种通用的记忆策略. 

### 3.3 三种架构变体

Titans 将短期记忆(滑动窗口注意力)和长期记忆(神经记忆模块)组合为三种架构: 

**MAC(Memory as Context)** : 将长期记忆的输出作为注意力机制的上下文补充. 输入序列分片后，先从长期记忆中检索相关信息，再与当前输入拼接，送入注意力模块. 

**MAG(Memory as Gating)** : 通过门控机制动态融合滑动窗口注意力的输出和长期记忆的输出. 门控权重是可学习的，模型自主决定何时相信短期记忆(注意力)vs 长期记忆. 

**MAL(Memory as Layer)** : 将神经记忆作为深度神经网络中的独立一层，直接参与信息处理. 

### 3.4 关键实验结果

Titans 在多个任务上表现出色: 

**大海捞针(NIAH)** : 在 16K 上下文下，Titans 的所有变体均优于 Transformer 和 Mamba，MAC 变体表现最佳. 

**BABILong(长文档推理)** : Titans 在需要跨长文档进行多步推理的任务上显著优于基线. 

**理论优势**: 论文证明了 Titans 能够解决超出 $TC^0$ 复杂度的问题，而 Transformer、对角线性循环模型和 DeltaNet 都被限制在 $TC^0$ 内. 这意味着 Titans 在理论上具有更强的表达能力. 

### 3.5 未解决的挑战

Titans 目前尚未开源，测试时训练的计算成本仍然较高. 2025 年 10 月有一篇论文尝试复现了 MAC 变体，发现冻结 backbone 只更新记忆 MLP 时存在稳定性挑战. Google 是否会在内部走通这条路线，或最终选择开源，将决定 Titans 的行业影响力. 

## 4. TTT vs Titans: 两种记忆哲学

| 维度 | TTT | Titans |
|:----|:----|:-------|
| **核心机制** | 每 token 一步梯度下降 | 基于惊奇度量的选择性更新 |
| **隐藏状态** | 线性模型或 MLP 的权重 | 多层 MLP |
| **更新频率** | 每个 token 都更新 | 仅对"惊奇"的 token 更新 |
| **与注意力关系** | 替代注意力层 | 与注意力协同工作 |
| **开源状态** | 未大规模开源 | 未开源 |
| **理论创新** | 统一 RNN/线性注意力/参数化记忆 | 证明超出 $TC^0$ 的表达能力 |
| **主要瓶颈** | I/O 优化，wall-clock 速度 | 训练稳定性，计算成本 |

## 5. 对行业的影响

TTT 和 Titans 代表了一个重要的方向转变: **从"设计更好的固定架构"到"让架构在测试时自适应调整". **

这个方向的潜在影响是深远的: 

1. **长上下文的真正解决方案**: 如果测试时训练的效率问题能被解决，模型将不再受限于固定上下文窗口——它可以"边推理边学习"，持续吸收新信息. 

2. **终身学习的架构基础**: TTT/Titans 的元学习范式——在测试时通过梯度更新调整隐藏状态——为真正的终身学习提供了架构层面的支持. 

3. **与混合注意力互补**: 2026 年的主流是混合注意力(Gated Attention + GLA/SSD)，TTT/Titans 代表另一条路线. 两者并不互斥——未来可能出现融合两者的架构. 

4. **工程挑战仍然巨大**: 测试时训练需要额外的计算开销，I/O 优化和硬件适配是实际部署的核心瓶颈. TTT 论文中专门开发了 ThunderKittens GPU 内核来缓解这一问题，说明这条路线的工程门槛远高于纯注意力变体. 

## 6. 总结

TTT 和 Titans 是 2025-2026 年最值得关注的"第三波"架构创新. 它们不满足于优化注意力机制，而是重新定义了序列模型处理信息的基本方式. 虽然工程化落地还有距离，但它们提出的"测试时学习"范式，为 LLM 突破固定上下文窗口的限制、实现真正的终身学习提供了理论上的可行路径.
