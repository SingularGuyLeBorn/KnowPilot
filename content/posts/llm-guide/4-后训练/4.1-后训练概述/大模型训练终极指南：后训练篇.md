# 大模型训练终极指南：后训练篇

> 本文系统梳理后训练阶段的核心技术——从监督微调(SFT)的高级技巧、奖励模型训练到 PPO 算法的工程实现，为构建高质量对齐模型提供可落地的实践指南. 

---

## 1. 监督微调(SFT)的高级技巧

### 1.1 高质量 SFT 数据构建

**数据质量筛选的核心指标**：
1. **输入-输出相关性**：prompt 和 response 的语义匹配度
2. **输出完整性**：response 是否完整回答了 prompt
3. **格式规范性**：JSON/XML/代码格式是否正确
4. **长度合理性**：避免过短(<50 tokens)或过长(>max_length)

**数据多样性保障**：
- 任务类型覆盖：问答、摘要、翻译、代码、推理、创意写作
- 难度分布：简单:中等:困难 = 3:5:2
- 领域分布：通用知识、STEM、人文社科、编程、多语言

### 1.2 SFT 训练技巧

**学习率策略**：
- 峰值学习率：1e-5 到 5e-5(比预训练低 1-2 个数量级)
- 衰减策略：线性衰减或余弦衰减
- Warmup：前 5-10% 的步数线性增加到峰值

**数据排序**：
- **长度排序**：将相近长度的样本分组，减少 padding 浪费
- **难度排序(Curriculum Learning)** ：先训简单样本，再逐步增加难度
- **随机打乱**：避免模型过拟合特定任务顺序

**多轮对话处理**：
- 完整的对话历史作为输入，最后一轮作为预测目标
- 对历史轮次使用 loss mask，只计算最后一轮的 loss
- 系统提示始终参与 loss 计算

## 2. 奖励模型训练与 RLHF

### 2.1 奖励模型(Reward Model)

**数据构建**：
- 偏好数据：同一 prompt 的多个 response，人工标注排序
- 比较对：(x, y_w, y_l)，其中 y_w 优于 y_l
- 标注质量：至少 3 人独立标注，一致性 > 70% 才采用

**训练目标(Bradley-Terry 模型)** ：

L_RM = -E[(x, y_w, y_l)] [ log sigmoid( r_phi(x, y_w) - r_phi(x, y_l) ) ]

**关键技巧**：
- 使用预训练模型(如 SFT 后的模型)初始化 RM
- RM 通常比策略模型小(如 7B RM 用于 70B 策略模型)
- 添加 margin loss 防止 RM 对所有 response 打相似分数

### 2.2 RLHF 完整流程

Step 1: 收集偏好数据(Prompt + Response A + Response B + 人类偏好)
Step 2: 训练奖励模型(Bradley-Terry 损失)
Step 3: 使用 PPO 优化策略模型，最大化奖励模型打分
Step 4: 迭代：用优化后的模型生成新 response，扩展偏好数据

## 3. PPO 算法的工程实现

### 3.1 PPO 核心公式

**策略梯度**：

grad J(theta) = E[x ~ D, y ~ pi_theta] [ sum_t min( r_t(theta) * A_t, clip(r_t(theta), 1-eps, 1+eps) * A_t ) ]

其中：
- r_t(theta) = pi_theta(y_t|x, y_<t) / pi_theta_old(y_t|x, y_<t)：新旧策略概率比
- A_t：优势函数估计(通常用 GAE)
- eps：裁剪阈值(通常 0.1 或 0.2)

**KL 惩罚**：

L_KL = beta * E[x,y] [ KL( pi_theta(y|x) || pi_ref(y|x) ) ]

### 3.2 工程实现要点

**四个模型并行**：
1. Policy Model(策略模型)：被优化的主模型
2. Reference Model(参考模型)：提供 KL 基准
3. Reward Model(奖励模型)：评估生成质量
4. Value Model(价值模型)：估计状态价值

**内存优化**：
- Policy 和 Reference 可共享部分参数
- Value Model 通常比 Policy Model 小
- 使用 LoRA 只训练 adapter

**训练稳定性**：
- 梯度裁剪：max_norm = 1.0
- 学习率：比 SFT 更低(1e-6 到 5e-6)
- Early stopping：KL 超过阈值时停止

### 3.3 PPO 的替代方案

| 算法 | 核心改进 | 优势 | 劣势 |
|:-----|:--------|:-----|:-----|
| GRPO | 移除 Value Model，组内均值估计基线 | 省显存 | 方差大 |
| DAPO | 动态调整裁剪阈值 | 更稳定 | 调参复杂 |
| ORPO | 直接用偏好数据训练，无需 RM | 端到端 | 效果弱于 RLHF |
| Rejection Sampling | 采样多个输出选最优 SFT | 简单稳定 | 无法超越采样分布 |

## 4. 后训练的评估与调试

### 4.1 评估维度

**有用性(Helpfulness)** ：任务完成率、信息准确性、输出质量
**安全性(Harmlessness)** ：有害内容率、越狱鲁棒性、偏见检测
**诚实性(Honesty)** ：幻觉率、不确定性表达、知识边界认知

### 4.2 常见失败模式

**模式崩溃(Mode Collapse)** ：
- 现象：模型总是生成相似的 response
- 原因：KL 惩罚过强或奖励模型过度优化
- 解决：降低 KL 系数、增加采样温度

**奖励黑客(Reward Hacking)** ：
- 现象：模型找到奖励模型漏洞，生成高分低质内容
- 解决：迭代更新奖励模型、多维度评估

**灾难性遗忘(Catastrophic Forgetting)** ：
- 现象：RL 后通用任务性能下降
- 解决：保留 SFT 数据混合训练、限制 KL 散度

## 5. 总结

| 阶段 | 目标 | 关键技术 | 评估重点 |
|:-----|:-----|:--------|:--------|
| SFT | 学习指令遵循 | 数据筛选、多轮对话、学习率调优 | 指令遵循率 |
| RM 训练 | 建立偏好评估 | 偏好数据、Bradley-Terry、Margin Loss | 排序准确率 |
| RL | 优化生成质量 | PPO/GRPO/DAPO、KL 惩罚 | 奖励提升、安全性 |
