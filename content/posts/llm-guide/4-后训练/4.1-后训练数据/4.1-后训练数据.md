# 4.1 后训练数据

> 后训练(Post-Training)阶段使用的数据，是决定模型对齐质量和 specialized 能力的关键因素. 本章深入分析 SFT 数据、偏好数据、合成数据的构建方法与核心挑战，涵盖数据来源、质量筛选、配比策略与最新研究进展. 

---

## 1. SFT 数据构建

监督微调(Supervised Fine-Tuning, SFT)是后训练的第一阶段，目标是让模型学会遵循指令、完成各类任务. 

### 1.1 数据来源

SFT 数据的主要来源可分为以下几类：

**1. 公开指令数据集**

| 数据集 | 规模 | 来源 | 特点 |
|:------|:----|:-----|:-----|
| ShareGPT | ~100K | 用户与 ChatGPT 的真实对话 | 多样性高，但存在质量噪音 |
| OpenAssistant 对话 | ~161K | 人工编写 | 开源、多轮对话、许可友好 |
| Dolly 2.0 | ~15K | Databricks 员工编写 | 高质量人工标注，7 类任务 |
| Alpaca | ~52K | Self-Instruct 合成 | 种子任务+Text-Davinci-003 生成 |
| LIMA | ~1K | 人工精选 | 极少量但高质量，"少即是多" |
| No Robots | ~10K | 人工编写 | 高质量，多任务覆盖 |
| UltraChat | ~1.5M | 多轮合成 | 大规模合成数据，多样性强 |
| SlimOrca | ~518K | 过滤版 OpenOrca | 质量筛选后的中大规模数据 |

**2. 自建指令数据**

实际生产中，多数团队会结合自身场景自建 SFT 数据：
- **任务定义**：先定义模型需要支持的任务类型(问答、写作、代码、翻译、推理等)
- **种子示例**：由领域专家编写少量高质量种子示例(通常 100~500 条)
- **模板扩展**：基于种子示例设计模板，快速扩展到数千条
- **持续积累**：从线上用户反馈中收集真实指令，经审核后加入训练集

**3. 合成数据**

合成数据(Synthetic Data)已成为 SFT 数据构建的核心手段：
- **Self-Instruct**：种子任务 → LLM 生成指令 → LLM 生成回复 → 质量过滤
- **Evol-Instruct**(WizardLM)：从简单指令逐步演化出复杂指令(深度演化+宽度演化)
- **反向指令生成**：给定文本段落 → LLM 生成对应的指令/问题
- **多轮对话展开**：单轮 → LLM 扩展为多轮对话

> 研究表明，**合成数据+少量人工精选数据**的组合效果往往优于纯人工数据或纯合成数据. 

### 1.2 质量控制的关键方法

SFT 数据的质量比数量重要得多. LIMA(1K 数据超越 50K+ 数据)和之后的许多研究都验证了这一观点. 

**核心质控方法：**

**1. 去重与近义合并**
- 语义去重：基于 embedding 相似度移除高度相似的指令
- 任务均衡：确保各类任务分布合理，避免单一任务主导

**2. 多样性采样**
- 基于指令类型/任务类别的分层采样
- 基于指令 embedding 的最大边际相关性(MMR)采样
- 难度分层：简单/中等/困难指令保持合理比例

**3. 回复质量审核**
- 长度过滤：过短(< 50 字符)或过长(> 4096 字符)回复需要人工复核
- 格式检查：回复是否符合预期的格式要求(如代码要有注释)
- 事实性校验：涉及事实类问题时核对来源
- 安全性审核：确保回复不包含有害内容

**4. Deita 等自动筛选方法**

Deita(2024)提出了一种无需人工标注的数据筛选方法：
- **指令复杂度评分**：利用 LLM 评估指令的难度和多样性
- **回复质量评分**：基于模型自身的困惑度或奖励模型评分
- **综合筛选**：结合复杂度×质量进行 top-k 采样

实验表明，Deita 仅需 6K 数据即可达到全量数据集的 SFT 效果. 

### 1.3 数据配比策略

SFT 数据涉及多种任务类型的混合，合理的配比至关重要：

| 任务类型 | 推荐比例 | 说明 |
|:--------|:-------|:-----|
| 通用对话 | 30~40% | 日常问答、开放对话、闲聊 |
| 写作创作 | 15~20% | 文章、邮件、报告、创意写作 |
| 代码编程 | 10~15% | 代码生成、调试、解释 |
| 阅读理解 | 10~15% | 文档问答、摘要、信息提取 |
| 推理计算 | 5~10% | 数学、逻辑、常识推理 |
| 专业领域 | 5~10% | 医疗、法律、金融等垂直领域 |
| 安全对齐 | 3~5% | 拒绝回答、边界测试、无害性 |

**关键发现**：
- **数据配比比数据总量更重要**——即使总量很大，如果配比失衡，模型在低占比任务上表现会很差
- **同一比例下多样性更重要**——1000 条多样指令 > 10000 条相似指令
- **需要持续迭代**——SFT 阶段通常是多轮、渐进式的，每轮根据评估结果调整配比

## 2. 偏好数据构建

偏好数据用于 RLHF(基于人类反馈的强化学习)或其替代方案(DPO、GRPO 等)，是模型对齐的关键. 

### 2.1 偏好数据的类型

**1. 比较型偏好数据(Pairwise)** 

最主流的形式，对比两条回复让标注者选优：
```
指令：什么是量子计算？
回复 A：量子计算是...(详细解释)
回复 B：量子计算是利用量子力学...(简洁回答)
偏好：A(更详细、更准确)
```

特点：
- **标注成本较低**：对比选择比绝对评分更容易
- **数据利用率高**：一个 pair 同时提供正负信号
- **主流数据集**：Anthropic HH-RLHF(~170K)、OpenAI WebGPT(~20K)、Nectar(~180K)

**2. 评分型偏好数据(Scalar)** 

对每条回复给出绝对评分(如 1~5 分)：
- 优点：支持更细粒度的奖励模型训练
- 缺点：标注一致性差(不同标注者的评分标准不同)
- 适用场景：奖励模型预训练、质量评估

**3. 排序型偏好数据(Ranking)** 

对多条回复进行排序(通常是 4~8 条)：
- 优点：一条标注产生多个比较对，信息量更大
- 缺点：标注复杂度高，标注疲劳效应显著
- 代表工作：InstructGPT 论文中的排序标注

### 2.2 人类标注的挑战与实践

**标注者管理**：
- **标注者筛选**：通过测试题筛选符合标准的标注者
- **培训标准化**：统一标注指导手册，定期校准会议
- **标注一致性检测**：在标注集中混入已知标准答案的校验题
- **交叉审核**：约 10~20% 的标注由多人标注，计算一致性指标(如 Cohen's Kappa)

**常见标注偏差**：
- **长度偏差**：标注者倾向于选择更长的回复(即使质量更低)
- **冗余偏差**：包含安全免责声明的回复更容易被选为坏回复
- **顺序偏差**：排在前面的回复更容易被选中
- **流行偏差**：标注者倾向于选择看起来更"专业"的表述

**缓解策略**：
- 回复匿名化(消除模型来源信息)
- 回复长度尽量均衡(或控制变量)
- 随机打乱呈现顺序
- 使用预训练奖励模型辅助标注，标注者仅做校验

### 2.3 AI 反馈与合成偏好数据

随着标注成本上升和质量控制难度加大，AI 反馈(RLAIF)成为越来越主流的选择：

**Constitutional AI(Anthropic, 2023)** 
- 定义一组"宪法原则"(如"不要生成有害内容")
- 让模型自己生成回复并根据宪法原则自我评判
- 自我修订 + 偏好数据合成

**AI 标注器替代人类**：
- 使用强大的 LLM(如 GPT-4、Claude 3.5)作为偏好判定器
- 人类仅标注少量种子数据用于校准
- 研究表明，RLAIF 在多数任务上可达与 RLHF 相当的对齐效果

**Self-Play 与迭代式数据生成**：
- **SPIN(Self-Play Fine-Tuning)** ：模型自己生成回复作为负样本，真实数据作为正样本，迭代训练
- **迭代 DPO**：每一轮训练后，用新模型生成新的对比数据再次训练
- **SimPO**：直接基于模型自身概率进行偏好优化，无需 reference model

> 2024~2025 年的趋势是：**人工标注偏好数据 → AI 辅助标注 → 完全自合成偏好数据**. DeepSeek-R1 和后续系列模型大量使用了 model-generated 的推理轨迹数据.

## 3. 数据数量与质量的权衡

### 3.1 Scaling Laws 与边际收益

SFT 数据的 scaling law 与预训练有很大不同：

| 论断 | 证据 |
|:----|:-----|
| **SFT 存在明显的边际收益递减** | LIMA 实验：1K→2K 有提升，2K→10K 提升很小，10K→50K 几乎无提升 |
| **但任务覆盖面仍与数据量正相关** | Tulu 2 实验：当任务数量增加时，需要更多样化的数据 |
| **质量始终优于数量** | Deita 实验：6K 精选 ≈ 全量 50K |
| **数据多样性是关键瓶颈** | < 100 个独特任务时，数据再多也无益 |

**实际建议**：
- **冷启动阶段**：重点提升数据质量(1K~5K 精选数据)
- **扩展阶段**：逐步增加任务类型覆盖(10K~50K 多样化数据)
- **饱和阶段**：关注数据配比优化和合成数据增强

### 3.2 数据飞轮

生产环境中，后训练数据的持续迭代构成了"数据飞轮"：

```
模型上线 → 收集用户反馈 → 识别 badcase → 补充高质量数据
    ↑                                              |
    └────────── 新模型上线 ← 重新训练 ←────────────┘
```

关键的反馈数据来源：
- **用户点赞/点踩**：最直接的偏好信号
- **对话留存率**：用户是否愿意继续对话
- **任务成功率**：在特定任务上的客观评估指标
- **安全审核记录**：被安全策略拦截的对话

## 4. 前沿方向与开放问题

### 4.1 合成数据的质量天花板

虽然合成数据极大扩展了 SFT 数据的规模，但其存在根本性的限制：
- **知识锚定**：合成数据来自 LLM 本身，无法引入超出模型已有知识的新信息
- **风格同质化**：合成数据往往呈现"平均化"的语言风格，缺乏多样性
- **错误传播**：合成数据中的错误会通过多轮生成被放大

应对策略包括：
- 引入外部知识源(文档库、维基百科、API)作为合成输入
- 使用不同的 prompt 策略和 temperature 生成多样化回复
- 人工对合成数据的 1~5% 进行抽查和修正

### 4.2 长 CoT 数据

随着推理模型(如 OpenAI o1/o3、DeepSeek-R1、Qwen3)的兴起，长思考链(Long Chain-of-Thought)数据的构建成为新的焦点：

- **数据特点**：数千~数万 token 的详细推理过程，包含尝试、回溯、验证
- **构建方式**：从强推理模型中蒸馏 → 人工校验 → 拒绝采样 → 迭代微调
- **核心挑战**：推理过程的正确性验证(长链中如何定位错误步骤)
- **数据规模**：量级通常在数万~数十万条(过多会导致推理风格固化)

### 4.3 多模态后训练数据

随着多模态模型的发展，后训练数据正在从纯文本扩展为多模态：
- 图文对齐数据(图像+文本指令)
- 视频理解数据(视频片段+时序理解指令)
- 跨模态推理数据(多张图像的对比推理)

> 2025 年的趋势是：后训练数据的重心从"完成数量"转向"构建质量飞轮"，从"人工标注"转向"模型合成+人工校验"的混合模式.

## 5. 数据处理流程总结

一个完整的后训练数据处理流程如下：

```
1. 需求定义
   ├ 确定目标能力(对话/代码/推理/安全...)
   └ 确定评估指标(人工评估/自动评估/用户指标)

2. 数据采集
   ├ 已有数据集筛选(公开数据集)
   ├ 种子数据编写(专家编写 100~500 条)
   ├ 合成数据生成(Self-Instruct / Evol-Instruct)
   └ 用户反馈数据(线上 badcase)

3. 质量控制
   ├ 去重与近义合并
   ├ 多样性采样(MMR / 分层采样)
   ├ 格式规范检查
   ├ 长度范围控制
   └ 安全性过滤

4. 数据配比
   ├ 任务类型配比
   ├ 难度等级配比
   ├ 来源混合策略
   └ 迭代调整(根据评估结果)

5. 格式标准化
   ├ 统一对话模板
   ├ token 化处理
   └ 验证 token 长度分布

6. 持续迭代
   ├ 模型评估 → 识别问题 → 补充数据
   └ 每轮训练的数据版本化管理
```

---

> **「后训练数据的质量决定了模型的天花板，预训练数据的规模决定了模型的地板. 」**
> 在实际工程中，后训练数据的投入产出比远高于预训练数据——提升后训练数据的质量，往往比扩大预训练数据量获得更直接的收益.
