# 03 GSPO 完全解析: 驯服大模型强化学习的“稳定诅咒”

## 1. 问题的起源 - 大模型强化学习的“稳定诅咒”

要理解 GSPO 的革命性, 我们必须首先回到问题的起点, 看看它到底解决了什么.

### 1.1 强化学习: LLM 能力飞跃的引擎

LLM 的训练通常分为预训练 (Pre-training), 监督微调 (Supervised Fine-Tuning, SFT) 和强化学习 (RLHF) 三个阶段. 前两个阶段赋予模型知识和对话能力, 而 RLHF 则负责教会模型何为“好”的回答, 提升其推理、遵循复杂指令等高阶能力.

- **精妙类比 (Analogy): 训练一位作家**

- **预训练**: 让他阅读古今中外所有书籍, 掌握语言规律和海量知识.

- **监督微调**: 给他看大量优秀范文 (问题-答案对), 让他模仿写作风格.

- **强化学习**: 让他写一篇文章 (生成一个回答), 由一位资深编辑 (奖励模型) 通读全文后给出评分和修改意见 (奖励信号). 作家根据这些反馈, 调整自己的写作思路 (模型参数), 力求下一篇文章写得更好.

这个过程的核心是让模型在探索中学习, 而**近端策略优化 (PPO)** 算法正是这一阶段最经典的“教练”.

### 1.2 从 PPO 到 GRPO: 摆脱价值模型的尝试

PPO 的核心思想是“小步快跑”, 它通过一个名为“**重要性采样 (Importance Sampling)** ”的机制, 利用旧模型产生的数据来安全地更新当前模型, 同时通过“**裁剪 (Clipping)** ”防止步子迈得太大导致模型“学坏”.

但 PPO 有一个巨大的实践难题: 它严重依赖一个额外的**价值模型 (Value Model)** . 这个模型需要像策略模型一样庞大, 它的任务是给模型生成的每一个词元 (token) 打一个“预期未来收益分”, 这本身就极难训练且耗费资源.

为了摆脱这个昂贵的“累赘”, DeepSeek 团队提出了 **GRPO (组相对策略优化)** . GRPO 的想法非常巧妙:

- **核心思路**: 对于同一个问题, 让模型生成一组不同的回答 (例如 4 个).

- **相对评分**: 用一个奖励模型给这 4 个回答分别打一个总分, 然后计算每个回答与组内平均分的差距, 这个差距就是“**优势 (Advantage)** ”.

- **优势**: 高于平均分的回答获得正优势, 低于的则为负优势.

通过这种方式, GRPO 成功绕过了价值模型, 直接利用组内排名进行优化, 大大简化了流程.

### 1.3 GRPO 的“阿喀琉斯之踵”: 逐 Token 优化的原罪

尽管 GRPO 很聪明, 但 Qwen 团队发现它在理论根基上埋下了一个巨大的隐患. 问题出在它应用**重要性采样**的方式上.

- **问题根源**: GRPO 继承了 PPO 的做法, 在**每一个单独的 Token 级别**上计算重要性权重.

- **视觉化描述 (Visual Description)** : 想象一下, 强化学习的优化过程就像是在一片崎岖的地形上导航. **重要性采样**本应是一张地图, 告诉你旧数据的位置和新地形的偏差, 从而让你安全地走下一步. 而 GRPO 的做法, 相当于把这张地图撕成了无数碎片, 每走一小步 (生成一个 token) 就只看脚下那块碎片.

- **后果**: 单个 token 的概率波动是充满随机性的. 在单个 token 上应用重要性采样, 不仅无法修正分布偏差, 反而向梯度中注入了**高方差的噪声**. 这种噪声会随着序列变长而不断累积, 最终像滚雪球一样, 导致训练过程不可预测, 引发灾难性的**模型崩溃**.

这个问题在训练庞大而复杂的 **MoE (混合专家) 模型**时尤为致命, 因为 MoE 模型的“专家路由”本身就具有随机性, 进一步放大了 token 级别的噪声.

**引导性问题 (Guiding Question):** 既然奖励是针对整个回答给出的, 为什么之前的算法会普遍选择在更细的 token 粒度上进行优化呢? 这背后可能有哪些历史原因或实现上的便利性考量?

## 2. GSPO 的核心思想与设计哲学

面对 GRPO 的根本性缺陷, Qwen 团队提出的 GSPO 并非简单的修补, 而是回归第一性原理的重构.

### 2.1 核心原则: 优化的单元必须与奖励的单元匹配

这是 GSPO 设计哲学的基石. 既然奖励模型是通读整个回答后给出一个**整体分数**, 那么模型的学习信号和优化过程也应该在**序列 (Sequence)** 这个宏观层面上进行.

### 2.2 解决方案: 从“逐字打分”到“通篇评审”

GSPO 彻底抛弃了 GRPO 那种“逐字打分”的模式.

- **GRPO 的做法 (逐字打分)** : 对于一个获得高分的回答, GRPO 依然会去计较其中每个词元概率的微小变化, 用充满噪声的权重去指导更新.

- **GSPO 的做法 (通篇评审)** : 对于一个获得高分的回答, GSPO 会给出一个统一的、宏观的正面评价 (序列级权重), 然后用这个一致的信号去鼓励生成该回答的所有词元. 所有词元“同甘共苦”, 共同为一个优秀的目标服务.

### 2.3 设计精髓: 序列级重要性采样与长度归一化

GSPO 的核心机制是定义了一个全新的**序列级重要性权重**. 它计算的是整个回答序列在新旧策略下的**似然度 (Likelihood) 之比**. 这才是对重要性采样原则的正确应用.

此外, 为了解决长短句概率值差异巨大的数值稳定性问题, GSPO 引入了一个精巧的设计: **长度归一化**. 在计算序列似然比后, 对其开序列长度的次方根, 相当于取**几何平均值**. 这能将权重值拉回到一个合理的、可比较的范围内, 极大地增强了算法的鲁棒性.

## 3. 算法详解与公式推导

现在, 让我们深入 GSPO 的数学心脏.

### 3.1 序列级重要性权重 (s_i(θ)) 的定义

GSPO 的灵魂在于其序列级重要性权重 $s_i(\theta)$, 其定义如下:

$s_i(\theta) = \left( \frac{ \pi_{\theta} (y_i | x) }{ \pi_{\theta_{old}} (y_i | x)} \right)^{\frac{1}{|y_i|}} = \exp \left( \frac{1}{|y_i|} \sum_{t=1}^{|y_i|} \log \frac{ \pi_{\theta} (y_{i,t} | x, y_{i,<t}) }{ \pi_{\theta_{old}} (y_{i,t} | x,y_{i,<t})} \right)$

- $\pi_{\theta} (y_i | x)$: 当前策略 (新模型) 生成整个序列 $y_i$ 的概率.- $\pi_{\theta_{old}} (y_i | x)$: 旧策略 (采样模型) 生成整个序列 $y_i$ 的概率.- $(\cdot)^{\frac{1}{|y_i|}}$: 这就是**长度归一化**操作. 它将不同长度序列的权重调整到相似的尺度, 防止长序列的概率连乘积过小导致数值不稳定.

### 3.2 GSPO 优化目标

基于上述权重, GSPO 的优化目标 (损失函数) 定义为:

$\mathcal{J}_{GSPO} (\theta) = \mathbb{E}_{x \sim \mathcal{D}, \{y_i\} \sim \pi_{\theta_{old}}} \left[ \frac{1}{G} \sum_{i=1}^{G} \min \left( s_{i}(\theta) \widehat{A}_{i}, \, \mathrm{clip} \left( s_{i}(\theta), 1 - \varepsilon, 1 + \varepsilon \right) \widehat{A}_{i} \right) \right]$

- $\widehat{A}_{i}$: 与 GRPO 类似, 是第 $i$ 个回答的**组内相对优势**.- $\mathrm{clip}(\cdot)$: 与 PPO/GRPO 一样, 是裁剪函数, 将重要性权重限制在 $(1-\varepsilon, 1+\varepsilon)$ 的小范围内, 防止策略更新过猛.

- **关键区别**: 这里的权重 $s_i(\theta)$ 和裁剪操作, 都是作用于**整个序列**的, 而不是像 GRPO 那样作用于每个 token.

### 3.4 GSPO-token: 兼顾灵活性与稳定性的变体

论文还提出了一个名为 **GSPO-token** 的变体, 用于多轮对话等可能需要更细粒度奖励的场景. 它巧妙地将序列级权重的**数值**应用到每个 token 上, 在理论上证明了当一个序列中所有 token 共享相同优势时, 其梯度与 GSPO 完全等价. 这在保留 GSPO 稳定性的同时, 提供了为每个 token 定制优势的灵活性.

## 4. 实验证据与关键洞察

理论的优雅需要实践的检验. Qwen 团队的实验结果雄辩地证明了 GSPO 的优越性.

### 4.1 实验结果: 更稳定、更高效、更强大

![](./03-GSPO-images/image_0.jpg)

图 1: GSPO 与 GRPO 的训练性能对比. 在多个基准上, GSPO (蓝色/绿色) 的性能曲线稳定地高于需要特殊技巧 (Routing Replay) 才能收敛的 GRPO (橙色).

实验表明, 在同等计算资源下, GSPO 在训练奖励和下游任务 (如数学, 编程) 性能上, 都稳定且持续地优于精心调优的 GRPO.

### 4.2 关键洞察一: 反直觉的“裁剪悖论”

实验中出现了一个惊人的现象: GSPO 裁剪掉 (即因偏差过大而丢弃) 的 token 比例远高于 GRPO, 两者相差近两个数量级.

- **精妙类比 (Analogy): 投资策略**

- **GRPO**: 像一个新手投资者, 对市场上 1000 个项目都觉得有点道理, 于是给每个都投了点钱. 大部分投资的微小收益被少数的巨大亏损所抵消, 整体回报率低. 它利用了所有数据, 但这些数据充满了噪声.

- **GSPO**: 像一位经验丰富的投资大师, 只挑选了 50 个最优质的项目进行重仓. 虽然它“浪费”了研究另外 950 个项目的时间, 但这 50 笔高质量的投资带来了惊人的回报.

这个现象雄辩地证明: **在强化学习中, 学习信号的质量远比数量重要.** GSPO 的序列级筛选机制确保了每一次更新都来自高信噪比的优质样本.

### 4.3 关键洞察二: MoE 训练的“特效药”

如前所述, MoE 模型在 RL 训练中存在“**专家激活不稳定性**”问题. GRPO 必须依赖一个名为“**路由重放 (Routing Replay)** ”的复杂技巧 (强制新旧策略使用相同的专家) 才能稳定训练.

GSPO 则完全不需要这个“拐杖”. 因为它关注的是宏观的序列似然度, 这个指标对底层具体的专家组合变化不那么敏感. 这就像一家大公司的 CEO, 他关心的是公司本季度的总体财报 (序列概率), 而不是某个具体部门的人员微调 (专家路由变化).

### 4.4 关键洞察三: RL 基础设施的简化

GSPO 对数值精度的差异容忍度更高, 这意味着可以直接使用推理引擎返回的序列似然进行优化, **避免了在训练引擎中重新计算似然的耗时步骤**. 这大大简化了 RL 的工程基础设施, 提高了训练效率.
