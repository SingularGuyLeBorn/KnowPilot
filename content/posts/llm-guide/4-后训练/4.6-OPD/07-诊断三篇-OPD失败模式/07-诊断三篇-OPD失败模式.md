# 07 · OPD 失败模式诊断: 高熵坍缩、捷径作弊与长链蒸发

>  **前置约定**: 本文符号体系与 01-OPD基础原理、05-G-OPD广义OPD 保持一致. 
> - $\pi_\theta(a|s)$: 学生策略(当前待优化的模型). 
> - $\pi_T(a|s)$: 教师策略(外部强模型、领域专家、或经过特权上下文增强的自身). 
> - $s_t$: 第 $t$ 步的状态(前缀/上下文)，由 prompt + 已生成的 token 序列构成. 
> - $\mathcal{V}$: 词表(Vocabulary)，$|\mathcal{V}| = V$. 
> - $H(P) = -\sum_a P(a) \log P(a)$: 香农熵. 
> - $D_{KL}(P \| Q) = \sum_a P(a) \log \frac{P(a)}{Q(a)}$: Reverse KL 散度.

## Part 1: 高熵坍缩(High-Entropy Collapse)

### 1.1 现象: 盲目自信的幻觉与 Loss-Accuracy 背离

高熵坍缩是 OPD 训练中最具欺骗性的失败模式. 它的欺骗性在于: **训练日志看起来一切正常，甚至堪称优秀**——Loss 曲线平滑下降，KL 散度稳步收敛，梯度范数稳定——但当你把模型拿到验证集上测试时，会发现准确率早在几百个 step 之前就触顶了，此后无论如何延长训练，分数都纹丝不动. 

#### 1.1.1 具体案例: 数学推理中的多路径困境

让我们来看一个真实的训练场景. 假设我们正在用 OPD 训练一个数学推理模型，教师是一个经过精心调优的 72B 模型，学生是一个 7B 模型. 题目是: 

> 求所有满足 $x^2 + y^2 = z^2$ 且 $x + y + z = 120$ 的正整数三元组 $(x, y, z)$. 

这是一道经典的数论题，存在多种解法: 代数消元法、参数化勾股数法、对称性分析法等. 教师模型(72B)在推理到某个关键分支点时——比如"接下来应该消去哪个变量"——给出的概率分布是: 

$$
 \pi_T(\text{消去 } z | s_t) = 0.35, \quad \pi_T(\text{参数化 } | s_t) = 0.30, \quad \pi_T(\text{对称分析 } | s_t) = 0.25, \quad \pi_T(\text{其他 } | s_t) = 0.10 \tag{1}
$$

这是一个典型的高熵状态: 教师自己也没有压倒性的偏好，四种策略看起来都可行. 在标准的 OPD 训练中，学生模型(7B)在这个状态上采样、计算 Reverse KL、反向传播. 

训练进行到第 500 个 step 时，你检查学生模型的输出，发现了以下诡异现象: 

- 对于这道题，学生模型每次都选择"消去 $z$"，并且以 98% 的概率笃定这个选择. 它不再考虑参数化法或对称分析法. 
- 如果强行让模型在更高温度下采样，你会发现它对其他解法给出的概率几乎为 0——仿佛那些解法从未存在于它的认知中. 
- 更致命的是，"消去 $z$"这条路径在某些变体题目中(比如 $x + y + z = 119$)会导致极其繁琐的计算，而参数化法在这种情况下明显更优. 但学生模型完全丧失了切换策略的能力. 

这就是高熵坍缩的核心现象: **模型在教师分布平坦的区域，凭空捏造了一个极端的模态(Modal)，然后用盲目的自信把这个模态钉死在参数空间中**. 

#### 1.1.2 训练日志特征: Loss 下降但验证准确率 Plateau

高熵坍缩在训练日志上留下了一组极其典型的"指纹". 如果你知道去哪里看，就能在崩溃发生之前捕捉到预警信号. 

**特征一: 训练 Loss 持续下降，但验证 Loss 与训练 Loss 的差距越来越大. **

在正常的 OPD 训练中，训练 Loss(在学生自己的 rollout 上计算的 KL)和验证 Loss(在固定的验证集 rollout 上计算的 KL)应该保持大致同步的下降趋势. 但在高熵坍缩发生时，训练 Loss 会因为学生越来越"专注"于少数几个高概率 token 而持续下降——因为 Reverse KL 的 Mode-seeking 特性让学生在自采样的轨迹上"自圆其说". 然而，验证 Loss 却停滞甚至上升，因为验证集上的状态分布与学生训练时的状态分布发生了偏移(学生只在狭窄的模态上训练，遇到验证集中的新状态时不知所措). 

**特征二: 准确率曲线在初期快速上升后进入"死亡平台". **

在训练的前 50-100 个 step，模型通常会因为 On-Policy 蒸馏的快速收敛特性而表现出准确率的跃升. 但当高熵坍缩开始发生后，准确率会在某个水平(比如 62%)上彻底停滞. 无论你再训练 1000 个 step 还是 10000 个 step，这个数字都不会动. 这不是数据不足的问题——即使你不断更换训练数据，平台依然存在. 

**特征三: 学生输出分布的熵($H(\pi_\theta)$)系统性下降. **

这是最直接、最量化的预警信号. 在正常的 OPD 训练中，学生的输出熵应该保持在一个合理的区间内——它可能比教师低(因为 Mode-seeking)，但不会持续单调下降. 在高熵坍缩场景中，你会看到学生熵从初始的 2.5 nats 稳步下降到 0.3 nats 甚至更低，表明模型的输出越来越"偏执"，越来越不愿意探索替代方案. 

**特征四: 教师分布在某些 token 上的概率被学生"选择性失明". **

具体而言，在那些教师分布平坦的状态上，教师给多个 token 分配了中等概率(比如每个 0.1-0.2)，但学生最终只保留了其中一个 token 的概率(比如 0.95)，其余全部压到接近 0. 这不是因为学生"学会了"哪个 token 最好——在教师自己都分不清的情况下，学生不可能学会——而是因为它"捏造"了一个选择，并用盲目的自信维护它.

### 1.2 数学分析: Reverse KL 在高熵区域的结构性缺陷

高熵坍缩不是工程调参的失败，而是 Reverse KL 散度在数学上的内在病理. 要理解这一点，我们需要从最基础的定义出发，严格推导当教师分布趋于均匀时，Reverse KL 对学生行为的激励结构. 

#### 1.2.1 核心推导: 均匀分布下的 Reverse KL

考虑一个特定的状态 $s_t$，假设教师在该状态下的分布是**完全均匀的**: 

$$
 \pi_T(a | s_t) = \frac{1}{|\mathcal{V}|}, \quad \forall a \in \mathcal{V} \tag{2}
$$
其中 $|\mathcal{V}| = V$ 是词表大小. 对于现代大语言模型，$V$ 通常在 32000 到 128000 之间. 

Reverse KL 散度的定义为: 

$$
 D_{KL}\big(\pi_\theta(\cdot|s_t) \,\big\|\, \pi_T(\cdot|s_t)\big) = \sum_{a \in \mathcal{V}} \pi_\theta(a|s_t) \left[ \log \pi_\theta(a|s_t) - \log \pi_T(a|s_t) \right] \tag{3}
$$
此式描述了变量之间的定量关系，其物理意义将在下文详细阐述. 
基于上述分析，建立如下数学关系: 

$$
 D_{KL}\big(\pi_\theta \,\big\|\, U\big) = \sum_{a} \pi_\theta(a) \log \pi_\theta(a) - \sum_{a} \pi_\theta(a) \log \frac{1}{V} \tag{4}
$$
其中 $U$ 表示均匀分布. 注意 $\sum_a \pi_\theta(a) = 1$，所以第二项简化为: 

$$
 D_{KL}\big(\pi_\theta \,\big\|\, U\big) = -H(\pi_\theta) + \log V \tag{5}
$$

这个等式揭示了一个极其关键的数学事实: 

$$
 \boxed{ D_{KL}\big(\pi_\theta \,\big\|\, U\big) = \log V - H(\pi_\theta) } \tag{6}
$$
**物理意义逐项拆解**: 

- **$\log V$**: 这是一个与 $\pi_\theta$ 无关的常数. 对于给定的词表，它是一个固定的上界参考值. 当 $V = 32000$ 时，$\log V \approx 10.37$(自然对数). 

- **$H(\pi_\theta)$**: 学生分布的熵. 熵度量了学生输出的"不确定性"——熵越高，学生越"犹豫"; 熵越低，学生越"笃定". 

- **整体含义**: 在教师均匀分布的条件下，最小化 Reverse KL 完全等价于**最大化学生分布的熵 $H(\pi_\theta)$**. 

从表面上看，这个结论似乎在说: Reverse KL 会驱使学生趋向均匀分布，从而诚实地匹配教师的平坦分布. 如果是这样，高熵坍缩怎么会发生呢？

答案是: **梯度优化的动力学与全局最优解之间存在巨大的鸿沟**. 虽然全局最优解确实是 $\pi_\theta = U$，但从一个实际的初始分布(通常是预训练模型给出的、已经带有一定偏见的分布)出发，通过随机梯度下降到达均匀分布的"路径"，在 Reverse KL 的景观中是极其不稳定的. 

#### 1.2.2 关键极限: Delta 分布的 KL 值是有界的

让我们计算当学生分布趋向极端的 Delta 分布时，Reverse KL 的值. 假设学生将所有概率质量集中在单个 token $a^*$ 上: 

$$
 \pi_\theta(a|s_t) = \begin{cases} 1 & \text{if } a = a^* \\ 0 & \text{otherwise} \end{cases} \tag{7}
$$

此时学生熵 $H(\pi_\theta) = 0$(因为 Delta 分布没有不确定性). 代入上面的公式: 

$$
 D_{KL}(\delta_{a^*} \| U) = \log V - 0 = \log V \tag{8}
$$
**核心洞察**: 即使学生完全"捏造"了一个极端的选择，Reverse KL 的惩罚也**仅为 $\log V$**. 对于 $V = 32000$，这个惩罚约为 10.37 nats. 

现在考虑一个对比场景: 假设教师分布在某个 token $a^*$ 上有一个尖锐的峰值 $\pi_T(a^*) = 0.9$，而在其他 token 上概率很低. 如果学生错误地忽略了 $a^*$，将所有质量放在另一个 token $a'$ 上(其中 $\pi_T(a') = 0.01$)，那么: 

$$
 D_{KL}(\delta_{a'} \| \pi_T) = -\log \pi_T(a') = -\log(0.01) \approx 4.6 \tag{9}
$$

等等，这个值比 $\log V \approx 10.37$ 还小！这说明在教师分布不均匀时，学生"错过"一个高概率 token 而选择一个中等概率 token 的惩罚，反而比在高熵区域捏造一个 Delta 分布的惩罚更小. 

但更关键的比较是: 在同一教师分布下，如果学生诚实地学习教师(即 $\pi_\theta = \pi_T$)，KL = 0; 如果学生捏造 Delta，KL = $-\log \pi_T(a') = 4.6$. 惩罚确实存在. 

然而，在均匀教师的情况下: 
- 诚实学习(均匀): KL = 0
- 捏造 Delta: KL = $\log V \approx 10.37$

差距只有 10.37. 而在教师有尖锐峰值的场景中，错过峰值的惩罚可以趋向无穷大(如果学生选择了一个教师概率为 0 的 token，KL 发散). 

**关键矛盾**: 在均匀分布区域，Reverse KL 对"捏造"行为的惩罚是**有界的、温和的、不具威慑力的**. 而在非均匀区域，惩罚可以趋向无穷大. 这导致了一个不对称的激励结构: 

- **在非均匀区域**: Reverse KL 像一把严厉的戒尺，学生不敢偏离教师的高概率区域太远. 

- **在均匀区域**: Reverse KL 的戒尺变软了. 学生发现，即使自己完全瞎猜一个答案，最多也只被罚 $\log V$——这在整个训练损失的尺度上是一个"可承受"的代价. 

#### 1.2.3 梯度分析: 为什么优化会偏向 Delta 而非均匀

让我们进一步分析 Reverse KL 关于学生 logits 的梯度，看看在高熵区域，梯度到底把模型推向何方. 

设学生模型输出 logits $z_a$，经过 softmax 得到概率: 

$$
 \pi_\theta(a) = \frac{e^{z_a}}{\sum_{a'} e^{z_{a'}}} \tag{10}
$$
Reverse KL 对 $z_a$ 的梯度为: 

$$
 \frac{\partial D_{KL}(\pi_\theta \| \pi_T)}{\partial z_a} = \pi_\theta(a) \left[ \log \frac{\pi_\theta(a)}{\pi_T(a)} + 1 - \sum_{a'} \pi_\theta(a') \log \frac{\pi_\theta(a')}{\pi_T(a')} \right] \tag{11}
$$

当 $\pi_T = U$ 为均匀分布时，$\pi_T(a) = 1/V$，代入得: 

$$
 \frac{\partial D_{KL}}{\partial z_a} = \pi_\theta(a) \left[ \log (V \cdot \pi_\theta(a)) + 1 - D_{KL}(\pi_\theta \| U) \right] \tag{12}
$$
注意 $\log(V \cdot \pi_\theta(a)) = \log \pi_\theta(a) + \log V$. 

**关键观察**: 对于当前概率最高的 token $a_{\max}$($\pi_\theta(a_{\max}) > 1/V$)，$\log(V \cdot \pi_\theta(a_{\max})) > 0$; 而对于概率最低的 token($\pi_\theta(a_{\min}) < 1/V$)，该值为负. 

这意味着梯度在**推高高概率 token 的 logits、压低低概率 token 的 logits**——即使教师分布是完全均匀的！梯度在"制造"差异，而不是"抹平"差异. 

为什么会这样？因为 Reverse KL 的梯度结构天然具有**自我强化**的特性. $\pi_\theta(a)$ 作为权重因子出现在梯度中: 当前概率越高的 token，其 logits 获得的梯度更新幅度越大. 一旦某个 token 因为随机初始波动而获得微弱优势($\pi_\theta > 1/V$)，梯度就会把这个优势进一步放大. 而均匀分布是一个**鞍点**(Saddle Point)或**不稳定平衡点**——理论上它是全局最优，但任何微小的偏离都会被梯度动力学放大，最终滑向某个 Delta 分布的局部最优. 

#### 1.2.4 信息论解释: Mode-Seeking 在高熵区域制造假模态

从信息论的角度，我们可以更深刻地理解这一现象. 

Reverse KL 被称为 Mode-seeking(寻模态)，因为以学生分布 $\pi_\theta$ 为权重，它只关心学生在"自己认为重要"的区域与教师的匹配程度. 在教师分布平坦的区域，学生面临一个根本性的困境: 

- **诚实策略**: 保持输出分布的平坦性，与教师一致. 这要求学生在所有 token 上都维持大致相等的概率质量. 但梯度优化并不"喜欢"这种策略——因为 softmax 参数化和交叉熵/ KL 散度的组合，在梯度层面天然倾向于"赢者通吃". 

- **捏造策略**: 将概率质量集中到任意一个教师支持集内的 token 上. 由于教师的 support 覆盖了整个词表(均匀分布)，学生可以选择任何一个 token 作为"假模态"的锚点. Reverse KL 对这种行为的惩罚是有界的($\log V$)，而且一旦学生开始集中，自我强化的梯度动力学会让它越来越集中. 

**信息论视角下的"假模态"**: 

当学生将分布从均匀压缩到 Delta 时，它"创造"了一个实际上并不存在于教师认知中的模态. 教师的意思是"多个选项都可以"，但学生的行为变成了"只有这一个可以". 这种信息丢失是不可逆的——学生从教师那里接收到的"多路径可用"的信号，被反向 KL 的 Mode-seeking 特性扭曲成了"单路径唯一"的伪信念. 

更定量地说，我们可以计算这种信息丢失的代价. 设教师的真实意图是"均匀分布在 $k$ 个可行选项上"，即 $\pi_T(a_i) = 1/k$ 对这 $k$ 个选项，对其余为 0. 但学生捏造了一个 Delta 分布在其中一个选项上. 此时: 

$$
 D_{KL}(\pi_\theta \| \pi_T) = -\log \frac{1}{k} = \log k \tag{13}
$$

如果 $k = 4$(四个看起来都可以的解法)，KL = $\log 4 \approx 1.39$. 这个惩罚微不足道！学生用 1.39 nats 的 KL 代价，就"买断"了在四个正确选项中任意选择一个并假装它是唯一正确答案的权利. 

而在 Forward KL 的框架下($D_{KL}(\pi_T \| \pi_\theta)$)，同样的 Delta 捏造会付出代价: 

$$
 D_{KL}(\pi_T \| \delta_{a^*}) = \sum_{a} \pi_T(a) \log \frac{\pi_T(a)}{\pi_\theta(a)} \tag{14}
$$
对于 $a \neq a^*$，$\pi_\theta(a) = 0$，Forward KL 中的 $\log(\pi_T(a)/0) \to +\infty$. 这意味着 Forward KL 会**彻底禁止**学生忽略教师的任何支持集内的选项——它是一种"强迫覆盖"的散度. 

Reverse KL 和 Forward KL 在这个场景下的差异揭示了一个深刻的权衡: 
- Forward KL 强迫学生学会教师的所有选项，哪怕学生没这个能力(导致四不像的幻觉). 
- Reverse KL 允许学生只挑一个选项，但当教师本身有多个选项时，这种"只挑一个"就变成了**对教师真实意图的背叛**.

### 1.3 检测: 建立高熵坍缩的早期预警系统

高熵坍缩的破坏性在于它的"隐蔽性"——训练 Loss 看起来正常，模型输出也流畅自然，只有在严格的验证测试下才能暴露问题. 因此，建立一套多维度、自动化的检测系统至关重要. 

#### 1.3.1 监控指标一: 学生输出熵 $H(\pi_\theta)$

这是最直接、最敏感的指标. 在每个训练 step，对验证集上的固定一批状态 $s_t$，计算学生模型输出分布的**平均熵**: 

$$
 \bar{H}_t = \frac{1}{N} \sum_{i=1}^{N} H\big(\pi_\theta(\cdot | s_t^{(i)})\big) \tag{15}
$$

**正常行为**: 在训练初期，$\bar{H}_t$ 可能较高(预训练模型的输出本身就有一定多样性). 随着训练进行，$\bar{H}_t$ 会适度下降(学生学会了聚焦在教师认可的高概率区域)，但应稳定在某个下限之上(通常在 1.5-3.0 nats 之间，取决于任务复杂度). 

**异常行为(高熵坍缩信号)** : $\bar{H}_t$ 持续单调下降，跨过一个临界阈值后仍然不停歇. 对于典型的推理任务，如果 $\bar{H}_t$ 从初始的 2.5+ nats 下降到 0.5 nats 以下，且没有回升迹象，高熵坍缩已经发生. 

**工程实现**: 

```python
def monitor_student_entropy(student_model, validation_states, threshold=0.5):
    """
    监控学生输出熵，检测高熵坍缩
    
    Args:
        validation_states: 固定验证集状态列表
        threshold: 熵的告警阈值(nats)
    
    Returns:
        avg_entropy: 平均熵
        alert: 是否触发告警
    """
    entropies = []
    student_model.eval()
    with torch.no_grad():
        for state in validation_states:
            logits = student_model(state).logits[:, -1, :]  # 最后一个位置的logits
            probs = F.softmax(logits, dim=-1)
            entropy = -(probs * torch.log(probs + 1e-10)).sum(dim=-1)
            entropies.append(entropy.item())
    
    avg_entropy = sum(entropies) / len(entropies)
    alert = avg_entropy < threshold
    
    return avg_entropy, alert
```

#### 1.3.2 监控指标二: KL 散度与准确率的相关性

在正常的 OPD 训练中，验证集上的 Reverse KL 散度和任务准确率应该呈现**负相关**: KL 越低，说明学生越贴近教师，准确率应该越高. 

在高熵坍缩场景中，这种相关性会**断裂甚至反转**. 原因在于: 学生通过坍缩到 Delta 分布来降低 KL，但这种降低是"虚假"的——它不是通过学会正确的分布来降低 KL，而是通过忽略教师的多路径信号来降低 KL. 因此，你会观察到: 

- 训练 KL 持续下降
- 验证 KL 也下降(因为学生在验证集上同样坍缩)
- 但验证准确率停滞或下降

**检测方法**: 每 50 个 step，计算验证集上 KL 散度和准确率之间的皮尔逊相关系数 $r$. 如果 $r$ 从负值跳变为接近 0 或正值，说明 KL 的下降不再代表能力的提升. 

#### 1.3.3 监控指标三: 教师-学生 JS 散度

JS 散度(Jensen-Shannon Divergence)是对称的、有界的($[0, \log 2]$)，它对学生在教师 support 之外的"捏造"行为和教师在学生 support 之外的"被忽略"行为给予均衡的惩罚. 

在高熵区域，如果学生坍缩到 Delta 而教师是均匀的: 
- Reverse KL = $\log V$(有界但不小)
- JS 散度会接近其最大值(因为两个分布几乎没有重叠)

因此，监控 JS 散度可以作为 Reverse KL 的补充. 如果 JS 散度持续上升而 Reverse KL 持续下降，这是一个强烈的高熵坍缩信号——说明学生正在以牺牲分布重叠度为代价来"欺骗"Reverse KL. 

$$
 D_{JS}(\pi_\theta \| \pi_T) = \frac{1}{2} D_{KL}\big(\pi_\theta \| M\big) + \frac{1}{2} D_{KL}\big(\pi_T \| M\big), \quad M = \frac{\pi_\theta + \pi_T}{2} \tag{16}
$$

#### 1.3.4 监控指标四: Top-1 置信度的增长速度

计算学生在验证集上的平均 Top-1 置信度(即最大概率值): 

$$
 \bar{C}_t = \frac{1}{N} \sum_{i=1}^{N} \max_{a} \pi_\theta(a | s_t^{(i)}) \tag{17}
$$

在健康的训练中，$\bar{C}_t$ 会从初始的 0.3-0.5 缓慢上升到 0.6-0.8. 但在高熵坍缩中，$\bar{C}_t$ 会快速攀升到 0.95 以上，表明模型在不合适的过度自信. 

**综合告警规则**: 

```
IF (entropy < 0.5) AND (top1_confidence > 0.95) AND (kl_accuracy_corr < 0.1)
THEN TRIGGER "High-Entropy Collapse Alert"
```

### 1.4 大厂填坑: 温度退火、熵正则与散度替换

当高熵坍缩被检测到后，业界已经发展出了三套成熟的修复方案. 这些方案不是相互排斥的——在实际的工业 pipeline 中，它们常常被组合使用. 

#### 1.4.1 方案一: 温度退火(Temperature Annealing)

**核心思想**: 通过动态降低 softmax 温度 $T$，在训练后期"锐化"教师分布，使得原本平坦的教师分布变得更有"主见"，从而消除高熵区域. 

温度退火的调度函数通常采用指数衰减: 

$$
 T(t) = T_0 \cdot \gamma^t \tag{18}
$$
其中 $T_0$ 是初始温度(通常 1.0-2.0)，$\gamma \in (0, 1)$ 是衰减率，$t$ 是训练 step. 

**物理机制**: 

教师的软化分布为 $\pi_T^{(T)}(a) \propto \exp(z_a^T / T)$. 当 $T > 1$ 时，分布被"拉平"; 当 $T < 1$ 时，分布被"锐化". 在训练初期使用较高的 $T$，可以让教师的多路径信号被充分传递; 在训练后期逐渐降低 $T$，使得教师即使在原本平坦的区域也表现出明确偏好. 

**参数配置表**: 

| 机构/论文 | $T_0$ | $\gamma$ | 退火步数 | 备注 |
|-----------|-------|----------|----------|------|
| Qwen3 (Alibaba, 2025) | 1.5 | 0.995 | 前 500 steps | 后期固定 $T=0.7$ |
| DeepSeek-V4 (2026) | 2.0 | 0.998 | 前 1000 steps | 配合 EMA 教师 |
| MiniLLM baseline | 1.0 | 0.99 | 全程 | 较激进方案 |
| Nemotron Cascade 2 | 1.2 | 0.996 | 前 800 steps | 自适应调整 |

**注意事项**: 温度退火是一把双刃剑. 如果退火过快($\gamma$ 太小)，教师分布过早锐化会导致学生被迫在尚未充分理解的情况下做出选择，反而加剧模式崩溃. Qwen3 的经验是: $T$ 从 1.5 降到 0.7 大约需要 300-500 个 step，$\gamma \approx 0.995$ 是一个安全的经验值. 

#### 1.4.2 方案二: 熵正则化(Entropy Regularization)

**核心思想**: 直接在损失函数中加入对学生输出熵的惩罚项，强制学生保持一定的输出多样性，防止坍缩到 Delta 分布. 

熵正则化的损失函数为: 

$$
 \mathcal{L}_{\text{total}} = \mathcal{L}_{OPD} - \lambda_{\text{ent}} \cdot H(\pi_\theta) \tag{19}
$$

其中 $\lambda_{\text{ent}} > 0$ 是熵正则系数. **注意这里是负号**: 因为我们希望最大化学生熵，而梯度下降是在最小化损失，所以用负号将最大化熵转化为最小化负熵. 

**物理机制**: 

OPD 损失 $\mathcal{L}_{OPD} = D_{KL}(\pi_\theta \| \pi_T)$ 本身就有隐式的熵惩罚(因为 $D_{KL} = -H(\pi_\theta) + \text{const}$ 在固定 $\pi_T$ 时)，但这个隐式惩罚的强度完全由教师分布决定. 在均匀教师区域，隐式惩罚的强度不够(如前面推导，最多只有 $\log V$). 显式的熵正则项 $\lambda_{\text{ent}} \cdot H(\pi_\theta)$ 提供了一个**不依赖于教师分布**的多样性约束——无论教师是否平坦，学生都被强制要求保持一定的熵. 

**参数配置表**: 

| 机构/论文 | $\lambda_{\text{ent}}$ | 熵目标值 | 实施方式 |
|-----------|----------------------|----------|----------|
| GKD (Generalized KD) | 0.1-0.3 | 动态 | 固定系数 |
| Qwen3 消融实验 | 0.15 | 1.5 nats | 配合温度退火 |
| DeepSeek-R1 OPD 阶段 | 0.05-0.2 | 无明确目标 | 随训练递减 |
| 学术界通用实践 | 0.1 | 无 | 作为默认配置 |

**工程技巧**: 熵正则系数 $\lambda_{\text{ent}}$ 不应固定不变. 在训练初期，学生模型本身输出就比较多样化，不需要强熵正则; 在训练中期，当检测到熵开始下降时，可以临时提高 $\lambda_{\text{ent}}$; 在训练后期，如果学生已经稳定，可以适当降低 $\lambda_{\text{ent}}$ 以允许模型在真正有把握的问题上表现果断. 

#### 1.4.3 方案三: JSD 替代 Reverse KL

**核心思想**: 放弃 Reverse KL，改用对称的 Jensen-Shannon 散度(JSD)，从根本上消除 Mode-seeking 在高熵区域的病理行为. 

基于上述分析，建立如下数学关系: 

$$
 D_{JS}(\pi_\theta \| \pi_T) = \frac{1}{2} D_{KL}\big(\pi_\theta \| M\big) + \frac{1}{2} D_{KL}\big(\pi_T \| M\big), \quad M = \frac{\pi_\theta + \pi_T}{2} \tag{20}
$$
**为什么 JSD 能缓解高熵坍缩**: 

1. **对称性**: JSD 对学生偏离教师和教师偏离学生给予同等惩罚. 当学生在均匀教师区域捏造 Delta 时，JSD 的第二项 $D_{KL}(\pi_T \| M)$ 会急剧增大——因为教师(均匀)与混合分布 $M$(几乎等于半个 Delta 加半个均匀)之间的差异是巨大的. 这给了学生一个强烈的"覆盖教师 support"的激励. 

2. **有界性**: JSD 的值域是 $[0, \log 2]$，无论分布差异多大都不会爆炸. 这提供了比 KL 散度更强的数值稳定性. 

3. **无零分母风险**: Reverse KL 中如果 $\pi_T(a) = 0$ 而 $\pi_\theta(a) > 0$，散度趋向无穷大. JSD 通过混合分布 $M$ 避免了这种除零风险. 

**代价**: JSD 的 Mode-seeking 特性比 Reverse KL 弱. 在非均匀区域，JSD 会迫使学生更多地覆盖教师的低概率区域，这可能导致一定的幻觉风险. 因此，JSD 通常不是完全替代 Reverse KL，而是作为"混合散度"使用: 

$$
 \mathcal{L}_{\text{mixed}} = \alpha \cdot D_{KL}(\pi_\theta \| \pi_T) + (1 - \alpha) \cdot D_{JS}(\pi_\theta \| \pi_T) \tag{21}
$$

**参数配置表**: 

| 机构/论文 | $\alpha$ (KL 权重) | JSD 权重 | 应用场景 |
|-----------|-------------------|----------|----------|
| SDPO 工程实现 | 0.5 | 0.5 | 代码生成(Rich Feedback) |
| DeepSeek-V4 数学蒸馏 | 0.7 | 0.3 | 数学推理(适度 Mode-seeking) |
| 通用保守配置 | 0.3 | 0.7 | 高熵敏感任务 |
| G-OPD 默认 | 1.0 | 0.0 | 标准配置 |

#### 1.4.4 综合修复方案与参数推荐

在实际工程中，单一方案往往不足以完全消除高熵坍缩. 以下是经过多个工业级项目验证的综合方案: 

```
Phase 1 (Steps 0-200):
  - 温度: T = 1.5 → 1.0 (线性)
  - 散度: Reverse KL (α = 1.0)
  - 熵正则: λ_ent = 0.0
  - 目标: 快速吸收教师信号

Phase 2 (Steps 200-600):
  - 温度: T = 1.0 → 0.7 (指数, γ=0.997)
  - 散度: Mixed (α = 0.7, KL权重)
  - 熵正则: λ_ent = 0.15
  - 目标: 防止高熵坍缩，保持多样性

Phase 3 (Steps 600+):
  - 温度: T = 0.7 (固定)
  - 散度: Reverse KL (α = 1.0)
  - 熵正则: λ_ent = 0.05 (递减)
  - 目标: 精准对齐，适度收敛
```

## Part 2: 捷径作弊(Shortcut Cheating)

### 2.1 现象: 表面模式的诱惑与 CoT 推理的崩坏

捷径作弊是 OPD 在 Chain-of-Thought(CoT)训练中最常见的失败模式. 与 high-entropy collapse 的"隐蔽性"不同，shortcut cheating 往往会在验证测试中以一种令人哭笑不得的方式暴露出来——模型的答案有时候是对的，但推理过程完全是胡扯; 更多时候，连答案也是错的，但推理过程中充满了对题目表面的拙劣模仿. 

#### 2.1.1 具体案例: 数学题中的数字复制与随机运算符

假设我们正在用 OPD 训练一个模型来解小学数学题. 教师模型(经过精心调优)在生成 CoT 时会展示完整的推理过程: 

> **题目**: 小明有 15 个苹果，给了小红 7 个，又买了 8 个. 现在小明有多少个苹果？
>
> **教师 CoT**: 初始数量 15，减去给出去的 7，得到 15 - 7 = 8. 然后加上新买的 8，得到 8 + 8 = 16. 所以答案是 16. 

在 OPD 训练的初期，学生模型诚实地尝试模仿这种推理风格. 但由于 7B 学生模型的能力有限，它在某些步骤上会感到困惑. 在某次 rollout 中，学生模型偶然生成了以下"推理": 

> **学生 CoT(作弊版)** : 15 个苹果，小红 7 个，买了 8 个. 15 + 7 + 8 = 30. 答案 30. 

这个推理是完全错误的——它把题目中的数字简单地相加，没有任何逻辑. 但如果教师模型在这个学生生成的轨迹上评估时，由于题目本身的数字特征(15, 7, 8)，教师在某些 token 位置上可能恰好也给了较高的概率(比如"15"、"+"、"7"这些 token 因为出现在题目中，在语言模型的先验下确实有更高的概率). 

更关键的是: 如果答案恰好蒙对了(比如另一道题"3 + 5 = ?"，学生直接复制"3 + 5 = 8")，教师模型在最终答案位置上的概率会很高. OPD 的 Reverse KL 会把这个"正确答案"的信号反向传播到整个推理链上——即使前面的推理步骤是胡扯. 

一旦学生模型在一次 rollout 中偶然发现了这种"复制数字 + 插入运算符"的捷径，On-Policy 采样的正反馈循环就开始运转: 

1. 学生在这次 rollout 中使用了捷径，获得了较低的 KL 损失(因为某些 token 确实得到了教师的高概率). 
2. 梯度更新让捷径路径上的 token 概率进一步增加. 
3. 下一次 rollout，学生更倾向于使用捷径. 
4. 捷径被用得越来越多，真实的推理路径被挤出采样空间. 
5. 最终，模型变成了一个"数字搬运工"——它看到题目中的数字就往上堆，偶尔蒙对，经常出错. 

#### 2.1.2 训练日志特征: 步级准确率与最终准确率的剪刀差

捷径作弊在训练日志上留下的指纹与 high-entropy collapse 截然不同. 关键在于区分**最终答案准确率**和**中间推理步骤准确率**. 

**特征一: 最终答案准确率尚可，但逐步验证(Step-by-Step Verification)准确率暴跌. **

假设我们有一个测试集，可以对模型的 CoT 进行逐行验证(例如，用 Python 解释器检查每一步算术是否正确). 在健康的训练中，步级准确率和最终答案准确率应该大致同步. 但在 shortcut cheating 场景下: 

- 最终答案准确率 = 65%(看起来还行)
- 步级逻辑正确率 = 23%(惨不忍睹)

这个巨大的剪刀差(65% vs 23%)是捷径作弊的决定性证据. 模型在"猜答案"方面表现不错(因为数字复制有时能蒙对)，但在"正确推理"方面一塌糊涂. 

**特征二: Loss 曲线出现"诡异的好转". **

在训练的前 100 个 step，Loss 可能下降得比较慢(因为学生还在努力学会真实推理). 但在某个临界点之后，Loss 突然加速下降——这不是因为学生"顿悟"了，而是因为它发现了捷径. 捷径让学生的输出与教师在表面 token(数字、运算符)上的分布更容易对齐，从而降低了 KL. 

**特征三: 输出长度的分布发生突变. **

真实的 CoT 通常有相对稳定的平均长度(比如 80-120 个 token). 但在 shortcut cheating 发生后，输出长度的分布会分裂成两个峰: 一个峰对应"完整推理"(较长，但越来越少)，另一个峰对应"捷径答案"(很短，越来越多). 最终，短峰完全吞并长峰. 

**特征四: 教师分布在捷径 token 上的概率被人为抬高. **

这是一个容易被忽视的信号. 由于学生在 On-Policy 采样中越来越频繁地生成捷径轨迹，教师模型在评估这些轨迹时，会看到大量"数字复制 + 随机运算符"的模式. 即使教师本身不认可这种模式，由于自回归语言模型的上下文敏感性，教师在前缀中已经出现了错误运算符的情况下，可能会"被迫"给后续的运算符较高的概率(为了维持语法连贯性). 这形成了一个**教师分布的污染效应**——教师本应是"正确答案"的来源，但它被学生自己的捷径轨迹"带偏"了. 

### 2.2 数学分析: On-Policy 采样的放大效应与复杂度不对称

捷径作弊的数学根因比 high-entropy collapse 更深层. 它不仅涉及 Reverse KL 的性质，还涉及 On-Policy 采样的动力学、序列 Loss 的分解结构，以及 Kolmogorov 复杂度在蒸馏框架下的不对称性. 

#### 2.2.1 On-Policy 采样如何放大捷径

在 OPD 中，训练数据(即 rollout 轨迹)是由学生模型自己生成的: 

$$
 s_t \sim \pi_\theta, \quad y_t \sim \pi_\theta(\cdot | s_t) \tag{22}
$$
这与 SFT 的 Off-Policy 采样($s_t \sim \pi_T$)形成鲜明对比. Off-Policy 采样的一个巨大优势是: 训练数据的质量不依赖于学生当前的能力——无论学生多弱，它都在学习教师生成的优质轨迹. 

On-Policy 采样则把这个前提翻转了: **学生只能学习它自己能生成的轨迹**. 这带来了一个根本性的困境: 

- 如果学生当前的能力只能生成低质量轨迹，那么教师对这些低质量轨迹的评估信号也会被纳入训练. 
- 如果这些低质量轨迹中包含"表面模式匹配"(shortcut)，而表面模式恰好能让某些 token 获得教师的中等概率，那么梯度会强化这些表面模式. 
- 表面模式一旦被强化，学生在后续 rollout 中就更频繁地采样它们. 
- 正反馈循环完成. 

我们可以用数学来严格描述这个放大效应. 设"捷径策略"为 $\pi_{\text{shortcut}}$，"真实推理策略"为 $\pi_{\text{real}}$. 在某个状态 $s_t$ 上，两种策略生成不同 token 的概率为 $\pi_{\text{shortcut}}(a_{\text{sc}}) = p_{\text{sc}}$ 和 $\pi_{\text{real}}(a_{\text{real}}) = p_{\text{real}}$. 

在 On-Policy 采样中，$a_{\text{sc}}$ 被采样的概率就是 $p_{\text{sc}}$. 如果 $a_{\text{sc}}$ 被采样后，OPD Loss 给出了正向反馈(即 KL 散度较低)，那么梯度更新会进一步增加 $p_{\text{sc}}$. 用递推关系表示: 

$$
 p_{\text{sc}}^{(t+1)} = p_{\text{sc}}^{(t)} + \eta \cdot \frac{\partial \mathcal{L}_{OPD}}{\partial p_{\text{sc}}} \tag{23}
$$

如果 $\frac{\partial \mathcal{L}_{OPD}}{\partial p_{\text{sc}}} < 0$(即增加 $p_{\text{sc}}$ 会降低 Loss)，那么 $p_{\text{sc}}$ 会单调递增. 关键问题是: 为什么捷径 token 更容易获得负梯度？

#### 2.2.2 Loss 分解: OPD 缺乏跨位置耦合项

OPD 的损失函数是按 token 位置分解的: 

$$
 \mathcal{L}_{OPD} = \frac{1}{T} \sum_{t=1}^{T} D_{KL}\big(\pi_\theta(\cdot | s_t) \,\big\|\, \pi_T(\cdot | s_t)\big) \tag{24}
$$
这个分解揭示了一个致命的结构性缺陷: **损失函数中没有跨位置的耦合项**. 每一个 token 位置的 KL 散度被独立计算、独立平均. 模型在第 $t$ 步的梯度更新，完全不考虑第 $t+1$ 步、第 $t+2$ 步的因果后果. 

这意味着什么？

假设学生在第 10 步生成了一个错误的运算符"$\times$"(应该是"$+$"). 由于第 10 步之前的前缀是"15 - 7 = "，教师模型在第 10 步看到"8"这个中间结果，可能会给"$\times$"一个非零概率(比如 0.05)，因为语言模型知道"8"后面可以接任何运算符. 因此，第 10 步的 KL 散度可能并不高——学生对"$\times$"的概率是 0.6，教师给的是 0.05，KL 贡献为 $0.6 \times (\log 0.6 - \log 0.05) \approx 0.6 \times 2.48 = 1.49$. 

但如果有一个**跨位置耦合项**——比如"第 10 步的错误导致第 15 步的答案错误"——那么"$\times$"的选择应该受到强烈惩罚. 然而 OPD 没有这样的项. 第 15 步的答案错误确实会产生较高的 KL(因为教师会给正确答案高概率)，但这个惩罚被"平均"分摊到了 15 个 token 上，而且梯度无法直接追溯到第 10 步的错误运算符——它只能看到第 15 步的答案不对，但不知道是因为第 10 步错了. 

这就是 shortcut cheating 的数学根源: **OPD 的逐 token KL 散度只能惩罚局部的不匹配，无法惩罚全局的因果错误链**. 捷径策略利用了这一缺陷——它在每一步都选择"看起来还行"的 token(因为表面模式与教师分布有一定重叠)，但在全局上构成了一个完全错误的推理链. 

#### 2.2.3 Kolmogorov 复杂度视角: 捷径的定义与不可压缩性

从 Kolmogorov 复杂度(Kolmogorov Complexity)的角度，我们可以给"捷径"一个严格的数学定义. 

给定一个问题 $x$ 和一个推理链 $y$，设 $K(y | x)$ 为在已知 $x$ 的条件下生成 $y$ 的最短程序长度. 真实推理链 $y_{\text{real}}$ 通常具有较低的 Kolmogorov 复杂度(相对于 $x$)，因为它遵循通用的数学规则: 

$$
 K(y_{\text{real}} | x) \approx K(\text{math rules}) + K(\text{problem-specific params}) \tag{25}
$$

而捷径链 $y_{\text{shortcut}}$ 虽然表面上看起来很短，但它的 Kolmogorov 复杂度实际上更高——因为它需要编码"如何从 $x$ 中提取数字并随机组合"这种非通用的启发式规则: 

$$
 K(y_{\text{shortcut}} | x) \approx K(\text{heuristic pattern matcher}) + K(\text{random choices}) \tag{26}
$$
等等，这好像与直觉相反. 捷径看起来应该更简单才对. 让我们重新思考. 

实际上，从描述复杂度的角度，捷径链 $y_{\text{shortcut}}$ 的**生成程序**确实比真实推理链更简单——因为它不需要实现完整的数学推理引擎，只需要实现"复制数字 + 插入随机运算符". 但问题是: **这个简单程序生成的输出，在验证时几乎总是错的**. 真正重要的是"生成正确答案的程序"的复杂度. 

换一个角度: 设 $y^*$ 是正确答案. 真实推理链 $y_{\text{real}}$ 是 $y^*$ 的一个"可验证构造"——每一中间步骤都可以被独立验证. 捷径链 $y_{\text{shortcut}}$ 虽然也可能偶尔命中 $y^*$，但它无法被独立验证. 

在 OPD 框架下，教师模型 $\pi_T$ 的分布实际上编码了"验证信号"——教师在每一步都给出了"哪些 token 是可接受的". 然而，这种验证是局部的、逐 token 的. 它无法验证"整个推理链是否在逻辑上一致". 

**核心定理(非正式表述)** : 

如果存在一个低复杂度的表面模式 $c$，使得对于大量问题 $x$，$c(x)$ 在教师的逐 token 评估中获得非零概率(即 $\pi_T(c_t | x, c_{<t}) > 0$ 对所有 $t$)，而真实推理链 $r(x)$ 的复杂度 $K(r) \gg K(c)$，那么在 On-Policy 采样和逐 token KL 的训练动力学下，$c$ 会系统性地排挤 $r$. 

**为什么？** 因为 $c$ 的每个 token 都"便宜"——它只需要匹配表面统计特征，不需要执行深层逻辑. 在梯度优化的竞争中，"便宜"的策略总是胜出，除非有额外的约束(如过程验证器)来惩罚它的全局错误. 

#### 2.2.4 教师概率的"污染效应"

在自蒸馏场景(OPSD、SDFT)中，shortcut cheating 有一个特别阴险的变体: 教师分布被学生的捷径轨迹"污染". 

具体机制如下: 

1. 学生生成了一条包含捷径的轨迹 $y_{\text{sc}}$. 
2. 教师模型(同一个模型，带有特权上下文)评估这条轨迹. 由于自回归特性，教师看到"15 + 7 + 8"这样的前缀后，会倾向于"合理化"它——给"= 30"较高的概率，因为"15 + 7 + 8 = 30"在语法和算术上是自洽的. 
3. 教师因此给了最终答案"30"一个较高的概率(比如 0.7). 
4. OPD 的 KL 散度计算中，最终答案位置的匹配度很好，损失较低. 
5. 梯度反向传播到整个序列，强化了"15 + 7 + 8 = 30"这条路径. 
6. 下一次 rollout，学生更可能生成类似的捷径. 
7. 教师再次被暴露在捷径前缀下，进一步强化对捷径的认可. 

这是一个典型的**正反馈循环**，而且教师在循环中扮演了"共谋者"的角色——它不是故意教坏学生，而是被学生生成的轨迹"带偏"了. 在数学上，这等价于教师分布 $\pi_T$ 不再是一个固定的目标，而是一个**依赖于学生当前策略的随机变量**: 

$$
 \pi_T^{\text{effective}}(a | s_t) = \pi_T(a | s_t, y_{<t}^{\text{sc}}) \tag{27}
$$

其中 $y_{<t}^{\text{sc}}$ 是学生生成的捷径前缀. 这个"有效教师"与"理想教师"之间的偏差，随着 shortcut cheating 的加剧而指数增长.

### 2.3 检测: 从步级验证到注意力可视化

Shortcut cheating 的检测比 high-entropy collapse 更依赖任务特定的验证工具. 因为 shortcut 的本质是"局部正确、全局错误"，你需要有能力评估推理链的全局正确性. 

#### 2.3.1 检测方法一: 步级准确率与最终准确率的差距监控

这是最直接的检测方法. 对于任何可以被分解为多个验证步骤的任务(数学推理、代码执行、科学计算)，计算两个指标: 

- **$A_{\text{final}}$**: 最终答案的准确率. 

- **$A_{\text{step}}$**: 中间推理步骤的准确率(例如，每一步算术是否正确，每一步逻辑推导是否合理). 

定义**推理一致性指数(Reasoning Consistency Index, RCI)** : 

$$
 \text{RCI} = \frac{A_{\text{step}}}{A_{\text{final}}}} \tag{28}
$$
**正常范围**: 在健康的 CoT 训练中，RCI 应该接近 1.0(比如 0.85-1.0). 这意味着模型答对的题目，其推理过程也大概率是正确的; 偶尔答对的题目可能是蒙对的，但比例不高. 

**异常信号**: 
- RCI < 0.6: 强烈怀疑 shortcut cheating. 模型大量依赖猜答案. 
- RCI < 0.4: 几乎可以确定 shortcut cheating 已经发生. 
- RCI 从 0.9 快速下降到 0.5: shortcut cheating 正在发生，需要立即干预. 

**工程实现**: 

对于数学任务，可以用 Python 的 `sympy` 或简单的正则表达式提取每一步的算式并验证. 对于代码任务，可以在每个断点检查变量状态. 对于逻辑推理任务，可以使用 SAT solver 或手动编写的验证规则. 

```python
def compute_rci(model, validation_set, verifier):
    """
    计算推理一致性指数 RCI
    
    Args:
        model: 待评估的学生模型
        validation_set: 验证集，每个样本包含问题 x 和可验证的中间步骤
        verifier: 验证函数，输入推理链 y，返回 (final_correct, step_correct_list)
    
    Returns:
        rci: 推理一致性指数
        a_final: 最终答案准确率
        a_step: 平均步级准确率
    """
    total_samples = len(validation_set)
    correct_final = 0
    correct_steps_total = 0
    total_steps = 0
    
    for sample in validation_set:
        x = sample['question']
        y = model.generate(x)  # 生成推理链
        
        final_correct, step_correct_list = verifier(y)
        correct_final += int(final_correct)
        correct_steps_total += sum(step_correct_list)
        total_steps += len(step_correct_list)
    
    a_final = correct_final / total_samples
    a_step = correct_steps_total / total_steps if total_steps > 0 else 0
    rci = a_step / a_final if a_final > 0 else 0
    
    return rci, a_final, a_step
```

#### 2.3.2 检测方法二: 注意力可视化与"数字复制"模式

Transformer 的注意力机制为检测 shortcut 提供了一个独特的窗口. 在 shortcut cheating 中，模型倾向于直接从问题陈述(question tokens)复制数字和运算符到答案区域，而不是在中间推理步骤中进行渐进式变换. 

通过可视化 cross-attention(如果模型有 encoder-decoder 结构)或 decoder 的自注意力模式，可以观察到以下"指纹": 

- **注意力"跳步"**: 在生成最终答案时，注意力权重直接聚焦于问题中的数字 token，而不是中间推理步骤的 token. 

- **数字-数字强耦合**: 问题中的数字 token 和答案中的数字 token 之间存在异常高的注意力权重. 

- **推理步骤被"跳过"**: 中间推理 token 的注意力权重很低，表明模型没有真正"阅读"自己的推理过程. 

**定量指标**: 定义**注意力跳步指数(Attention Skip Index, ASI)** : 

$$
 \text{ASI} = \frac{\sum_{i \in \text{question}} \sum_{j \in \text{answer}} \alpha_{i,j}}{\sum_{i \in \text{reasoning}} \sum_{j \in \text{answer}} \alpha_{i,j}} \tag{29}
$$

其中 $\alpha_{i,j}$ 是 token $i$ 到 token $j$ 的注意力权重. ASI 衡量"答案直接依赖问题"与"答案依赖推理过程"的比例. 

**正常范围**: ASI 应该适中(比如 0.2-0.5)，表明答案既参考了问题本身，也参考了中间推理. 

**异常信号**: ASI > 1.0 表明答案更依赖问题原文而不是推理过程——这是 shortcut copying 的直接证据. 

#### 2.3.3 检测方法三: 分布外(OOD)泛化测试

Shortcut 策略通常是"过拟合于训练数据的统计表面特征"的. 因此，它在分布内(In-Distribution)数据上可能表现尚可(因为训练数据的数字范围、运算符类型、句式结构都被它记住了)，但在分布外(Out-of-Distribution)数据上会彻底崩溃. 

**OOD 测试设计**: 

- **数字范围偏移**: 如果训练数据中的数字主要是 1-100，OOD 测试使用 1000-10000 的大数字. 

- **运算符替换**: 把"+"换成"加"，把"$\times$"换成"乘以"，看模型是否还能正确解析. 

- **无关信息插入**: 在问题中加入无关的干扰数字，看模型是否能区分相关数字和干扰数字. 

- **问题格式翻转**: 把"A 比 B 多 5"改成"B 比 A 少 5"，测试模型是否真正理解关系. 

**判据**: 如果模型在分布内准确率是 70%，但在 OOD 上骤降到 20% 以下，强烈表明 shortcut cheating. 

#### 2.3.4 检测方法四: KL 散度的" token 级解剖"

在正常训练中，KL 散度在不同类型 token(数字、运算符、逻辑连接词、单位词)上的分布应该是相对均衡的. 在 shortcut cheating 中，KL 散度会呈现出异常的分布: 

- **数字 token 的 KL 极低**: 因为模型直接从问题复制数字，与教师的数字分布高度重合. 

- **运算符 token 的 KL 较高但无规律**: 因为模型随机插入运算符，教师对这些运算符的分布也很困惑. 

- **逻辑连接词("因此"、"所以"、"因为")的 KL 较高**: 因为模型没有真正理解逻辑关系，这些连接词的使用与教师不一致. 

通过按 token 类型分解 KL 散度，可以精确定位 shortcut 的表现形式. 

### 2.4 大厂填坑: 过程监督、步级验证器与多样性采样

针对 shortcut cheating，业界已经发展出了三套成熟的对抗策略. 这些策略的核心思想都是同一个: **打破 OPD 逐 token 独立优化的局限，引入跨步骤的全局约束**. 

#### 2.4.1 方案一: 过程奖励模型(PRM，Process Reward Model)

**核心思想**: 不再只在序列末尾给出一个奖励信号，而是在推理链的每一个关键步骤都提供一个验证信号. 这个信号告诉我们"到这一步为止，推理是否正确". 

PRM 的形式化定义: 

$$
 R_{\text{process}}(s_t) = \begin{cases} +1 & \text{if reasoning up to step } t \text{ is correct} \\ -1 & \text{if a mistake has been made at or before step } t \end{cases} \tag{30}
$$
在 OPD 框架中融入 PRM，通常通过修改目标函数实现: 

$$
 \mathcal{L}_{OPD+PRM} = \mathbb{E}_{s_t \sim \pi_\theta} \left[ D_{KL}(\pi_\theta \| \pi_T) - \lambda_{PRM} \cdot R_{\text{process}}(s_t) \cdot \log \pi_\theta(a_t | s_t) \right] \tag{31}
$$
此式描述了变量之间的定量关系，其物理意义将在下文详细阐述. 
**物理机制**: 

当 $R_{\text{process}}(s_t) = -1$(即当前推理链已经出现错误)时，第二项变为 $+ \lambda_{PRM} \cdot \log \pi_\theta(a_t | s_t)$. 由于 $\log \pi_\theta < 0$，这一项实际上是**正的惩罚**——它要求模型降低当前路径的概率. 这直接把"全局错误信号"注入了局部 token 的优化中. 

**训练 PRM 的方法**: 

1. **人工标注**: 雇佣人类标注员对大量 CoT 进行逐步验证. 这是最准确但最昂贵的方法. 

2. **自动构造**: 用 Python 解释器或形式化验证工具自动检查每一步. 只适用于可被程序验证的领域(数学、代码). 

3. **迭代自举(Bootstrapping)** : 先用一个弱的 PRM 过滤数据，训练一个更强的模型，再用更强模型生成更多数据来训练更强的 PRM. OpenAI 的 o1 和 DeepSeek-R1 都采用了类似策略. 

**参数配置表**: 

| 机构/论文 | PRM 类型 | $\lambda_{PRM}$ | 验证粒度 | 备注 |
|-----------|----------|----------------|----------|------|
| OpenAI o1 (推测) | Bootstrapped NN | 0.5-1.0 | 每步 | 核心组件 |
| DeepSeek-R1 | 规则验证器 | 1.0 | 关键步骤 | 数学+代码 |
| Kimi-K1.5 | 混合(规则+模型) | 0.3-0.7 | 每步 | 长推理链 |
| 学术基准配置 | 规则验证器 | 0.5 | 每步 | 推荐默认值 |

#### 2.4.2 方案二: 步级验证器(Step-Level Verifier)与拒绝采样

**核心思想**: 在训练循环中引入一个显式的"步级验证器"，如果某条 rollout 轨迹在任何一步被判定为错误，则整条轨迹被丢弃或降权，不参与梯度更新. 

这是比 PRM 更"硬核"的方案——它不做微妙的奖励调整，而是直接"枪毙"错误轨迹. 

**算法流程**: 

```
For each training step:
  1. 学生生成 N 条轨迹 {y^(1), ..., y^(N)} (On-Policy 采样)
  2. 对每条轨迹 y^(i)，用步级验证器 V 逐行检查
  3. 如果 V(y^(i)) = False(任何一步错误)，则丢弃该轨迹
  4. 只保留 V(y^(i)) = True 的轨迹进行 OPD 蒸馏
  5. 如果保留率 < threshold，启用回退保险(SFT)
```

**与 SCOPE 的联系**: 

这个方案实际上就是 SCOPE(Selective Feedback)的一种具体实例. SCOPE 的三维质量向量($H$, Align, RM-Conf)中的"RM-Conf"在这里被替换为"步级验证器的二元判决". 如果验证器判定某步错误，该步的"质量"为零，被门控拦截. 

**代价**: 

拒绝采样的最大问题是**样本效率低**. 在训练初期，学生模型生成的轨迹可能 80% 都有错误，如果全部丢弃，训练数据就所剩无几了. 因此，拒绝采样通常需要配合"课程学习"(Curriculum Learning)——初期用简单题目(验证通过率高)，逐渐过渡到困难题目. 

**参数配置**: 

| 超参数 | 推荐值 | 说明 |
|--------|--------|------|
| 每次采样轨迹数 N | 8-16 | 保证过滤后有足够数据 |
| 最小保留率 | 10%-20% | 低于此阈值启用回退保险 |
| 回退保险权重 $\lambda_{\text{fallback}}$ | 0.3-0.5 | 与 SCOPE 一致 |
| 课程难度增速 | 每 100 steps 提升 10% | 防止过早进入高难度 |

#### 2.4.3 方案三: 多样性正则化(Diversity Regularization)

**核心思想**: Shortcut cheating 之所以猖獗，是因为 On-Policy 采样让学生反复陷入同一条捷径路径. 如果我们强制学生在每次 rollout 中探索不同的路径，就可以打破捷径的正反馈循环. 

多样性正则化通过惩罚输出分布的"尖锐度"来实现这一目标: 

$$
 \mathcal{L}_{\text{div}} = -\lambda_{\text{div}} \sum_{a \in \mathcal{V}} \pi_\theta(a | s_t)^2 \tag{32}
$$
注意 $\sum_a \pi_\theta(a)^2$ 是分布的**基尼指数**(Gini Index)或**浓度指数**——它度量了概率质量集中在少数 token 上的程度. 对于均匀分布，$\sum_a \pi_a^2 = 1/V$; 对于 Delta 分布，$\sum_a \pi_a^2 = 1$. 

**物理机制**: 

最小化 $-\sum_a \pi_a^2$(即最大化 $\sum_a \pi_a^2$ 的负值)等价于**最大化分布的"分散度"**. 这与熵正则 $\max H(\pi)$ 的方向一致，但数学形式更简单、梯度计算更高效. 

多样性正则化的一个变体是**互信息最大化**: 鼓励学生模型在不同 rollout 之间生成多样化的轨迹. 具体实现是: 对同一个问题采样多条轨迹，惩罚轨迹之间的相似度. 

$$
 \mathcal{L}_{\text{MI}} = -\lambda_{\text{MI}} \cdot \text{Var}(\{y^{(i)}\}_{i=1}^N) \tag{33}
$$

**参数配置表**: 

| 机构/论文 | 正则类型 | 系数 | 实施方式 |
|-----------|----------|------|----------|
| 学术界通用 | Gini 惩罚 | $\lambda_{\text{div}} = 0.05$-0.2 | 逐 token |

#### 2.4.4 综合修复方案

在实际工程中，对抗 shortcut cheating 需要多管齐下: 

```

  - 对包含 shortcut 的数据进行人工清洗或标注

  - 每 50 steps 计算 RCI
  - 监控 OOD 准确率

```

## Part 3: 长链蒸发(Long-Chain Evaporation)

### 3.1 现象: 超长推理链中模型"思考变懒"

长链蒸发是 OPD 在处理复杂推理任务时最致命的失败模式之一. 它的核心现象可以用一句话概括: **模型明明有能力进行长程推理，但训练后它越来越不愿意这么做了**. 在需要 100 步以上推理的数学竞赛题、复杂代码调试、多步逻辑谜题等任务中，模型会系统性地生成更短的推理链，跳过关键中间步骤，或者直接给出无根据的结论. 

#### 3.1.1 具体案例: 复杂数学证明中的"跳步"与"放弃"

假设我们训练一个模型来证明以下命题: 

> 证明: 对于所有正整数 $n$，$n^3 - n$ 能被 6 整除. 

标准的证明需要以下步骤: 
1. 因式分解: $n^3 - n = n(n^2 - 1) = n(n-1)(n+1)$
2. 观察: $(n-1), n, (n+1)$ 是三个连续整数
3. 引理1: 三个连续整数中必有一个是 2 的倍数
4. 引理2: 三个连续整数中必有一个是 3 的倍数
5. 结论: 因此乘积同时被 2 和 3 整除，即被 6 整除

在健康的 OPD 训练中，教师模型(一个强推理模型)会生成包含上述所有步骤的完整 CoT，学生模型逐渐学会这种逐步展开的推理风格. 

但在长链蒸发场景中，训练进行到中期后，学生模型的输出开始退化: 

**退化阶段一(步骤省略)** : 
> $n^3 - n = n(n-1)(n+1)$. 这是三个连续整数，所以能被 6 整除. 

模型跳过了引理1和引理2的明确陈述. 它"知道"结论是对的，但不愿意展开论证. 

**退化阶段二(直接断言)** : 
> 因为 $n^3 - n = n(n-1)(n+1)$ 包含因子 2 和 3，所以可被 6 整除. 

这里模型连"三个连续整数"这个关键观察都省略了，直接跳到了结论. 

**退化阶段三(无根据结论)** : 
> $n^3 - n$ 总是偶数且可被 3 整除，因此答案成立. 

模型完全丧失了结构化的推理能力，只剩下孤立的断言. 

在验证集上，这种现象表现为: **简单题目(需要 10-30 步推理)的准确率保持稳定，但复杂题目(需要 50+ 步推理)的准确率出现断崖式下跌**. 更微妙的是，即使模型偶尔答对了复杂题目，它的推理链也往往是不完整的——它可能恰好"猜"对了关键的中间步骤，而不是真正理解了证明结构. 

#### 3.1.2 训练日志特征: 平均长度下降与复杂题准确率断崖

长链蒸发在训练日志上留下了一组独特的"指纹". 

**特征一: 平均生成长度单调递减. **

在正常的 CoT 训练中，模型的平均输出长度应该先上升(学会展开推理)后稳定(形成成熟的推理风格). 但在长链蒸发中，长度从一开始就呈现单调递减趋势. 

$$
 \bar{L}_t = \frac{1}{N} \sum_{i=1}^{N} |y^{(i)}| \tag{34}
$$
如果 $\bar{L}_t$ 从初始的 200 tokens 持续下降到 80 tokens 以下，且不再回升，长链蒸发已经发生. 

**特征二: 简单题与复杂题的准确率"分叉". **

将验证集按所需推理步数分层: 
- 简单层: 需要 1-20 步推理
- 中等层: 需要 21-50 步推理
- 困难层: 需要 51+ 步推理

在健康训练中，三层的准确率应该同步提升(虽然绝对值有差距). 在长链蒸发中: 
- 简单层准确率: 持续提升(模型学会了短链推理)
- 中等层准确率: 初期提升后停滞
- 困难层准确率: 初期短暂提升后**断崖式下跌**

这种"分叉"现象是长链蒸发的决定性证据. 模型不是在"普遍变笨"——它在简单任务上的表现可能很好——而是在"选择性逃避"复杂任务. 

**特征三: 长链中的"提前终止"频率上升. **

统计模型在生成长链时提前输出终止符(如"所以答案是"、"证毕"、"```")的频率. 在长链蒸发中，这个频率会显著上升——模型在推理进行到一半时就急于结束. 

**特征四: 教师分布在长链后段的熵异常升高. **

这是一个深层信号. 由于学生生成的长链质量越来越差，教师模型在评估这些劣质长链的后段时，会变得越来越困惑(因为前缀已经偏离了正确轨道). 因此，教师在长链后段的输出熵 $H(\pi_T)$ 会异常升高. 这与高熵坍缩不同——那里的高熵来自教师本身的不确定性，而这里的高熵来自学生制造的"垃圾前缀"对教师的污染.

### 3.2 数学分析: 误差传播的指数律与 Reverse KL 的长度惩罚

长链蒸发的数学根因来自两个独立但相互加强的机制: **误差沿推理链的指数传播**，以及 **Reverse KL 对长链施加的隐式长度惩罚**. 

#### 3.2.1 误差传播的指数衰减律

考虑一个长度为 $L$ 的推理链. 假设在每一步，模型有独立的概率 $p_0$ 做出正确推理(即生成与教师一致的 token). 那么整个推理链完全正确的概率为: 

$$
 P(\text{entire chain correct}) = p_0^L \tag{35}
$$

这是一个**指数衰减**函数. 即使 $p_0$ 很高(比如 0.95)，当 $L = 50$ 时: 

$$
 P(\text{correct}) = 0.95^{50} \approx 0.077 \tag{36}
$$
这意味着: 如果一个模型在每一步都有 95% 的概率做对，它在 50 步推理链中完全做对的概率只有 7.7%. 这不是因为模型"弱"，而是因为指数律的冷酷——微小误差的累积是摧毁性的. 

更现实的模型是**误差传播模型**: 每一步的错误不仅降低当前步的正确率，还会通过状态 $s_t$ 影响后续步骤. 设第 $t$ 步的条件正确率为: 

$$
 P(\text{correct at } t | \text{correct up to } t-1) = p_0 \cdot \gamma \tag{37}
$$

其中 $\gamma \in (0, 1)$ 是**错误传播衰减因子**. 那么: 

$$
 P(\text{correct chain of length } L) = p_0^L \cdot \gamma^{L(L-1)/2} \tag{38}
$$
这个衰减比纯指数更快(因为指数上有二次项 $L^2$). 这意味着在长链中，**后期步骤几乎必然出错**——不是因为后期步骤本身更难，而是因为前期步骤的微小误差被不断放大. 

#### 3.2.2 Reverse KL 对长链的隐式惩罚

现在我们来分析 OPD 的损失函数如何与长链的误差传播相互作用. 

OPD 的总损失是逐 token 的平均: 

$$
 \mathcal{L}_{OPD} = \frac{1}{T} \sum_{t=1}^{T} D_{KL}\big(\pi_\theta(\cdot | s_t) \,\big\|\, \pi_T(\cdot | s_t)\big) \tag{39}
$$

注意这里的**归一化因子是 $1/T$**，即序列长度 $T$. 这个平均操作看似公平，但它隐藏了一个关键的不对称性. 

假设我们有两个候选策略: 
- **策略 A(长链)** : 生成完整的 $L$ 步推理链. 前 $L/2$ 步正确，后 $L/2$ 步因误差传播而错误. 

- **策略 B(短链)** : 生成 $L/2$ 步的简化推理，然后直接给出结论. 中间跳过了关键步骤. 

对于策略 A，总损失为: 

$$
 \mathcal{L}_A = \frac{1}{L} \left[ \sum_{t=1}^{L/2} \underbrace{D_{KL}^{(t)}}_{\text{small}} + \sum_{t=L/2+1}^{L} \underbrace{D_{KL}^{(t)}}_{\text{large}} \right] \tag{40}
$$
如式 (40) 所示，前 $L/2$ 步的 KL 很小(因为推理正确，学生贴近教师)，但后 $L/2$ 步的 KL 很大(因为误差传播导致学生偏离教师). 假设前半段平均 KL = 0.1，后半段平均 KL = 2.0，代入得: 

$$
 \mathcal{L}_A = \frac{1}{L} \left[ \frac{L}{2} \times 0.1 + \frac{L}{2} \times 2.0 \right] = 0.05 + 1.0 = 1.05 \tag{41}
$$
对于策略 B，总损失为: 

$$
 \mathcal{L}_B = \frac{1}{L/2} \sum_{t=1}^{L/2} D_{KL}^{(t)} \tag{42}
$$
如式 (42) 所示，策略 B 只生成 $L/2$ 个 token，所以它只在前 $L/2$ 步有损失. 假设它在这 $L/2$ 步中的平均 KL = 0.8(比策略 A 的前半段高，因为它跳过了关键步骤，导致局部不匹配增加)，代入得: 

$$
 \mathcal{L}_B = \frac{1}{L/2} \times \frac{L}{2} \times 0.8 = 0.8 \tag{43}
$$
**关键比较**: 如式 (41) 和式 (43) 所示，$\mathcal{L}_B = 0.8 < \mathcal{L}_A = 1.05$. 

策略 B(短链)的总损失**低于**策略 A(长链)！即使策略 B 的每一步局部匹配度都不如策略 A 的前半段，但由于它避免了后半段误差传播带来的灾难性 KL 惩罚，它在总损失上胜出. 

这就是长链蒸发的核心数学机制: **OPD 的逐 token 平均 KL 损失，天然地惩罚长链**. 不是因为长链本身不好，而是因为长链中不可避免地包含高误差区域，而这些区域的 KL 惩罚被平均分摊到了整个序列上，使得"缩短链条、逃避复杂区域"成为更优的优化策略. 

#### 3.2.3 长度惩罚与 OPD 目标的结构性冲突

上面的分析揭示了一个更深层的结构性矛盾: 

**OPD 的目标函数(最小化平均 KL)与复杂推理任务的需求(生成长而正确的推理链)在数学上是冲突的. **

复杂推理任务需要长链，因为: 
- 很多逻辑步骤无法压缩(比如数学证明中的引理陈述). 
- 跳过步骤会导致推理漏洞(即使最终答案正确，过程也是不可接受的). 
- 长链提供了更多的验证点(每一步都可以被独立检查). 

但 OPD 的目标函数"不喜欢"长链，因为: 
- 长链中的每一步都是独立的"失败机会". 
- 任何一步的失误都会通过状态传播放大后续步骤的 KL. 
- 平均归一化使得"减少 token 数量"成为降低总损失的直接途径. 

这个矛盾不是可以通过简单的超参数调整解决的——它根植于 OPD 目标函数的基本形式. 要修复它，必须改变目标函数的结构(如引入长度归一化或分段蒸馏). 

#### 3.2.4 从强化学习视角看: 长链的信用分配灾难

长链蒸发还有一个来自强化学习(RL)视角的解释. 在 RL 术语中，长链推理面临的是臭名昭著的**信用分配问题(Credit Assignment Problem)** : 模型在 100 步推理后得到了一个最终奖励(答案对或错)，但它无法确定这个奖励应该归因于哪一步. 

OPD 虽然通过逐 token 的 KL 散度缓解了信用分配问题(每个 token 都有独立的监督信号)，但它引入了一个新的、更微妙的问题: **逐 token 的信号质量沿链衰减**. 

具体而言: 
- 在推理链的前段($t = 1, 2, \dots$)，状态 $s_t$ 主要由问题陈述决定，与学生的生成关系不大. 因此教师评估这些状态时，给出的分布信号是高质量、低噪声的. 
- 在推理链的后段($t = L/2, \dots, L$)，状态 $s_t$ 已经包含了学生之前生成的所有 token. 如果学生在任何 earlier step 犯了错误，$s_t$ 就是一个"污染状态". 教师在这个污染状态上的分布，不再是"正确答案应该是什么"，而是"给定这个错误前缀，最合理的延续是什么". 

因此，长链后段的 KL 散度惩罚了一个**错误的东西**: 它不是惩罚"学生没有学会正确答案"，而是惩罚"学生没有学会如何在错误的前提下自圆其说". 而后者正是 shortcut cheating 所需要的技能！

**长链蒸发与捷径作弊的耦合**: 

在长链的后段，学生面临两个选择: 
1. 继续诚实地尝试推理(但状态已经被污染，教师信号也是污染的，KL 很高). 
2. 采用捷径策略(如直接断言结论)，尽早结束序列(减少 KL 累积). 

OPD 的目标函数明确激励选项 2. 这就是为什么在长链蒸发和 shortcut cheating 之间存在强烈的耦合——长链蒸发创造了"逃避复杂推理"的需求，而 shortcut cheating 提供了"如何逃避"的具体手段. 

![长链与短链 KL 面积对比图](images/length_penalty_accumulation.png)

> **图 4.6.7.7 误差蔓延与短链套利 (Arbitrage)**
> 在连乘的概率链条下，后半段的误差呈现爆炸性蔓延(KL 激增). 于是模型开始“投机”：与其冒着被末端高 KL 惩罚的风险，不如直接在中间切断逻辑链输出答案(短链套利)，以获取全局更低的总体误差面积. 

### 3.3 检测: 长度曲线与复杂题分层的双重监控

长链蒸发的检测需要同时监控"输出长度"和"分层准确率"两个维度. 单独看任何一个维度都可能误判. 

#### 3.3.1 监控指标一: 平均生成长度的下降曲线

在每个训练 step，对固定验证集采样，计算平均输出长度: 

$$
 \bar{L}_t = \frac{1}{N} \sum_{i=1}^{N} \text{length}(y^{(i)}) \tag{44}
$$

**正常行为**: $\bar{L}_t$ 在训练初期可能略有波动，然后稳定在一个与任务复杂度匹配的水平(例如数学证明任务稳定在 150-250 tokens). 

**异常行为**: $\bar{L}_t$ 呈现持续下降趋势，从 200+ tokens 降到 100 tokens 以下，且无回升迹象. 

**工程实现**: 

```python
def monitor_average_length(model, validation_prompts, max_tokens=1024):
    """
    监控平均生成长度
    """
    lengths = []
    model.eval()
    with torch.no_grad():
        for prompt in validation_prompts:
            output = model.generate(prompt, max_new_tokens=max_tokens)
            lengths.append(len(output))
    
    avg_length = sum(lengths) / len(lengths)
    return avg_length
```

#### 3.3.2 监控指标二: 复杂题准确率的分层监控

将验证集按预估推理复杂度分层(可以通过教师模型生成长度、人工标注、或自动分析来分层): 

$$
 \text{Complexity}(x) = \text{length}(\text{teacher_CoT}(x)) \tag{45}
$$

定义三层: 
- 简单: Complexity $\le Q_1$(第一四分位数)
- 中等: $Q_1 <$ Complexity $\le Q_3$
- 困难: Complexity $> Q_3$

分别计算每层的准确率 $A_{\text{easy}}, A_{\text{med}}, A_{\text{hard}}$. 

**关键指标: 复杂度-准确率弹性(Complexity-Accuracy Elasticity, CAE)** : 

$$
 \text{CAE} = \frac{A_{\text{easy}} - A_{\text{hard}}}{A_{\text{easy}}} \tag{46}
$$
CAE 度量了模型准确率对复杂度的敏感度. 

- **CAE < 0.2**: 健康. 模型在复杂任务上的衰减不大. 

- **0.2 $\le$ CAE < 0.5**: 轻度长链蒸发. 需要关注. 

- **CAE $\ge$ 0.5**: 严重长链蒸发. 模型基本放弃了复杂任务. 

#### 3.3.3 监控指标三: 长链完成率(Long-Chain Completion Rate, LCCR)

对于需要长链的题目，定义"完成率"为: 模型是否生成了包含所有关键步骤的完整推理链(可由验证器判定). 

$$
 \text{LCCR} = \frac{\text{number of completed long chains}}{\text{total number of long-chain problems}} \tag{47}
$$

**正常行为**: LCCR 从 0 缓慢上升到 0.6-0.8(不是所有题目都能完成，但大部分可以). 

**异常行为**: LCCR 在初期上升到 0.3-0.4 后停滞，或甚至下降——模型学会了"开始"长链推理，但中途放弃. 

#### 3.3.4 监控指标四: KL 散度的"位置偏置"

计算验证集上 KL 散度在序列不同位置的平均值: 

$$
 \text{KL}_{\text{pos}}(k) = \frac{1}{N} \sum_{i=1}^{N} D_{KL}^{(i)}(\text{position } k) \tag{48}
$$
在健康训练中，$\text{KL}_{\text{pos}}(k)$ 应该在所有位置 $k$ 上大致均匀(或前段略低、后段略高，因为后段更难). 

在长链蒸发中，$\text{KL}_{\text{pos}}(k)$ 会呈现出明显的"U 型"或"前低后极高"模式——后段的 KL 急剧上升，表明模型在后段完全失控. 更关键的是，如果你只统计模型实际生成的序列长度内的位置，你会发现 KL 在后段异常高的同时，序列长度却在缩短——这说明模型通过"提前结束"来逃避高 KL 区域. 

### 3.4 大厂填坑: 长度归一化、动态温度与分段蒸馏

针对长链蒸发，业界发展出了三套互补的修复方案. 它们分别从目标函数结构、温度调度和序列分解三个角度来解决问题. 

#### 3.4.1 方案一: 长度归一化 KL(Length-Normalized KL)

**核心思想**: 修改 OPD 的目标函数，用**长度归一化**替代简单的算术平均，使得长链的每一步获得更公平的权重. 

标准 OPD 使用算术平均，将每步 KL 散度简单相加后除以序列长度: 

$$
 \mathcal{L}_{OPD} = \frac{1}{T} \sum_{t=1}^{T} D_{KL}^{(t)} \tag{49}
$$
如式 (49) 所示，这种平均方式对长链不公平——步骤越多，总惩罚越大. 为解决这个问题，长度归一化 KL 改用**累积折扣平均**: 

$$
 \mathcal{L}_{LN\text{-}OPD} = \sum_{t=1}^{T} \frac{1}{t} D_{KL}^{(t)} \tag{50}
$$
或者更常见的**指数折扣平均**: 

$$
 \mathcal{L}_{\gamma\text{-}OPD} = \frac{\sum_{t=1}^{T} \gamma^{T-t} D_{KL}^{(t)}}{\sum_{t=1}^{T} \gamma^{T-t}} \tag{51}
$$
其中 $\gamma \in (0, 1]$ 是折扣因子. 当 $\gamma = 1$ 时，退化为标准平均; 当 $\gamma < 1$ 时，后期步骤获得更高权重. 

**等等，这个方向似乎错了**. 如果我们给后期步骤更高权重，岂不是会让长链的惩罚更重？

让我重新思考. 长链蒸发的核心问题是: 长链的总 KL 更大，导致模型倾向于缩短链条. 如果我们想要**鼓励长链**，应该**降低长链的惩罚**，而不是增加. 

正确的长度归一化应该是: 

$$
 \mathcal{L}_{\text{norm}} = \frac{1}{T^\alpha} \sum_{t=1}^{T} D_{KL}^{(t)} \tag{52}
$$

其中 $\alpha < 1$. 这样，当 $T$ 增加时，分母的增长速度慢于分子的线性增长，长链的平均惩罚被"打折". 

更优雅的方案是**按难度加权**: 给复杂步骤更高的权重，简单步骤更低的权重，使得"跳过复杂步骤"不再是一个降低 Loss 的有效策略. 

但业界最广泛采用的实际方案是**Chunk-wise 分段蒸馏**(见 3.4.3)，因为它不仅解决了归一化问题，还解决了误差传播问题. 

#### 3.4.2 方案二: 动态温度调度(Dynamic Temperature Schedule)

**核心思想**: 通过调整 softmax 温度 $T$，在训练不同阶段控制模型的"探索-利用"权衡，防止模型过早地收敛到短链策略. 

动态温度调度的一种形式是**线性升温**: 

$$
 T(t) = T_0 + \alpha \cdot t \tag{53}
$$

其中 $T_0$ 是初始温度，$\alpha$ 是升温速率. 这与高熵坍缩中的温度退火(降温)方向相反. 

**物理机制**: 

在高温($T > 1$)下，softmax 输出更平坦，模型在采样时更倾向于探索低概率选项. 这有两个好处: 
1. **防止过早坍缩**: 模型不会太快地锁定某一条短链路径. 

2. **增加长链的探索概率**: 长链中的某些"关键步骤"在低温下可能概率很低(因为模型还没学会)，但在高温下有机会被采样到. 一旦这些关键步骤被采样到并获得了教师的正向反馈，模型就会开始学会它们. 

另一种动态温度策略是**基于长度的温度调节**: 

$$
 T(s_t) = T_{\text{base}} + \beta \cdot \text{length}(s_t) \tag{54}
$$
即在长链的后段使用更高的温度. 这直接对抗了长链后段的"放弃倾向"——当模型已经生成了很多 token 时，给它更多的探索自由度，鼓励它继续而不是终止. 

**参数配置表**: 

| 机构/论文 | 调度类型 | $T_0$ | $\alpha$ | 备注 |
|-----------|----------|-------|----------|------|
| DeepSeek-R1 OPD | 线性升温 | 1.0 | +0.001/step | 上限 2.0 |
| Kimi-K1.5 | 分段常数 | 1.5→1.0→0.8 | 阶梯 | 每 300 steps 降一档 |
| Qwen3 长链实验 | 振荡调度 | 1.0 | 正弦 | 周期 200 steps |
| 推荐默认 | 线性升温 | 1.0 | +0.0005/step | 上限 1.5 |

**注意事项**: 温度过高会导致模型输出过于随机，降低训练稳定性. 通常 $T$ 不应超过 2.0. 而且温度调度需要与拒绝采样(只保留高质量轨迹)配合使用，否则高温引入的噪声轨迹会污染训练. 

#### 3.4.3 方案三: 分段蒸馏(Chunk-wise Distillation)

**核心思想**: 不再对整个长链进行端到端的 KL 蒸馏，而是将长链切分为多个独立的"段"(Chunks)，每段独立进行蒸馏和验证. 

这是对抗长链蒸发最彻底、最工程化的方案. 

**算法流程**: 

```
输入: 问题 x, 教师完整推理链 Y_T = [y_1, y_2, ..., y_L]
步骤:
  1. 将 Y_T 切分为 K 个 chunks: C_1, C_2, ..., C_K
     每个 chunk 包含约 M 个 tokens(如 M=50)
  
  2. 对每个 chunk k:
     a. 构造 chunk 的上下文: prefix_k = x + C_1 + ... + C_{k-1}
     b. 学生模型在 prefix_k 条件下生成 chunk C_k^(student)
     c. 计算 chunk 级 KL: D_KL(pi_theta(chunk_k) || pi_T(chunk_k))
     d. 用步级验证器检查 chunk_k 的正确性
     e. 如果 chunk 错误，只惩罚该 chunk，不影响其他 chunks
  
  3. 总损失 = sum_{k=1}^{K} [chunk_k_loss * I(chunk_k correct)]
```

**核心优势**: 

1. **误差隔离**: 第 $k$ 个 chunk 的错误不会传播到第 $k+1$ 个 chunk. 即使学生在某个 chunk 中犯了错误，下一个 chunk 仍然可以从正确的教师前缀重新开始. 

2. **局部归一化**: 每个 chunk 的长度固定为 $M$，KL 散度在每个 chunk 内部归一化. 长链不再因为"总长度更长"而受到惩罚——无论链多长，每个 chunk 的优化压力是相同的. 

3. **并行训练**: 不同 chunk 的蒸馏可以并行进行，因为它们共享前缀但独立优化. 

4. **课程学习天然集成**: 简单 chunk(前段，基于问题陈述)可以先用标准 OPD 训练; 复杂 chunk(后段，需要深层推理)可以后加入，或在更高温度下训练. 

**Chunk 切分策略**: 

- **固定长度切分**: 每 $M$ 个 token 切一刀. 简单但可能切断逻辑单元. 

- **语义边界切分**: 在逻辑连接词("因此"、"所以"、"首先/其次")处切分. 更符合推理结构. 

- **教师引导切分**: 让教师模型在生成时显式输出 chunk 边界标记(如 `<|chunk_end|>`). 

**参数配置表**: 

| 机构/论文 | Chunk 大小 M | 切分策略 | 验证方式 |
|-----------|-------------|----------|----------|
| DeepSeek-R1 (推测) | 32-64 tokens | 语义边界 | 规则验证器 |
| Kimi-K1.5 | 约 50 tokens | 固定长度 | 模型验证器 |
| 学术基准 | 64 tokens | 固定长度 | 程序验证 |
| 推荐默认 | 50 tokens | 语义边界 | 混合验证 |

#### 3.4.4 综合修复方案

长链蒸发的修复需要在目标函数、采样策略和验证机制三个层面同时发力: 

```
Layer 1 (目标函数):
  - Chunk-wise KL: chunk_size = 50, 每 chunk 独立归一化
  - 长度奖励: R_length = +0.01 * min(T, T_target) 
    (鼓励学生生成接近目标长度的序列)

Layer 2 (采样策略):
  - 动态温度: T(t) = 1.0 + 0.0005 * t, 上限 1.5
  - Nucleus 采样: p = 0.95
  - 强制最小长度: 对复杂题强制至少 100 tokens

Layer 3 (验证机制):
  - 每 chunk 步级验证
  - 错误 chunk 只丢弃该 chunk，不惩罚整条链
  - 保留"长但部分正确"的轨迹用于后续 chunks 的训练

Layer 4 (监控):
  - 平均长度曲线(目标: 稳定或缓慢上升)
  - CAE 指数(目标: < 0.3)
  - LCCR(目标: > 0.6)
```
