# 4.6 OPD 深度全解: 在线自蒸馏与 On-Policy 蒸馏统一框架

> **⚠️ 前言与勘误**
> 
> 在讨论大模型后训练(Post-Training)时，很多人会将 OPD 与 DPO (Direct Preference Optimization) 混淆. 实际上，**OPD (Online Preference/Policy Distillation, 或泛指 Online Self-Distillation 在线自蒸馏)** 是一套完全不同于基于成对偏好强化学习的方法. 
> 它解决的核心痛点是: **如何在没有海量人类反馈(RLHF)，甚至没有更强外部教师(Teacher)的情况下，通过模型自己的探索(On-Policy)和自我蒸馏(Self-Distillation)，打破能力天花板？**

---

## 1. 钩子与痛点: 为什么我们需要 OPD？

当大语言模型(LLM)完成 SFT(监督微调)后，通常会进入强化学习阶段(如 PPO 或 GRPO)以提升推理和遵循指令的能力. 然而，RL 的代价是极其昂贵的: 
1. **PPO 的复杂性**: 你需要 Actor、Critic、Reward Model、Reference Model 四个模型在显存中同时跳舞. 

2. **奖励稀疏 (Sparse Reward)** : 模型生成几百个 token 后，只能得到一个标量奖励(如 1 或 -1). 这种信号太微弱了，模型很难知道到底哪一句话是对的，哪一句话是错的(Credit Assignment Problem). 

3. **SFT 的局限 (Exposure Bias)** : 如果只是单纯用强模型(如 GPT-4)生成数据进行 SFT(这叫 Off-Policy Distillation)，学生模型只能被动模仿，从未在自己生成的轨迹中“试错”. 一旦在推理时偏离了教师的分布，就会一步错步步错. 

**破局之道: 在线自蒸馏 (OPD)** . 
如果让模型自己去生成(**On-Policy**，解决 Exposure Bias)，但在每生成一个 token 时，都通过自己更深思熟虑的上下文(或者稍强一点的辅助状态)来给自己打分，提供每个 token 级别的密集监督(**Dense Supervision**)，这就是 OPD 的物理直觉. 

---

## 2. 直觉类比: 考场上的“自查”与“抄作业”

我们可以用学生考试来类比三种后训练范式: 

- **SFT (Off-Policy 蒸馏)** : 学生(小模型)照着学霸(GPT-4)的满分试卷抄一遍. 考场上遇到学霸没写过的题，学生当场懵逼(这就是 Exposure Bias，分布偏移). 

- **RL (强化学习)** : 学生自己做了一套卷子，老师只在最后给个总分(+10 分). 学生需要通过几万次考试去瞎猜到底哪一步做对了(奖励稀疏). 

- **OPD (在线自蒸馏)** : 学生自己做卷子(On-Policy). 但在检查每一道题时，学生突然进入了“绝对专注”模式(给予更多的上下文或思考时间)，并把专注模式下的解题思路(Dense Logits)作为监督信号，教导常态下的自己. **整个过程没有满分答案，全靠自己探索和自我校准. **

---

## 3. OPD 的数学基础: Reverse KL 与 f-Divergence

在学术界，OPD 通常被建模为最小化学生策略 $\pi_\theta$ 和教师/自建目标策略 $\pi_T$ 之间的散度. 

### 3.1 为什么不能用传统的 Forward KL？

传统知识蒸馏使用 **Forward KL (前向 KL 散度)** : 

$$ D_{KL}(\pi_T || \pi_\theta) = \mathbb{E}_{x \sim \pi_T} [\log \pi_T(x) - \log \pi_\theta(x)] \tag{1} $$

- **特点**: 它是 **Mean-Seeking(求均值)** 的. 如果老师觉得两个答案都行，学生会倾向于在两个答案中间找一个四不像的答案. 

- **致命弱点**: 它强迫学生去**覆盖老师所有的概率空间**，哪怕学生根本没能力学会. 这叫“打肿脸充胖子”，会导致严重的幻觉. 

### 3.2 Reverse KL: 量力而行的精准模仿

OPD 通常使用的是 **Reverse KL (逆向 KL 散度)** ，公式仅仅是把位置调换了一下: 

$$ D_{KL}(\pi_\theta || \pi_T) = \mathbb{E}_{x \sim \pi_\theta} [\log \pi_\theta(x) - \log \pi_T(x)] \tag{2} $$
- **注意这里的期望** $\mathbb{E}_{x \sim \pi_\theta}$: 这是 **On-Policy** 的灵魂所在！数据是**由学生模型自己生成**的. 

- **特点**: 它是 **Mode-Seeking(求众数/寻模态)** 的. 如果老师给出了多个可行解，学生只要挑一个自己最擅长、最符合自己知识结构的解去贴近即可. 

- **优势**: 学生不去瞎猜老师不确定的长尾概率，只在自己生成的轨迹上，用老师的高概率信号去收敛. 这极大地减少了幻觉(Hallucination).

## 4. 在线自蒸馏 (Online Self-Distillation) 的工程实现

在没有任何更强模型(没有 GPT-4)的情况下，模型如何当自己的老师？

**核心思路: 用“高信息量/深思熟虑”的上下文预测，去蒸馏“低信息量/直觉”的上下文预测. **

### 4.1 双路生成机制

假设我们正在教模型回答问题. 我们构造两个 Prompt: 
1. **Teacher Context (高维)** : `"你是一个专家，请一步步仔细思考以下问题: [问题]"`
2. **Student Context (低维)** : `"回答问题: [问题]"`

**Step 1: On-Policy 采样 (Rollout)** 
学生模型(带低维上下文)或带有高维上下文的模型自行生成一条轨迹(Trajectory) $y = (y_1, y_2, ..., y_T)$. 

**Step 2: 计算 Logits**
将这条轨迹同时喂给模型两次(一次带 Teacher Prompt，一次带 Student Prompt)，获得每个 token 位置上的输出概率分布: 
- 老师分布: $P_T(y_i | x_{high}, y_{<i})$
- 学生分布: $P_S(y_i | x_{low}, y_{<i})$

**Step 3: Reverse KL 损失反向传播**
$$ \mathcal{L}_{OPD} = \sum_{i=1}^T P_S(y_i | x_{low}, y_{<i}) \left[ \log P_S(y_i | x_{low}, y_{<i}) - \log P_T(y_i | x_{high}, y_{<i}) \right] \tag{3} $$

*(在实际工程中，通常使用单样本近似截断，或者用 JSD 来避免除零异常). *

## 5. 50行 PyTorch 简化实现

我们用一段简化代码来展示 OPD 的精髓: 如何计算每一层的 Dense 监督信号. 

```python
import torch
import torch.nn.functional as F

def opd_self_distillation_step(model, inputs_student, inputs_teacher, temperature=1.0):
    """
    在线自蒸馏核心逻辑简化版
    inputs_student: shape (B, L) 学生上下文
    inputs_teacher: shape (B, L) 教师上下文 (例如带有 detailed system prompt)
    """
    
    # [1] On-Policy 采样: 学生自己生成轨迹 (这里省略自回归生成步骤，假设已生成 y)
    # student_trajectory = model.generate(inputs_student)
    
    # 假设输入中已经包含了学生生成的序列，我们将其分别带入高低维上下文
    model.train()
    
    # [2] 获取学生 (Student) 分布 - 开启梯度
    logits_S = model(inputs_student).logits
    
    # [3] 获取自建教师 (Teacher) 分布 - 不传导梯度
    with torch.no_grad():
        logits_T = model(inputs_teacher).logits
    
    # 软化分布 (Temperature Scaling)
    probs_S = F.log_softmax(logits_S / temperature, dim=-1)
    probs_T = F.softmax(logits_T / temperature, dim=-1)
    
    # [4] 计算 Reverse KL Loss
    # 注意: torch 的 kl_div 默认计算 KL(target || input)，
    # 如果要算 Reverse KL: KL(Student || Teacher), 需要让 target=Teacher
    # 理论公式为 E_S [log S - log T]，这里使用批量 token 上的平均近似
    loss_kl = F.kl_div(
        input=probs_S,          # log S
        target=probs_T,         # T
        reduction='batchmean'
    ) * (temperature ** 2)
    
    # [工程优化] 很多时候还会加上传统的 NLL Loss 以防止分布灾难性崩溃
    # labels = ... (从轨迹中获取真实 token)
    # loss_nll = F.cross_entropy(logits_S.transpose(1,2), labels)
    # total_loss = loss_nll + alpha * loss_kl
    
    return loss_kl
```

> **>>> 注释解析**: 
> 这段代码完美体现了“在线自蒸馏”: 我们没有使用第二个模型，`model` 即是学生也是老师. 区别仅在于输入的上下文不同. 通过 `torch.no_grad()` 切断了 Teacher 的梯度，让 Student 的分布强行去向 Teacher 的分布拟合.

## 6. OPD vs SFT vs RL 全面对比

为什么 Qwen3 的技术报告中，OPD 只用了 RL 十分之二的算力，却达到了更高的分数？请看下表: 

| 维度 | SFT / Off-Policy | RL (PPO/GRPO) | OPD (在线自蒸馏) |
|------|------------------|---------------|------------------|
| **采样策略** | 静态外部数据集 | 动态 **On-Policy** | 动态 **On-Policy** |
| **监督粒度** | Token-level (硬标签) | Sequence-level (稀疏标量) | **Token-level (密集分布)** |
| **探索能力** | 无 (纯模仿) | 强 (但容易 Reward Hacking) | 中等 (在自我分布边界探索) |
| **显存占用** | 低 (1x Model) | 极高 (3x-4x Models) | 中 (1.5x-2x，因为需双路前向计算) |
| **收敛速度** | 快 | 极慢 (需万级 step 探索) | **极快** (因为梯度信号是稠密的) |
| **分布崩溃** | 严重 (Exposure Bias) | 模式坍缩 (Mode Collapse) | 相对稳定 (Reverse KL 兜底) |

## 7. 2025-2026 前沿演进与未来探索

现在的 OPD 已经不仅仅是“同一个模型跑两遍”，而是演化出了一系列复杂的“自我对抗”机制: 

1. **SDPO (Self-Distillation Policy Optimization)** : 摒弃了 KL 散度，将自蒸馏生成的隐式偏好概率，直接带入 DPO 的对数似然损失中，将 OPD 和 DPO 在数学上统一. 

2. **G-OPD (广义 OPD)** : 将奖励模型(Reward Model)的输出与自蒸馏分布耦合，创造了一个动态的、随训练过程自我演化的“虚拟超模”作为 Teacher. 

3. **SCOPE (Selective Feedback)** : 并非所有的自我生成都是好数据. SCOPE 引入了选择性拒绝，如果 Teacher Prompt 生成的结果依然比不过某些阈值，则将该条轨迹扔掉，避免模型学习自己的愚蠢错误. 

4. **多教师 OPD(DeepSeek-V4 实践)** : DeepSeek-V4 的后训练将传统混合 RL 阶段完全替换为 OPD. 首先在数学、代码、Agent、指令遵循等领域分别训练独立专家(SFT + GRPO)，然后用十余个教师模型对统一学生模型进行多教师 OPD. 关键工程细节在于采用**全词汇 logit 蒸馏**而非 token-level KL 估计: 保留完整的 logit 分布计算反向 KL 散度，获得更稳定的梯度估计和更忠实的蒸馏效果. 学生模型在自身生成的轨迹上采样，通过 logits 级别的对齐将多个物理分离的专家权重 consolidate 到统一参数空间中，实质性地规避了传统权重合并或混合 RL 的性能退化. 

## 8. 总结

1. **OPD 的本质不是对齐外部人类偏好，而是榨干模型内部知识. **
2. **On-Policy** 解决了面对陌生场景不会推断的问题. 

3. **Dense Supervision (密集分布监督)** 让它比传统的强化学习收敛快得多，代价小得多. 

4. **Reverse KL** 确保了模型只学习自己能理解的高概率知识，避免了盲目模仿导致的幻觉. 

在线自蒸馏正在取代笨重的 PPO 管道，成为长逻辑链(如 o1-like reasoning)训练的下一个主战场.
