# 11.3 资源索引

## 1. 知名课程

### 1.1 预训练与基础架构

**Stanford CS25: Transformers United**
- 链接: https://web.stanford.edu/class/cs25/
- 内容: 由Lucas Beyer、Noam Shazeer等Transformer核心作者主讲的系列讲座，涵盖Transformer架构、Vision Transformer、多模态模型等前沿主题. 
- 适合人群: 希望从设计者视角理解Transformer演进的研究者. 

**Stanford CS336: Large Language Models**
- 链接: https://stanford-cs336.github.io/spring2025
- 内容: 从数据收集清洗、预训练、模型构建到部署前评估的完整实践课程，代码量极大，要求不使用高层框架从零实现关键组件. 
- 适合人群: 想深入理解预训练工程细节的工程师. 

**Princeton COS 597G: Understanding Large Language Models**
- 讲师: 陈丹琦(Danqi Chen)
- 内容: 高级研究生课程，聚焦LLM的理论基础、推理能力和知识存储机制. 课程资料以幻灯片和论文阅读为主. 
- 适合人群: 已有ML/NLP基础，希望进行理论深化的高年级研究生. 

### 1.2 对齐与后训练

**Berkeley CS285: Deep Reinforcement Learning**
- 链接: https://rail.eecs.berkeley.edu/deeprlcourse/
- 内容: Sergey Levine主讲的深度强化学习课程，涵盖策略梯度、Actor-Critic、Model-Based RL等. RLHF的底层RL方法在此有系统讲解. 
- 适合人群: 想理解PPO、TRPO等算法数学原理的读者. 

**OpenAI Spinning Up in Deep RL**
- 链接: https://spinningup.openai.com/
- 内容: OpenAI出品的强化学习入门资料，以清晰的数学推导和可运行代码著称. 
- 适合人群: RL零基础但希望快速上手并理解核心概念的工程师. 

### 1.3 多模态与视觉

**Stanford CS231n: Convolutional Neural Networks for Visual Recognition**
- 链接: http://cs231n.stanford.edu/
- 内容: 虽然主要聚焦CNN，但其对视觉特征提取、注意力机制在视觉中的应用讲解极为扎实，是理解VLM视觉Encoder 的基础. 
- 适合人群: 从NLP转向多模态、需要补齐视觉基础的研究者. 

### 1.4 中文课程

**李沐《动手学深度学习》**
- 链接: https://zh.d2l.ai/
- 内容: 从MLP到Transformer的完整中文教程，配套视频和可运行代码，是中文社区最广泛使用的深度学习入门资料. 
- 适合人群: 中文母语、偏好系统化教材而非论文的初学者. 

**深度学习技术前沿(B站/知乎系列)** 
- 多个UP主和专栏作者持续产出大模型相关的中文解读，适合作为英文论文的预习材料.

## 6. 学习路径建议

### 6.1 初学者路径(0-3个月)

1. **数学基础复习**: 线性代数(矩阵乘法、特征分解)、概率统计(条件概率、MLE)、优化(梯度下降). 

2. **框架入门**: PyTorch官方Tutorial → 李沐《动手学深度学习》→ 用Hugging Face Transformers跑通第一个文本分类任务. 

3. **Transformer原理**: 阅读Attention Is All You Need原文 → 观看Stanford CS25相关讲座 → 用nanoGPT手写一次推理. 

### 6.2 进阶者路径(3-12个月)

1. **预训练深入**: 阅读GPT-3、Llama 2、Llama 3论文 → 了解分布式训练(DeepSpeed/Megatron)→ 尝试在小规模数据上复现预训练. 

2. **对齐技术**: 精读InstructGPT和DPO论文 → 用TRL库跑通SFT + DPO流程 → 理解Reward Hacking和训练稳定性问题. 

3. **推理优化**: 学习量化原理(GPTQ/AWQ)→ 用vLLM部署一个微调模型 → 进行吞吐和延迟压测. 

### 6.3 专家路径(1年+)

1. **前沿跟踪**: 订阅arXiv daily mailings，每周精读3-5篇相关论文. 

2. **开源贡献**: 为vLLM、Transformers、DeepSpeed等核心项目提交PR(Bug修复、新功能、文档改进). 

3. **领域深耕**: 选择一个细分方向(如长上下文、多模态Agent、模型安全)进行系统性研究，产出博客或论文.

## 7. 总结

LLM领域的学习资源呈指数级增长，但质量参差不齐. 建议读者建立自己的"核心资源集"——3-5门高质量课程、10-15篇反复精读的论文、2-3个活跃的技术社区，然后将大部分注意力放在动手实践上. 理论指导实践，实践反哺理论理解，这是在内卷的大模型赛道中建立差异化竞争力的唯一路径.
