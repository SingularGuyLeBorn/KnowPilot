# 3.3.2 通用基准

> 在困惑度之外，通用基准(Benchmark)提供了衡量大模型实际能力的"标准化考场". 本节系统梳理当前业界最权威的综合性评测基准，帮助理解如何通过标准化考试来横向对比不同模型的通用智能水平. 

---

## 1. 为什么需要通用基准？

困惑度(PPL)虽然能反映模型对语言分布的建模能力，但它存在明显局限：

- **与人类感知脱节**：PPL 降低 10 个点，未必意味着模型生成质量提升了 10%
- **无法衡量特定能力**：数学推理、代码生成、常识理解等能力，PPL 无法直接反映
- **数据泄露风险**：训练集与测试集重叠时，PPL 会虚低

通用基准(Benchmarks)正是为了解决这些问题而设计的标准化评测集. 它们通过精心设计的测试任务，系统性地评估模型的某方面能力，让不同模型能在 **公平的"考场"** 上进行比较. 

---

## 2. 评测集矩阵

下表梳理了当前业界最主流的综合性评测基准，涵盖知识理解、数学推理、代码生成、常识推理等多个维度：

| 评测集 | 核心考察能力 | 任务形式 | 题量 | 典型例子 |
|:------|:-----------|:--------|:---:|:--------|
| **MMLU** | 多学科知识广度 | 4选1选择题 | 15,908 | "青霉素是由哪种微生物产生的？" |
| **GSM8K** | 数学推理 | 数学应用题 | 8,500 | "小红有3个苹果，小明有5个...一共有几个？" |
| **MATH** | 高阶数学推理 | 竞赛级数学题 | 5,000 | 微积分/线性代数/数论问题 |
| **HumanEval** | 代码生成 | 函数补全 | 164 | "编写一个函数计算斐波那契数列" |
| **MBPP** | 代码生成 | 编程问题 | 974 | "编写一个函数检查回文" |
| **HellaSwag** | 常识推理 | 续写4选1 | 10,042 | 给定情境选择最合理的后续 |
| **ARC** | 科学推理 | 自然科学选择题 | 7,787 | 小学~高中科学知识问答 |
| **WinoGrande** | 代词消歧 | 二选一 | 44,000 | "奖杯太重了，__ 拿不动. "(谁？) |
| **TruthfulQA** | 事实准确性 | 开放问答 | 817 | 检测模型是否生成常见误解 |
| **BBH (Big Bench Hard)** | 复杂推理 | 23个高难子任务 | 23 tasks | 逻辑推理/时间理解/多步推理 |

这些评测集按照难度递增的层次，构成了大模型评测的 **"通用能力金字塔"**：

```
                   🏆 综合推理
               (BBH/MATH/MMLU-Pro)
              🧮 专业能力
          (GSM8K/HumanEval/ARC)
         🧠 常识理解
     (HellaSwag/WinoGrande)
    📚 知识广度
  (MMLU/TruthfulQA)
```

## 3. 重点评测集详解

### 3.1 MMLU — 多学科知识广度"奥运会"

**全称**：Massive Multitask Language Understanding

**基本信息**：
- 发布机构：UC Berkeley(2020年)
- 题量：15,908 道 4 选 1 选择题
- 覆盖学科：**57个学科**，分为 STEM(科学、技术、工程、数学)、人文、社会科学 3 大类

**学科覆盖示例**：

| 学科类别 | 代表学科 | 题量 |
|:--------|:--------|:----:|
| STEM | 物理学、化学、生物学、计算机科学、数学 | ~8,000 |
| 人文 | 历史、法律、哲学、文学、语言学 | ~4,000 |
| 社会科学 | 经济学、心理学、政治学、地理学 | ~4,000 |

**评测方式**：
```
输入：The antibiotic penicillin was first discovered by:
A) Alexander Fleming
B) Louis Pasteur
C) Robert Koch
D) Joseph Lister

输出：A) Alexander Fleming  ✅
```

**典型难度曲线**：

| 学科 | GPT-3 (175B) | GPT-4 | LLaMA 2 70B | 人类专家 |
|:----|:-----------:|:-----:|:----------:|:-------:|
| 美国历史 | 55.3% | **84.2%** | 72.1% | ~85% |
| 高中物理 | 40.8% | **72.6%** | 58.3% | ~80% |
| 临床知识 | 52.1% | **86.0%** | 70.5% | ~90% |
| 宏观经济学 | 58.2% | **85.7%** | 76.8% | ~85% |
| **整体平均** | 43.9% | **86.4%** | 68.9% | ~89.8% |

**意义**：MMLU 是目前引用最广泛的通用知识评测集之一. 一个模型如果在 MMLU 上达到 **80%+** 以上，可以认为它在知识广度上达到了"人类专家水平". MMLU-Pro(2024年发布)进一步提高了难度，去除了容易猜测的题目. 

### 3.2 GSM8K — 数学推理的"基础门槛"

**全称**：Grade School Math 8K

**基本信息**：
- 发布机构：OpenAI(2021年)
- 题量：8,500 道数学应用题
- 难度等级：小学水平(3~8年级)
- 评测方式：自由生成答案，自动比对最终数字

**典型题目**：
```
题目：Janet买了一只鸭子，每天收16个蛋. 她每天早餐用3个蛋做煎饼，
     用4个蛋做蛋糕. 她把剩下的蛋卖给邻居，每个2美元. 
     她一周能赚多少钱？

正确推理：
- 每天收蛋：16个
- 每天自用：3 + 4 = 7个
- 每天剩余：16 - 7 = 9个
- 每天收入：9 × 2 = 18美元
- 一周收入：18 × 7 = 126美元

答案：126
```

**GSM8K 的评测价值**：

| 版本 | 说明 |
|:----|:-----|
| **GSM8K (Base)** | 标准版，纯文本数学应用题 |
| **GSM8K-Symbolic** | 符号替换版，将题目中数字换为字母，检测是否真正理解而非记忆 |
| **GSM-Plus** | 变体版，修改题目条件，检测模型对推理链的鲁棒性 |

许多早期模型(< 100B)在 GSM8K 上准确率不足 **20%**，而 GPT-4 达到了 **92%**. GSM8K 已成为衡量基础数学推理能力的"及格线". 

### 3.3 HumanEval & MBPP — 代码生成能力

**HumanEval**(OpenAI, 2021)：
- 164 道手写编程题
- 每个问题包含函数签名、文档字符串、测试用例
- 评测指标：**pass@k**(生成 k 个样本，至少通过测试的概率)
- 语言：Python

**MBPP**(Google, 2021)：
- 974 道编程题
- 更偏向日常编程任务
- 语言：Python

**典型 HumanEval 题目**：
```python
def find_missing(arr):
    """给定一个包含 n-1 个不重复整数的数组，
       这些整数在 1 到 n 范围内，找到缺失的那个数. 
    """
    n = len(arr) + 1
    expected = n * (n + 1) // 2
    actual = sum(arr)
    return expected - actual
```

**代表性模型表现**：

| 模型 | HumanEval pass@1 | MBPP pass@1 |
|:----|:----------------:|:-----------:|
| GPT-3 (175B) | 17.7% | ~30% |
| Codex (12B) | 28.8% | ~47% |
| GPT-4 | **67.0%** | **73.5%** |
| CodeLLaMA 34B | 53.7% | 62.4% |
| DeepSeek-Coder 33B | 65.2% | 74.4% |

### 3.4 HellaSwag & ARC — 常识推理

**HellaSwag**(AI2, 2019)：
- 全称：Harder Endings, Longer contexts, and Activity descriptions
- 10,042 道 4 选 1 续写题
- 对抗性生成：用 GPT-2 生成干扰项，使任务对模型更具挑战性
- 典型场景："一个人走进厨房，打开冰箱..." → 最合理的下一步？

**ARC(AI2 Reasoning Challenge, 2018)** ：
- 7,787 道科学选择题(小学~高中水平)
- 分为 **Challenge Set**(2,590 道难题)和 **Easy Set**(5,197 道简单题)
- 需要世界知识和多步推理

| 模型 | HellaSwag | ARC-C (Challenge) |
|:----|:---------:|:-----------------:|
| 人类基线 | ~95% | ~95% |
| GPT-3 | 78.9% | 51.3% |
| LLaMA 65B | 86.5% | 59.4% |
| GPT-4 | **95.3%** | **96.3%** |
| LLaMA 3 70B | 90.2% | 82.1% |

> **关键洞察**：HellaSwag 一度被认为是最难突破的基准之一. 早期语言模型得分仅 30~40%，远低于人类. 当模型在 HellaSwag 上超过 **85%** 时，通常意味着它具有了较强的常识理解能力. 

### 3.5 BBH — 复杂推理"精英测试"

**Big-Bench Hard(BBH, 2022)** ：
- 从 Big-Bench 中筛选出的 **23 个最难子任务**
- 涵盖：逻辑推理、时间理解、因果判断、多步规划等
- 任务示例：
  - **逻辑推理**："所有 A 都是 B，所有 B 都是 C，那么 A 是 C 吗？"
  - **时间理解**："如果周三前三天是周二，那么周三后五天是？"
  - **矛盾检测**："两段文字是否互相矛盾？"

| 模型 | BBH (CoT) |
|:----|:---------:|
| GPT-3 | 40.2% |
| PaLM 540B | 62.7% |
| GPT-4 | **86.6%** |
| Gemini Ultra | 83.6% |
| DeepSeek-V3 | 82.3% |

## 5. 使用通用基准的注意事项

### 5.1 数据泄露风险

当评测集题目出现在训练数据中时，得分会被严重高估. 典型例子：
- GPT-3 发布时声称在 MATH 上准确率仅 ~6%
- 多个后续模型通过**数据清洗**(去重训练集中出现的评测题)将准确率提升至 **50%+**
- **应对方案**：使用更新发布的评测集(如 MMLU-Pro、LiveBench)，或使用私有 held-out 测试集

### 5.2 评测方式的影响

| 因素 | 影响 | 典型差异 |
|:----|:----|:--------|
| **少样本 vs 零样本** | 少样本通常更好 | 0-shot → 5-shot: +5~15% |
| **CoT(思维链)** | 对推理类任务提升显著 | GSM8K: +20~40% |
| **多数投票** | 多次采样取多数 | pass@1 可能翻倍 |
| **采样数量** | pass@k vs pass@1 | pass@80 可比 pass@1 高 30%+ |

### 5.3 评测集的局限性

- **题目过时**：MMLU 发布于 2020 年，部分知识已更新
- **选择题形式**：四选一更容易猜测(随机基准 25%)
- **英语为中心**：非英语评测集仍在发展中(如 C-Eval 中文评测)
- **无法衡量创造力**：标准化测试难以评估生成式 AI 真正的创造性和实用性

## 6. 总结

通用基准为大模型的能力评估提供了标准化的"度量衡"：

| 维度 | 代表基准 | 衡量什么 |
|:----|:--------|:--------|
| 📚 **知识广度** | MMLU / MMLU-Pro | 模型的知识储备是否全面 |
| 🧮 **数学推理** | GSM8K / MATH | 模型能否进行量化推理 |
| 💻 **代码生成** | HumanEval / MBPP | 模型是否具备编程能力 |
| 🧠 **常识推理** | HellaSwag / ARC | 模型对世界的理解是否合理 |
| 🔬 **复杂推理** | BBH | 模型的逻辑和因果推理深度 |

**核心原则是**：不要依赖单一基准，**组合使用**多个评测集才能获得对模型能力的全面理解. 评估时还需注意数据泄露、评测方式差异等潜在陷阱，确保对比的公平性.
