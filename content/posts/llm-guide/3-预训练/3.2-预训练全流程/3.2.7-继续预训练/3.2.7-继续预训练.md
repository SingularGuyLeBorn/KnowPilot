# 3.2.7 继续预训练：让基座模型"持续进化"

## 1. 继续预训练的应用场景

### 1.1 知识更新

最直观的应用场景是让模型"与时俱进". 例如：

- GPT-4 的训练数据截止于 2023 年 10 月
- 如果想让模型了解 2024 年发布的 Llama 3、GPT-4o 等新模型
- 可以通过继续预训练注入这些新知识

**关键挑战**：新知识必须与旧知识无缝融合，不能导致模型对已有知识的遗忘. 

### 1.2 领域适配

这是目前最广泛的继续预训练应用. 常见领域包括：

| 领域 | 数据来源 | 训练量级 | 典型案例 |
|:----|:--------|:--------|:--------|
| 医疗 | PubMed 论文、临床指南、电子病历 | 100B+ tokens | BioBERT, PubMedGPT, Med-PaLM |
| 法律 | 法律法规、判例文书、法律评论 | 50B+ tokens | LawGPT, LexLM |
| 金融 | 财报、研报、新闻、公告 | 100B+ tokens | BloombergGPT, FinGPT |
| 代码 | GitHub 代码库、技术文档 | 500B+ tokens | CodeLlama, StarCoder |
| 多语言 | 目标语言的语料库 | 100B+ tokens | BLOOM, Llama 3 中文增强 |

**领域适配的核心原则**：**数据质量远比数量重要**. 1B 高质量领域 token 的效果可能超过 10B 低质量领域 token. 

### 1.3 技能注入

某些技能需要大量数据才能习得，无法通过 SFT 完成：

- **代码生成能力**：需要大量代码语料的学习
- **数学推理能力**：需要大量数学证明和推导
- **长文本理解能力**：需要长序列的继续训练
- **特定工具使用能力**：需要特定格式的工具调用数据

### 1.4 多语言扩展

如果基座模型主要使用英文训练，可以通过继续预训练扩展到其他语言：

$$
P_{\text{new}}(x_{\text{target}}) = \text{CPT}(P_{\text{base}}, D_{\text{target}}) \tag{1}
$$

其中 $D_{\text{target}}$ 是目标语言的大规模语料. 例如：
- Llama 系列通过继续预训练扩展到中文(如 Chinese-Llama)
- BLOOM 在 46 种语言上进行了大规模多语言继续预训练

## 2. 核心挑战：灾难性遗忘

### 2.1 问题定义

继续预训练面临的最大挑战是**灾难性遗忘(Catastrophic Forgetting)** . 这一概念最早在连续学习(Continual Learning)领域被系统研究，其数学表述为：

**给定基座模型 $\theta_0$**，在新数据 $D_{\text{new}}$ 上训练后得到 $\theta_1$，我们希望在 $D_{\text{new}}$ 上性能提升的同时，在原始数据 $D_0$ 上的性能不显著下降：

$$
\theta_1 = \arg\min_\theta \mathcal{L}(\theta, D_{\text{new}}) \tag{2}
$$
$$
\text{subject to } \mathcal{L}(\theta_1, D_0) \approx \mathcal{L}(\theta_0, D_0) \tag{3}
$$

这个约束要求模型在新数据上训练后，在旧数据上的损失函数值与训练前基本持平. 然而在实际训练中，如果不加干预，模型在新数据上的学习会覆盖旧数据的表征，导致：

$$
\mathcal{L}(\theta_1, D_0) \gg \mathcal{L}(\theta_0, D_0) \tag{4}
$$

### 2.2 遗忘的根源

灾难性遗忘的深层原因可以从两个角度理解：

**从神经元激活的角度**：
模型的每个参数 $\theta_i$ 参与多个知识片段的表示. 当新知识需要调整参数时，该参数承载的旧知识也会被修改. 

$$
\Delta \theta_i = -\eta \frac{\partial \mathcal{L}(D_{\text{new}})}{\partial \theta_i} \tag{5}
$$

如果参数 $\theta_i$ 同时参与了旧知识 $K_0$ 的表示，那么：
$$
K_0(\theta_0 + \Delta \theta) \neq K_0(\theta_0) \tag{6}
$$
**从损失景观的角度**：
新数据 $D_{\text{new}}$ 的损失景观与旧数据 $D_0$ 的损失景观不同，梯度下降会把模型推向 $D_{\text{new}}$ 的最优点，偏离 $D_0$ 的最优点. 

### 2.3 遗忘的度量

在实际训练中，如何量化遗忘？常用的指标包括：

| 指标 | 定义 | 说明 |
|:----|:----|:----|
| **遗忘率** | $\frac{\text{Acc}_{old}(D_0) - \text{Acc}_{new}(D_0)}{\text{Acc}_{old}(D_0)}$ | 原始评测集上的准确率下降比例 |
| **KL 散度** | $D_{KL}(p_{\theta_0} \parallel p_{\theta_1})$ | 模型输出分布的变化程度 |
| **Loss 变化** | $\mathcal{L}(\theta_1, D_0) - \mathcal{L}(\theta_0, D_0)$ | 原始数据上的 Loss 上升幅度 |

**目标**：在新领域 Loss 下降的同时，原始领域 Loss 上升最小化.

## 3. 缓解遗忘的核心策略

### 3.1 更小的学习率

这是最简单也最有效的策略. 继续预训练的学习率通常比从零预训练小 **10~100 倍**：

| 训练阶段 | 典型学习率 | 说明 |
|:--------|:---------:|:----|
| 从零预训练 | $3 \times 10^{-4}$ | AdamW 默认配置 |
| 继续预训练 | $1 \times 10^{-5} \sim 5 \times 10^{-5}$ | 约为预训练的 1/10 |
| SFT | $1 \times 10^{-6} \sim 2 \times 10^{-5}$ | 更小的扰动 |

**为什么有效？**

从梯度更新的角度，学习率 $\eta$ 直接控制参数更新的步长：

$$
\theta_{t+1} = \theta_t - \eta \nabla \mathcal{L}(\theta_t) \tag{7}
$$

更小的 $\eta$ 意味着每次更新的幅度更小，减少了知识覆盖的风险. 直观上，**小学习率让模型"温和地"吸收新知识，而非"野蛮地"覆盖旧知识**. 

### 3.2 新旧数据混合

这是最核心的策略，也是实际中最有效的方案. 

**核心思想**：在继续训练时，将新数据与一部分原始预训练数据混合，让模型在学习新知识的同时，复习旧知识. 如果不混合旧数据，模型可能在新数据上快速收敛，但会灾难性地遗忘预训练阶段学到的通用语言知识和世界知识. 

混合损失函数将新旧数据的损失按权重 $\lambda$ 加权组合：

$$
\mathcal{L}_{\text{total}} = \lambda \cdot \mathcal{L}(D_{\text{new}}) + (1-\lambda) \cdot \mathcal{L}(D_{\text{old}}) \tag{8}
$$
其中 $\lambda$ 是新旧数据的混合比例，通常取 0.1~0.5. $\lambda$ 越大，模型越偏向新领域; $\lambda$ 越小，旧知识保留越好但新领域适应速度越慢. 常见的配置包括：

| 混合策略 | 新数据比例 | 旧数据比例 | 适用场景 |
|:--------|:---------:|:---------:|:--------|
| 高比例新数据 | 90% | 10% | 知识更新(新知识量大) |
| 均衡混合 | 50% | 50% | 领域适配(需保持通用能力) |
| 渐进混合 | 逐渐降低新数据比例 | 逐渐增加 | 长期连续训练 |

**实践建议**：

经验法则：**旧数据量至少为领域数据的 $1/3 \sim 1/2$**. 例如：

- 领域数据 100B tokens
- 混合通用数据 50B tokens
- 总训练数据 150B tokens

这样可以有效平衡"注入新知识"和"保持旧知识"之间的矛盾. 

### 3.3 学习率预热 + 余弦衰减

继续预训练通常采用与预训练类似的学习率调度，但参数不同：

```python
# 继续预训练的典型学习率调度
import math

def cosine_schedule_with_warmup(step, warmup_steps, total_steps, min_lr, max_lr):
    """余弦退火 + 预热"""
    if step < warmup_steps:
        # 线性预热
        return min_lr + (max_lr - min_lr) * step / warmup_steps
    else:
        # 余弦衰减
        progress = (step - warmup_steps) / (total_steps - warmup_steps)
        return min_lr + 0.5 * (max_lr - min_lr) * (1 + math.cos(math.pi * progress))

# 典型参数
config = {
    "max_lr": 5e-5,        # 最大学习率(比预训练小10倍)
    "min_lr": 5e-6,        # 最小学习率
    "warmup_steps": 500,   # 预热步数
    "total_steps": 50000,  # 总训练步数
}
```

### 3.4 弹性权重巩固(EWC)

弹性权重巩固(Elastic Weight Consolidation)是一种更高级的缓解遗忘方法，其核心思想是：**对重要参数的保护更强，不太重要的参数可以更自由地调整**. 

$$
\mathcal{L}_{\text{EWC}}(\theta) = \mathcal{L}(D_{\text{new}}, \theta) + \frac{\lambda}{2} \sum_i F_i (\theta_i - \theta_i^*) ^ 2 \tag{9}
$$

EWC 在标准的新数据损失上增加了一个二次正则项，该项惩罚对重要参数的偏离. 这里的 $\theta_i^*$ 是基座模型的原始参数，作为锚点防止模型偏离旧知识过远; $F_i$ 是 Fisher 信息矩阵的对角线元素，衡量每个参数对旧任务的重要性，重要性越高的参数受到的保护越强; $\lambda$ 则是正则化强度，控制新任务适应与旧知识保留之间的权衡. 

**优缺点**：

| 优点 | 缺点 |
|:----|:----|
| 可以对不同参数施加不同强度的约束 | 计算 Fisher 矩阵开销大 |
| 理论上更精确地控制遗忘 | 超参数 $\lambda$ 难以调节 |
| 适合小规模知识更新 | 大规模训练时不实用 |

在实际大规模继续预训练中，**新旧数据混合 + 小学习率** 已经足够有效，EWC 更多用于参数高效微调场景.

## 4. 继续预训练的工程实践

### 4.1 数据处理流程

继续预训练的数据处理与从零预训练类似，但有一些特殊考虑：

```python
# 继续预训练的数据处理 pipeline
def prepare_continued_pretrain_data(
    domain_data_path,     # 领域数据路径
    general_data_path,    # 通用数据路径
    mix_ratio=0.7,        # 领域数据比例
    max_seq_length=4096,  # 最大序列长度
    tokenizer=None        # tokenizer
):
    """准备继续预训练数据"""
    
    # 1. 加载数据
    domain_data = load_dataset(domain_data_path)
    general_data = load_dataset(general_data_path)
    
    # 2. 按比例混合
    domain_size = int(len(domain_data) * mix_ratio)
    general_size = int(len(general_data) * (1 - mix_ratio))
    
    # 3. 去重(避免数据污染)
    domain_data = deduplicate(domain_data)
    
    # 4. 质量过滤(领域数据质量至关重要)
    domain_data = filter_by_quality(domain_data, 
                                     min_score=0.8)  # 质量阈值
    
    # 5. 分块和 tokenize
    dataset = concatenate_and_chunk(
        domain_data + general_data,
        max_length=max_seq_length,
        tokenizer=tokenizer
    )
    
    return dataset
```

### 4.2 训练超参数推荐

以下是在 Llama 2 7B 上进行领域继续预训练的典型配置：

| 参数 | 推荐值 | 说明 |
|:----|:------|:----|
| 学习率 | $5 \times 10^{-5}$ | 预训练的 $1/6$ |
| Batch Size | 1024 ~ 4096 | 与预训练相同量级 |
| 预热步数 | 200 ~ 1000 | 占总步数的 1%~5% |
| 训练步数 | 5000 ~ 50000 | 视数据量而定 |
| 混合精度 | bf16 | 与预训练一致 |
| 梯度裁剪 | 1.0 | 防止个别 batch 导致大幅更新 |
| 权重衰减 | 0.1 | 与预训练一致 |
| 新数据比例 | 50%~90% | 根据遗忘程度调节 |

### 4.3 监控指标

继续预训练过程中需要特别关注的指标：

| 指标 | 监控方式 | 期望行为 |
|:----|:--------|:--------|
| 领域 Loss | 定期在领域评测集上计算 |  持续下降 |
| 通用 Loss | 定期在通用评测集上计算 |  轻微上升可接受，大幅上升需警惕 |
| 领域评测 | MMU/专业知识评测 |  稳步提升 |
| 通用评测 | MMLU/HellaSwag 等 |  下降不超过 2%~5% |
| 梯度范数 | 监控梯度的 L2 范数 |  骤增可能表示灾难性遗忘 |

**决策阈值**：
基于上述分析，建立如下数学关系：
$$
\text{若 } \frac{\Delta \text{Loss}_{\text{general}}}{\Delta \text{Loss}_{\text{domain}}} > 0.2 \text{，需降低学习率或增加通用数据比例} \tag{10}
$$
此式描述了变量之间的定量关系，其物理意义将在下文详细阐述.

## 5. 案例分析

### 5.1 BloombergGPT：金融领域适配

BloombergGPT 是 Bloomberg 基于 50 层 Transformer 训练的金融领域 LLM，其继续预训练策略是**从零训练而非继续训练**，但它的数据配置极具参考价值：

| 数据来源 | 比例 | 规模 |
|:--------|:----:|:----:|
| 金融数据 | 52.9% | 363B tokens |
| 通用数据 | 47.1% | 324B tokens |
| **总计** | **100%** | **687B tokens** |

**关键启示**：即使是从零训练，金融数据也仅占 53%，说明**通用知识的保留极其重要**. 

### 5.2 CodeLlama：代码能力增强

Meta 的 CodeLlama 在 Llama 2 基础上进行了代码领域的继续预训练：

1. **第一阶段**：在 500B 代码 tokens 上继续预训练(学习率 $5 \times 10^{-5}$)
2. **第二阶段**：在代码 + 通用数据混合上继续训练
3. **第三阶段**：代码 SFT

**结果**：HumanEval 得分从 29.4%(Llama 2 70B)提升到 **53.5%**(CodeLlama 34B)，同时通用 NLP 性能几乎无下降. 

### 5.3 中文 Llama 增强

将 Llama 扩展到中文的典型流程：

| 阶段 | 训练数据 | 学习率 | 训练步数 |
|:----|:--------|:-----:|:--------:|
| 1. 中文词表扩展 | 新增中文 token 嵌入层 | — | — |
| 2. 中文语料预热 | 10B 中文通用数据 | $1 \times 10^{-4}$ | 5000 |
| 3. 中文继续训练 | 100B 中文数据 + 50B 通用数据 | $5 \times 10^{-5}$ | 30000 |
| 4. 微调对齐 | 中文指令数据 | $2 \times 10^{-5}$ | 2000 |

**注意事项**：
- 中文词表扩展后需要**预热训练**让新嵌入层稳定
- 中文数据占比从低到高渐进增加
- 全程保留 30%~50% 的英文通用数据防止灾难性遗忘

## 6. 前沿方向与总结

### 6.1 前沿方向

继续预训练领域正在快速发展，以下是最值得关注的几个方向：

| 方向 | 核心思想 | 代表工作 |
|:----|:--------|:--------|
| **自适应混合** | 根据梯度信号动态调整新旧数据比例 | Adaptive CPT |
| **参数隔离** | 将领域知识存储在独立参数模块中 | LoRA-CPT, AdapterFusion |
| **知识蒸馏** | 用基座模型指导领域模型的训练 | Distill-CPT |
| **数据筛选** | 自动化选择最有价值的领域数据 | Data Selection for CPT |
| **持续学习框架** | 真正实现\"一次训练，持续更新\" | Continual LM |

其中：**LoRA-CPT** 是最有潜力的方向之一——通过低秩适配器(LoRA)来注入新知识，原始参数保持不变：

$$
\theta_{\text{new}} = \theta_0 + \Delta \theta_{\text{LoRA}}, \quad \Delta \theta = BA \tag{11}
$$

这种方法天然解决了灾难性遗忘问题，因为 $\theta_0$ 被冻结，永远不会被修改. 

### 6.2 总结

| 核心要点 | 关键建议 |
|:--------|:--------|
| **继续预训练 ≠ SFT** | CPT 注入知识，SFT 对齐行为 |
| **灾难性遗忘是最大挑战** | 旧知识在新知识学习过程中被覆盖 |
| **新旧数据混合** | 最有效的缓解策略，旧数据占比 $1/3 \sim 1/2$ |
| **小学习率** | 控制在预训练学习率的 $1/10 \sim 1/6$ |
| **渐进式训练** | 从低新数据比例开始，逐步增加 |
| **监控通用性能** | 通用 Loss 上升超过阈值需立即调整 |
| **未来方向** | 参数隔离、自适应混合、持续学习框架 |
