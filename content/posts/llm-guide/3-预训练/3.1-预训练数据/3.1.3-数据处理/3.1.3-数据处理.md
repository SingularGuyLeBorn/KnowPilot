# 3.1.3 数据处理

> 未经处理的原始数据只是噪声，精心处理的数据才是模型的燃料. 

## 概述

在上一节中，我们探讨了如何收集海量的预训练数据——从网络爬虫抓取万亿级Token，到引入代码、数学、多语言数据. 然而，原始数据中充斥着重复、噪声、低质量和有害内容，**直接把未经处理的数据喂给模型训练，是通往失败最快的方式**. 

本节将深入探讨数据处理的三个核心环节：**数据配比(Data Mixture)** 、**合成数据(Synthetic Data)** 和 **质量筛选(Quality Filtering)** . 这三个环节共同决定了大语言模型知识体系的质量和广度. 

---

## 1. Data Mixture(数据配比)

### 1.1 为什么需要数据配比

大语言模型的训练数据通常来自多个领域：网页文本(Common Crawl)、学术论文(ArXiv)、代码(GitHub)、书籍、数学推理数据、多语言语料等. 不同领域的数据在数量、质量和语义密度上差异巨大. **数据配比的核心问题**是：如何确定各领域数据的比例，使得模型在多个任务上达到最优性能？

### 1.2 经验配比与理论方法

**LLaMA 系列的配比实践**：
- LLaMA-1：67% 英文网页(C4)+ 15% 书籍 + 4.5% 学术论文 + 4.5% 代码等
- LLaMA-2：90% 英文数据 + 10% 其他语言，强化了代码和数学比例
- LLaMA-3：进一步提升代码和数学数据的占比(>15%)，以增强推理能力

**DoReMi(Data Re-Mixing)** ：
Google 提出的数据配比优化方法，其核心思想是：使用一个小模型先跑一轮训练，观察模型在各个领域上的困惑度(Perplexity)变化，然后根据"哪些领域模型学得最差"来重新分配权重. 具体而言，通过 **Group DRO(Distributionally Robust Optimization)** 方法，给困难领域的样本分配更高的权重. 实验表明，使用 DoReMi 优化配比后，在同等的计算预算下，模型在多个下游任务上取得了一致的性能提升. 

**p(y|x) 方法**：
另一种思路是从"下一个Token预测"任务本身出发，分析不同领域数据对 Loss 的贡献. 如果某个领域数据对 Loss 降低的边际效益递减，则应减少其占比. 这种方法更为动态，但计算开销也更大. 

### 1.3 常见的配比策略对比

| 策略 | 核心思想 | 优点 | 缺点 |
|:----|:--------|:----|:----|
| **均匀采样** | 各领域数据按原始比例混合 | 实现简单 | 噪声领域(如网页)占比过大 |
| **上采样** | 人工调整高质量领域(代码、数学)的采样权重 | 针对性强 | 过度上采样可能导致过拟合 |
| **DoReMi** | 基于小模型的训练动态调整 | 数据驱动，效果有保障 | 需要额外的训练实验 |
| **课程学习** | 训练前期多用通识文本，后期增加代码/数学 | 模拟人类学习节奏 | 调度策略难以确定 |

### 1.4 数据配比的实际案例

**DeepSeek-V2/V3** 的数据配比策略值得关注：
- 中文与英文数据比例约为 3:7
- 代码和数学数据的占比显著高于早期的 GPT 系列，反映了对推理能力的重视
- 引入了"数据质量分数"作为配比的辅助指标——质量越高的领域，不一定数量越多，但采样权重越高

## 2. Synthetic Data(合成数据)

合成数据是近年来大模型训练领域最热门的方向之一. 当高质量的自然数据被"耗尽"时，合成数据成为扩展训练数据的核心手段. 

### 2.1 Self-Instruction

**Self-Instruction** 方法由 Stanford Alpaca 项目推广. 其流程如下：
1. 使用一个种子集(175条人工编写的指令-回答对)
2. 调用强模型(如 GPT-3.5/4)生成新的指令
3. 再让强模型为这些新指令生成回答
4. 将生成的数据加入训练集

这种方法的关键在于**指令多样化**——如果生成的指令过于相似，模型的泛化能力会受限. 后续的改进包括使用 In-context Learning 引导生成不同难度的指令，以及引入约束条件(如要求生成特定领域的指令). 

### 2.2 Evolving(数据演化)

**Evol-Instruct** 由 WizardLM 提出，比 Self-Instruction 更进一步. 它通过**演化操作**提升指令的复杂度：

- **In-depth Evolving**：增加约束条件(如"请给出至少3个理由")、加深推理深度(如"请结合XX理论分析")、增加复杂性(如"请比较A和B的优缺点")
- **In-breadth Evolving**：生成全新的、不同领域的指令，扩大覆盖范围

Evol-Instruct 的核心洞察是：简单的指令只能训练出简单的能力，**指令的复杂度决定了模型能力的天花板**. 后续的 WizardCoder、WizardMath 等工作将该方法推广到了代码和数学领域. 

### 2.3 Distillation(蒸馏)

蒸馏是合成数据的另一种重要形式，其核心思想是：**用强模型生成数据，让小模型学习**. 

**DeepSeek 的实践**：
DeepSeek 系列模型在训练中大量使用了蒸馏技术. 具体做法包括：
- 使用 DeepSeek-R1(基于强化学习训练的推理模型)生成推理链数据
- 将这些数据用作 DeepSeek-V3 和更小模型的训练数据
- 在数学推理和代码生成任务上，蒸馏数据的质量甚至超过了原始标注数据

**教材式合成数据**(Textbook-like Data)：
Phi 系列模型提出了"教科书质量"的合成数据概念. 他们让 GPT-3.5/4 生成带有详细解释、逐步推理的"教科书式"内容，而非简单的问答对. 这些数据的特点是：**逻辑严谨、解释清晰、覆盖全面**. Phi-1 仅用 50 亿 Token 的教科书质量数据，就在代码生成任务上超越了使用万亿级数据训练的模型. 

### 2.4 合成数据的关键挑战

| 挑战 | 表现 | 缓解策略 |
|:----|:----|:--------|
| **模式坍塌** | 生成的数据过于相似，多样性不足 | 引入温度采样、In-context Learning 多样化 |
| **错误放大** | 强模型生成的内容存在错误，小模型学习后放大 | 多模型审核、人工抽样检查 |
| **偏差固化** | 模型学会合成数据中的固有偏见 | 多元化的种子集、对抗性过滤 |
| **边际效益递减** | 合成数据量超过一定阈值后效果不再提升 | 结合真实数据混合训练 |

## 3. Quality Filtering(质量筛选)

### 3.1 去重(Deduplication)

数据重复是预训练中一个被严重低估的问题. 研究表明：
- **重复数据会降低训练效率**：模型反复看到同样的句子，但对"新知识"的增益为零
- **重复数据导致记忆效应**：模型会"背诵"特定文本，而非理解其语义
- 去除重复后，在相同计算预算下，模型的困惑度可降低 5-10%

**MinHash LSH(局部敏感哈希)** ：
这是最常用的大规模文本去重方法. 工作流程如下：
1. **Shingling**：将文档拆分为重叠的 N-gram(通常为 5-gram)
2. **MinHashing**：用多个哈希函数将 N-gram 集合压缩为固定长度的"签名"
3. **LSH(Locality Sensitive Hashing)** ：将签名分桶，签名相似的文档落入同桶
4. **比较**：仅在桶内进行两两比较，大大降低了 O(n²) 的计算复杂度

经过 MinHash LSH 去重后，Common Crawl 数据通常可减少 20-40% 的冗余. 

**去重的层次**：
- **文档级去重**：去除完全重复或近似重复的文档
- **段落级去重**：去除文档内部或跨文档的重复段落
- **句子级去重**：精确到句子级别的 N-gram 去重(如 Exact Substring Dedup)

### 3.2 启发式规则过滤

启发式规则是最基础但也最有效的过滤手段：

| 规则类型 | 过滤目标 | 示例 |
|:--------|:--------|:----|
| **语言过滤** | 非目标语言文档 | fastText 语言分类器 |
| **长度过滤** | 过短/过长文档 | 少于50个Token或超过10万的文档 |
| **困惑度过滤** | 低质量文档(高困惑度 = 质量差) | 用 KenLM 计算 n-gram 困惑度 |
| **特殊字符过滤** | 乱码/HTML残留 | 非字母字符占比 > 40% 的文档 |
| **广告/导航过滤** | 非内容正文 | 提取 main-content 区域 |
| **毒性过滤** | 有害内容 | 关键词匹配 + 分类器 |

### 3.3 基于模型的质量分类

**分类器过滤**是启发式规则的升级版：
1. 人工标注一批高质量和低质量的文档样本
2. 训练一个二分类器(常使用 BERT/RoBERTa)
3. 用分类器对所有文档评分
4. 设定阈值过滤低质量文档

这种方法效果比启发式规则更好，但需要高质量的标注数据. GPT-3 的报告中提到，他们训练了一个质量分类器来筛选 WebText 数据. 

**FastText 质量分类**：
CCNet(Common Crawl Neural Network)项目使用 fastText 对文档进行质量评分，设置多级阈值：
- 过滤：去除评分 < 0.3 的文档
- 低质量：0.3 ~ 0.6(可选择性保留)
- 高质量：> 0.6(核心训练数据)

### 3.4 毒性过滤与安全筛选

现代大模型训练中，毒性过滤已成为标准流程：
- **关键词过滤**：基于黑名单关键词移除含暴力、色情、仇恨言论的文档
- **分类器过滤**：使用 Perspective API 或其他毒性检测模型评分
- **PII(个人身份信息)移除**：正则匹配移除邮箱、电话、地址等信息

## 4. 数据处理的全流程

将以上三个环节串联起来，一个典型的数据处理 Pipeline 如下：

```
原始数据 → 语言过滤 → 去重(文档级) → 质量筛选 → 毒性过滤 → 去重(句子级) → 数据配比 → 合成数据扩增 → 混洗 → 训练
```

**数据处理的计算成本**不可忽视. 对于万亿级的预处理流程，通常需要数千 CPU 核心运行数天. 因此，**数据处理的效率优化**本身就是一个工程问题——包括使用 MapReduce 框架进行分布式去重、使用 GPU 加速质量分类器等. 

---

## 总结

数据处理不是预训练前的"预处理杂务"，而是决定模型能力上限的**战略性工程**. 三个核心环节各有侧重：

- **数据配比**决定了模型的"知识结构"——偏重推理还是偏重通识
- **合成数据**解决了"数据天花板"问题——当自然数据枯竭时继续扩展
- **质量筛选**守住了"数据底线"——低质量数据的渗入会拖垮整个训练
