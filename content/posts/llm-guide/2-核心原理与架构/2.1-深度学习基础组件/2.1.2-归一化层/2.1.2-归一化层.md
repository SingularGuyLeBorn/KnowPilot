# 2.1.2 优化曲面的平滑剂：归一化层

在深度学习的早期，训练深层网络几乎像一门玄学. 我们小心翼翼地调整初始化、学习率和梯度裁剪，只希望梯度信号能活着穿过几十层网络，而不要在中途彻底消失或爆炸. 那时的损失曲面像一片崎岖山脉，优化器稍有不慎就会跌进局部陡坡，训练稳定性几乎完全依赖经验. 

归一化层的出现，改变了这个局面. 它不直接增加模型容量，也不直接提升某个 benchmark 分数，但它通过约束中间表示的数值分布，让优化过程变得更平滑、更可控、更容易扩展到深层网络. 换句话说，归一化层像一个「数值稳定器」：它不替代 Attention、FFN 或位置编码，但它让这些核心模块有机会在更安全的数值范围里工作. 

对于现代大语言模型来说，归一化层的重要性反而比早期更高. 模型越深、参数越多、上下文越长、精度越低，对数值稳定性的要求就越严苛. BatchNorm、LayerNorm、RMSNorm、Pre-Norm、Post-Norm 这些看似只是实现细节的选择，最终都会影响训练是否稳定、推理是否高效、量化是否可用、长上下文是否能真正跑通. 

本文围绕五个问题展开：

1. 归一化层到底在稳定什么. 
2. 为什么 BatchNorm 在视觉里成功，却不适合 LLM 主干. 
3. 为什么 LayerNorm 成为 Transformer 时代的核心，而 RMSNorm 又进一步替代它. 
4. 为什么 Pre-Norm 几乎成为大模型默认结构. 
5. 归一化层为什么会成为训练、量化、长上下文和工程部署里的隐性关键模块. 

## 1. 归一化层到底在解决什么问题

最常见的粗略说法是：归一化层可以「减少内部协变量偏移」. “协变量”这里可以简单理解成**参与描述输入或中间表示分布的特征变量**; “偏移”则是说这些变量的统计范围和形态在训练过程中不断变化. 这个说法有启发，但并不够准确. 更稳妥的理解是：归一化层通过约束中间激活的尺度与分布，让**优化景观变得更平滑**，从而让梯度下降更可预测、学习率更宽容、深层训练更稳定. 

这件事可以从三个层面理解. 第一，激活值如果层层放大或层层塌缩，后面的模块就会在异常数值范围里工作. 所谓「放大」，就是层输出的幅值越来越大; 所谓「塌缩」，就是层输出越来越接近零. 这会导致注意力 logits、FFN 中间态或残差主干不断漂移. 这里的 `logits` 可以简单理解成**模块在做 softmax 或离散决策前给出的原始分数**; 而「残差主干」指的是**跨层直接相加的那条主路径**，它负责把早期信息稳定地传到更深层. 

第二，混合精度训练会把这类数值问题进一步放大，因为 BF16、FP16 乃至更低精度对异常幅值更敏感. 第三，网络越深，这些小的分布偏移越容易在主干上逐层累计，最终破坏整个优化过程. 这里的「分布偏移」，指的就是激活值的均值、方差或整体尺度在层与层之间不断变化，后面的模块不得不持续适应新的输入范围. 

因此，归一化层的工作不是「让所有层都长一样」，而是让它们不要失控. 它像一个持续在线的数值约束器，把原本可能任意发散的表示，重新拉回优化器还能处理的范围之内.

## 2. BatchNorm：卷积时代的关键发明，为什么不适合 LLM 主干

[Ioffe & Szegedy (2015)](https://arxiv.org/abs/1502.03167) 提出的 Batch Normalization，简称 **BatchNorm**，是深度学习训练史上的里程碑. 这里的 `mini-batch` 可以简单理解成**一次参数更新时同时送入模型的一小批样本**. 它的核心想法是：利用当前 mini-batch 的统计量，对每个特征维度做标准化，再通过可学习参数恢复表示能力. 

给定一个批次输入 $B=[x_1, ..., x_m]$，对某个特征维度，BatchNorm 先计算批次均值与方差：

$$
\mu_B=\frac{1}{m}\sum_{i=1}^{m}x_i
$$

这里 $\mu_B$ 表示当前 batch 在这一维度上的平均水平. 它的作用是先定位当前这批样本的中心位置，为后续标准化提供参照. 

$$
\sigma_B^2=\frac{1}{m}\sum_{i=1}^{m}(x_i-\mu_B)^2
$$

这里 $\sigma_B^2$ 表示当前 batch 在这一维度上的离散程度，也就是这批样本围绕均值波动有多大. 只有知道中心和波动范围，后面才能把不同 batch 拉回相近尺度. 

然后执行标准化与可学习仿射变换：

$$
\hat{x}_i=\frac{x_i-\mu_B}{\sqrt{\sigma_B^2+\epsilon}}
$$

这一步把输入重新压到更稳定的数值范围里. 这里 $\epsilon$ 是一个很小的数值稳定项，用来避免分母过小甚至为零. 

$$
y_i=\gamma \hat{x}_i+\beta
$$

这里 $\gamma,\beta$ 是可学习参数. 直观上，BatchNorm 的作用是把当前 batch 的该维特征重新拉回「零均值、单位方差」的尺度附近，再允许模型自己学会恢复最合适的缩放和平移. 如果没有这一步，归一化会把表示固定得过死，反而削弱网络表达能力. 

在卷积网络中，这种方法非常有效. 因为图像任务里，同一层特征在 batch 内往往有较强统计一致性，且卷积结构本身也更容易从 batch 统计里获益. BatchNorm 因而显著提升了 CNN 的训练稳定性和收敛速度. 

但它并不适合作为 LLM 主干的归一化方案，原因主要有三个. 第一，NLP 序列长度动态变化，padding token 很容易污染 batch 统计量. 第二，自回归推理是逐 token 生成，推理时很难获得与训练一致的 batch 统计. 第三，大模型训练和部署阶段常常使用很小的 micro-batch，这会让 BatchNorm 的统计量本身变得噪声很大. 

所以 BatchNorm 在视觉模型里是皇帝，在 LLM 主干里却基本退出舞台. 理解这一点很重要，因为它说明了一个核心事实：**归一化不是“随便选一种都行”，而是强依赖数据结构和推理模式. **

## 3. LayerNorm：为什么 Transformer 时代改成了按特征归一化

[Ba, Kiros, & Hinton (2016)](https://arxiv.org/abs/1607.06450) 提出的 **Layer Normalization**，简称 **LayerNorm**，可以视为归一化轴的一次关键旋转：它不再跨 batch 统计，而是对**单个样本内部的特征维度**做归一化. 

对于一个 token 表示 $x\in\mathbb{R}^d$，LayerNorm 先计算该 token 的层内均值与方差：

$$
\mu_L=\frac{1}{d}\sum_{j=1}^{d}x_j
$$

这里 $\mu_L$ 是当前 token 在所有特征维度上的平均值. 它回答的问题不是“这一批样本平均怎样”，而是“这一个 token 自己当前整体偏到哪里去了”. 

$$
\sigma_L^2=\frac{1}{d}\sum_{j=1}^{d}(x_j-\mu_L)^2
$$

这里 $\sigma_L^2$ 是当前 token 内部各维特征的波动范围. 它衡量的是这个表示向量本身是否过度发散，而不是 batch 内不同样本之间是否一致. 

然后进行标准化与仿射变换：

$$
\text{LN}(x)=\frac{x-\mu_L}{\sqrt{\sigma_L^2+\epsilon}}\odot \gamma+\beta
$$

其中 $\gamma,\beta\in\mathbb{R}^d$ 是逐维度可学习参数，$\odot$ 表示逐元素乘法. 和 BatchNorm 相比，LayerNorm 的关键变化在于：它完全不依赖 batch 统计，因此对 batch size 不敏感，对 padding 噪声也不敏感，训练和推理阶段的行为更加一致. 

这恰好击中了 NLP 和 Transformer 的核心需求. 每个 token 都有自己的表示，每个位置都可以独立完成归一化，不必依赖同批次中其他样本的分布. 这使 LayerNorm 成为 Transformer 时代真正意义上的基础设施.

## 4. RMSNorm：为什么大模型开始抛弃均值中心化

[Zhang & Sennrich (2019)](https://arxiv.org/abs/1910.07467) 提出的 **RMSNorm**，可以看作对 LayerNorm 的一次极简化：它保留了尺度归一化，但去掉了均值中心化. 

其核心定义是：

$$
\text{RMS}(x)=\sqrt{\frac{1}{d}\sum_{j=1}^{d}x_j^2+\epsilon}
$$

这里的 RMS 可以理解成「均方根尺度」，也就是这个向量整体有多大. 它关心的是表示的模长是否稳定，而不是均值是否恰好落在零点附近. 

$$
\text{RMSNorm}(x)=\frac{x}{\text{RMS}(x)}\odot g
$$

这里 $g\in\mathbb{R}^d$ 是逐维度可学习增益参数. 与 LayerNorm 相比，RMSNorm 不再计算均值 $\mu_L$，也不再执行减均值操作，只保留「按均方根缩放」这一步. 

为什么这样做居然有效？因为在很多高维表示空间里，真正重要的是**向量的尺度是否可控**，而不是均值是否恰好被移到零点. 对大模型来说，去掉均值分支意味着：

- 少一组 reduce 统计
- 少一次减法路径
- kernel 更容易融合
- 数值路径更短

从工程角度看，这正是大模型时代非常需要的简化. RMSNorm 因此成为 Llama、Qwen 等现代大模型的主流选择.

## 5. Pre-Norm 为什么几乎成为大模型默认结构

归一化层不只是谁来做、怎么算的问题，还涉及它**放在残差结构的哪里**. 这就是 Transformer 发展史上的一个关键分歧：**Post-Norm** 与 **Pre-Norm**. 前者可以先粗略理解成「先做子层和残差相加，再归一化」，后者则是「先归一化，再送入子层」. 

Post-Norm 写成：

$$
x_{l+1}=\text{Norm}(x_l+F(x_l))
$$

Pre-Norm 写成：

$$
x_{l+1}=x_l+F(\text{Norm}(x_l))
$$

这两个公式看起来只是顺序交换，但训练行为差异很大. Pre-Norm 的真正优势在于，它给残差主干保留了一条更清晰的「恒等高速路」. 把式子递归展开，就能看到输入信号可以更直接地跨层累积，而不必在每层都先穿过一个归一化层再回传梯度. 

这也是为什么模型越深，Pre-Norm 越占优. 对百层级网络来说，训练最怕的不是单层有一点不稳定，而是小的不稳定被主干重复累计. Pre-Norm 更容易维持残差路径的可训练性，因此几乎成为现代大模型的默认选择. 

## 6. 归一化层为什么会成为工程隐性瓶颈

归一化层不是参数大头，也不是 FLOPs 大头，但它常常是**数值稳定性、精度路径和 kernel 融合**的关键节点. 这里的 `kernel` 可以简单理解成 **GPU 上执行某一步计算的底层算子**; 所谓「融合」，就是把多步本来分开的操作拼成一次更大的底层计算，减少中间读写与调度开销. 很多时候，真正的工程难点不在于“归一化本身复杂”，而在于它刚好卡在所有主干模块的输入输出之间. 

先看训练阶段. 归一化层会参与每一次前向、每一次反向，而且它的 `reduce` 操作天然比简单逐元素算子更敏感. 这里的 `reduce` 指的是**沿某个维度把很多数聚合成少量统计量**，例如求和、求均值、求方差. 若统计量不在 FP32 中累加，或 `fused kernel` 与 `reference implementation` 存在微小差异，这些误差会在长训练中被放大. 这里的 `reference implementation` 就是**最标准、最直接、通常也最慢的对照实现**，它常被用来检查高性能实现有没有算偏. 

再看推理阶段. 归一化层虽然单层不重，但层数很多，而且常常需要和 residual add、dropout、linear projection 一起融合，才能避免 `kernel launch` 过碎. 这里的 `kernel launch` 可以理解成**GPU 每调用一次底层算子都要付出的调度成本**; 步骤拆得太碎，调度本身就会变贵. 对大模型部署来说，归一化层往往属于「单层不贵、总量不小、并且很容易被忽视」的那类模块. 

最后看量化. 归一化层本质上是在管理激活的尺度分布. 只要这一步失真，后面的 FFN、Attention 或输出层量化误差都会被系统性放大. 所以归一化层虽然不是量化压缩的主要对象，却常常是量化成败的隐形前提. 

## 7. 训练、量化与长上下文里该重点看什么

归一化层的工程价值，最终会体现在一组非常具体的检查项上. 训练时，最值得监控的是：

1. 归一化前后激活 RMS 或方差是否稳定. 
2. 增益参数 $\gamma$ 是否出现异常漂移. 
3. LayerNorm / RMSNorm 输出分布是否在深层持续扩张或塌缩. 
4. 混合精度下是否出现 reduce 路径溢出. 
5. fused kernel 与 reference 实现是否数值一致. 

量化时，应重点关注：

1. 归一化输出分布是否仍与高精度版本接近. 
2. norm 层是否需要在更高精度中执行. 
3. 量化前后余弦相似度是否在深层明显恶化. 

长上下文时，则更应关注：

1. 归一化输出尺度是否随长度逐步漂移. 
2. 残差主干幅值是否累积放大. 
3. 深层 attention / FFN 前的输入分布是否越来越极端. 

这些指标看起来琐碎，但它们往往比单看全局 loss 更接近问题本源. 这里的 `loss` 就是**模型当前输出与目标之间的总体误差**; 它能告诉你“整体好不好”，却不一定能告诉你“是哪一层先坏了”. 很多「模型怎么突然不稳了」的问题，最后都会在归一化层的统计上留下痕迹.

## 8. 极简实现：LayerNorm 与 RMSNorm 的 PyTorch 映射

下面用极简代码说明这两类归一化层在实现上的核心差别：

```python
import torch
import torch.nn as nn


class SimpleLayerNorm(nn.Module):
    def __init__(self, dim: int, eps: float = 1e-5):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(dim))
        self.bias = nn.Parameter(torch.zeros(dim))
        self.eps = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        mean = x.mean(dim=-1, keepdim=True)
        var = (x - mean).pow(2).mean(dim=-1, keepdim=True)
        normed = (x - mean) / torch.sqrt(var + self.eps)
        return normed * self.weight + self.bias


class SimpleRMSNorm(nn.Module):
    def __init__(self, dim: int, eps: float = 1e-5):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(dim))
        self.eps = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return (x / rms) * self.weight
```

这段代码清楚说明了两点. 第一，LayerNorm 比 RMSNorm 多出「求均值 -> 减均值」这一分支. 第二，RMSNorm 的简化并不是偷懒，而是有意识地把归一化保留在“尺度控制”这个最必要的层面上. 这里的 `affine` 可以简单理解成**最后那一步可学习的缩放和平移变换**，它让模型保留“必要时把归一化部分撤回来”的自由度.

## 9. 技术前瞻：归一化层还会怎么演化

归一化层不是一个已经完全收敛的话题. 至少还有四条趋势值得继续关注. 

第一，面向 fused kernel 的归一化设计会继续增强. 未来很多归一化方案是否能成为主流，不只取决于数学解释，还取决于它是否容易被高效融合进推理路径. 

第二，低比特与混合精度会继续反向塑造归一化设计. 随着 FP8、FP6、FP4 路线推进，归一化的统计路径、缩放路径和精度路径会变得更敏感. 

第三，极深网络与长上下文会继续放大主干稳定性问题，这意味着 Pre-Norm、残差幅值控制和归一化层的协同设计仍然是核心课题. 

第四，多模态模型会继续挑战「同一种归一化适合所有模态」这个假设. 文本、图像、音频、视频共享主干之后，归一化层很可能承担更多模态对齐职责，而不只是数值稳定职责. 

## 10. 本节小结

归一化层不是大模型里最性感的模块，但它几乎是最「不容失手」的模块之一. 这里的 `benchmark` 可以简单理解成**一组标准化评测任务上的分数**. 它的价值不在于单独抬高某个 benchmark 分数，而在于为整个系统提供一个可扩展、可训练、可量化、可部署的数值地基. 

从 BatchNorm 到 LayerNorm，再到 RMSNorm 与 Pre-Norm 路线，归一化层的演化逻辑其实非常一致：**谁更符合现代模型的训练方式、推理方式和硬件约束，谁就会成为主流. ** 对 LLM 而言，归一化层不是小配件，而是决定模型能否「跑得深、跑得稳、跑得长、跑得省」的关键基础设施之一.
