# 2.1.1 静态知识的存储层：前馈网络 FFN 与激活函数

在很多入门材料里，Transformer 的前馈网络 Feed-Forward Network，简称 **FFN**，常被介绍成「注意力后面跟着的两层 MLP」. 这句话没有错，但**严重低估了 FFN 的地位**. 对于现代大语言模型来说，FFN 不是一个附属零件，而是决定模型容量、知识内化、训练稳定性、推理成本和后续压缩路线的核心模块之一. 若把注意力机制看成「决定当前 token 该从哪里取信息」，那么 FFN 更接近「决定这些信息进入当前表示后该被怎样加工、放大、筛选和固化」. 

这也是为什么工程实践里，真正拉开模型差距的往往不只是注意力头数、位置编码或上下文长度，而是 **FFN 的宽度设计、激活函数形态、门控结构、量化策略和稀疏化方案**. BERT、GPT-2、GPT-3 那一代模型主要采用基于 GELU 的标准 FFN; PaLM、Llama、Qwen、Mistral、DeepSeek 这一代主流大模型则大规模转向 SwiGLU 等门控 FFN; 再进一步，MoE 路线几乎总是把稀疏专家安在 FFN 上，而不是注意力层上. 这不是偶然，而是因为 FFN 天然承载了**参数大头**和**表达大头**. 

本文不满足于把 FFN 讲成「升维、激活、降维」三个词，而是从 **演进脉络、数学公式、几何直觉、工程成本、训练诊断、量化与 MoE 路线**几条线并行展开，回答四个问题：

1. FFN 在 Transformer 里究竟承担什么职责. 
2. 为什么激活函数不是细枝末节，而是 FFN 的真正决策点. 
3. 为什么现代模型纷纷从 ReLU、GELU 走向 SwiGLU. 
4. 为什么 FFN 经常才是训练、推理、量化与稀疏化的主战场. 

## 1. FFN 在 Transformer 中到底是什么

Transformer 的一个标准块通常由两部分组成：**注意力子层**和 **FFN 子层**. 注意力子层负责跨 token 交互，FFN 子层则对每个 token 的隐藏状态独立做高容量非线性变换. 这里的「独立」很容易被误解. 它不是说 FFN 与上下文无关，而是说 FFN **不直接比较不同 token**; 它处理的输入已经是「吸收过上下文的 token 表示」. 换句话说，注意力把别处的信息搬过来，FFN 决定这些信息在当前 token 身上最终变成什么. 

若输入序列中的某个 token 原本只是一个粗糙表示，经过注意力聚合后，它可能同时携带事实知识、语法约束、任务指令、风格提示、长程依赖和局部模式. FFN 的职责就是把这堆混合信息重新投影到一个更大的特征空间里，通过非线性激活做筛选和重组，再压回模型维度. 这一过程既像「特征加工」，也像「参数化记忆检索」，还是「局部计算中心」. 所以 FFN 不是纯粹的后处理层，它往往决定了模型能否把注意力带来的上下文真正变成有用表示. 

从参数规模上看，FFN 也远比很多人直觉里更重要. 以标准 Transformer 为例，注意力部分包含 $Q,K,V,O$ 四个投影矩阵，而标准 FFN 至少包含两个大矩阵 $W_1,W_2$; 当隐藏维度扩展到 $d_{ff}=4d$ 时，FFN 参数量通常已经超过单层总参数的一半. 若使用 SwiGLU 这类门控 FFN，虽然宽度会做参数守恒调整，但 FFN 仍然常常是单层参数大头. 也正因此，很多模型压缩、量化、蒸馏、专家化工作最终都**绕不过 FFN**. 

## 2. 演进脉络：从 MLP 到 ReLU、GELU 再到 SwiGLU

### 2.1 MLP 的基础：通用逼近定理解决了「能不能」，没有解决「怎么训好」

FFN 的祖先是最传统的多层感知机 MLP. MLP 的理论根基来自通用逼近定理. [Cybenko (1989)](https://link.springer.com/article/10.1007/BF02551274) 和后续 [Hornik (1991)](https://www.sciencedirect.com/science/article/pii/089360809190009T) 等工作证明：只要有至少一个隐藏层并配备非线性激活，MLP 就能以任意精度逼近紧致空间上的连续函数. 这个结论非常重要，因为它告诉我们，神经网络不是只能做线性映射，它有足够强的函数表达能力. 

若只看一个最基本的单隐藏层 MLP，它通常写成：

$$
\text{MLP}(x)=W_2\,\phi(W_1x+b_1)+b_2
$$

其中 $x$ 是输入向量，$W_1,b_1$ 负责把输入映射到隐藏层，$\phi$ 是 ReLU、Tanh、Sigmoid、GELU 这类非线性激活函数，$W_2,b_2$ 再把隐藏表示映射为输出. 这个公式的关键点不在于它有两层线性变换，而在于中间必须插入非线性 $\phi$; 没有这一步，多层线性映射仍然可以合并成单层线性变换，模型表达能力不会真正提升. 

但通用逼近定理只回答了「表示能力存在」，并没有回答深度学习真正困难的问题：怎样的激活函数更容易训练、怎样的结构更稳定、怎样的参数配置更适合大规模优化、怎样才能在百亿参数量级上既保留表达力又不把硬件成本炸穿. 也正因此，神经网络结构的真正演化并不是从「不能表达」走向「可以表达」，而是从「理论上可行」走向「在大规模系统里可训练、可部署、可压缩、可扩展」. 

Transformer 继承了 MLP 的基本骨架，但做了两个关键改造. 第一，FFN 不在 token 之间混合，而是对每个 token 的隐藏状态独立应用同一组参数，这形成了 position-wise 结构. 第二，隐藏宽度被显著放大，通常取 $d_{ff}=4d$，让模型在更高维空间里展开特征后再做非线性选择. 换言之，Transformer 的 FFN 不是对 MLP 的简单复用，而是在「逐 token 独立计算」的约束下，把 MLP 改造成了一个高维特征加工器. 

### 2.2 ReLU：训练深网络的第一代实用激活

ReLU，Rectified Linear Unit，定义为：

$$
\text{ReLU}(x)=\max(0,x)=
\begin{cases}
x, & x>0 \\
0, & x\le 0
\end{cases}
$$

它的优势非常直接：实现简单、导数简单、在正区间梯度恒为 1，不像 Sigmoid 和 Tanh 那样容易在大幅值输入下饱和. AlexNet 之后，ReLU 几乎成为深层网络的默认激活函数，因为它显著缓解了梯度消失问题. 

在 FFN 语境里，ReLU 的作用可以理解为「硬阈值门」. 当某个方向上的响应为负时，直接压到零; 当响应为正时，保留原值. 这样一来，FFN 会自然产生较高稀疏性. 大量神经元在单次前向中完全不激活，只有一部分通道真正参与计算. 这种稀疏性既带来一定的表达分解能力，也让模型在某种意义上更像「选择性触发的规则集合」. 

但 ReLU 的问题同样鲜明. 最典型的是 dead ReLU，即神经元如果长期落在负区间，就可能永久输出零，导致该通道几乎失活. 对于大模型里的宽 FFN 来说，少量死神经元不一定立刻致命，但如果叠加不良初始化、过大学习率或分布漂移，它会影响层内有效容量. 此外，ReLU 的非平滑折点也使得模型在零附近的数值行为不够柔和，这在极深网络和更复杂优化条件下并不总是理想.

### 2.3 GELU：从「硬阈值」走向「软筛选」

GELU，Gaussian Error Linear Unit，最常见定义为：

$$
\text{GELU}(x)=x\Phi(x)=x\cdot \frac{1}{2}\left(1+\text{erf}\left(\frac{x}{\sqrt{2}}\right)\right)
$$

它和 ReLU 的根本区别在于：GELU 不是把负值一刀切掉，而是用一个平滑概率门去调制输入. 输入越大，被保留的概率越高; 输入为负时也不是绝对清零，而是渐进地衰减. 可以把它理解成「软版本的特征保留策略」. 

GELU 的价值不只是平滑. 更重要的是，它保留了负区间的微弱梯度，从而减少神经元永久失活的风险; 同时又不像 Tanh 那样在大区间里过度饱和. 在 BERT、GPT-2、GPT-3 这一代标准 dense Transformer 中，GELU 成为非常常见的选择，核心原因就是它在大规模预训练中展现出更稳定的优化行为. 

但 GELU 仍然有一个结构性限制：它对所有通道使用的是同一条激活曲线. 也就是说，虽然它解决了「阈值太硬」的问题，却没有解决「能否根据输入内容动态调节不同支路开关程度」的问题. 通道仍然是在统一曲线下被逐元素处理，而不是通过另一条支路显式决定「这条支路此刻应开到多大」. 这就为更强的门控结构留下了空间. 

### 2.4 SwiGLU：从激活函数升级为门控结构

[Shazeer (2020)](https://arxiv.org/abs/2002.05202) 提出的 SwiGLU 可以写成：

$$
\text{SwiGLU}(x)=\left(\text{SiLU}(xW_g)\odot (xW_u)\right)W_d
$$

其中 $\text{SiLU}(z)=z\sigma(z)$，$\odot$ 表示逐元素乘法. 和标准 FFN 相比，SwiGLU 最关键的变化不是把 GELU 换成 SiLU，而是引入了双支路与乘性门控：一条支路产生门控信号，另一条支路产生候选信息，二者逐元素相乘后再映射回模型维度. 

这件事的含义非常大. 标准 ReLU/GELU 更像「每个神经元各自过一遍统一非线性」; SwiGLU 则更像「先计算内容，再计算这个内容应被放行多少」. 模型不再只是在通道内做被动激活，而是在通道层面做条件化调制. 对于复杂任务输入，这种乘性门控通常能提供更细致的表示选择能力. 

现代大模型大规模转向 SwiGLU，不是偶然，也不是为了追求更炫的结构，而是因为它在同等参数预算下往往能提供更好的下游性能、更强的条件选择性，以及更适合大模型训练的表达能力. Llama、Qwen、PaLM、Mistral、DeepSeek 等模型都在不同程度上体现了这一趋势. 

## 3. 数学形式：标准 FFN、GELU 与 SwiGLU 怎么写

### 3.1 标准 FFN

最标准的 FFN 形式是：

$$
\text{FFN}_{std}(x)=\phi(xW_1+b_1)W_2+b_2
$$

其中 $x\in\mathbb{R}^{1\times d}$ 是单个 token 的隐藏状态，$W_1\in\mathbb{R}^{d\times d_{ff}}$ 把输入投影到更高维度，$W_2\in\mathbb{R}^{d_{ff}\times d}$ 再把结果压回模型维度. $\phi$ 是激活函数，常见取 ReLU 或 GELU. 现代大语言模型里，偏置项经常被省略，以减少参数和简化数值行为，所以也常见写成：

$$
\text{FFN}_{std}(x)=\phi(xW_1)W_2
$$

若没有中间的非线性 $\phi$，那么两个矩阵乘法本质上可以合并为一个大矩阵，整个网络退化成线性变换. 也就是说，FFN 的灵魂并不在「有两个矩阵」，而在「这两个矩阵中间必须有非线性」.

图 6：标准 FFN 的数据流. 输入表示先从模型维度 $d$ 升到中间维度 $d_{ff}$，经过非线性筛选后，再压回 $d$，并且这一过程对每个 token 独立重复执行. 

### 3.2 ReLU 的分段线性形式

若在标准 FFN 中取 $\phi=\text{ReLU}$，则有：

$$
\text{FFN}_{\text{ReLU}}(x)=\text{ReLU}(xW_1+b_1)W_2+b_2
$$

其中 ReLU 本身是一个典型的分段线性函数：

$$
\text{ReLU}(x)=
\begin{cases}
x, & x>0 \\
0, & x\le 0
\end{cases}
$$

这也是为什么 ReLU 常被视作「硬阈值门」. 它不会像 GELU 那样做平滑概率筛选，而是直接把负响应截断为零，把正响应原样放行. 表达上足够直接，优化上也足够稳定，但代价是负区间梯度完全消失. 

### 3.3 GELU 的工程近似

[Hendrycks & Gimpel (2016)](https://arxiv.org/abs/1606.08415) 给出的 GELU 精确形式涉及误差函数 $\text{erf}$，实际工程实现里常用 tanh 近似：

$$
\text{GELU}(x)\approx 0.5x\left(1+\tanh\left[\sqrt{\frac{2}{\pi}}\left(x+0.044715x^3\right)\right]\right)
$$

这个近似的重点不在于记住常数，而在于理解它的目的：在保留平滑软门特性的同时，降低直接计算误差函数的成本. 对训练者来说，最重要的不是推导这些常数，而是意识到 GELU 的本质是「按输入大小连续调节保留程度」，而不是「正的留、负的全砍」. 

### 3.4 SwiGLU 的三矩阵结构

SwiGLU 之所以常被写成 gate、up、down 三个投影，是因为它不再是单一路径：

$$
g=xW_g,\quad u=xW_u,\quad h=\text{SiLU}(g)\odot u,\quad y=hW_d
$$

这里 $W_g$ 产生门控支路，$W_u$ 产生候选特征支路，$W_d$ 负责降维回到模型维度. 与标准 FFN 相比，矩阵数从两个变成三个，因此如果仍让中间宽度保持 $4d$，参数量会明显增加. 为控制参数预算，现代实现通常会把中间宽度下调到接近 $8d/3$. 

### 3.5 为什么 SwiGLU 常取 $8d/3$

标准 FFN 若取 $d_{ff}=4d$，参数量约为：

$$
P_{std}=d\cdot 4d+4d\cdot d=8d^2
$$

SwiGLU 若设中间维度为 $m$，参数量约为：

$$
P_{swiglu}=d\cdot m+d\cdot m+m\cdot d=3dm
$$

为了与标准 FFN 大致保持同量级参数预算，令 $3dm\approx 8d^2$，得到：

$$
m\approx \frac{8d}{3}
$$

这就是为什么很多实现把 SwiGLU 的中间维度设为 $8d/3$ 左右. 它不是神秘常数，而是「在三矩阵结构下保持总参数量可接受」的直接结果. 

如果只看大 O 记号，标准 FFN 的主计算量大约是两次大矩阵乘法，复杂度近似为 $O(dd_{ff})$; SwiGLU 则是三次投影加一次逐元素乘法，主复杂度也还是被投影矩阵主导. 理论上这是高效的密集线性代数问题，现实里却不总是算力瓶颈，因为推理阶段尤其是单 token 解码时，带宽和权重搬运常常更早成为限制. 这也是下一节工程分析的核心切入点：FFN 的难点不只是「算多少」，更是「搬多少」. 

## 4. 几何直觉：升维、折叠、门控与键值记忆

从几何视角看，$W_1$ 的作用不是简单“做大矩阵乘法”，而是把输入表示投影到一个更高维空间. 原本在低维模型空间里纠缠在一起的语义模式，到了更高维的 $d_{ff}$ 空间后，更有机会在不同方向上被分开. 你可以把它理解成把一团挤在一起的信息拉开，方便后续非线性做选择和裁剪. 

如果没有这个高维展开空间，模型只能在原始维度里做有限混合，那么很多本应被拆开的特征会继续挤在一起. FFN 的“宽”本质上就是用空间换表达能力：给模型一个足够大的中间空间，让它把不同语义因子、风格因子和任务因子先拆开，再决定保留什么. 

激活函数的直觉可以理解为空间折叠. ReLU 会把负半轴折叠到零，形成很多分段线性的决策边界; GELU 会以更平滑方式对这些边界做软处理; SwiGLU 则不只是折叠，而是先算门、再算值、最后做乘性调制. 这意味着它不只是“把某个方向变弯”，而是让不同方向之间发生条件耦合. 

这也是为什么从表达能力角度看，SwiGLU 常被认为比简单逐元素激活更强. 它允许一条支路决定另一条支路的放行程度，本质上相当于把“是否保留某个模式”做成了输入条件相关的函数，而不是固定阈值. 

![](./images/image_6.png)

图 7：FFN 的几何直觉. 升维先把纠缠特征拉开，激活函数再对这些方向做折叠、截断或门控选择，从而形成更强的可分离表示. 

[Geva et al. (2020)](https://aclanthology.org/2021.emnlp-main.446/) 的经典工作把 Transformer 的 FFN 解释成键值记忆. 其核心直觉是：$W_1$ 的列向量可以看成很多“键”，它们负责检测输入是否包含某种模式; $W_2$ 的行向量可以看成对应的“值”，它们负责把匹配到的模式映射为输出修正. 形式上可以写成：

$$
\text{FFN}(x)\approx \sum_{i=1}^{d_{ff}}\phi(x\cdot k_i)\,v_i
$$

这里 $k_i$ 可理解为第 $i$ 个模式探测器，$v_i$ 是与之对应的输出写回向量. 输入若与某些键高度匹配，这些键对应的值就会被加权提取出来. 这一解释极具启发性，因为它说明 FFN 不只是通用函数拟合器，它还像一个「分布式静态知识库」：事实、风格、局部变换规则，往往都可以以这种键值方式储存在参数里. 

![](./images/image_3.png)

图 8：FFN 的键值记忆视角. 输入表示先在 $W_1$ 中与多组“键”做匹配，再从 $W_2$ 中读取对应“值”，最终叠加成输出修正. 

Attention 和 FFN 看起来都在“变换表示”，但它们解决的是完全不同的问题. Attention 决定该关注哪些 token、如何在序列维度上传播信息; FFN 决定一个 token 在吸收完这些信息之后应怎样被重新编码. 前者更像外部检索与路由，后者更像内部加工与写回. 

若没有 Attention，模型很难做高质量的上下文交互; 若没有 FFN，模型只能把注意力拉来的信息做近似线性整合，缺乏足够强的本地非线性加工能力. 因此「Attention 很聪明，FFN 只是补一点非线性」这种说法是错误的. 更准确的说法是：Attention 负责跨 token 信息流，FFN 负责单 token 高容量变换，二者缺一不可.

图 9：Attention 更像跨 token 的信息调度系统，FFN 更像单 token 的高容量加工与知识内化系统. 

## 5. FFN 为什么经常是工程成本中心

在工程里，「最重要的模块」往往不是最显眼的模块，而是最贵的模块. FFN 之所以反复成为焦点，一个很现实的原因是它参数占比高. 对大模型而言，参数多意味着三件事：存储成本高、训练时梯度和激活成本高、推理时权重搬运成本高. 一个模块只要同时踩中这三点，就天然会成为优化的优先目标. 

很多人直觉里会觉得大矩阵乘法一定首先卡算力，但在单 token 解码场景中，真实瓶颈常常不是「乘加太慢」，而是「权重搬不动」. 原因在于：

1. batch 很小，矩阵并不总能把硬件吞吐吃满. 
2. FFN 权重很大，每生成一个 token 都要反复读取. 
3. 解码阶段复用度低，算子调度空间也更窄. 

所以你会看到一种常见现象：GPU 利用率并不高，但 HBM 带宽接近满载，单 token 延迟仍然很高. 这时盯着 FLOPs 做优化常常不够，真正应该优先处理的是 FFN 的量化、分块、融合和访存模式. 

![](./images/image_1.png)

图 10：在单 token 解码中，FFN 的主要压力经常来自权重搬运，而不是理论 FLOPs 本身. 

训练时的问题与推理不同. 推理更关心带宽，训练还要关心反向传播. FFN 的中间激活维度很大，尤其是标准 $4d$ 或 SwiGLU 的高宽度中间态，会显著抬高显存占用. 如果层数很多，这种中间态的保存与回传代价并不低. 因此 activation checkpointing、重计算、混合精度和张量并行往往都绕不开 FFN. 

此外，FFN 的梯度链也更长. 标准 FFN 至少有两次大投影和一次非线性，SwiGLU 则多一条支路和一次乘法门控. 只要输入分布、初始化或学习率设置不稳，FFN 层级统计就更容易先出现异常. 

## 6. 训练视角：FFN 怎样影响可训练性

从前向角度看，FFN 的第一件事是扩展表示空间，第二件事是通过激活或门控筛选特征，第三件事是把被筛选过的模式压回模型维度. 这一过程里，真正重要的不是「变大又变小」本身，而是哪些模式在高维空间里被显式分开、哪些模式被放大、哪些模式被抑制. 

对于代码、数学、对话、多语言、结构化输出等复杂任务，一个 token 往往携带多种纠缠语义. FFN 的宽度越充足、门控越细致，模型越有机会把这些模式拆开并做差异化处理. 因此，在大模型里 FFN 的表达质量往往直接影响最终输出是否稳定、是否遵从指令、是否保留领域风格. 

反向传播时，激活函数的导数是 FFN 的梯度阀门. ReLU 在负区间把梯度彻底切断，因此容易造成局部失活; GELU 和 SiLU 则在负区间保留细小但连续的梯度，使得模型更容易从不良状态中恢复. 门控结构还会让梯度同时依赖「内容支路」和「门控支路」的数值状态，这使 SwiGLU 既更强也更复杂. 

这也是为什么在超大模型训练里，FFN 不只是「表达模块」，还是「可训练性模块」. 一个理论上更强的门控 FFN，如果数值范围控制不好，也可能成为训练不稳定的源头. 

若你在训练一个大模型，FFN 相关指标很值得单独监控：

1. 激活稀疏率：判断通道是否大面积沉默. 
2. 中间态最大值与分位数：快速定位异常尖峰层. 
3. gate 输出均值和方差：检查门控是否长期塌缩或饱和. 
4. 每层 FFN 梯度范数：识别哪些层学不动，哪些层在振荡. 
5. 权重范数漂移：检查是否出现异常扩张或塌缩. 
6. 量化前后输出余弦相似度：评估部署阶段量化可承受度. 

真正有经验的训练团队很少只看全局 loss 曲线. 很多问题在全局 loss 上并不明显，但在 FFN 的层级统计上会非常醒目. 

![](./images/image_7.png)

图 11：训练视角下的 FFN. 前向传播决定特征怎样被放大或抑制，反向传播决定梯度如何穿过激活与门控结构，而训练诊断通常就围绕这些关键节点展开. 

## 7. 失效模式：FFN 会在哪些地方出问题

若 $d_{ff}$ 太小，模型在中间空间里可分离的模式不够，表达能力下降; 若太大，虽然表示能力上升，但参数、带宽、显存和训练不稳定风险都增加，而且边际收益递减. 所以 $4d$、$8d/3$ 这些配置不是神圣常数，却是大量工程折中后留下的经验稳定点. 

ReLU 的风险是死神经元，SwiGLU 的风险则更常表现为门控塌缩. 若某些 gate 长期接近零，对应通道虽然参数存在，但在实际前向中几乎失去作用. 门控饱和也会让梯度传播质量恶化. 因此现代模型训练时，除了关心总体 loss，也要观察门控统计. 

FFN 中间态常常比人们预期更“野”. 某些层会出现极端尖峰，这在高位宽训练里未必立即致命，但到了低比特量化阶段就会暴露出来. 只要校准区间没覆盖好，尖峰被截断，输出就会发生系统性偏移. 这类问题在标准 ReLU/GELU FFN 和门控 FFN 中都可能出现，但门控乘法会进一步放大量化误差. 

## 8. FFN 与量化、蒸馏、LoRA、MoE 的关系

FFN 参数大、读得勤，所以量化收益通常很直接. 把 FFN 权重从 FP16 压到 INT8、INT4，本质上是在降低每一步推理所需搬运的数据量. 很多时候，量化 FFN 获得的收益不只是「省显存」，而是直接降低单 token 延迟. 

但 FFN 量化也有自己的坑. 最典型的是门控乘法误差放大，以及不同矩阵对精度敏感度并不一致. gate、up、down 三类投影往往需要不同量化策略，激活也可能需要按通道或按组做更精细校准. 

蒸馏常常被直观理解为「学生去模仿老师的输出」或「去学注意力分布」，但真正难学的部分，往往是老师在 FFN 中完成的高容量变换. 小模型最困难的不是复制某个 attention score，而是在更小容量里复现老师的非线性加工能力. 因此，无论蒸馏目标是 hidden states 还是 logits，FFN 仍然是学生模型能否真正学到老师行为的关键环节. 

在很多领域适配任务里，只给 Attention 打 LoRA 并不总够. 原因很简单：Attention 决定信息路由，FFN 才是大量知识和风格变换真正发生的地方. 若任务差异主要体现在术语系统、输出风格、特定领域规则或结构化格式上，FFN 往往比 Attention 更值得进入适配范围. 

MoE 的核心思想是：既然 FFN 是最大的容量池，那就把它拆成多个专家，并让不同 token 只激活其中少数几个. 这样，模型总容量可以很大，但单次前向成本只增长有限. MoE 很少首先长在 Attention 上，原因不是不能，而是性价比不高. Attention 主要控制交互结构，而 FFN 更适合承载「模式探测器 + 变换器」的专家化分工. 

这也是为什么 [Dai et al. (2024)](https://arxiv.org/abs/2401.06066) 所代表的 DeepSeek-MoE 路线，以及 Mixtral、MiniMax 等模型，都会把稀疏专家设计围绕 FFN 展开. 因为从容量收益和工程落地角度看，这是最自然的一刀. 

![](./images/image_8.png)

图 12：MoE 为什么通常长在 FFN 上. 注意力结构保持共享，容量扩展主要发生在 FFN 专家池中，这样更符合「高容量变换器」的角色分工，也更容易获得稀疏计算收益. 

## 9. 极简实现：标准 FFN 与 SwiGLU 的 PyTorch 映射

下面用最小 PyTorch 代码展示两种主流实现. 真正的大模型实现会加入并行、融合、混合精度和张量切分，但核心结构就是这些. 

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class StandardFFN(nn.Module):
    def __init__(self, dim: int, hidden_dim: int):
        super().__init__()
        self.up = nn.Linear(dim, hidden_dim, bias=False)
        self.down = nn.Linear(hidden_dim, dim, bias=False)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.down(F.gelu(self.up(x)))


class SwiGLUFFN(nn.Module):
    def __init__(self, dim: int, hidden_dim: int):
        super().__init__()
        self.gate = nn.Linear(dim, hidden_dim, bias=False)
        self.up = nn.Linear(dim, hidden_dim, bias=False)
        self.down = nn.Linear(hidden_dim, dim, bias=False)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        gated = F.silu(self.gate(x)) * self.up(x)
        return self.down(gated)
```

这段代码已经足够说明三件事. 第一，SwiGLU 的核心不是「换了个激活函数」，而是多出一条门控支路. 第二，SwiGLU 与标准 FFN 的主要开销差别来自额外投影和门控乘法. 第三，后续所有工程问题，包括量化、融合、稀疏化、并行切分，本质上都围绕这些矩阵和这次门控乘法展开. 

![](./images/image_2.png)

图 13：标准 FFN 与门控 FFN 的直观差别. 前者更像单路特征放缩，后者更像「内容支路 + 门控支路」的条件调制结构. 

## 10. 技术前瞻：FFN 还会怎样演化

[Ramachandran et al. (2017)](https://arxiv.org/abs/1710.05941) 通过搜索得到的 Swish / SiLU 路线已经证明，激活函数并不是“选一个常见的就行”，而是会真实影响训练稳定性、量化友好性和表达能力. 未来面向超低比特硬件的新激活函数仍然很可能继续出现. 

与此同时，因子化和低秩结构会继续深入. 既然 LoRA 已经证明很多 FFN 变化可以用低秩更新捕捉，那么在预训练阶段直接引入因子化 FFN 结构就是合理方向. 再进一步，FFN 与状态空间模型、线性 RNN 和动态卷积之间的边界也可能继续被重写：Mamba、RWKV、RetNet 等路线都在尝试把「局部非线性加工」和「跨位置状态更新」做更深融合. 

最后，MoE 的可微路由和更稳定的稀疏训练仍是开放问题. Top-$k$ 离散路由好用但不可导，软路由可导但会损失稀疏加速. FFN 稀疏化的下一步，不只是更多专家，而是更聪明的专家调度. 

## 11. 本节小结

FFN 在 Transformer 里从来不是一个「顺手加上的两层 MLP」. 它是模型的 **静态知识库**、**局部非线性加工器**、**参数主体**、**训练稳定性关键节点**，也是量化、蒸馏、LoRA 与 MoE 的核心落点. Attention 决定「看哪里」，FFN 决定「看完以后怎么变」; 前者负责信息调度，后者负责知识内化. 

理解 FFN，不能只停留在「升维再降维」这一抽象定义上，而要把它放进完整的大模型工程里看：为什么它要这么宽，为什么激活函数会影响可训练性，为什么门控结构会成为新一代主流，为什么它常常才是带宽瓶颈、量化重点和专家化中心. 只有把这些问题连起来，才算真正理解了 FFN 在现代大模型中的位置.
