# 2.1 深度学习基础组件：从感知机到Transformer的每一块砖

## 1. 背景与定位：为什么这一章是整座大厦的地基

在深入探讨 Transformer 那令人惊叹的自注意力机制之前，我们必须先回答一个根本性的问题：**一个现代大语言模型(LLM)到底是由什么构成的？**

如果把 GPT-4、Claude 或 DeepSeek 这样的模型比作一座摩天大楼，那么 Transformer 架构是这座楼的钢结构框架，而本章将要介绍的**深度学习基础组件**则是框架中的每一块砖、每一根钢筋、每一袋水泥. 没有这些基础组件的精确配合，再高明的架构设计也只是空中楼阁. 

**家谱定位**：本章位于"核心原理与架构"的最前端，是整个 LLM 技术图谱的"第零层". 后续 2.2 节的 Attention 机制、2.3 节的 Transformer 块、乃至 2.4 节的位置编码，全都建立在本章所述概念之上. 

**核心目标**：
1. 为已经具备本科学历但非深度学习专业的读者，补齐理解 LLM 所需的全部数学与工程基础. 
2. 建立一个统一的符号体系和物理直觉，使得后续章节中的复杂推导不再突兀. 
3. 阐明每一个基础组件"为什么存在"——不是为了堆砌知识点，而是为了让你理解：如果去掉这个组件，模型会在训练或推理的哪个环节崩溃. 

## 2. 神经网络基础：从感知机到反向传播

### 2.1 感知机(Perceptron)：一切的起源

感知机是神经网络最原始的祖先. 1958 年，Frank Rosenblatt 提出的这个简单模型，至今仍是理解所有深度学习算法的最佳起点. 

#### 数学定义

为了将输入向量转化为可学习的预测，感知机需要建立一个带参数的映射关系. 给定输入 $\mathbf{x} = [x_1, x_2, \dots, x_d]^T \in \mathbb{R}^d$，感知机通过权重向量 $\mathbf{w} \in \mathbb{R}^d$ 为每个输入维度分配影响力，再通过偏置 $b \in \mathbb{R}$ 平移决策边界，最后经激活函数 $\sigma(\cdot)$ 引入非线性：

$$
\hat{y} = \sigma\left( \mathbf{w}^T \mathbf{x} + b \right) = \sigma\left( \sum_{i=1}^{d} w_i x_i + b \right) \tag{1}
$$
这里，$\mathbf{w}$ 决定了各输入维度对最终决策的贡献权重; $b$ 允许决策边界偏离原点——没有它，无论权重如何调整，分类边界都将被迫经过坐标原点，极大限制了表达能力; $\sigma(\cdot)$ 则是整个网络能够逼近非线性函数的关键，没有激活函数时，多层堆叠仍等价于单一的线性变换，连异或(XOR)这类基本问题都无法解决. 

#### 物理意义

感知机的几何解释极其直观：它在输入空间中画出了一个**超平面(hyperplane)** ，将空间一分为二. $\mathbf{w}^T \mathbf{x} + b = 0$ 就是这道分界线的方程. 正样本落在一侧，负样本落在另一侧. 

![感知机分类决策边界](images/perceptron_boundary.png)

#### 感知机的致命局限与多层网络的诞生

Minsky 和 Papert 在 1969 年证明了：**单个感知机只能解决线性可分问题**. 异或(XOR)这种简单的非线性模式，单个感知机永远无法学会. 这一发现导致了第一次 AI 寒冬. 

但解决方案出奇地简单——**堆叠多个感知机，形成多层网络(Multi-Layer Perceptron, MLP)** . 当多个线性变换之间夹入非线性激活函数时，网络就获得了对任意复杂函数的逼近能力(Universal Approximation Theorem). 这正是深度学习的核心前提. 

#### 代码示例：感知机的向量化实现

```python
import torch
import torch.nn as nn

class Perceptron(nn.Module):
    """
    单层感知机
    对应公式: y_hat = sigma(w^T @ x + b)
    """
    def __init__(self, input_dim: int, output_dim: int):
        super().__init__()
        # w: (output_dim, input_dim), b: (output_dim,)
        self.weight = nn.Parameter(torch.randn(output_dim, input_dim) * 0.01)
        self.bias = nn.Parameter(torch.zeros(output_dim))
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (batch_size, input_dim)
        返回: (batch_size, output_dim)
        """
        # torch.matmul 对应矩阵乘法 w @ x^T, 结果转置后加偏置
        return torch.matmul(x, self.weight.T) + self.bias
        # 注意：激活函数 sigma 通常在外部单独施加，便于灵活替换

# 验证维度逻辑
batch_size, input_dim, output_dim = 4, 10, 3
x = torch.randn(batch_size, input_dim)
model = Perceptron(input_dim, output_dim)
out = model(x)
assert out.shape == (batch_size, output_dim), f"期望 (4, 3), 得到 {out.shape}"
```

### 2.2 激活函数(Activation Functions)：非线性的灵魂

激活函数是神经网络之所以"深"且"有用"的根本原因. 没有非线性激活，1000 层的网络也只是一个巨大的矩阵乘法，其表达能力与单层线性回归完全等价. 

#### Sigmoid：从生物神经元到概率输出

基于前述物理直觉，给出数学形式：
$$
\sigma(x) = \frac{1}{1 + e^{-x}} \tag{2}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
Sigmoid 将任意实数压缩到 $(0, 1)$ 区间，历史上曾被用来模拟生物神经元的"全有或全无"放电特性. 它的导数有一个优雅的性质：

推导过程如下：
$$
\sigma'(x) = \sigma(x) \cdot (1 - \sigma(x)) \tag{3}
$$
Sigmoid 输出可以被直接解释为概率. 在二分类问题的最后一层，它回答的是"样本属于正类的概率是多少". 

**致命缺陷——梯度消失**：当 $|x|$ 很大时，$\sigma(x)$ 饱和在 0 或 1，导数趋近于 0. 在深度网络中，这意味着靠近输入层的参数几乎接收不到梯度信号，网络"冻住"了. 这就是为什么现代深度网络(尤其是 LLM)** 几乎从不在隐藏层使用 Sigmoid**. 

#### ReLU：深度学习复兴的功臣

基于上述分析，建立如下数学关系：
$$
\text{ReLU}(x) = \max(0, x) \tag{4}
$$
此式将上述约束形式化，各项分别对应输入变换、非线性激活与输出生成. 
ReLU(Rectified Linear Unit)是 2012 年 AlexNet 带来的革命. 它的导数在正区间恒为 1，彻底解决了深层网络中的梯度消失问题. 

ReLU 模拟了生物神经元的"稀疏激活"特性——在任何给定时刻，大脑中大约只有 1% 到 4% 的神经元处于活跃状态. ReLU 将所有负输入"掐断"为 0，使得网络中的神经元呈现出高度的稀疏性，这带来了计算效率和特征选择的双重好处. 

**ReLU 的缺陷——Dying ReLU**：如果一个神经元在训练初期被"推"到了负半轴，并且其输入分布始终为负，那么这个神经元将永远输出 0，梯度也为 0，相当于"死亡"了. 整个网络的有效容量因此下降. 

#### GELU：Transformer 时代的默认选择

基于上述分析，建立如下数学关系：
$$
\text{GELU}(x) = x \cdot \Phi(x) = x \cdot \frac{1}{2}\left[ 1 + \text{erf}\left( \frac{x}{\sqrt{2}} \right) \right] \tag{5}
$$
其中 $\Phi(x)$ 是标准正态分布的累积分布函数. 在实际工程中，GELU 常用一个光滑的近似：

为建立定量关系，定义如下表达式：
$$
\text{GELU}(x) \approx 0.5x \left( 1 + \tanh\left[ \sqrt{\frac{2}{\pi}} \left( x + 0.044715x^3 \right) \right] \right) \tag{6}
$$
此式描述了变量之间的定量关系，其中每一项对应特定的物理操作. 
与 ReLU 的"硬截断"不同，GELU 对负值进行**概率性的软衰减**. 它基于这样一个思想：在 Dropout 正则化下，一个神经元的输入应该以输入值的大小为概率被保留. GELU 本质上是在做"随机正则化的期望"，因此比 ReLU 更平滑、更稳定. 

**为什么 Transformer 选择 GELU**：在 BERT、GPT 系列等 Transformer 模型中，隐藏层默认使用 GELU 而非 ReLU. 原因是 Transformer 的预训练任务(掩码语言建模、自回归生成)对激活函数的平滑性要求极高，GELU 在负区间的非零梯度避免了 Dying ReLU 问题，同时其光滑性使得优化器能在更复杂的损失地形中找到更好的路径. 

#### Swish 与 SwiGLU：现代 LLM 的进化

$$
\text{Swish}(x) = x \cdot \sigma(x) = \frac{x}{1 + e^{-x}} \tag{7}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
Swish 通过将输入与自身的 Sigmoid 激活相乘，在正区间近似恒等、在负区间平滑衰减，比 ReLU 更光滑且处处可导. 

SwiGLU 是将 Swish 激活与门控线性单元(Gated Linear Unit)结合的结构：

$$
\text{SwiGLU}(x, W, V) = \text{Swish}(xW) \otimes (xV) \tag{8}
$$
其中 $\otimes$ 表示逐元素乘法，$W$ 和 $V$ 是两个独立的投影矩阵. SwiGLU 在 PaLM、LLaMA 系列中取代了传统的 ReLU/GELU + FFN 结构，带来了约 5% 到 10% 的下游任务提升. 

SwiGLU 引入了**门控机制(gating)** . 网络的隐藏层不再是一个固定的非线性变换，而是被输入数据自身动态调节. $xV$ 这个分支充当了"信息阀门"，决定前一分支的哪些信息可以通过. 这种自适应的信息筛选，非常契合自然语言中上下文依赖的复杂模式. 

#### 激活函数对比代码

```python
import torch
import torch.nn as nn
import math

def gelu_exact(x: torch.Tensor) -> torch.Tensor:
    """
    GELU 精确实现
    对应公式: x * 0.5 * (1 + erf(x / sqrt(2)))
    """
    return x * 0.5 * (1 + torch.erf(x / math.sqrt(2)))

def gelu_approx(x: torch.Tensor) -> torch.Tensor:
    """
    GELU tanh 近似(Hendrycks & Gimpel, 2016)
    对应公式: 0.5x * (1 + tanh(sqrt(2/pi) * (x + 0.044715x^3)))
    """
    return 0.5 * x * (1 + torch.tanh(
        math.sqrt(2 / math.pi) * (x + 0.044715 * x ** 3)
    ))

def swish(x: torch.Tensor) -> torch.Tensor:
    """
    Swish 激活: x * sigmoid(x)
    """
    return x * torch.sigmoid(x)

class SwiGLU(nn.Module):
    """
    SwiGLU 门控前馈层
    对应公式: Swish(xW) * (xV)
    输出维度为 hidden_dim, 需要内部投影到 2*hidden_dim
    """
    def __init__(self, dim: int, hidden_dim: int):
        super().__init__()
        # 注意：SwiGLU 需要三个投影矩阵来保持参数量平衡
        self.w = nn.Linear(dim, hidden_dim, bias=False)
        self.v = nn.Linear(dim, hidden_dim, bias=False)
        self.w2 = nn.Linear(hidden_dim, dim, bias=False)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # xW 经过 Swish, xV 作为门控
        return self.w2(swish(self.w(x)) * self.v(x))

# 数值验证：观察不同激活函数在负区间的行为
x = torch.linspace(-5, 5, 100)
print("ReLU(-3) =", torch.relu(torch.tensor(-3.0)).item())       # 0.0
print("GELU(-3) =", gelu_exact(torch.tensor(-3.0)).item())        # ~-0.04(非零！)
print("Sigmoid(-3) =", torch.sigmoid(torch.tensor(-3.0)).item())  # ~0.05(梯度几乎为0)
```

### 2.3 反向传播(Backpropagation)：梯度是如何流动的

反向传播是现代神经网络训练的数学引擎. 没有它，就没有深度学习. 

#### 链式法则：反向传播的数学心脏

假设我们有一个复合函数 $y = f(g(x))$，链式法则告诉我们：

$$
\frac{\partial y}{\partial x} = \frac{\partial y}{\partial g} \cdot \frac{\partial g}{\partial x} \tag{9}
$$
在神经网络中，这意味着：**损失函数对第 $l$ 层参数的梯度，等于损失对第 $l$ 层输出的梯度，乘以该层输出对参数的梯度. **

#### 反向传播的完整推导

考虑一个两层网络(为简化，省略偏置). 前向传播中，输入首先经过第一层线性变换与激活：

$$
\mathbf{z}^{[1]} = \mathbf{W}^{[1]} \mathbf{x}, \quad \mathbf{a}^{[1]} = \sigma(\mathbf{z}^{[1]}) \tag{10}
$$
然后经过第二层得到最终输出：

将上述约束转化为数学表达：
$$
\mathbf{z}^{[2]} = \mathbf{W}^{[2]} \mathbf{a}^{[1]}, \quad \hat{\mathbf{y}} = \sigma(\mathbf{z}^{[2]}) \tag{11}
$$
损失函数为均方误差 $\mathcal{L} = \frac{1}{2}(\hat{y} - y)^2$. 为了更新参数，我们需要计算损失对各层参数的梯度，这通过链式法则从输出层逐层反向传递. 

输出层的误差信号度量了预测与真实值的差距经激活函数缩放后的结果：

$$
\delta^{[2]} = \frac{\partial \mathcal{L}}{\partial \mathbf{z}^{[2]}} = (\hat{y} - y) \odot \sigma'(\mathbf{z}^{[2]}) \tag{12}
$$
其中 $\odot$ 表示逐元素乘法. 

隐藏层的误差信号则是将输出层误差反向传播回隐藏层空间的结果——$\mathbf{W}^{[2]}$ 的转置将输出层误差映射回隐藏层维度，再与激活函数的局部导数相乘，得到实际的参数更新信号：

基于前述物理直觉，给出数学形式：
$$
\delta^{[1]} = \frac{\partial \mathcal{L}}{\partial \mathbf{z}^{[1]}} = \left( (\mathbf{W}^{[2]})^T \delta^{[2]} \right) \odot \sigma'(\mathbf{z}^{[1]}) \tag{13}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
有了各层的误差信号，参数梯度可以直接通过矩阵外积得到：

$$
\frac{\partial \mathcal{L}}{\partial \mathbf{W}^{[2]}} = \delta^{[2]} (\mathbf{a}^{[1]})^T, \quad \frac{\partial \mathcal{L}}{\partial \mathbf{W}^{[1]}} = \delta^{[1]} \mathbf{x}^T \tag{14}
$$
如式 (3) 所示，输出层梯度直接依赖于预测误差; 如式 (4) 所示，隐藏层梯度通过权重矩阵的转置将误差逐层回传; 如式 (5) 所示，最终的参数更新量由误差信号与对应层输入的外积决定. 

#### 数值走查：反向传播的一步

假设输入 $x = 2.0$，真实标签 $y = 1.0$. 网络参数：$w^{[1]} = 0.5, w^{[2]} = 1.5$，使用 Sigmoid 激活. 

| 步骤 | 计算 | 结果 |
|------|------|------|
| 前向：隐藏层输入 | $z^{[1]} = 0.5 \times 2.0 = 1.0$ | $1.0$ |
| 前向：隐藏层输出 | $a^{[1]} = \sigma(1.0) = 0.731$ | $0.731$ |
| 前向：输出层输入 | $z^{[2]} = 1.5 \times 0.731 = 1.097$ | $1.097$ |
| 前向：预测 | $\hat{y} = \sigma(1.097) = 0.750$ | $0.750$ |
| 损失 | $\mathcal{L} = 0.5 \times (0.750 - 1.0)^2$ | $0.0313$ |
| 反向：输出误差 | $\delta^{[2]} = (0.750 - 1.0) \times \sigma'(1.097) = -0.25 \times 0.187$ | $-0.0468$ |
| 反向：隐藏误差 | $\delta^{[1]} = (1.5 \times -0.0468) \times \sigma'(1.0) = -0.0702 \times 0.197$ | $-0.0138$ |
| 梯度 $w^{[2]}$ | $\partial \mathcal{L} / \partial w^{[2]} = -0.0468 \times 0.731$ | $-0.0342$ |
| 梯度 $w^{[1]}$ | $\partial \mathcal{L} / \partial w^{[1]} = -0.0138 \times 2.0$ | $-0.0276$ |

可以看到，越靠近输入层，梯度的绝对值越小($-0.0468 \to -0.0138$). 当网络更深时，这种衰减会被多次累积，最终导致**梯度消失**. 

#### 反向传播的代码实现

```python
import torch
import torch.nn.functional as F

def sigmoid(x):
    return 1 / (1 + torch.exp(-x))

def sigmoid_derivative(x):
    """Sigmoid 导数: sigma'(x) = sigma(x) * (1 - sigma(x))"""
    s = sigmoid(x)
    return s * (1 - s)

def mse_loss(y_pred, y_true):
    """对应公式: L = 0.5 * (y_pred - y_true)^2"""
    return 0.5 * (y_pred - y_true) ** 2

# 手动实现反向传播(与 PyTorch autograd 对比)
x = torch.tensor(2.0)
y_true = torch.tensor(1.0)
w1 = torch.tensor(0.5, requires_grad=True)
w2 = torch.tensor(1.5, requires_grad=True)

# 前向传播
z1 = w1 * x          # 隐藏层线性变换
a1 = sigmoid(z1)     # 隐藏层激活
z2 = w2 * a1         # 输出层线性变换
y_pred = sigmoid(z2) # 输出层激活
loss = mse_loss(y_pred, y_true)

# 手动反向传播
# delta2 = (y_pred - y_true) * sigma'(z2)
delta2 = (y_pred - y_true) * sigmoid_derivative(z2)
# delta1 = w2 * delta2 * sigma'(z1)
delta1 = w2 * delta2 * sigmoid_derivative(z1)

# 参数梯度
grad_w2_manual = delta2 * a1
grad_w1_manual = delta1 * x

# PyTorch autograd 自动计算
loss.backward()
print(f"手动计算  dw1: {grad_w1_manual.item():.6f},  dw2: {grad_w2_manual.item():.6f}")
print(f"PyTorch autograd dw1: {w1.grad.item():.6f}, dw2: {w2.grad.item():.6f}")
# 两者应当完全一致(误差 < 1e-6)
```

## 3. 优化基础：如何让梯度真正更新参数

有了梯度，下一步是如何利用它来更新参数. 这不仅仅是"往负梯度方向走一步"那么简单——步长多大、方向如何修正、如何利用历史信息，这些问题共同构成了优化算法的研究领域. 

### 3.1 SGD：最朴素的梯度下降

$$
\theta_{t+1} = \theta_t - \eta \nabla_\theta \mathcal{L}(\theta_t) \tag{15}
$$
其中 $\eta$ 是学习率(learning rate)，控制每一步的步长. 

SGD 就像一个人在浓雾中下山，只能感觉到脚下的坡度，然后沿着最陡的方向迈出一步. 它简单、有效，但存在三个致命问题：

1. **学习率敏感**：太大则震荡不收敛，太小则收敛极慢. 

2. **陷入局部最优与鞍点**：在高维非凸损失地形中，SGD 很容易被困在局部极小值或鞍点(梯度为零但非最优的点). 

3. **同一学习率对所有参数**：稀疏特征(如罕见的单词)和密集特征(如常见的停用词)被迫使用相同的更新幅度，这是极度低效的. 

#### SGD 代码示例

```python
import torch

# 最简 SGD 手动实现
def sgd_update(params, grads, lr=0.01):
    """
    对应公式: theta_{t+1} = theta_t - eta * grad_theta L
    """
    for param, grad in zip(params, grads):
        param.data -= lr * grad

# PyTorch 内置 SGD
model = torch.nn.Linear(10, 2)
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

loss = model(torch.randn(4, 10)).sum()
loss.backward()
optimizer.step()  # 执行 theta = theta - lr * grad
optimizer.zero_grad()  # 清空梯度，防止累积
```

### 3.2 Momentum：赋予梯度"惯性"

Momentum 的灵感来自物理学. SGD 在峡谷地形中会来回震荡，因为它每一步都只考虑当前的坡度. 如果给优化过程加上"动量"，就像一个滚下山的球——它会积累速度，在一致的梯度方向上加速，在震荡方向上减速. 

$$
\mathbf{v}_{t+1} = \gamma \mathbf{v}_t + \nabla_\theta \mathcal{L}(\theta_t) \tag{16}
$$
该关系式明确了系统的数学约束，可直接用于后续优化推导. 
如式 (26) 所示，速度向量在当前梯度与历史动量之间做指数加权平均，$\gamma$ 越大，过去梯度对当前方向的影响越持久. 

推导过程如下：
$$
\theta_{t+1} = \theta_t - \eta \mathbf{v}_{t+1} \tag{17}
$$
其中：
- $\mathbf{v}_t$ 是速度向量(动量积累项)，初始为 0. 
- $\gamma \in [0, 1)$ 是动量系数，通常设为 0.9. 它决定了历史梯度的衰减速度. 

$
abla_\theta \mathcal{L}(\theta_t)$ 是当前时刻的"加速度"(力除以质量)，$\gamma \mathbf{v}_t$ 是上一时刻速度的惯性保留. 在梯度方向长期一致的区域(如平缓的斜坡)，速度会不断累加，使得参数更新越来越快; 而在梯度方向频繁反转的区域(如峡谷的两侧壁)，速度会相互抵消，震荡被抑制. 

**展开视角**：将 Momentum 的更新式递归展开：

基于前述物理直觉，给出数学形式：
$$
\mathbf{v}_t = \sum_{i=1}^{t} \gamma^{t-i} \nabla_\theta \mathcal{L}(\theta_i) \tag{18}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
这意味着当前的速度实际上是**历史梯度的指数加权平均**. 一个 10 步之前的梯度对当前速度的贡献是当前的 $\gamma^{10} \approx 0.35$ 倍(当 $\gamma=0.9$). 

### 3.3 Adam：自适应学习率的王者

Adam(Adaptive Moment Estimation)是目前深度学习中最流行的优化器，Transformer 的预训练几乎清一色使用 Adam 或其变体(AdamW). 

Adam 维护两个状态来分别捕捉梯度的方向趋势和波动幅度. 一阶矩 $\mathbf{m}_t$ 类似于 Momentum 中的速度积累，记录梯度的指数移动平均，为优化提供方向惯性; 二阶矩 $\mathbf{v}_t$ 则记录梯度平方的指数移动平均，用于度量每个参数历史梯度的波动幅度：

$$
\mathbf{m}_t = \beta_1 \mathbf{m}_{t-1} + (1 - \beta_1) \mathbf{g}_t \tag{19}
$$
此式将物理直觉转化为精确的数学语言，可直接用于算法实现. 

推导过程如下：
$$
\mathbf{v}_t = \beta_2 \mathbf{v}_{t-1} + (1 - \beta_2) \mathbf{g}_t^2 \tag{20}
$$
其中 $\mathbf{g}_t = \nabla_\theta \mathcal{L}(\theta_t)$ 是第 $t$ 步的梯度，$\beta_1$ 通常设为 0.9，$\beta_2$ 通常设为 0.999. 

由于 $\mathbf{m}_0$ 和 $\mathbf{v}_0$ 初始化为 0，前几步的估计会系统性偏向 0，因此 Adam 引入偏差修正来消除这种初始偏差：

$$
\hat{\mathbf{m}}_t = \frac{\mathbf{m}_t}{1 - \beta_1^t}, \quad \hat{\mathbf{v}}_t = \frac{\mathbf{v}_t}{1 - \beta_2^t} \tag{21}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
最终参数更新将修正后的一阶矩作为更新方向，并用二阶矩的平方根对学习率进行自适应缩放——历史梯度稳定的参数获得更大的有效步长，而波动剧烈的参数则被抑制：

基于上述分析，建立如下数学关系：
$$
\theta_{t+1} = \theta_t - \frac{\eta}{\sqrt{\hat{\mathbf{v}}_t} + \epsilon} \hat{\mathbf{m}}_t \tag{22}
$$
如式 (6) 所示，一阶矩通过指数移动平均积累梯度方向; 如式 (7) 所示，二阶矩积累梯度平方以度量波动; 如式 (8) 所示，偏差修正消除了初始化带来的系统性低估; 如式 (9) 所示，最终更新将方向惯性(分子)与自适应步长(分母)相结合，使得每个参数拥有独立的学习率调度. 其中 $\epsilon$(通常 $10^{-8}$)是防止除以零的数值稳定项. 

**为什么 Adam 特别适合 Transformer**：Transformer 的参数空间极其异质——Embedding 层处理的是离散的 token 索引，注意力层的 Query/Key/Value 投影处理的是高维语义空间，而 FFN 层则在做复杂的非线性变换. 不同层的梯度尺度差异巨大. Adam 的自适应学习率机制，使得每个参数都有自己的"个性化"更新步长，极大地稳定了这种异质网络的大规模训练. 

#### Adam 完整代码实现

```python
import torch
import math

class Adam:
    """
    Adam 优化器简化实现
    对应公式:
      m_t = beta1 * m_{t-1} + (1-beta1) * g_t
      v_t = beta2 * v_{t-1} + (1-beta2) * g_t^2
      m_hat = m_t / (1 - beta1^t)
      v_hat = v_t / (1 - beta2^t)
      theta = theta - lr * m_hat / (sqrt(v_hat) + eps)
    """
    def __init__(self, params, lr=1e-3, beta1=0.9, beta2=0.999, eps=1e-8):
        self.params = list(params)
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.eps = eps
        self.t = 0
        
        # 初始化一阶矩和二阶矩
        self.m = [torch.zeros_like(p) for p in self.params]
        self.v = [torch.zeros_like(p) for p in self.params]
    
    def step(self):
        self.t += 1
        for i, param in enumerate(self.params):
            if param.grad is None:
                continue
            g = param.grad
            
            # 更新一阶矩和二阶矩
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * g
            self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * (g ** 2)
            
            # 偏差修正
            m_hat = self.m[i] / (1 - self.beta1 ** self.t)
            v_hat = self.v[i] / (1 - self.beta2 ** self.t)
            
            # 参数更新
            param.data -= self.lr * m_hat / (torch.sqrt(v_hat) + self.eps)
    
    def zero_grad(self):
        for param in self.params:
            if param.grad is not None:
                param.grad.zero_()

# 与 PyTorch 内置 AdamW 对比(Transformer 训练的标准选择)
# AdamW 是 Adam 的变体，将权重衰减(L2 正则)从梯度中解耦，直接作用在参数上
model = torch.nn.Linear(100, 10)
optimizer_torch = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)
```

## 4. 正则化：防止模型"死记硬背"

神经网络的参数量常常远超训练样本数(例如 LLM 有数千亿参数，但训练语料只有数万亿 token)，这使得模型具有极强的记忆能力——如果不加约束，它会直接把训练集"背诵"下来，而在新数据上表现糟糕. 正则化就是对抗这种过拟合的武器. 

### 4.1 Dropout：训练时的"随机失活"

Dropout 由 Hinton 等人于 2012 年提出，其核心思想极其粗暴而有效：**在每次前向传播时，以概率 $p$ 随机将一部分神经元的输出置为 0**. 

#### 数学定义

在训练时，对于某层的输出 $\mathbf{h} \in \mathbb{R}^d$，Dropout 构造一个随机掩码向量 $\mathbf{m}$：

$$
m_i \sim \text{Bernoulli}(1 - p), \quad \tilde{\mathbf{h}} = \mathbf{m} \odot \mathbf{h} \tag{23}
$$
其中 $p$ 是丢弃概率(例如 0.1 表示 10% 的神经元被丢弃). 为了补偿训练时神经元数量减少带来的尺度漂移，输出需要缩放：

$$
\tilde{\mathbf{h}} = \frac{\mathbf{m} \odot \mathbf{h}}{1 - p} \tag{24}
$$
在推理时，Dropout 完全关闭，所有神经元都参与计算. 

#### 物理意义：集成学习的近似

Dropout 的深层解释是**模型集成(Model Ensemble)** . 每一次前向传播，Dropout 都从原始网络中采样出一个"子网络". 训练 $N$ 个 batch，相当于训练了 $N$ 个不同的子网络. 最终的完整网络，可以看作是这 $N$ 个子网络的指数级大集成的近似平均. 

**另一种直觉**：Dropout 强迫神经元不能"依赖"其他特定的神经元. 因为任何协作者都可能在下一刻被随机抹除，每个神经元必须学会独立提取鲁棒的特征. 这大大减少了神经元之间的"共谋"(co-adaptation)，迫使模型学习更本质的表示. 

**Transformer 中的 Dropout**：在 Transformer 中，Dropout 通常应用于三个位置：
1. Embedding 层输出(防止对特定 token 的过拟合). 
2. 注意力层的 Softmax 输出(Attention Dropout，随机掩盖部分注意力连接). 
3. FFN 层的输出. 

现代 LLM(如 LLaMA 3)倾向于使用较小的 Dropout 比率(0.0 到 0.1)，因为预训练数据量极大，过拟合风险相对较小. 但在微调阶段，Dropout 是防止灾难性遗忘和过拟合的关键手段. 

#### Dropout 代码实现

```python
import torch
import torch.nn as nn

class Dropout(nn.Module):
    """
    Dropout 手动实现
    训练时: h_tilde = m * h / (1 - p)
    推理时: h_tilde = h(恒等映射)
    """
    def __init__(self, p: float = 0.5):
        super().__init__()
        self.p = p
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        if not self.training or self.p == 0.0:
            return x
        # 生成 Bernoulli 掩码: 1 表示保留, 0 表示丢弃
        mask = (torch.rand_like(x) > self.p).float()
        # 缩放以保持期望值不变: E[mask] = 1 - p
        return x * mask / (1 - self.p)

# 验证期望值守恒
x = torch.ones(1000, 1000)
dropout = Dropout(p=0.3)
dropout.train()
out = dropout(x)
print(f"输入均值: {x.mean().item():.4f}")           # 1.0
print(f"输出均值: {out.mean().item():.4f}")         # 约 1.0(缩放补偿了丢弃)

# PyTorch 内置 Dropout
torch_dropout = nn.Dropout(p=0.1)
```

### 4.2 Batch Normalization(BN)：稳定数据分布的统计利器

Batch Normalization(Ioffe & Szegedy, 2015)是深度卷积网络的里程碑式发明. 它通过对每一层的输入进行标准化，极大地加速了训练收敛. 

#### 数学定义

对于一个 mini-batch 的输入 $\mathbf{x} \in \mathbb{R}^{B \times d}$($B$ 是 batch size，$d$ 是特征维度)，BN 先计算 batch 内的均值和方差：

$$
\mu_B = \frac{1}{B} \sum_{i=1}^{B} \mathbf{x}_i, \quad \sigma_B^2 = \frac{1}{B} \sum_{i=1}^{B} (\mathbf{x}_i - \mu_B)^2 \tag{25}
$$
此式将上述约束形式化，各项分别对应输入变换、非线性激活与输出生成. 
此式给出了精确的数学定义，为算法实现提供了理论基础. 
如式 (31) 所示，BN 在 batch 内计算每个特征维度的经验均值和方差，为后续标准化提供局部统计参考. 

然后标准化：

$$
\hat{\mathbf{x}}_i = \frac{\mathbf{x}_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} \tag{26}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
如式 (32) 所示，减去均值并除以标准差将每个特征维度缩放到零均值、单位方差，消除不同特征间的尺度差异. 

最后进行**可学习的仿射变换**：

将上述约束转化为数学表达：
$$
\mathbf{y}_i = \gamma \odot \hat{\mathbf{x}}_i + \beta \tag{27}
$$
其中 $\gamma$ 和 $\beta$ 是可学习的参数. 这个"标准化后再还原"的操作看似多此一举，实则至关重要——它赋予了网络**选择性**：如果某一层发现标准化反而有害，它可以通过学习 $\gamma = \sigma_B$ 和 $\beta = \mu_B$ 来近似恢复原始输入. 

#### 物理意义：对抗内部协变量偏移

在深度网络训练过程中，每一层的参数都在不断变化，导致其输出分布也在不断漂移. 后层网络必须不断适应前层分布的变化，这种现象被称为**内部协变量偏移(Internal Covariate Shift)** . BN 通过强制每一层的输出都服从均值为 0、方差为 1 的分布，极大地缓解了这个问题. 

**为什么 BN 在 NLP / Transformer 中不被使用**：

BN 有一个致命的假设——batch size 必须足够大，才能准确估计均值和方差. 但在 NLP 中：
1. 序列长度差异大，padding 会污染 batch 统计量. 
2. 推理时无法获得 batch 统计量，只能依赖训练时累积的滑动平均，这与训练时的分布可能不一致. 
3. Transformer 的预训练 batch size 虽然大，但序列维度的变化使得"跨样本统计"失去意义. 

因此，Transformer 使用的是 Layer Normalization，而非 Batch Normalization. 

### 4.3 Layer Normalization(LN)：为序列模型量身定制的归一化

Layer Normalization(Ba et al., 2016)是为 RNN 和 Transformer 这类序列模型设计的归一化方法. 

#### 数学定义

LN 不在 batch 维度上做统计，而是**对每个样本的每个时间步，独立地在特征维度上计算均值和方差**：

对于单个样本的单个时间步 $\mathbf{x} \in \mathbb{R}^d$：

$$
\mu = \frac{1}{d} \sum_{j=1}^{d} x_j, \quad \sigma^2 = \frac{1}{d} \sum_{j=1}^{d} (x_j - \mu)^2 \tag{28}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
如式 (34) 所示，LN 在单个样本的特征维度上独立计算统计量，完全不依赖 batch 中其他样本. 

$$
\text{LN}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x} - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta \tag{29}
$$
如式 (35) 所示，可学习的 $\gamma$ 和 $\beta$ 让网络在标准化后恢复对原始分布的表达能力，避免强制归一化破坏有用信息. 

#### 物理意义

LN 的核心思想是：**每个样本应该被独立归一化，不依赖 batch 中其他样本的信息**. 这对于序列数据至关重要，因为：

1. **序列长度可变**：不同句子长度不同，batch 统计量会被 padding 严重污染. 

2. **推理时一致性**：推理通常是单条或少量样本，LN 不需要维护全局统计量，训练和推理行为完全一致. 

3. **位置独立性**：句子中每个位置的语义角色不同(主语 vs 宾语)，跨位置做统计没有物理意义. 

**Transformer 中的 LN 位置**：在原始 Transformer 论文中，LN 放在残差连接之后(Post-LN). 但在 GPT-3、LLaMA 等现代架构中，普遍采用 Pre-LN——将 LN 放在子层(Attention 或 FFN)的输入端. Pre-LN 使得梯度在反向传播时不需要穿过 LN 层和残差分支，极大地稳定了深层网络的训练. 

#### LN vs RMSNorm：现代 LLM 的选择

RMSNorm(Root Mean Square Normalization)是 LN 的简化变体，去掉了均值减法的步骤：

推导过程如下：
$$
\text{RMSNorm}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x}}{\sqrt{\frac{1}{d} \sum_{j=1}^{d} x_j^2 + \epsilon}} \tag{30}
$$
RMSNorm 假设输入的均值已经接近 0(在深层网络中，ReLU/GELU 等激活函数的输出确实大致对称)，因此省去了计算均值的步骤. 这不仅减少了计算量，还在实证中展现出与 LN 相当甚至更优的性能. LLaMA、Qwen 等主流开源模型都采用了 RMSNorm. 

#### Layer Normalization 代码实现

```python
import torch
import torch.nn as nn

class LayerNorm(nn.Module):
    """
    Layer Normalization 手动实现
    对应公式: LN(x) = gamma * (x - mu) / sqrt(sigma^2 + eps) + beta
    """
    def __init__(self, normalized_shape: int, eps: float = 1e-5):
        super().__init__()
        self.eps = eps
        # gamma 和 beta 是可学习的参数
        self.gamma = nn.Parameter(torch.ones(normalized_shape))
        self.beta = nn.Parameter(torch.zeros(normalized_shape))
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (batch_size, seq_len, hidden_dim) 或任意最后维为 hidden_dim 的张量
        """
        # 在最后维度(特征维度)上计算均值和方差
        mean = x.mean(dim=-1, keepdim=True)        # (B, T, 1)
        var = x.var(dim=-1, keepdim=True, unbiased=False)  # (B, T, 1)
        
        # 标准化
        x_norm = (x - mean) / torch.sqrt(var + self.eps)
        
        # 可学习的仿射变换
        return self.gamma * x_norm + self.beta

class RMSNorm(nn.Module):
    """
    RMSNorm 实现(LLaMA 等模型使用)
    对应公式: RMSNorm(x) = gamma * x / sqrt(mean(x^2) + eps)
    """
    def __init__(self, dim: int, eps: float = 1e-6):
        super().__init__()
        self.eps = eps
        self.gamma = nn.Parameter(torch.ones(dim))
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # rms = sqrt(mean(x^2))
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return self.gamma * (x / rms)

# 维度验证
batch_size, seq_len, hidden_dim = 2, 10, 64
x = torch.randn(batch_size, seq_len, hidden_dim)
ln = LayerNorm(hidden_dim)
rms = RMSNorm(hidden_dim)
assert ln(x).shape == x.shape
assert rms(x).shape == x.shape

# 统计特性验证：LayerNorm 输出在最后维度的均值≈0，方差≈1
out_ln = ln(x)
print(f"LN 输出均值: {out_ln.mean(dim=-1).abs().mean().item():.6f}")  # ~0
print(f"LN 输出标准差: {out_ln.std(dim=-1).mean().item():.6f}")      # ~1
```

## 5. 嵌入层：从离散符号到连续语义空间

自然语言处理的核心困境在于：计算机只能处理数字，但语言是离散的符号系统. **嵌入层(Embedding Layer)** 就是跨越这道鸿沟的桥梁. 

### 5.1 Word Embedding：把单词变成向量

#### 数学定义

假设词汇表大小为 $V$，嵌入维度为 $d$. Word Embedding 就是一个可学习的查找表(lookup table)$\mathbf{E} \in \mathbb{R}^{V \times d}$：

$$
\mathbf{e}_i = \mathbf{E}[i] \in \mathbb{R}^d \tag{31}
$$
其中 $i \in \{0, 1, \dots, V-1\}$ 是 token 的整数索引. 在实际实现中，我们通常用 one-hot 向量与嵌入矩阵相乘来模拟查找：

推导过程如下：
$$
\mathbf{e} = \mathbf{x}_{\text{one-hot}} \mathbf{E} \tag{32}
$$
如式 (38) 所示，one-hot 向量与嵌入矩阵相乘等价于按索引提取对应行向量，是查找表操作在线性代数中的等价表达. 

#### 物理意义：分布式语义假设

Word Embedding 的理论基础是**分布式假设(Distributional Hypothesis)** ："一个词的含义，由它周围经常出现的词决定. "

在嵌入空间中：
- 语义相似的词(如"king"和"queen")在向量空间中距离很近. 
- 词与词之间的向量差可以编码语义关系：$\vec{\text{king}} - \vec{\text{man}} + \vec{\text{woman}} \approx \vec{\text{queen}}$. 
-  Embedding 空间的每个维度没有人类可解释的标签(不像"长度"、"颜色")，但整个向量共同编码了丰富的语义和语法信息. 

**现代 LLM 中的 Embedding**：在 Transformer 中，输入首先经过 Word Embedding，将每个 token ID 映射为 $d$ 维向量. 这个嵌入矩阵通常与输出层的分类头**共享权重**(Weight Tying)，这一技巧由 Press & Wolf (2017) 提出，能在不增加参数量的情况下提升性能，并加速收敛. 

#### Word Embedding 代码实现

```python
import torch
import torch.nn as nn

class TokenEmbedding(nn.Module):
    """
    Token Embedding 层
    对应公式: e = E[token_id]
    """
    def __init__(self, vocab_size: int, d_model: int):
        super().__init__()
        # 嵌入矩阵: (vocab_size, d_model)
        self.embedding = nn.Embedding(vocab_size, d_model)
        # 缩放因子(Transformer 原始论文使用 sqrt(d_model))
        self.scale = d_model ** 0.5
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (batch_size, seq_len) 的 token ID(整数)
        返回: (batch_size, seq_len, d_model)
        """
        # nn.Embedding 内部就是一个可学习的查找表
        return self.embedding(x) * self.scale

# 验证
vocab_size, d_model = 50000, 768
token_ids = torch.randint(0, vocab_size, (2, 128))  # 2 个样本, 每个 128 个 token
emb = TokenEmbedding(vocab_size, d_model)
out = emb(token_ids)
assert out.shape == (2, 128, 768)

# 展示语义相似度：随机初始化时，向量是均匀分布的
# 训练后，相似语义的词会有更高的余弦相似度
def cosine_similarity(a, b):
    return torch.dot(a, b) / (torch.norm(a) * torch.norm(b))
```

### 5.2 Position Embedding：为序列注入顺序信息

Word Embedding 有一个致命的盲区：**它对输入顺序完全不敏感**. 如果把句子中的词顺序打乱，Embedding 层的输出只是相应行向量的重新排列，模型本身无法区分"猫追老鼠"和"老鼠追猫". 

位置编码(Position Embedding / Position Encoding)就是为了解决这个问题. 

#### 绝对位置编码：Sinusoidal 与可学习 Embedding

Transformer 原始论文使用了基于正弦和余弦的固定位置编码：

$$
PE_{(pos, 2i)} = \sin\left( \frac{pos}{10000^{2i/d_{model}}} \right) \tag{33}
$$
此式将上述约束形式化，各项分别对应输入变换、非线性激活与输出生成. 

为了量化这一效应，定义：
$$
PE_{(pos, 2i+1)} = \cos\left( \frac{pos}{10000^{2i/d_{model}}} \right) \tag{34}
$$
其中 $pos$ 是位置索引，$i$ 是维度索引. 

这种编码的核心设计意图是**相对位置的可线性表达**. 利用三角恒等式，可以证明：

$$
PE_{pos + k} = f(PE_{pos}, PE_k) \tag{35}
$$
即位置 $pos + k$ 的编码可以由位置 $pos$ 和偏移 $k$ 的编码通过线性变换得到. 这意味着模型可以通过学习简单的注意力模式，来捕捉"相距 $k$ 个位置"的依赖关系，而不必为每个绝对位置单独记忆一种模式. 

**可学习位置编码**是另一种主流方案(BERT、GPT 使用)：直接将位置索引像 token 一样查表，$\mathbf{p}_{pos} = \mathbf{P}[pos]$，其中 $\mathbf{P} \in \mathbb{R}^{L_{max} \times d}$ 是可学习的矩阵. 它的优点是灵活，缺点是无法外推到训练时未见过的更长序列. 

#### RoPE：现代 LLM 的旋转位置编码

RoPE(Rotary Position Embedding，Su et al., 2021)是当前 LLM 的事实标准(LLaMA、Qwen、Baichuan 等均采用). 它不在输入端添加位置向量，而是**将位置信息编码到注意力计算的 Query 和 Key 向量中**. 

核心思想：将每对维度 $(d_{2i}, d_{2i+1})$ 视为二维平面上的一个向量，然后按照位置索引 $pos$ 进行旋转：

$$
\text{RoPE}(\mathbf{x}, pos) = \mathbf{x} \odot e^{i \cdot pos \cdot \theta} \tag{36}
$$
其中 $\theta$ 是一组预设的旋转频率. 写成矩阵形式，对于位置 $m$ 的向量 $\mathbf{x}_m$：

$$
\mathbf{x}_m^{(rot)} = \mathbf{R}_m \mathbf{x}_m \tag{37}
$$
其中 $\mathbf{R}_m$ 是一个分块对角矩阵，每个 $2 \times 2$ 块是一个旋转矩阵：

为建立定量关系，定义如下表达式：
$$
\mathbf{R}_m^{(i)} = \begin{pmatrix} \cos(m\theta_i) & -\sin(m\theta_i) \\ \sin(m\theta_i) & \cos(m\theta_i) \end{pmatrix} \tag{38}
$$
该关系式明确了系统的数学约束，可直接用于后续优化推导. 
RoPE 的精妙之处在于，它将位置信息编码为**向量空间的旋转**. 
- 绝对位置：通过旋转角度体现. 
- 相对位置：两个向量的内积只依赖于它们的相对距离 $m - n$，而与绝对位置无关($\mathbf{q}_m^T \mathbf{k}_n = f(\mathbf{x}_m, \mathbf{x}_n, m-n)$). 
- 长序列外推：通过调整旋转频率(NTK-aware 外推、YaRN 等)，RoPE 天然支持比训练时更长的序列.

#### Position Embedding 代码实现

```python
import torch
import torch.nn as nn
import math

class SinusoidalPositionEncoding(nn.Module):
    """
    Transformer 原始正弦位置编码
    对应公式:
      PE[pos, 2i]   = sin(pos / 10000^(2i/d))
      PE[pos, 2i+1] = cos(pos / 10000^(2i/d))
    """
    def __init__(self, d_model: int, max_len: int = 5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
        # 分母项: 10000^(2i/d) = exp(2i * -log(10000) / d)
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * 
            (-math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        # 注册为 buffer(不需要梯度)
        self.register_buffer('pe', pe.unsqueeze(0))  # (1, max_len, d_model)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (batch_size, seq_len, d_model)
        返回: (batch_size, seq_len, d_model)
        """
        return x + self.pe[:, :x.size(1), :]

class RotaryPositionalEmbedding(nn.Module):
    """
    RoPE 简化实现(仅展示核心旋转逻辑)
    对应公式: 将 Q/K 的每对维度视为复数，乘以 e^(i*m*theta)
    """
    def __init__(self, dim: int, max_seq_len: int = 2048, base: float = 10000.0):
        super().__init__()
        # 计算每个维度的旋转频率 theta_i = base^(-2i/dim)
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        self.register_buffer('inv_freq', inv_freq)
    
    def forward(self, x: torch.Tensor, seq_len: int) -> torch.Tensor:
        """
        x: (batch_size, n_heads, seq_len, head_dim)
        返回旋转后的 x(与输入同形状)
        """
        # 创建位置索引
        t = torch.arange(seq_len, device=x.device).type_as(self.inv_freq)
        # freqs: (seq_len, dim/2)
        freqs = torch.einsum('i,j->ij', t, self.inv_freq)
        # 将 freqs 扩展为复数指数形式
        # 实际工程中通常使用高效的复数运算或块对角矩阵乘法
        return x  # 简化示意，完整实现见后续 Attention 章节

# 验证正弦位置编码的相对位置性质
pe = SinusoidalPositionEncoding(d_model=64, max_len=100)
emb = pe.pe.squeeze(0)  # (100, 64)
# 位置 5 和位置 10 的编码
diff = emb[10] - emb[5]
# 注意：直接减法不是线性关系，但它们的内积模式仅依赖于距离 5
```

## 6. 损失函数：模型学习的"风向标"

损失函数定义了"什么是一个好的预测". 不同的任务需要不同的损失函数，而损失函数的选择直接决定了模型学到什么样的行为模式. 

### 6.1 交叉熵损失(Cross-Entropy Loss)：分类任务的核心

交叉熵是分类问题(包括语言模型)的默认损失函数. 它度量的是模型预测的概率分布与真实分布之间的"距离". 

#### 数学定义

对于单个样本，假设真实标签为 $y \in \{1, 2, \dots, C\}$，模型的输出 logits 经过 Softmax 后得到预测分布 $\hat{\mathbf{p}} = [\hat{p}_1, \dots, \hat{p}_C]$：

$$
\hat{p}_i = \frac{e^{z_i}}{\sum_{j=1}^{C} e^{z_j}} \tag{39}
$$
如式 (45) 所示，Softmax 通过指数归一化将任意实值 logits 转化为合法的概率分布，指数操作确保所有输出严格为正且和为 1. 

交叉熵损失为：

$$
\mathcal{L}_{CE} = -\sum_{i=1}^{C} p_i \log \hat{p}_i = -\log \hat{p}_y \tag{40}
$$
其中 $p_i$ 是真实分布(one-hot 向量，只有 $p_y = 1$，其余为 0). 因此公式简化为负对数似然(Negative Log-Likelihood, NLL). 

#### 物理意义

交叉熵的深层来源是信息论. $-\log \hat{p}_y$ 度量的是：当真实事件是 $y$ 时，使用模型预测的概率分布 $\hat{\mathbf{p}}$ 进行编码所需要的额外比特数(surprise). 

- 如果模型对正确答案非常确信($\hat{p}_y \approx 1$)，则 $-\log(1) = 0$，损失几乎为零. 
- 如果模型对正确答案完全没信心($\hat{p}_y \approx 0$)，则 $-\log(0) \to +\infty$，损失爆炸. 

这种"不对就严惩"的特性，迫使模型在训练时迅速找到对正确类别的高置信度预测. 

**标签平滑(Label Smoothing)** ：在大型 Transformer 训练中，原始的 one-hot 交叉熵容易导致模型过度自信. 标签平滑将目标分布从 $[0, 0, 1, 0]$ 修改为 $[\epsilon/(C-1), \epsilon/(C-1), 1-\epsilon, \epsilon/(C-1)]$，即给非正确类别分配少量概率质量. 这相当于对模型进行正则化，防止它对训练数据"死记硬背". 

#### 交叉熵损失代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

def cross_entropy_manual(logits: torch.Tensor, targets: torch.Tensor) -> torch.Tensor:
    """
    交叉熵手动实现
    对应公式: L = -log(softmax(logits)[target])
    """
    # logits: (batch_size, num_classes)
    # targets: (batch_size,) 的整数类别索引
    
    # Softmax 数值稳定版: 减去最大值
    max_logits = logits.max(dim=-1, keepdim=True).values
    stabilized = logits - max_logits
    
    # exp 和归一化
    exp_logits = torch.exp(stabilized)
    probs = exp_logits / exp_logits.sum(dim=-1, keepdim=True)
    
    # 取目标类别的概率
    batch_size = logits.size(0)
    target_probs = probs[torch.arange(batch_size), targets]
    
    # 负对数似然
    loss = -torch.log(target_probs + 1e-10)
    return loss.mean()

# 与 PyTorch 内置函数对比
batch_size, num_classes = 8, 1000
logits = torch.randn(batch_size, num_classes)
targets = torch.randint(0, num_classes, (batch_size,))

loss_manual = cross_entropy_manual(logits, targets)
loss_builtin = F.cross_entropy(logits, targets)
print(f"手动 CE: {loss_manual.item():.6f}")
print(f"PyTorch CE: {loss_builtin.item():.6f}")
# 两者几乎完全一致
```

### 6.2 均方误差(MSE)：回归任务的基石

推导过程如下：
$$
\mathcal{L}_{MSE} = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i)^2 \tag{41}
$$
MSE 惩罚的是预测值与真实值之间的平方误差. 平方操作使得大误差受到更严厉的惩罚——一个偏差为 10 的预测，其损失是偏差为 5 的预测的 4 倍. 

**在 LLM 中的应用**：虽然分类是语言模型的主要任务，但 MSE 在以下场景中出现：
- **值函数估计**：强化学习中，critic 网络用 MSE 拟合回报值. 

- **知识蒸馏**：有时用 MSE 匹配教师和学生模型的 logits(软标签蒸馏的变体). 

- **Embedding 对齐**：对比学习中，用 MSE 拉近正样本对的表示. 

### 6.3 对比损失(Contrastive Loss)：学习"谁像谁"

对比学习的核心思想是：**模型应该学会将相似的样本拉得更近，将不相似的样本推得更远**. 

#### InfoNCE：对比学习的标准形式

InfoNCE(Noise Contrastive Estimation)损失是现代对比学习(如 SimCLR、CLIP)的基础：

基于上述分析，建立如下数学关系：
$$
\mathcal{L}_{\text{InfoNCE}} = -\log \frac{e^{\text{sim}(\mathbf{z}_i, \mathbf{z}_i^+) / \tau}}{e^{\text{sim}(\mathbf{z}_i, \mathbf{z}_i^+) / \tau} + \sum_{j} e^{\text{sim}(\mathbf{z}_i, \mathbf{z}_j^-) / \tau}} \tag{42}
$$
其中：
- $\mathbf{z}_i$ 是锚点(anchor)样本的表示. 
- $\mathbf{z}_i^+$ 是与锚点配对的正样本表示. 
- $\mathbf{z}_j^-$ 是负样本表示. 
- $\text{sim}(\cdot, \cdot)$ 通常是余弦相似度或点积. 
- $\tau$ 是温度参数，控制分布的"尖锐程度". 

#### 物理意义

InfoNCE 本质上是一个**多分类问题**——锚点需要在 $\{1^+, 1^-, 2^-, 3^-, \dots\}$ 这个"类别集合"中，正确识别出自己的正样本. 温度参数 $\tau$ 起到了关键的调节作用：

- **$\tau$ 很小(如 0.07)** ：Softmax 分布变得非常尖锐，模型被迫将正样本的相似度推得极高，负样本推得极低. 这增强了区分度，但可能导致优化困难. 

- **$\tau$ 很大**：分布变得平缓，负样本之间的差异被模糊，训练更稳定但学到的表示区分度下降. 

**在 LLM 中的应用**：对比损失在 LLM 的后训练阶段扮演重要角色：
- **RLHF 中的奖励模型**：学习区分"人类偏好的回答"和"不想要的回答". 

- **Embedding 模型训练**：如 OpenAI 的 text-embedding-ada-002、BGE 系列，都是用对比损失训练，使得语义相似的句子在向量空间中距离更近. 

- **DPO(Direct Preference Optimization)** ：将 RLHF 简化为一个对比损失问题，直接偏好优化. 

#### 对比损失代码实现

```python
import torch
import torch.nn.functional as F

def info_nce_loss(anchor: torch.Tensor, positive: torch.Tensor, 
                  negatives: torch.Tensor, temperature: float = 0.07) -> torch.Tensor:
    """
    InfoNCE 损失简化实现
    对应公式: -log( exp(sim(a,p)/tau) / (exp(sim(a,p)/tau) + sum_j exp(sim(a,n_j)/tau)) )
    
    anchor: (d,)
    positive: (d,)
    negatives: (num_negatives, d)
    """
    # 计算余弦相似度
    sim_pos = F.cosine_similarity(anchor.unsqueeze(0), positive.unsqueeze(0), dim=-1)
    sim_negs = F.cosine_similarity(anchor.unsqueeze(0), negatives, dim=-1)
    
    # 拼接所有相似度: [sim(a,p), sim(a,n1), sim(a,n2), ...]
    all_similarities = torch.cat([sim_pos, sim_negs]) / temperature  # (1 + num_negatives,)
    
    # 正样本位于索引 0
    loss = -F.log_softmax(all_similarities, dim=0)[0]
    return loss

# 数值示例：温度参数的影响
anchor = torch.randn(128)
positive = anchor + torch.randn(128) * 0.1  # 与锚点相似
negatives = torch.randn(10, 128)  # 随机负样本

for tau in [0.01, 0.07, 1.0]:
    loss = info_nce_loss(anchor, positive, negatives, temperature=tau)
    print(f"温度 τ={tau:.2f}, Loss={loss.item():.4f}")
# 通常：tau 越小，loss 越大(因为需要更极端的区分度)
```

## 7. 从 RNN/LSTM 到 Transformer：为什么需要一场革命

在 Transformer 出现之前，序列建模的王者是循环神经网络(RNN)及其改进版 LSTM/GRU. 理解它们的局限，是理解 Transformer 为什么被发明的关键. 

### 7.1 RNN：顺序处理的困境

RNN 的核心思想是用一个隐藏状态 $\mathbf{h}_t$ 来累积序列的信息：

基于前述物理直觉，给出数学形式：
$$
\mathbf{h}_t = \tanh(\mathbf{W}_{hh} \mathbf{h}_{t-1} + \mathbf{W}_{xh} \mathbf{x}_t + \mathbf{b}_h) \tag{43}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 

为建立定量关系，定义如下表达式：
$$
\hat{\mathbf{y}}_t = \mathbf{W}_{hy} \mathbf{h}_t + \mathbf{b}_y \tag{44}
$$
该关系式明确了系统的数学约束，可直接用于后续优化推导. 
$\mathbf{h}_t$ 是一个"记忆容器". 在每个时间步，它读取当前输入 $\mathbf{x}_t$，并结合上一时刻的记忆 $\mathbf{h}_{t-1}$，更新自己的状态. 这种顺序依赖性使得 RNN 天然适合处理序列——理论上，$\mathbf{h}_t$ 包含了从第一个词到第 $t$ 个词的全部信息. 

**但 RNN 有三个致命的工程缺陷**：

1. **无法并行**：计算 $\mathbf{h}_t$ 必须先等 $\mathbf{h}_{t-1}$ 算完. 对于一个长度为 $T$ 的序列，时间复杂度是 $O(T)$，且无法利用 GPU 的并行能力. 在现代 LLM 动辄处理 128K token 的上下文中，这种串行性是不可接受的. 

2. **长程依赖的梯度消失/爆炸**：误差信号从时间步 $T$ 传回时间步 $t$，需要经过 $T-t$ 次矩阵乘法：

$$
\frac{\partial \mathcal{L}}{\partial \mathbf{h}_t} = \frac{\partial \mathcal{L}}{\partial \mathbf{h}_T} \cdot \prod_{k=t}^{T-1} \frac{\partial \mathbf{h}_{k+1}}{\partial \mathbf{h}_k} \tag{45}
$$
该关系式明确了系统的数学约束，可直接用于后续优化推导. 
如果 Jacobian 矩阵的最大特征值小于 1，这个连乘会指数级衰减到 0(梯度消失); 如果大于 1，则会指数级爆炸. 无论哪种情况，距离当前位置较远的历史信息都无法有效影响参数更新. 

3. **信息瓶颈**：整个序列的历史信息，被压缩到一个固定维度的隐藏向量 $\mathbf{h}_t$ 中. 当序列很长时，早期的信息会被后续信息"淹没"，造成信息丢失. 

### 7.2 LSTM：用门控机制延缓死亡

LSTM(Long Short-Term Memory)通过引入三个门控和一个独立的细胞状态来缓解梯度消失. 遗忘门 $\mathbf{f}_t$ 决定从旧细胞状态中丢弃哪些信息，输入门 $\mathbf{i}_t$ 决定吸收哪些新信息，候选细胞状态 $\tilde{\mathbf{C}}_t$ 则生成待写入的新内容：

$$
\mathbf{f}_t = \sigma(\mathbf{W}_f [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f) \tag{46}
$$
该关系式明确了系统的数学约束，可直接用于后续优化推导. 

为建立定量关系，定义如下表达式：
$$
\mathbf{i}_t = \sigma(\mathbf{W}_i [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i) \tag{47}
$$
此式描述了变量之间的定量关系，其中每一项对应特定的物理操作. 

基于前述物理直觉，给出数学形式：
$$
\tilde{\mathbf{C}}_t = \tanh(\mathbf{W}_C [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_C) \tag{48}
$$
此式将上述直觉形式化，各项分别对应输入变换、非线性激活与输出生成. 
细胞状态的更新是 LSTM 的核心创新——它采用加法路径而非连乘，使得梯度可以在时间维度上"高速公路式"地反向传播：

基于上述分析，建立如下数学关系：
$$
\mathbf{C}_t = \mathbf{f}_t \odot \mathbf{C}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{C}}_t \tag{49}
$$
输出门 $\mathbf{o}_t$ 控制从细胞状态中提取什么信息作为当前隐藏状态，最终隐藏状态通过对细胞状态的非线性变换和门控筛选得到：

$$
\mathbf{o}_t = \sigma(\mathbf{W}_o [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o) \tag{50}
$$

将上述约束转化为数学表达：
$$
\mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{C}_t) \tag{51}
$$
如式 (10) 所示，遗忘门通过 sigmoid 输出 0 到 1 之间的值，选择性保留旧信息; 如式 (11) 所示，输入门控制新信息的流入比例; 如式 (13) 所示，细胞状态的加法更新是避免梯度消失的关键——与 RNN 的连乘不同，梯度在这里可以直接通过加法路径无损传递; 如式 (15) 所示，最终隐藏状态是细胞状态经 tanh 压缩后再由输出门筛选的结果. 

**但 LSTM 仍然无法解决并行性问题**. 它的门控计算仍然依赖于 $\mathbf{h}_{t-1}$，这意味着无论如何优化，RNN/LSTM 类模型的训练速度都被序列长度牢牢锁死. 在 2017 年之前，训练一个能处理长文档的模型，需要数周甚至数月的时间. 

### 7.3 Transformer 的破局：注意力即一切

2017 年，Google 的《Attention Is All You Need》彻底改写了规则. Transformer 抛弃了循环结构，用**自注意力机制(Self-Attention)** 来建模序列依赖. 

自注意力的核心思想是：**序列中任意两个位置之间的依赖，都可以通过一次直接交互来计算，而不需要经过中间状态的逐步传递**. 

对于序列中的每个位置，模型生成三个向量：
- **Query(查询)** ："我在寻找什么信息？"
- **Key(键)** ："我拥有什么信息？"
- **Value(值)** ："我实际携带的信息内容是什么？"

然后通过点积计算 Query 和 Key 的匹配程度，用 Softmax 归一化得到注意力权重，最后对 Value 进行加权求和：

$$
\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left( \frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}} \right) \mathbf{V} \tag{52}
$$
该关系式明确了系统的数学约束，可直接用于后续优化推导. 
**为什么是革命性的**：

1. **完全并行**：序列中所有位置的 Query、Key、Value 可以同时计算. 对于长度为 $T$ 的序列，注意力矩阵的计算可以在 GPU 上一次性完成，时间复杂度从 $O(T)$ 降到了 $O(1)$(按层数计). 

2. **恒定长度的依赖路径**：在 RNN 中，位置 $i$ 和位置 $j$ 的信息交互需要经过 $|i-j|$ 步传递. 在 Transformer 中，无论两个 token 相距多远，它们的信息交换都只需要 **一层注意力 + 一层 FFN** 即可完成. 这使得捕捉长程依赖从"尽可能缓解"变成了"天然支持". 

3. **可解释性**：注意力权重矩阵 $\text{softmax}(\mathbf{Q}\mathbf{K}^T)$ 直接展示了序列中每个词"关注"了哪些其他词. "it" 的注意力权重在 "cat" 上飙升，这种直观的可解释性是 RNN 的隐藏状态所无法提供的. 

**代价**：天下没有免费的午餐. Transformer 的并行化以 $O(T^2)$ 的内存和计算复杂度为代价——注意力矩阵的大小是 $T \times T$. 对于长度为 128K 的上下文，这意味着一个 128K $\times$ 128K 的矩阵. 这是后续章节中 FlashAttention、稀疏注意力、线性注意力等算法试图解决的核心问题.

## 8. 总结与承上启下

本章作为"核心原理与架构"的入口，系统回顾了构建现代大语言模型所需的全部深度学习基础组件：

| 组件类别 | 核心概念 | 在 Transformer / LLM 中的作用 |
|---------|---------|---------------------------|
| 神经网络基础 | 感知机、激活函数、反向传播 | 构成 FFN 和输出层的计算基础; GELU/SwiGLU 是隐藏层的默认选择 |
| 优化器 | SGD、Momentum、Adam | AdamW 是预训练的标准配置; 动量加速收敛，自适应学习率处理异质参数 |
| 正则化 | Dropout、LayerNorm、BatchNorm | Dropout 防止过拟合; Pre-LN / RMSNorm 稳定深层训练; BN 不适用于 NLP |
| 嵌入层 | Word Embedding、Position Embedding | 将离散 token 映射到连续空间; RoPE 为注意力注入相对位置信息 |
| 损失函数 | CrossEntropy、MSE、对比损失 | CE 是语言建模的核心; 对比损失驱动 RLHF 和 Embedding 训练 |
| 序列建模演进 | RNN $\to$ LSTM $\to$ Transformer | 从顺序计算到完全并行，从梯度衰减到恒定路径，注意力机制颠覆一切 |

**关键物理直觉回顾**：

1. **没有非线性激活，就没有深度学习**. GELU 的平滑性和 SwiGLU 的门控机制，是 Transformer 能堆叠数百层的先决条件. 

2. **Adam 的自适应学习率不是锦上添花，而是必需品**. 在拥有 Embedding 层、注意力层、FFN 层这三类梯度尺度天差地别的网络中，统一的学习率会导致某些层无法收敛. 

3. **LayerNorm 的位置比形式更重要**. Pre-LN 将归一化移到残差分支之前，使得梯度可以畅通无阻地从深层流回浅层——这是训练 100+ 层模型的关键. 

4. **从 RNN 到 Transformer 不是渐进式改良，而是范式革命**. 当序列长度超过一定阈值后，$O(T)$ 的串行计算无论如何优化，都无法与 $O(1)$ 的并行计算竞争. 

**承上启下**：

掌握了这些基础组件后，我们终于具备了理解 Transformer 核心机制的全部先验知识. 在下一节(2.2 Attention 机制)中，我们将把本章学到的向量空间、Softmax、矩阵乘法、位置编码等概念全部串联起来，详细剖析 Query-Key-Value 的数学本质、多头注意力的并行计算、以及因果掩码(Causal Masking)如何在自回归生成中保证因果性.
