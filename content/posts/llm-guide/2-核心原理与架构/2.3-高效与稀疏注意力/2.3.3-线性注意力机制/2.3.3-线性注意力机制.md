# 2.3.3-线性注意力机制

## 1. 演进脉络与技术演进

Transformer 的自注意力机制将序列建模从 RNN 的 $O(n)$ 串行依赖改造为 $O(n^2)$ 的矩阵并行计算, 在 GPU 批量运算特性上获得了数量级优势. 但当序列长度从 1K 扩展至 128K 甚至 1M, 注意力层的计算量与内存占用随长度平方增长——$128K^2 = 1.6 \times 10^{10}$ 个注意力分数的存储与计算, 使标准 Transformer 在长序列场景下沦为物理不可行. 

FlashAttention 系列通过 tiling 与 softmax safe 重排将 $O(n^2)$ 的 HBM 访存降至 $O(n)$, 但计算量(FLOPs)仍为 $O(n^2)$. 稀疏注意力(MoBA、NSA)通过放弃全局注意力的假设, 将复杂度压至 $O(n \log n)$ 或 $O(n \sqrt{n})$, 但稀疏模式的设计本身成为超参数负担, 且部分任务的全局依赖无法被局部窗口捕获. 

线性注意力的诞生源于一个根本追问：能否从数学形式上改造注意力, 使其复杂度真正与序列长度呈线性关系？这一追问在 2020-2024 年间催生了三条平行路线：

| 年份 | 方法 | 核心思想 | 复杂度 |
|:----|:-----|:--------|:------|
| 2020 | Linear Transformer (Katharopoulos et al.) | 核技巧分解 Softmax, 矩阵乘法结合律重排 | $O(n)$ |
| 2021 | Performer / FAVOR+ (Choromanski et al.) | 随机正交特征映射近似 Softmax 核 | $O(n)$ |
| 2021 | RWKV (Peng et al.) | 将 time-mixing 写成 RNN 递归形式, 消除二次注意力 | $O(n)$ |
| 2023 | Mamba (Gu & Dao) | 选择性状态空间模型(Selective SSM), 输入依赖的状态转移 | $O(n)$ |
| 2024 | Mamba-2 (Dao & Gu) | Structured Attention + SSM 统一视角, SSD 算法加速 | $O(n)$ |

前代方法的局限构成线性注意力的设计动机：

- Softmax 的 $O(n^2)$ 瓶颈：$\text{softmax}(QK^T/\sqrt{d})$ 要求先计算完整的 $n \times n$ 分数矩阵, 再逐行归一化. 这一操作无法分解为序列上的增量更新. 
- 长序列 KV Cache 膨胀：标准注意力的 KV Cache 随序列线性增长, 在 $n = 128K$、$d = 4096$、batch = 32 时占用 32GB, 逼近单卡显存上限. 
- 因果掩码的串行依赖：自回归生成时, 每个新 token 必须 attend 到全部历史 token, 导致解码阶段无法像训练那样并行. 

线性注意力的设计初衷是打破 Softmax 的不可分解性, 将注意力写成线性代数中的结合律形式, 使 $(\phi(Q) \phi(K)^T) V$ 重排为 $\phi(Q) (\phi(K)^T V)$, 从而将 $n \times n$ 的中间矩阵消去. 更进一步, Mamba 和 RWKV 将注意力隐式地编码为状态向量, 使序列建模退化为 RNN 式的状态转移, 实现真正的长度无关复杂度. 

## 2. 工业价值与应用场景

线性注意力已从学术概念进入工业部署, 在特定场景下成为标准注意力的替代方案. 

| 模型/产品 | 采用的线性注意力技术 | 解决的关键指标 | 部署规模 |
|:---------|:-------------------|:-------------|:--------|
| MiniMax-01 | 线性注意力 + Lightning Attention | 4M 上下文窗口的推理延迟从分钟级降至秒级 | 生产环境 |
| RWKV-5/6 | RWKV time-mixing + token-shift | 在消费级 GPU 上运行 14B 模型的 32K 上下文 | 开源社区 + 边缘部署 |
| Mamba / Mamba-2 | Selective SSM + SSD | 基因组序列(100K-1M tokens)建模速度提升 5x | 生信研究 + 长序列预训练 |
| Jamba (AI21 Labs) | Mamba + Transformer 混合层 | 256K 上下文的吞吐量提升 3x, 同时保持 Transformer 精度 | 企业 API |
| Falcon Mamba | 纯 Mamba 架构 | 7B 参数在 1M 序列上的推理显存 < 20GB | 开源模型 |

关键工业指标改善：

- 吞吐量：在序列长度 $n > 16K$ 时, 线性注意力的解码吞吐量不再随 $n$ 衰减. 标准注意力在 $n = 64K$ 时吞吐量降至 $n = 1K$ 的 3% 以下, Linear Transformer 保持在 85% 以上. 
- 显存占用：Mamba-2 在 1M 序列上的 KV Cache 等价物(状态向量)占用约 50MB, 而 Llama-3 的 KV Cache 在同等条件下占用 128GB. 
- 训练稳定性：RWKV 的 RNN 形式消除了梯度裁剪对注意力的依赖, 在超长序列(>100K)预训练时 Loss 曲线更稳定. 

适用场景边界：

- 序列长度 > 8K 且以自回归解码为主：线性注意力的优势区间. 
- 需要全局精度的下游任务(如代码补全中的跨文件依赖检索)：混合架构(Jamba)更优. 
- 短序列(< 2K)高并发推理：标准 Transformer + FlashAttention 仍具优势, 线性注意力的额外 kernel 开销无法摊销. 

## 3. 物理直觉与核心机制

### 3.1 核技巧：将内积映射到特征空间

标准注意力的 Softmax 操作 $a_{ij} = \exp(q_i^T k_j / \sqrt{d})$ 在数学上是一种相似度核. 核技巧的核心洞察：若存在特征映射 $\phi$ 使得 $\kappa(q, k) = \phi(q)^T \phi(k)$, 则无需显式计算 $n \times n$ 的核矩阵, 可通过矩阵乘法的结合律绕过. 

物理类比：Softmax 注意力如同在 $n$ 个历史 token 中逐一比较相似度, 再按相似度加权求和. 核技巧将"逐一比较"转化为"将每个 token 投影到固定维度的特征空间, 再累加特征向量". 比较操作从 $O(n^2)$ 降为 $O(n)$, 因为每个 token 只被处理一次, 其特征向量被累加到全局状态. 

### 3.2 递归状态：从 Attention 到 RNN

Linear Transformer 的核心思想是：与其在每一步重新计算所有历史 token 的注意力权重, 不如维护一个持续更新的隐状态, 将历史信息以压缩形式存储. 具体地, 引入累积状态矩阵 $s_t \in \mathbb{R}^{m \times d}$ 来编码截至时间步 $t$ 的全部 key-value 信息, 同时用归一化累积向量 $z_t \in \mathbb{R}^m$ 保证输出概率和为 1, 得到如式 (1) 所示的累积形式：

$$
s_t = s_{t-1} + \phi(k_t)^T v_t, \quad o_t = \frac{\phi(q_t) s_t}{\phi(q_t) z_t} \tag{1}
$$
这里, $s_{t-1}$ 是上一时间步的状态矩阵; $\phi(k_t) \in \mathbb{R}^m$ 是当前 token 经特征映射后的 key 向量; $v_t \in \mathbb{R}^d$ 是当前 value 向量; $q_t \in \mathbb{R}^d$ 是当前 query 向量. 式 (1) 的物理意义十分清晰：左侧子式 $s_t = s_{t-1} + \phi(k_t)^T v_t$ 执行状态增量更新, 将当前 token 的 key-value 外积累加到全局状态矩阵, 每一步仅需 $O(md)$ 计算且与历史长度无关; 输出 $o_t$ 的分子 $\phi(q_t) s_t$ 通过 query 与累积状态矩阵相乘, 一次性检索全部历史的加权 value 总和, 分母 $\phi(q_t) z_t$ 则提供全局归一化因子, 确保最终输出是合法的加权平均. 这一变换将 $O(n^2)$ 的注意力矩阵计算替换为 $O(md)$ 的状态更新, 是线性注意力实现线性复杂度的核心代数操作. 该形式与 RNN 的隐状态更新 $h_t = f(h_{t-1}, x_t)$ 同构——注意力不再是"回头看所有历史并重新分配权重", 而是"维护一个压缩了全部历史的隐状态, 每步只读取当前输入并更新状态". 

物理类比：标准注意力如同每次做决策前翻阅整本日记; 线性注意力如同维护一份持续更新的摘要笔记, 决策时只看笔记而非原始日记. 

### 3.3 选择性遗忘：Mamba 的门控机制

标准 SSM 的状态转移矩阵 $A$ 是输入无关的常数, 这导致所有 token 以相同速率遗忘历史, 无法区分关键信息与噪声. Mamba 的核心改进是将离散化参数 $B, C, \Delta$ 全部参数化为当前输入 $x_t$ 的函数, 使每个 token 拥有独立的状态更新策略. 如式 (2) 所示, 离散时间步 $t$ 的隐状态 $h_t$ 由历史状态的输入依赖衰减与当前输入的输入依赖写入共同决定：

推导过程如下：
$$
h_t = \bar{A}_t h_{t-1} + \bar{B}_t x_t, \quad y_t = C_t h_t \tag{2}
$$

其中 $\bar{A}_t = \exp(\Delta_t A)$ 是输入依赖的离散状态转移矩阵, $\bar{B}_t$ 是输入依赖的离散输入耦合矩阵, $C_t$ 是输入依赖的输出观测矩阵. 高 $\Delta_t$ 时 $\bar{A}_t$ 使历史状态快速衰减, 模型聚焦当前输入; 低 $\Delta_t$ 时历史信息被长期保留. 输出 $y_t = C_t h_t$ 通过输入依赖的观测矩阵 $C_t$ 决定从状态中检索哪些维度生成输出. 式 (2) 将固定参数的线性 SSM 升级为输入自适应的动态系统：不同 token 触发不同的遗忘/写入/读取模式, 实现"有选择的记忆", 这是 Mamba 在线性复杂度与表达能力之间取得平衡的关键. 

$\bar{A}_t = \exp(\Delta_t A)$ 中的 $\Delta_t$ 由输入 $x_t$ 经线性投影 + softplus 生成. 高 $\Delta_t$ 意味着"聚焦当前输入, 快速遗忘历史"(类似注意力中的高权重); 低 $\Delta_t$ 意味着"保留历史状态, 忽略当前扰动". 

物理类比：$\Delta_t$ 是镜头的快门速度. 面对快速运动场景(信息密度高), 提高快门速度(高 $\Delta_t$)捕捉细节; 面对静态场景(信息冗余), 降低快门速度(低 $\Delta_t$)保持画面稳定. 

### 3.4 RWKV 的 time-mixing：通道级门控

RWKV 将注意力的 query-key-value 交互拆解为时间混合(time-mixing)与通道混合(channel-mixing). Time-mixing 为每个维度独立维护衰减率 $w$ 和门控 $u$, 使不同特征通道以不同时间尺度更新. 这模拟了生物神经元的多种时间常数——某些神经元对瞬时刺激敏感, 另一些对累积信号敏感. 

## 4. 数学推导与公式对比

### 4.1 标准注意力的不可分解性

为了理解线性注意力对标准注意力的改进, 首先回顾标准自注意力的矩阵形式. 给定输入序列矩阵 $X \in \mathbb{R}^{n \times d_{\text{model}}}$, 通过可学习的投影矩阵 $W_Q, W_K, W_V$ 分别生成 Query 矩阵 $Q = XW_Q \in \mathbb{R}^{n \times d_k}$、Key 矩阵 $K = XW_K \in \mathbb{R}^{n \times d_k}$ 和 Value 矩阵 $V = XW_V \in \mathbb{R}^{n \times d_v}$, 其中 $n$ 为序列长度, $d_k$ 为 query/key 维度. 标准自注意力的输出矩阵 $O \in \mathbb{R}^{n \times d}$ 如式 (3) 所示：

$$
O = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V \tag{3}
$$
式 (3) 的计算流程包含四个关键环节：首先, $QK^T$ 计算所有 $n \times n$ 个 query-key 点积, 生成注意力分数矩阵; 随后除以 $\sqrt{d_k}$ 进行缩放, 防止方差随维度增加而膨胀; 接着 $\text{softmax}(\cdot)$ 对每行进行归一化, 将分数转化为注意力概率分布; 最后用该概率分布对 $V$ 的各行加权求和, 得到每个位置的上下文表示. Softmax 的逐行归一化操作使 $(QK^T)V$ 无法利用矩阵乘法结合律重排为 $Q(K^T V)$, 这正是标准注意力 $O(n^2)$ 复杂度的数学根源. 

其中 $S = QK^T / \sqrt{d_k} \in \mathbb{R}^{n \times n}$ 为注意力分数矩阵, $S_{ij}$ 表示第 $i$ 个 query 对第 $j$ 个 key 的原始分数. Softmax 的逐行归一化操作如式 (4) 所示：

$$
\text{softmax}(S)_{ij} = \frac{\exp(S_{ij})}{\sum_{l=1}^{n} \exp(S_{il})} \tag{4}
$$
该关系式明确了系统的数学约束, 可直接用于后续优化推导. 
式 (4) 中, 分子 $\exp(S_{ij})$ 将单个分数映射为正实数并放大差异, 使注意力分布更加尖锐; 分母 $\sum_{l=1}^{n} \exp(S_{il})$ 对第 $i$ 行所有分数的指数求和, 充当归一化配分函数, 确保每个位置的注意力权重是其相对"吸引力"占总吸引力的比例, 且满足 $\sum_j \text{softmax}(S)_{ij} = 1$. 关键在于, 分母必须遍历第 $i$ 行的全部 $n$ 个元素, 这使得 softmax 无法分解为增量形式——每新增一个历史 token, 整个分母都需要重新计算, 从而阻止了 $O(n^2)$ 向 $O(n)$ 的复杂度下降. 

分母 $\sum_l \exp(S_{il})$ 依赖于整行 $n$ 个元素, 使 $(QK^T)V$ 无法重排为 $Q(K^T V)$. 这是 $O(n^2)$ 复杂度的数学根源. 

### 4.2 线性注意力的核分解推导

线性注意力的核心步骤是将 Softmax 核 $\kappa(q, k) = \exp(q^T k)$ 替换为可分解核 $\kappa(q, k) = \phi(q)^T \phi(k)$. 

Katharopoulos et al. (2020) 证明：对任意核函数, 若存在特征映射 $\phi: \mathbb{R}^d \to \mathbb{R}^m$ 使得 $\kappa(q, k) = \phi(q)^T \phi(k)$, 则：

推导过程如下：
$$
\begin{aligned}
O_i &= \sum_{j=1}^{n} \frac{\phi(q_i)^T \phi(k_j) v_j}{\sum_{l=1}^{n} \phi(q_i)^T \phi(k_l)} \\
&= \frac{\phi(q_i)^T \sum_{j=1}^{n} \phi(k_j) v_j}{\phi(q_i)^T \sum_{l=1}^{n} \phi(k_l)} \\
&= \frac{\phi(q_i)^T S_n}{\phi(q_i)^T Z_n}  \tag{5} \end{aligned}
$$
其中 $S_n = \sum_{j=1}^{n} \phi(k_j)^T v_j \in \mathbb{R}^{m \times d}$, $Z_n = \sum_{l=1}^{n} \phi(k_l) \in \mathbb{R}^m$. 矩阵乘法的结合律将 $O(n^2 d)$ 的 $(QK^T)V$ 重排为 $O(n m d)$ 的增量累积. 

特征映射 $\phi$ 的选择：

| 方法 | 特征映射 $\phi(x)$ | 维度 $m$ | 核函数近似 |
|:-----|:-----------------|:--------|:---------|
| Linear Transformer | $\text{elu}(x) + 1$ | $d$ | 无显式核, 经验有效 |
| Performer (FAVOR+) | $\frac{1}{\sqrt{m}} \exp(W_r x - \frac{\|x\|^2}{2})$ | $r$(随机特征数) | $\exp(q^T k)$ |
| Performer (正交) | 上述 + 正交随机矩阵 | $r$ | 降低方差 |

在因果掩码约束下(即自回归生成场景), 线性注意力可以写成更适合增量计算的递归形式. 如式 (6) 所示, 时间步 $t$ 的累积状态矩阵 $S_t$ 由上一时间步的状态 $S_{t-1}$ 与当前 token 的 key-value 外积 $\phi(k_t)^T v_t$ 相加得到, 同时归一化向量 $Z_t$ 也同步累积当前 key 的特征映射 $\phi(k_t)$：

$$
S_t = S_{t-1} + \phi(k_t)^T v_t, \quad Z_t = Z_{t-1} + \phi(k_t) \tag{6}
$$
式 (6) 的物理意义在于：$S_t$ 以压缩形式保留全部历史信息, 每步只需将当前 token 的贡献增量式地加入, 无需回溯重写; $Z_t$ 则持续维护归一化所需的累积量, 用于后续输出计算. 每个时间步的计算量仅为 $O(md)$, 与历史长度 $t$ 完全无关. 这一增量形式使自回归解码无需存储随序列增长的 KV Cache, 仅需维护固定大小的状态矩阵 $S_t \in \mathbb{R}^{m \times d}$ 和向量 $Z_t \in \mathbb{R}^m$, 将显存占用从 $O(nd)$ 降至 $O(md)$. 

每个时间步 $t$ 的计算量仅为 $O(md)$, 与历史长度无关. 

### 4.3 Performer 的随机特征映射

Performer (Choromanski et al., 2021) 的核心是 FAVOR+(Fast Attention Via Orthogonal Random Features)：用随机傅里叶特征近似高斯核 $\kappa(q, k) = \exp(-\|q - k\|^2 / 2)$. 

Performer 的随机特征近似建立在 Bochner 定理之上：任何平移不变核函数 $\kappa(q, k)$ 都可以表示为随机频率向量 $w$ 的复指数期望. 具体地, 从标准正态分布 $w \sim \mathcal{N}(0, I)$ 采样随机频率, 计算其与 query-key 差 $q - k$ 的内积 $w^T (q - k)$, 再通过复指数 $\exp(i w^T (q - k))$(由欧拉公式可分解为 $\cos + i\sin$ 形式)编码该投影, 最后对所有可能的随机频率取期望, 如式 (7) 所示：

$$
\kappa(q, k) = \mathbb{E}_{w \sim \mathcal{N}(0, I)} [\exp(i w^T (q - k))] \tag{7}
$$
此式将上述直觉形式化, 各项分别对应输入变换、非线性激活与输出生成. 
式 (7) 是 Performer 的理论基石：Bochner 定理保证该期望恰好等于高斯核 $\exp(-\|q-k\|^2/2)$. 在实际计算中, 无法用无限多个 $w$ 求期望, 因此 Performer 用有限个随机频率的采样平均值来近似这一期望, 从而将不可分解的指数核转化为可分解的线性特征映射. 

基于式 (7) 的理论保证, Performer 构造了如式 (8) 所示的随机傅里叶特征映射 $\phi: \mathbb{R}^d \to \mathbb{R}^m$, 将输入向量 $x$(query 或 key)映射到 $m$ 维特征空间：

$$
\phi(x) = \frac{1}{\sqrt{m}} \exp\left(-\frac{\|x\|^2}{2}\right) [\exp(i w_1^T x), \dots, \exp(i w_m^T x)]^T \tag{8}
$$
式 (8) 包含三个关键设计：高斯振幅调制项 $\exp(-\|x\|^2/2)$ 用于抵消核函数中的输入自相关项, 确保平移不变性; $[\exp(i w_1^T x), \dots, \exp(i w_m^T x)]^T$ 是在 $m$ 个独立采样的随机频率 $w_1, \dots, w_m \sim \mathcal{N}(0, I)$ 下的复指数特征, 将输入编码到随机频率空间; 归一化因子 $1/\sqrt{m}$ 保证 $\phi(q)^T \phi(k)$ 的期望等于真实核值. 通过这一映射, 原本不可分解的指数核 $\exp(q^T k)$ 被近似为有限维内积 $\phi(q)^T \phi(k)$, 从而实现了线性复杂度. 

FAVOR+ 进一步引入正交随机特征(ORF)：将随机矩阵 $W \in \mathbb{R}^{m \times d}$ 约束为正交矩阵的倍数, 使特征估计的方差从 $O(1/m)$ 降至 $O(1/m^2)$. 这在 $m = 256$ 时即可达到与精确 Softmax 在 GLUE 上无显著差异的精度. 

Performer 的复杂度：

- 训练：$O(n \cdot m \cdot d)$, $m \ll n$ 时获得线性加速
- 解码：$O(m \cdot d)$ 每步, KV Cache 等价物为 $S_t \in \mathbb{R}^{m \times d}$

### 4.4 RWKV 的 time-mixing 公式

RWKV-4 的 time-mixing 层(等效于自注意力)定义如下：

$$
\begin{aligned}
r_t &= W_r \cdot (\mu_r x_t + (1 - \mu_r) x_{t-1}) \\
k_t &= W_k \cdot (\mu_k x_t + (1 - \mu_k) x_{t-1}) \\
v_t &= W_v \cdot (\mu_v x_t + (1 - \mu_v) x_{t-1}) \\
w_{tk} &= \exp(-\exp(\omega) \cdot (t - k) + u) \\
a_t &= \frac{\sum_{k=1}^{t} \exp(r_t^T k_k + u) \cdot v_k}{\sum_{k=1}^{t} \exp(r_t^T k_k + u)}  \tag{9} \end{aligned}
$$
此式给出了精确的数学定义, 为算法实现提供了理论基础. 
$\mu_r, \mu_k, \mu_v$ 是可学习的 token-shift 系数, $\omega$ 是通道级衰减率, $u$ 是即时位置偏置. RWKV-5/6 将上述形式进一步改写为真正的 RNN 递归, 消除了求和上限对 $t$ 的依赖. 

RWKV-6 的 RNN 形式：

基于上述分析, 建立如下数学关系：
$$
\begin{aligned}
s_t &= s_{t-1} \odot \exp(-w) + \exp(k_t) \odot v_t \\
a_t &= \sigma(r_t) \odot \frac{s_t}{\exp(k_t) \cdot a_{t-1} + b_t}  \tag{10} \end{aligned}
$$
其中 $s_t$ 是累加器状态, $w$ 是数据无关的衰减向量. 该形式与 SSM 的状态更新方程同构, 证明了 RWKV 本质上是特定参数化的线性 SSM. 

### 4.5 Mamba 的选择性状态空间方程

Mamba 的数学基础是经典控制理论中的连续时间状态空间模型. 该系统用隐状态向量 $h(t) \in \mathbb{R}^N$ 描述内部动态, 其中 $\dot{h}(t) = dh/dt$ 表示状态对时间的导数. 状态转移矩阵 $A \in \mathbb{R}^{N \times N}$ 控制无外部输入时系统的自然演化(如指数衰减), 输入耦合矩阵 $B \in \mathbb{R}^{N \times 1}$ 决定外部信号 $x(t)$ 如何驱动状态变化, 输出观测矩阵 $C \in \mathbb{R}^{1 \times N}$ 则将状态的某些线性组合映射为可观测输出 $y(t)$. 如式 (11) 所示：

为了量化这一效应, 定义：
$$
\dot{h}(t) = A h(t) + B x(t), \quad y(t) = C h(t) \tag{11}
$$
式 (11) 左侧子式 $\dot{h}(t) = A h(t) + B x(t)$ 是一个标准线性常微分方程(ODE)：$A h(t)$ 描述状态自演化, $B x(t)$ 提供外部驱动, 两者之和决定状态变化率; 右侧 $y(t) = C h(t)$ 从状态空间中读取输出. 这是经典 SSM 的连续时间形式, Mamba 的创新在于将原本固定的 $A, B, C$ 以及离散化步长 $\Delta$ 全部改为输入依赖, 从而实现选择性动态. 

为了在数字计算机上执行连续时间模型, 需要将其离散化. 采用零阶保持(Zero-Order Hold, ZOH)假设：输入 $x(t)$ 在每个采样区间 $[k\Delta, (k+1)\Delta)$ 内保持常数. 在此假设下, 连续 ODE $\dot{h} = Ah + Bx$ 可转化为离散递推形式, 其中离散化参数如式 (12) 所示：

$$
\bar{A} = \exp(\Delta A), \quad \bar{B} = (\Delta A)^{-1}(\bar{A} - I) \cdot \Delta B \tag{12}
$$
此式将上述约束形式化, 各项分别对应输入变换、非线性激活与输出生成. 
此式给出了精确的数学定义, 为算法实现提供了理论基础. 
式 (12) 中, $\bar{A} = \exp(\Delta A)$ 将连续时间衰减矩阵 $A$ 按步长 $\Delta$ 离散化, 得到一步状态转移的衰减因子, 在标量情形下等价于 $e^{\Delta a}$; $\bar{B}$ 的表达式包含零阶保持的精确校正：$(\Delta A)^{-1}(\bar{A} - I)$ 是将连续输入在区间 $[0, \Delta]$ 上均匀分布的假设转化为离散等效的校正因子, 再乘以 $\Delta B$ 完成输入项的离散化. 整体而言, 式 (12) 将连续 ODE 转化为 $h_t = \bar{A} h_{t-1} + \bar{B} x_t$, 使 SSM 可在数字计算机上逐步执行. 

Mamba 相对于标准 SSM 的核心创新在于：将原本输入无关的参数 $B, C, \Delta$ 全部参数化为当前输入 $x_t$ 的函数, 使每个 token 拥有独立的选择性遗忘、更新与读取策略. 如式 (13) 所示：

$$
\Delta_t = \text{softplus}(W_\Delta x_t + b_\Delta), \quad B_t = W_B x_t, \quad C_t = W_C x_t \tag{13}
$$
式 (13) 中, $\Delta_t$ 由输入经线性投影 $W_\Delta x_t + b_\Delta$ 后再通过 softplus 激活(即 $\log(1+e^x)$, 保证输出严格为正)生成, 大 $\Delta_t$ 对应"快速更新、聚焦当前输入", 小 $\Delta_t$ 对应"缓慢演化、保留历史"; $B_t = W_B x_t$ 使输入耦合矩阵与当前内容相关, 决定 $x_t$ 的哪些特征维度应被写入状态; $C_t = W_C x_t$ 使输出观测矩阵同样输入依赖, 决定从状态中检索哪些特征生成输出. 通过将 $(\Delta, B, C)$ 全部改为输入函数, Mamba 实现了选择性状态空间——不同 token 触发不同的状态转移模式. 

将输入依赖的参数代入离散化框架, 得到 Mamba 选择性 SSM 的最终递推形式. 如式 (14) 所示, 离散时间步 $t$ 的隐状态 $h_t$ 由历史状态经输入依赖衰减 $\bar{A}_t h_{t-1}$ 与当前输入经输入依赖写入 $\bar{B}_t x_t$ 相加得到, 输出则由输入依赖观测矩阵 $C_t$ 从状态中检索：

$$
h_t = \bar{A}_t h_{t-1} + \bar{B}_t x_t, \quad y_t = C_t h_t \tag{14}
$$
式 (14) 中, $\bar{A}_t = \exp(\Delta_t A)$ 是输入依赖的离散状态转移矩阵, 高 $\Delta_t$ 时使历史状态快速衰减以聚焦当前信息, 低 $\Delta_t$ 时长期保留历史; $\bar{B}_t$ 决定 $x_t$ 的哪些维度进入状态; $C_t$ 决定从状态中检索哪些信息生成输出 $y_t$. 该方程将固定参数的线性 SSM 升级为输入自适应的动态系统, 不同 token 触发不同的遗忘/写入/读取模式, 实现"有选择的记忆", 这是 Mamba 在线性复杂度与表达能力之间取得平衡的关键. 

$\bar{A}_t = \exp(\Delta_t A)$ 是对角矩阵的指数. 由于 $\Delta_t$ 输入依赖, $\bar{A}_t$ 随 token 变化, 实现"选择性遗忘". 

Mamba-2 的 SSD(State Space Duality)框架证明：选择性 SSM 等价于一种结构化掩码的线性注意力, 其中注意力掩码 $M_{ij} = \prod_{k=i+1}^{j} \bar{a}_k$(累积乘积形式). 这一发现使 Mamba-2 能用 FlashAttention 风格的块级并行算法加速 SSM 计算, 在硬件效率上追平甚至超越标准 Transformer.

## 5. 算力成本与数值走查

以 $d = 4096$、$n_{heads} = 32$、$d_{head} = 128$ 的单层注意力为例, 对比不同序列长度下标准注意力与线性注意力的 FLOPs 和显存占用. 

### 5.1 FLOPs 对比

为了量化线性注意力带来的计算收益, 以 $d = 4096$、$n_{heads} = 32$、$d_{head} = 128$ 的单层注意力为例, 对比不同序列长度下的 FLOPs. 标准注意力前向传播(忽略 Q/K/V/O 投影矩阵)的 FLOPs 如式 (15) 所示：

$$
\text{FLOPs}_{\text{std}} = 2 \cdot n_{heads} \cdot n^2 \cdot d_{head} = 2 \cdot 32 \cdot n^2 \cdot 128 = 8192 \cdot n^2 \tag{15}
$$
式 (15) 中, $n^2$ 来自 $QK^T$ 的 $n \times n$ 矩阵乘法, 是二次复杂度的唯一来源; $d_{head}$ 是每个点积涉及的乘加次数; $n_{heads}$ 是多头并行数; 系数 $2$ 表示每次乘加计为 2 FLOPs. 当序列长度 $n$ 从 1K 增长到 1M 时, $n^2$ 项使总 FLOPs 增长 $10^6$ 倍, 标准注意力在超长序列上趋于物理不可行. 

线性注意力通过矩阵结合律消去了 $n \times n$ 中间矩阵, 将序列复杂度从二次降为一次. 采用特征映射维度 $m = 256$ 时, 线性注意力前向 FLOPs 如式 (16) 所示：

$$
\text{FLOPs}_{\text{lin}} = 2 \cdot n_{heads} \cdot n \cdot m \cdot d_{head} = 2 \cdot 32 \cdot n \cdot 256 \cdot 128 = 2,097,152 \cdot n \tag{16}
$$
此式给出了形式化的数学定义, 建立了输入与输出之间的定量关系. 
此式给出了精确的数学定义, 为算法实现提供了理论基础. 
式 (16) 中, $n$ 替代了式 (15) 中的 $n^2$, 这是线性复杂度的核心来源; $m$ 是特征映射维度, 在核分解中替代了点积维度 $d_{head}$ 的角色. 总复杂度为 $O(n \cdot m \cdot d_{head})$, 当 $m \ll n$ 时获得线性加速. 例如 $n = 128K$、$m = 256$ 时, 线性注意力的 FLOPs 约为标准注意力的 $256/128K \approx 1/500$, 实现数百倍理论加速. 

在 $n = 1K$ 时, 线性注意力的优势不明显, 甚至因特征映射的额外计算略慢. 在 $n = 128K$ 时, 481x 的理论加速比使线性注意力成为长序列建模的必选项. 

### 5.2 显存占用对比

标准注意力的显存消耗(FP16)：

- 注意力分数矩阵：$n^2 \cdot n_{heads} \cdot 2 \text{ bytes} = 64 \cdot n^2$ bytes
- KV Cache：$2 \cdot n \cdot d \cdot 2 \text{ bytes} = 16 \cdot n \cdot d$ bytes

线性注意力的显存消耗：

- 状态矩阵 $S_t$：$n_{heads} \cdot m \cdot d_{head} \cdot 2 \text{ bytes}$
- 归一化向量 $Z_t$：$n_{heads} \cdot m \cdot 2 \text{ bytes}$

| 序列长度 $n$ | 标准 Attention 显存 | 线性 Attention 显存 | 压缩比 |
|:-----------|:------------------|:-------------------|:------|
| 1K | 131MB | 4.2MB | 31x |
| 8K | 8.2GB | 4.2MB | 1958x |
| 32K | 131GB | 4.2MB | 31250x |
| 128K | 2,097GB | 4.2MB | 500000x |
| 1M | 134TB | 4.2MB | $3.2 \times 10^7$x |

线性注意力的状态存储与序列长度完全无关. 在 $n = 128K$ 时, 标准注意力的 KV Cache 已达 2TB, 线性注意力仍为 4.2MB——这一差距决定了硬件可行性. 

### 5.3 带宽瓶颈分析

在评估算法实际效率时, 需要判断其受限于算力还是受限于内存带宽. 以 A100 为例, FP16 Tensor Core 峰值算力为 312 TFLOPS, HBM 显存带宽为 2 TB/s. roofline 模型的计算强度阈值如式 (17) 所示：

$$
I_{\text{crit}} = \frac{\text{FLOPs}}{\text{Bytes}} = \frac{312 \times 10^{12}}{2 \times 10^{12}} = 156 \tag{17}
$$
式 (17) 中, 分子 $312 \times 10^{12}$ 代表 GPU 每秒能执行的浮点运算次数上限, 即算力天花板; 分母 $2 \times 10^{12}$ 代表 GPU 每秒能从 HBM 读取的字节数上限, 即带宽天花板; 比值 $156$ 的含义是：每读取 1 byte 数据必须执行至少 156 次浮点运算, 才能充分利用算力, 否则 GPU 将因等待数据而空闲. 该阈值是判断算法处于计算受限区还是带宽受限区的核心参数. 

标准注意力的实际计算强度可由其总 FLOPs 除以总访存量得到. 在 $n = 8K$ 时, 如式 (18) 所示：

$$
I_{\text{std}} = \frac{2 n^2 d_{head}}{4 n d_{head} + 2 n^2} \approx \frac{n}{2} = 4000 \tag{18}
$$
式 (18) 的分子 $2 n^2 d_{head}$ 是注意力计算的总运算量, 其中 $n^2$ 项占主导; 分母中, $4 n d_{head}$ 是读取 Q、K、V 和写回 O 的总字节数(FP16, 每个元素 2 bytes), $2 n^2$ 是中间注意力分数矩阵 $S$ 的读写字节数(在标准实现中通常被物化到 HBM). 当 $n \gg d_{head}$ 时, $n^2$ 项同时主导分子和分母, 因此可约简为 $n/2$. 结果 $4000$ 远大于式 (17) 的临界阈值 $156$, 理论上标准注意力处于计算受限区. 但由于非融合 kernel 的额外访存开销, 实际有效强度远低于理论值, FlashAttention 正是通过 kernel 融合将有效强度拉回计算受限区. 

线性注意力的计算强度同样可由其总 FLOPs 除以总访存量得到. 在 $n = 8K$、$m = 256$ 时, 如式 (19) 所示：

$$
I_{\text{lin}} = \frac{2 n m d_{head}}{2 n d_{head} + 2 m d_{head}} \approx m = 256 \tag{19}
$$
此式描述了变量之间的定量关系, 其中每一项对应特定的物理操作. 
式 (19) 的分子 $2 n m d_{head}$ 是线性注意力的总运算量, 与序列长度 $n$ 呈线性关系; 分母 $2 n d_{head} + 2 m d_{head}$ 是总访存量, 第一项来自输入 Q、K、V 的读取, 第二项来自固定大小状态矩阵 $S_t$ 的读写. 当 $n \gg m$ 时, $n$ 项主导分母, 分子分母中的 $n$ 约去, 结果趋近于 $m$. 结果 $256$ 仍高于式 (17) 的临界阈值 $156$, 但已接近带宽受限边界; 当特征维度 $m$ 较小(如 64-128)时, 线性注意力将落入内存带宽受限区, 需要 kernel 融合优化来维持效率. 

仍高于阈值, 但更接近边界. 在 $m$ 较小(64-128)时, 线性注意力可能落入内存带宽受限区, 需要 kernel 融合优化. 

## 6. 极简实现与源码映射

以下代码实现 Linear Transformer 的核心机制, 注释直接映射到上方公式. 

```python
import torch
import torch.nn as nn
import math

class LinearAttention(nn.Module):
    """
    Linear Attention (Katharopoulos et al., 2020)
    将 Softmax(QK^T)V 改写为 phi(Q)(phi(K)^T V), 实现 O(n) 复杂度. 
    """
    def __init__(self, d_model=512, n_heads=8, feature_dim=256):
        super().__init__()
        self.d_model = d_model
        self.n_heads = n_heads
        self.d_head = d_model // n_heads
        self.feature_dim = feature_dim  # 特征映射维度 m

        # Q/K/V 投影矩阵
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def feature_map(self, x):
        """
        特征映射 phi(x) = elu(x) + 1
        对应公式: phi(x) = max(0, x) + exp(min(0, x))
        确保输出严格正, 避免除零. 
        """
        return nn.functional.elu(x) + 1.0

    def forward(self, x, state=None, use_recurrent=False):
        """
        Args:
            x: (batch, n, d_model)
            state: 用于自回归解码的累积状态 (S, Z)
            use_recurrent: True 时启用 RNN 式增量更新
        Returns:
            out: (batch, n, d_model)
            state: 更新后的 (S, Z)
        """
        batch, n, _ = x.shape

        # 1. 投影并分头: Q = XW_q, K = XW_k, V = XW_v
        q = self.W_q(x).view(batch, n, self.n_heads, self.d_head)
        k = self.W_k(x).view(batch, n, self.n_heads, self.d_head)
        v = self.W_v(x).view(batch, n, self.n_heads, self.d_model // self.n_heads)

        # 2. 特征映射: phi(Q), phi(K)
        # 对应公式: phi(q)^T phi(k) 近似 softmax(q^T k)
        q = self.feature_map(q)
        k = self.feature_map(k)

        if not use_recurrent:
            # 3. 训练模式: 利用矩阵乘法结合律重排
            # O = phi(Q)(phi(K)^T V) / (phi(Q)(phi(K)^T 1))
            # 对应公式: S_n = sum_j phi(k_j)^T v_j, Z_n = sum_l phi(k_l)
            kv = torch.einsum('bnhd,bnhe->bhde', k, v)
            # kv: (batch, heads, d_head, d_head)  —— 即 S_n 矩阵

            k_sum = torch.einsum('bnhd->bhd', k)
            # k_sum: (batch, heads, d_head)  —— 即 Z_n 向量

            out = torch.einsum('bnhd,bhde->bnhe', q, kv)
            # out: (batch, n, heads, d_head) = phi(Q) * S_n

            norm = torch.einsum('bnhd,bhd->bnh', q, k_sum).unsqueeze(-1)
            # norm: (batch, n, heads, 1) = phi(Q) * Z_n

            out = out / (norm + 1e-6)
        else:
            # 4. 自回归解码模式: RNN 式状态更新
            # S_t = S_{t-1} + phi(k_t)^T v_t
            # Z_t = Z_{t-1} + phi(k_t)
            # o_t = phi(q_t) S_t / (phi(q_t) Z_t)
            if state is None:
                S = torch.zeros(batch, self.n_heads, self.d_head, self.d_model // self.n_heads,
                                device=x.device, dtype=x.dtype)
                Z = torch.zeros(batch, self.n_heads, self.d_head,
                                device=x.device, dtype=x.dtype)
            else:
                S, Z = state

            outputs = []
            for t in range(n):
                qt = q[:, t]      # (batch, heads, d_head)
                kt = k[:, t]      # (batch, heads, d_head)
                vt = v[:, t]      # (batch, heads, d_head)

                S = S + torch.einsum('bhd,bhe->bhde', kt, vt)
                Z = Z + kt

                ot = torch.einsum('bhd,bhde->bhe', qt, S)
                nt = torch.einsum('bhd,bhd->bh', qt, Z).unsqueeze(-1)
                outputs.append(ot / (nt + 1e-6))

            out = torch.stack(outputs, dim=1)

        # 5. 合并多头并输出投影
        out = out.view(batch, n, self.d_model)
        out = self.W_o(out)

        return out, (S, Z) if use_recurrent else None


class PerformerAttention(nn.Module):
    """
    Performer / FAVOR+ (Choromanski et al., 2021)
    使用随机正交特征映射近似 Softmax 核. 
    """
    def __init__(self, d_model=512, n_heads=8, num_features=256):
        super().__init__()
        self.d_head = d_model // n_heads
        self.n_heads = n_heads
        self.num_features = num_features

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

        # 随机正交特征矩阵, 训练时固定
        self.register_buffer('Omega', torch.randn(num_features, self.d_head))

    def feature_map(self, x):
        """
        随机傅里叶特征映射:
        phi(x) = exp(-||x||^2 / 2) * [exp(i * Omega @ x)]
        实现中取实部: [cos(Omega x), sin(Omega x)] / sqrt(m)
        """
        x_norm_sq = (x ** 2).sum(dim=-1, keepdim=True) / 2
        proj = x @ self.Omega.T  # (..., num_features)
        return torch.exp(-x_norm_sq) * torch.cat([torch.cos(proj), torch.sin(proj)], dim=-1) / math.sqrt(self.num_features)

    def forward(self, x):
        batch, n, _ = x.shape
        q = self.feature_map(self.W_q(x).view(batch, n, self.n_heads, self.d_head))
        k = self.feature_map(self.W_k(x).view(batch, n, self.n_heads, self.d_head))
        v = self.W_v(x).view(batch, n, self.n_heads, self.d_head)

        kv = torch.einsum('bnhm,bnhd->bhmd', k, v)
        k_sum = torch.einsum('bnhm->bhm', k)
        out = torch.einsum('bnhm,bhmd->bnhd', q, kv)
        norm = torch.einsum('bnhm,bhm->bnh', q, k_sum).unsqueeze(-1)
        out = out / (norm + 1e-6)
        return self.W_o(out.view(batch, n, -1))
```

关键代码-公式映射：

| 代码行 | 对应公式 | 说明 |
|:------|:--------|:-----|
| `feature_map` | $\phi(x) = \text{elu}(x) + 1$ | 正特征映射, 确保可除性 |
| `kv = einsum(...)` | $S_n = \sum_j \phi(k_j)^T v_j$ | 状态矩阵累积 |
| `k_sum = einsum(...)` | $Z_n = \sum_l \phi(k_l)$ | 归一化累积 |
| `out = einsum(...)` | $\phi(q_i)^T S_n$ | 查询与状态的内积 |
| `S = S + einsum(kt, vt)` | $S_t = S_{t-1} + \phi(k_t)^T v_t$ | 递归状态更新 |
| `Performer.feature_map` | $\phi(x) = \frac{\exp(-\|x\|^2/2)}{\sqrt{m}} [\cos, \sin]$ | 随机傅里叶特征 |

## 7. 边界条件与失效模式

线性注意力并非万能替代方案, 其数学改造引入了新的失效模式. 

| 失效场景 | 物理根源 | 典型症状 | 缓解策略 |
|:--------|:--------|:--------|:--------|
| 近似精度损失 | 核特征映射 $\phi$ 对 Softmax 的近似存在固有误差 | 下游任务(如翻译、摘要)BLEU/ROUGE 下降 2-5 点 | 增大特征维度 $m$; 在顶层保留标准注意力 |
| 长程依赖退化 | 线性注意力的权重缺乏 Softmax 的 sharp peak, 难以聚焦 distant token | 长文档问答中跨段落引用失败 | RWKV 的 time-shift; Mamba 的选择性门控 |
| 因果掩码梯度问题 | 增量累积 $S_t$ 在反向传播时产生长程梯度路径 | 训练不稳定, Loss 震荡 | 使用 chunk-wise 梯度 checkpoint; Mamba-2 的并行扫描 |
| 特征映射数值溢出 | $\exp(-\|x\|^2/2)$ 在 FP16 下易下溢为零 | Performer 输出全零 | 切换到 FP32 计算特征映射; 使用数值稳定的 elu+1 |
| 状态维度瓶颈 | $S_t \in \mathbb{R}^{m \times d}$ 的容量有限 | 复杂多跳推理任务表现劣于标准注意力 | 增大 $m$; 使用多层状态(Hierarchical SSM) |
| 初始化敏感 | RWKV/Mamba 的衰减参数 $w$ 对初始化极度敏感 | 训练初期梯度消失, Loss 不下降 | 采用 SSM 专用初始化(HiPPO, S4D); 渐进式衰减率训练 |
| 短序列负收益 | 线性注意力的 kernel 开销在小 $n$ 时无法摊销 | $n < 1K$ 时速度反而慢于标准注意力 | 动态路由：短序列走标准注意力, 长序列走线性 |
| 无全局注意力模式的任务 | 某些任务(如代码依赖图)需要显式的图结构注意力 | 线性注意力的隐式加权无法建模结构化稀疏 | 混合架构(Jamba)：奇数层 Mamba, 偶数层 Transformer |

精度损失的深层原因：Softmax 的指数函数产生尖锐的注意力分布, 使模型能够"聚焦"到单个关键 token. 线性注意力的核映射 $\phi$ 通常输出平滑分布, 丢失了这种聚焦能力. Performer 在 $m = 256$ 时能在多数任务上逼近 Softmax, 但在需要精确定位(如指代消解)的任务上仍有差距. 

Mamba 的选择性门控虽缓解了长程退化, 但引入了输入依赖的状态转移矩阵, 使并行训练复杂化. Mamba-2 的 SSD 算法通过将序列分块并在块内并行、块间递归, 解决了这一问题, 但块大小成为新的超参数. 

## 8. 技术前瞻与演进

线性注意力领域仍在高速迭代, 2024-2025 年的关键方向包括：

**Mamba-2 与状态空间对偶性**

Mamba-2 (Dao & Gu, 2024) 提出 SSD(State Space Duality)框架, 证明选择性 SSM 等价于一种结构化掩码的线性注意力. 这一理论突破使 Mamba-2 能够：

- 使用 FlashAttention 风格的块级并行算法, 在 A100/H100 上达到与 Transformer 相当的训练吞吐
- 将状态维度从 Mamba-1 的 16 扩展至 64-128, 提升表达能力而不增加计算量
- 支持张量并行与序列并行, 适配大规模分布式训练

**混合架构：Jamba 与 Zamba**

纯线性注意力在需要全局精确定位的任务上仍逊于标准注意力. Jamba (AI21, 2024) 采用 Mamba 层与 Transformer 层交替堆叠：Mamba 层处理局部上下文与长程压缩, Transformer 层提供全局注意力能力. 实验表明, 在 256K 上下文上, Jamba 的吞吐量是纯 Transformer 的 3 倍, 精度损失 < 1%. 

Zamba (Zyphra, 2024) 进一步优化混合比例, 将 Mamba 层占比提升至 75%, 仅在最后几层保留标准注意力. 其 7B 模型在 MMLU 上追平 Llama-2-13B, 推理速度提升 2.5x. 

**线性注意力的硬件协同设计**

标准 Transformer 的 $O(n^2)$ 计算在 $n$ 较小时(< 4K)完全受限于 GPU 算力, FlashAttention 的优化空间在于减少访存. 线性注意力的 $O(n)$ 计算在 $n$ 较大时退化为内存带宽受限, 其优化空间在于：

- 降低状态矩阵 $S_t$ 的精度(INT8/FP8)以减少带宽
- 设计专门的 SSM 硬件单元(类似 NVIDIA Tensor Core 的矩阵乘法单元)
- 利用线性注意力的序列无关性, 实现无限的上下文长度(理论上仅受状态容量限制)

**MiniMax-01 与 Lightning Attention**

MiniMax-01 (2025) 提出 Lightning Attention, 将线性注意力与标准注意力的混合推向极致：在底层使用线性注意力处理长程上下文, 在顶层使用标准注意力聚焦局部细节. 其 456B 总参数模型支持 4M tokens 上下文, 在"大海捞针"测试中实现 100% 召回. 

**未解之谜**

- 状态容量的理论极限：$S_t \in \mathbb{R}^{m \times d}$ 的信息容量是否存在上界？是否可借用信息论中的率失真理论量化？
- 线性注意力的涌现能力：标准 Transformer 的涌现能力(如上下文学习、思维链)是否依赖于 Softmax 的 sharp attention 模式？线性注意力能否在同等规模下复现涌现？
- 多模态扩展：视觉-语言模型中的图像 patch 数(如 1024x1024 图像产生 4096 patches)使线性注意力的优势更加显著, 但空间注意力与序列注意力的混合机制尚未成熟.

## 10. 补充专题：Gated-DeltaNet 与 Qwen3-Next 混合注意力架构

Gated-DeltaNet 代表了 2025 年线性注意力机制的前沿高峰, 它成功消除了传统线性注意力(如 Performer, Favor+)的记忆模糊问题, 并在 Qwen3-Next-80B-A3B 等工业界旗舰模型中实现了生产级落地. 

### 10.1 线性注意力范式演进与 Gated-DeltaNet 动力学

核函数相似度映射将 $QK^T$ 的计算顺序不可交换性剥离, 将复杂度压回 $O(L \cdot d^2)$. 但在长序列下, 基础线性注意力由于状态矩阵 $S_t$ 容量上限限制, 新 token 会无差别稀释历史信息, 导致"记忆模糊"：

$$
S_t = S_{t-1} + \phi(k_t) v_t^T \tag{4}
$$

为了正面解决记忆模糊, DeltaNet 引入了 **TTT (Test-Time Training)** 的梯度更新思想. 其目标是让状态矩阵 $S_t$ 能够实现将历史 Key 精确映射 to 正确的 Value(即 $S_t k_j \approx v_j$). 通过定义局部的误差损失函数：

$$
\mathcal{L}_t(S) = \frac{1}{2} \|S k_t - v_t\|^2 \tag{5}
$$

在每一步前向输入时, 对状态矩阵进行一次梯度更新步长 $\beta_t$ 的校正更新：

$$
S_t = S_{t-1} - \beta_t
abla \mathcal{L}_t(S_{t-1}) \implies S_t = S_{t-1}(\mathbf{I} - \beta_t k_t k_t^T) + \beta_t v_t k_t^T \tag{6}
$$

这一更新策略被称为 **Delta Rule (德尔塔规则)**. 其物理直觉极其精妙：在将新的 key-value 信息写入记忆前, 先从状态矩阵中剔除与当前 key 相关的误差预测分量 $(\mathbf{I} - \beta_t k_t k_t^T)$, 再加入新的有效增量 $\beta_t v_t k_t^T$. 这使得状态矩阵能够动态选择性地更新与覆盖, 消除了信息饱和稀释. 

**Gated-DeltaNet** 进一步在 Delta Rule 基础上引入了可学习的衰减系数 $\alpha_t \in (0, 1)$, 实现了输入相关的自适应遗忘与记忆深度门控：

$$
S_t = \alpha_t S_{t-1}(\mathbf{I} - \beta_t k_t k_t^T) + \beta_t v_t k_t^T \tag{7}
$$

### 10.2 Qwen3-Next 架构解析与 3:1 混合注意力设计

Qwen3-Next-80B-A3B-Instruct 采用了一种极具创新的 **3:1 混合架构配比**：每 3 层 Gated DeltaNet 层后, 接 1 层 Gated Attention 层. 全部 48 层网络均采用高度稀疏的 MoE FFN 层进行网络容量扩充. 

#### 10.2.1 Gated DeltaNet 模块架构流

对于输入的 Hidden States $x_t \in \mathbb{R}^{B \times S \times 2048}$：
1. **Depthwise Conv1D**：首先通过一个卷积核宽度为 4、pad=3、分组数 groups=8192 的深度可分离因果一维卷积. 这极大地增强了模型的局部 token 级别时序感知与局部依赖建模能力：
   $$
\tilde{x}_t = \text{Conv1D}([x_{t-3}, x_{t-2}, x_{t-1}, x_t]) \tag{8}
$$
2. **非对称 Parameter Grouping 策略**：投影产生 $Q$ (16 heads, 128 dim), $K$ (16 heads, 128 dim), $V$ (32 heads, 128 dim). 在计算时, 对 $Q$ 和 $K$ 应用 `repeat_interleave` 复制为 32 个头, 从而在保持 32 头 Value 高表达容量的同时, 使 KV 状态矩阵的参数存储开销降低 50%. 
3. **Mamba-style 门控融合**：最终输出经过门控机制与 RMSNorm 进行信道非线性融合, 其形式类似于：
   $$
o_t = W_o \left( \text{Sigmoid}(W_x x_t) \odot \text{RMSNorm}(S_t \phi(q_t)) \right) \tag{9}
$$

#### 10.2.2 3:1 混合的设计直觉

纯线性注意力在超长程的"精确定位与多跳推理"(如跨文档指代消解、复杂数学推理)上由于缺乏 Softmax 尖锐概率峰值仍存在微弱劣势. Qwen3-Next 通过 75% 的线性注意力(维持 $O(N)$ 复杂度和 256K 超长上下文窗口)加 25% 的 Gated Attention(提供全局精细 Softmax 局部相关性检索), 在效率与能力两端达成了帕累托最优. 

### 10.3 Zero-Centered RMSNorm 机制

在深度线性注意力网络中, 为了提高长程自回归递推的数值稳定性和梯度稳定性, Qwen3-Next 引入了 **Zero-Centered RMSNorm**. 其参数初始化机制将权重 $\gamma$ 初始化为零, 相比传统 RMSNorm (初始化为 1) 拥有更加优越的梯度流和收敛稳定性：

```python
class Qwen3NextRMSNorm(nn.Module):
    def __init__(self, dim: int, eps: float = 1e-6):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.zeros(dim))  # 初始化为 0, 而非传统 RMSNorm 的 1

    def forward(self, x):
        # 归一化计算
        output = x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
        # Zero-centered 标度调整：y = (1 + gamma) * normalized_x
        output = output * (1.0 + self.weight.float())
        return output.type_as(x)
```

### 10.4 Chunkwise Parallel 训练与推理分块递推机制

在模型训练与推理 Prefill 阶段, 为了维持 GPU Tensor Core 的并行矩阵乘法高利用率, Qwen3-Next 采用 **Chunkwise Parallel** 机制. 其核心思想是将序列切分为长度为 $C$ 的小块(Chunk)：
- **块内并行计算**：块内使用类似 Full Attention 的矩阵计算形式, 计算 $QK^T$ 并乘以 $V$, 计算量为 $O(C^2)$; 
- **块间递推更新**：块与块之间, 通过线性注意力的状态递推公式, 将块级状态矩阵 $S$ 逐块向下游传递. 
总时间复杂度在序列长度 $L$ 下为 $O(L \cdot C) + O(L \cdot d^2 / C)$. 当 $C \approx d$ 时, 分块递推在维持并行计算效率的同时, 消除了二次复杂度, 达成了完美的算力与内存带宽平衡. 
## 11. 为什么线性注意力不是“更快的 softmax attention”那么简单

很多人第一次接触线性注意力时, 会把它理解成“把 attention 近似一下, 所以更快”. 这个理解不够深. 

线性注意力真正做的, 是把原本依赖完整 $QK^T$ 注意力矩阵的计算图, 重写成一种 **可以通过状态累积或特征映射分解的结构**. 这不仅仅是速度优化, 而是对注意力本体的一次重新参数化. 

也就是说, 线性注意力不是简单减少一点乘法, 而是在问：

- 我们能否不显式保存所有 pairwise 交互？
- 我们能否把“看全部历史”改写成“维护一个足够好的摘要状态”？
- 我们能否把全局依赖从矩阵乘法变成递归更新？

所以, 线性注意力更接近一种“序列建模范式转换”, 而不是一条普通性能 patch. 

## 12. 线性注意力的共同代价：你省掉了矩阵, 但丢掉了什么

线性注意力之所以诱人, 是因为它从根上把复杂度压下来了. 但这份诱人是有代价的. 

标准 softmax attention 的一个巨大优势是, 它允许每个 query 为全部 key 重新分配一遍权重. 这个过程代价高, 但表达力也极强.   
线性注意力一旦通过核映射、递归状态或选择性状态空间把历史信息压进某种摘要结构, 就意味着你不再能“零损失地重新访问全部历史”. 

因此, 线性注意力的核心 trade-off 永远是：

- 更小的状态成本
- 更低的时间复杂度
- 更强的长序列可扩展性

对抗：

- 信息压缩损失
- 表达偏置改变
- 某些检索型任务上精度下降

这也是为什么工业界很少直接把线性注意力当成“无脑替代”. 更多时候, 它被用在长上下文敏感、吞吐敏感、边缘部署敏感的场景, 或者和标准 attention 组成混合结构. 

## 13. Performer / FAVOR+ 的真正意义

Performer 一类方法的价值, 不只是提出了随机特征映射, 而是首次系统性地告诉大家：softmax kernel 也可以被近似分解成一类可结合的特征内积. 

这一步非常关键, 因为它让“线性 attention”第一次不只是工程直觉, 而是有了明确的核方法数学支撑. 虽然这些方法在工业界没有成为绝对主流, 但它们开了一个口子：softmax 并不是不可碰的黑箱, 相似度函数本身也可以被改写. 

很多后续工作不一定继续沿 FAVOR+ 的具体形式走, 但它们都继承了同一个问题意识：如果目标是长序列, 我们必须思考 attention 内积结构本身是否可以被重新设计. 

## 14. RWKV、Mamba 为什么会被归到这里

严格来说, RWKV 和 Mamba 并不总被所有人叫作“线性注意力”. 但把它们放进这一章是合理的, 因为它们都在回答同一个根问题：**如何以线性成本处理长序列, 同时保留尽可能强的序列建模能力. **

RWKV 的做法是把注意力隐含成 time-mixing 与状态递推.   
Mamba 的做法是把状态空间模型做成输入依赖的 selective 更新系统.   

二者都不再显式保留完整注意力矩阵, 也都把“全历史交互”改写为“状态式摘要 + 当前输入更新”. 所以从能力账本而不是公式形式看, 它们属于同一大类路线. 

## 15. 为什么工业界常常选择混合架构而不是纯线性架构

纯线性注意力的优点和缺点都非常明显. 它擅长降复杂度, 但不一定总能保住所有任务上的精细表达能力.   
于是现实中更流行的一条路径是混合架构：

- 一部分层保留标准 attention
- 一部分层换成线性模块或状态模型
- 再通过 MoE、长上下文外推和多头压缩做联合优化

Jamba、Falcon-Mamba、Qwen3-Next 一类路线本质上都在说明一个事实：**工业界真正想要的不是“彻底告别 attention”, 而是“在该省的地方省, 在该保的地方保”. **

这也是为什么线性注意力更适合作为架构工具箱的一部分, 而不是宗教式替代方案.

## 17. 本节小结

线性注意力机制的真正价值, 不在于“把 attention 做成一个更便宜的近似”, 而在于它重新思考了长序列建模到底应不应该显式保存完整两两交互. 

它提供了一整条路线, 从核技巧分解, 到递归状态摘要, 再到输入依赖的选择性状态更新, 把原本随着长度迅速失控的注意力系统, 改写成了更适合超长序列和流式推理的结构. 

当然, 这份收益从来不是免费的. 你省掉了显式矩阵和长缓存, 也就接受了某种信息压缩与表达偏置变化. 因此, 线性注意力最正确的定位不是“彻底替代一切”, 而是成为长上下文时代注意力系统工具箱里的关键分支.
