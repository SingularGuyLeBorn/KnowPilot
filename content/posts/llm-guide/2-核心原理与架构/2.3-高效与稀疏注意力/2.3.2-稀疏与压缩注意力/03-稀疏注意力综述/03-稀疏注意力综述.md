# 03 驯服二次方猛兽: 从稀疏矩阵到动态路由注意力的数学演义

Transformer 架构的出现标志着自然语言处理(NLP)进入了一个新纪元,但其核心组件——自注意力机制(Self-Attention)——携带了一个原罪: **二次方复杂度**. 随着序列长度 $L$ 的增加,计算量和显存占用以 $O(L^2)$ 的速度爆炸. 

这是一场数学家与物理限制的博弈. 从早期的**静态图论掩码**,到对**GPU 访存特性**的深刻反思,再到如今**DeepSeek NSA** 和 **Moonshot MoBA** 这种**动态路由**的百花齐放,我们试图在保持全局感受野的同时,将计算复杂度压回线性. 

本文将剥离工程实现的表象,从线性代数、概率论和硬件算术强度的视角,重构这一演进历程. 

## 1. 维度的诅咒与稠密注意力的代价

一切的起点是标准的多头自注意力(MHA). 我们需要先量化“代价”究竟在哪里. 

### 1.1 全注意力机制的数学形式

给定输入序列 $X \in \mathbb{R}^{L \times d}$,我们通过线性投影得到 $Q, K, V$. 标准的缩放点积注意力定义为: 

$\text{Attention}(Q, K, V) = \text{Softmax}\left( \frac{QK^T}{\sqrt{d}} \right) V$

其中 $A = QK^T$ 生成了一个 $L \times L$ 的注意力矩阵. 

### 1.2 显存墙: 屋顶线模型 (Roofline Model) 的视角

在推理的解码(Decoding)阶段,真正的瓶颈往往不是计算(FLOPs),而是**访存(Memory Access)** . 这是理解稀疏注意力必要性的物理基础. 

定义 **算术强度 (Arithmetic Intensity)** $I$ 为每字节内存传输所进行的浮点运算次数: 

$I = \frac{\text{FLOPs}}{\text{Bytes Access}}$

在自回归解码中,对于第 $t$ 步,我们需要读取所有历史的 $K, V$(大小为 $2t \cdot d \cdot \text{precision}$),但只进行 $q_t k_i^T$ 的点积运算(计算量极小). 
当 $t \to \infty$ 时,KV Cache 变得巨大. 由于 GPU 的 HBM(高带宽内存)带宽是有限的,系统迅速进入 **Memory Bound (带宽受限)** 区域. 

$\lim_{L \to \infty} \text{Throughput} \propto \frac{\text{Bandwidth}}{L}$

这就是我们必须引入稀疏性的物理动力: **我们必须大幅减少读取的数据量(Sparse Access),而不仅仅是减少计算量. **

> 图 1: GPU 内存金字塔（HBM→SRAM→寄存器）；长序列 decode 算术强度极低，系统落在带宽受限区而非算力受限区。

- $\lim_{L\to\infty}$ 吞吐 $\propto \mathrm{Bandwidth}/L$ — 仅减 FLOPs 不够，必须减 KV 访问量。

## 2. 静态稀疏的尝试: 图论视角的图谱

早期的尝试(2020-2022,如 Longformer, BigBird)集中于设计固定的稀疏模式. 这本质上是在**图论**框架下寻找全连接图的高效近似. 

### 2.1 稀疏掩码通用公式

我们将注意力视为一个有向图,其中节点是 Token. 稀疏注意力通过引入一个掩码矩阵 $M \in \{0, -\infty\}^{L \times L}$ 来限制连接: 

$A_{\text{sparse}} = \text{Softmax}(QK^T + M)V$

其中 $M_{ij} = 0$ 表示 $i$ 关注 $j$,否则为 $-\infty$. BigBird 证明了结合 **滑动窗口 (Sliding Window)** 、**全局节点 (Global Tokens)** 和 **随机连接 (Random Connections)** 的稀疏图是图灵完备的. 

### 2.2 硬件的鸿沟: 为什么随机稀疏很难快？

尽管 BigBird 等算法在理论上将复杂度降到了 $O(L)$,但在实际 GPU 训练中,它们往往比全注意力还慢. 

**原因: 访存不连续性 (Non-Coalesced Access)** 
GPU 的内存架构是为**稠密计算**设计的.

- **HBM 访问粒度**: GPU 从显存读取数据时,是以 **Cache Line**(通常 128 字节)为单位的.

- **合并访存 (Coalesced Access)** : 如果线程束 (Warp) 中的线程请求连续的内存地址,这些请求会被合并为一个 Transaction. 

如果在 $A_{ij}$ 中 $j$ 的索引是随机分布的(Token-level Sparsity),GPU 需要发起大量离散的内存请求. 假设我们只读取了一个 bfloat16 (2 bytes),但 GPU 必须传输整个 128 bytes 的 Cache Line,导致 **有效带宽利用率** 极低: 

$\text{Efficiency} = \frac{\text{Useful Bytes}}{\text{Transferred Bytes}} \ll 1$

因此,现代稀疏算法(如 DeepSeek NSA 和 MoBA)的一个核心原则是: **Block-wise Sparsity (块级稀疏)** . 稀疏性的粒度必须是 Block 而非 Token,以保证访存连续性.

## 3. DeepSeek NSA: 硬件亲和的原生稀疏注意力

DeepSeek 在 2025 年提出的 **NSA (Native Sparse Attention)** 是对稀疏注意力的一次“工业级”重构. 它不仅关注算法层面的稀疏,更关注**硬件对齐 (Hardware Alignment)** 和 **原生训练 (Native Training)** . 

### 3.1 核心架构: 三支路的分层建模

NSA 摒弃了单一的注意力计算,将其分解为三个并行的物理分支,分别捕捉不同粒度的信息. 对于查询 $q_t$,输出 $o_t^*$ 定义为: 

$o_t^* = \sum_{c \in \{cmp, slc, win\}} g_t^c \cdot \text{Attn}(q_t, \tilde{K}_t^c, \tilde{V}_t^c)$

其中 $g_t^c$ 是门控系数. 这三个分支分别是: 

1. **压缩分支 (Token Compression)** : 捕捉**粗粒度 (Coarse-grained)** 的全局信息.

2. **选择分支 (Token Selection)** : 捕捉**细粒度 (Fine-grained)** 的长程相关性(这是稀疏的核心).

3. **滑动窗口分支 (Sliding Window)** : 捕捉**局部 (Local)** 上下文.

- **块级稀疏**保证 GPU 合并访存；token 级随机稀疏在此被抛弃。

### 3.2 压缩分支: 信息的物理降维

为了以低成本获取全局视野,NSA 将连续的 $l$ 个 Key/Value 聚合为一个“压缩 Token”. 
定义块大小为 $l$,压缩函数 $f^{cmp}$ 为一个带有位置编码的 MLP: 

$\tilde{K}_t^{cmp} = \{ \varphi(\mathbf{k}_{id+1:id+l}) \mid 0 \le i < \lfloor (t-l)/d \rfloor \}$

这在数学上相当于对历史信息进行了**下采样 (Downsampling)** . 虽然丢失了高频细节,但保留了低频的语义轮廓. 

### 3.3 选择分支: 基于粗粒度引导的精细检索

这是 NSA 最精妙的部分. 如何决定哪些块是重要的？NSA 利用**压缩分支的中间结果**来引导选择,避免了额外的辅助网络开销. 

**重要性评分 (Importance Score)** :
利用压缩后的 Key $\tilde{K}^{cmp}$ 计算注意力分数: 

$\mathbf{p}_t^{cmp} = \text{Softmax}(q_t^T \tilde{K}_t^{cmp})$

这个分数 $\mathbf{p}_t^{cmp}$ 代表了 Query 对每个“压缩块”的关注程度. 
如果压缩块和选择块的粒度一致,我们直接使用 $\mathbf{p}_t^{cmp}$ 作为选择依据. 如果粒度不一致,则进行插值或聚合. 

**Top-k 块选择**:
根据重要性评分,选择 Top-$n$ 个块的索引集合 $\mathcal{I}_t$: 

$\mathcal{I}_t = \{ i \mid \text{rank}(\mathbf{p}_t^{cmp}[i]) \le n \}$

然后,**只从 HBM 中加载**这些被选中的**原始精细** Key/Value 块: 

$\tilde{K}_t^{slc} = \text{Concat}(\{ \mathbf{k}_{il'+1:(i+1)l'} \mid i \in \mathcal{I}_t \})$

**数学本质**: 这是一种**分层近似 (Hierarchical Approximation)** . 先用低秩(压缩)信息进行全量扫描,再用高秩(原始)信息进行稀疏计算. 

### 3.4 硬件对齐: 为什么 NSA 快？

NSA 的设计完全是为了迎合 Tensor Core.

- **Block-wise**: 所有的选择和计算都是以 Block 为单位. 这意味着内存读取是**连续**的 (Coalesced Access),矩阵乘法是**稠密**的 (Dense GEMM on Blocks).

- **Triton Kernel**: DeepSeek 专门编写了 Triton 内核,在 Grid 级别调度 Query Group,在 Inner Loop 中加载共享的 KV Block. 这使得 NSA 在训练和推理阶段都能跑满 GPU 的算力.

## 4. Moonshot MoBA: 混合块注意力的动态路由

如果说 NSA 是侧重于“压缩与选择”,那么 Moonshot AI 的 **MoBA (Mixture of Block Attention)** 则是将 **MoE (混合专家模型)** 的思想彻底贯彻到了注意力机制中. 

### 4.1 历史即专家 (History as Experts)

在标准 MoE 中,我们根据输入选择不同的 FFN 专家. 在 MoBA 中,**历史上下文被切分为块,每个块被视为一个“专家”**. 
当前的 Query $q_t$ 不再关注所有历史,而是通过一个**门控网络 (Gating Network)** 动态路由到最相关的历史块. 

### 4.2 门控机制与亲和度计算

将上下文划分为 $n$ 个块,第 $i$ 个块的范围是 $I_i$. 
首先计算 Query 与第 $i$ 个块的**亲和度 (Affinity Score)** $s_i$. 
为了降低计算量,MoBA 并不计算与块内每个 Token 的点积,而是计算与**块均值 (Block Mean)** 的点积: 

$\bar{K}_i = \text{mean\_pool}(K[I_i])$

$s_i = \langle q, \bar{K}_i \rangle$

### 4.3 Top-k 门控与因果性

门控值 $g_i$ 定义为: 

$g_i = \begin{cases} 
1 & \text{if } s_i \in \text{Topk}(\{s_j\}, k) \\
0 & \text{otherwise}
\end{cases}$

**因果性约束 (Causality)** : 
为了保证自回归性质,MoBA 强制规定: 

1. 当前块 (Current Block) **必须**被选中 ($g_{\text{curr}} = 1$). 2. 未来块的亲和度设为 $-\infty$. 

最终的注意力输出是选中的块与当前块的并集: 

$\text{MoBA}(q, K, V) = \text{Softmax}\left( q K[\mathcal{I}]^T \right) V[\mathcal{I}]$

其中 $\mathcal{I} = \bigcup_{g_i=1} I_i$ 是所有被激活块的索引集合.

- **当前块强制入选** + 因果屏蔽未来块，保证自回归合法。

### 4.4 MoBA 与 NSA 的异同

从数学形式上看,MoBA 与 NSA 的 Selection 分支高度相似,都采用了“粗粒度评分 + 细粒度计算”的范式.

- **评分依据**: MoBA 使用 mean_pool,NSA 使用学习到的压缩 Token.

- **训练范式**: MoBA 探索了从 Full Attention 平滑过渡到 Sparse Attention 的混合训练策略,以及层级混合(底层 MoBA,顶层 Full Attention).

## 5. 推理时的稀疏: KV Cache 的驱逐策略

如果我们在训练时没有使用稀疏注意力(如 Llama 3),能否在推理时强行稀疏化以节省显存？这就是 **Inference-Time Sparsity**. 

### 5.1 Attention Sinks: Softmax 的非零陷阱

MIT 的 StreamingLLM 发现了一个有趣的数学现象: 如果直接丢弃早期的 Token(如只保留最近的窗口),PPL 会瞬间爆炸. 

**数学解释**: 
Softmax 函数具有归一化特性:  $\sum_{j} \exp(q_t k_j^T) = Z$. 
在训练初期,作为序列开头的 Token(如 [BOS])充当了 **Attention Sink (注意力汇)** . 当当前窗口内的 Token 与 Query 都不相关(分数都很低)时,Softmax 依然需要把概率分配出去. 模型倾向于将这些“无处安放”的概率分配给序列的起始 Token. 

$\text{If } \forall j \in \text{Local}, q \cdot k_j \ll 0 \implies \text{Attention} \to \text{Sink Token}$

如果丢弃它们,分母 $Z$ 剧烈减小,导致注意力分布崩塌. 因此,**保留前几个 Sink Token + 滑动窗口**,是实现无限流式生成的数学必要条件. 

### 5.2 H2O: 累积注意力的贪心驱逐

H2O (Heavy Hitter Oracle) 提出了一种基于**累积重要性**的 KV Cache 驱逐策略. 
定义 Token $j$ 的**累积注意力分数**: 

$S_j = \sum_{t > j} A_{tj}$

这个分数反映了 Token $j$ 对未来的重要性. 当 KV Cache 满时,H2O 建立一个小顶堆,驱逐 $S_j$ 最小的 Token. 
这在数学上等价于**保留注意力矩阵中的主成分**,是一种在线的、动态的低秩近似. 

## 6. 结语: 从 $O(N^2)$ 到 $O(N)$ 的圣杯

从 Longformer 的人工设计图案,到 NSA 的硬件感知分块,再到 MoBA 的动态路由,我们见证了注意力机制从**静态几何**向**动态流形**的演进. 

这一演进的终极目标是明确的: 
**我们希望模型能够像人类一样,拥有近乎无限的记忆容量(存储在 CPU 内存或 SSD 中),但在处理每一个当下时刻时,通过高效的索引机制,只将极少量的关键记忆块(Sparse Blocks)加载到 GPU 的显存中. **

NSA 和 MoBA 证明了: 通过分层建模(粗粒度索引 + 细粒度计算),我们可以在数学上逼近全注意力的性能,同时在物理上突破显存带宽的桎梏. 这不仅是算法的胜利,更是对硬件特性的极致利用.
