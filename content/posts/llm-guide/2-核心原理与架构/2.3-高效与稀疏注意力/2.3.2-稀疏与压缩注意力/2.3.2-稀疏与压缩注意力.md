## 1. 演进脉络与技术坐标

Transformer 自注意力的 $O(n^2)$ 复杂度在长序列场景下构成根本性的物理瓶颈. 当序列长度从 4K 扩展至 128K, 注意力层的计算量增长 1024 倍, KV Cache 的显存占用同步线性膨胀. FlashAttention 家族通过 IO-aware 的 tiling 重排将访存从 $O(n^2)$ 压至 $O(n)$, 但计算量的二次方本质未被触及. 线性注意力(Mamba、RWKV)从数学层面将复杂度降至 $O(n)$, 却需要完全抛弃 Softmax 的细粒度表达能力, 且无法直接复用现有预训练 Transformer 的权重. 

稀疏与压缩注意力代表第三条路线：保留 Softmax 注意力的表达能力与 Transformer 架构兼容性, 通过放弃"每个 query 必须关注所有 key"的全局假设, 将有效计算量压至 $O(n \log n)$ 甚至 $O(n)$. 该路线经历了三个阶段的演化：

| 阶段 | 时间 | 代表方法 | 核心思想 | 物理局限 |
|:-----|:-----|:--------|:--------|:--------|
| 静态稀疏掩码 | 2020-2022 | Longformer, BigBird, Sparse Transformer | 预定义滑动窗口 + 全局 token + 随机连接 | 随机访存模式导致 GPU 有效带宽利用率极低, 实际训练常慢于全注意力 |
| 推理时稀疏化 | 2023 | StreamingLLM, H2O, Quest | KV Cache 驱逐与压缩, 无需重新训练 | 基于预训练全注意力的后处理, 性能退化显著; 无法降低训练成本 |
| 原生动态路由 | 2024-2025 | NSA, MoBA | 端到端可训练的块级动态路由, 硬件对齐内核 | 对块大小、路由粒度的超参敏感; SFT 阶段可能出现稀疏梯度问题 |

静态稀疏掩码的研究(BigBird 等)在图论层面证明了稀疏注意力图在引入全局节点和随机边后具备通用逼近能力, 但 token-level 的随机稀疏模式与 GPU 的合并访存(Coalesced Access)机制冲突. GPU HBM 以 128B Cache Line 为单位传输数据, 若 query 随机访问分散的 key, 有效带宽利用率暴跌至理论值的 10% 以下. 这一硬件现实迫使 2024-2025 年的稀疏注意力研究集体转向**块级稀疏(Block-wise Sparsity)** ——稀疏的基本单元必须是连续的数据块, 而非单个 token. 

推理时稀疏化方法(StreamingLLM、H2O)在 2023 年提供了无需重新训练的捷径, 但存在根本性局限：这些方法基于预训练好的全注意力模型进行 KV Cache 的后处理压缩, 模型从未在稀疏模式下学习过路由行为. 当保留 20% 的注意力连接时, 实际恢复的注意力分数可能仅达 70%, 导致长文本 PPL 急剧上升. 这证明了**原生稀疏(Native Sparsity)** 的必要性——稀疏模式必须从预训练阶段就嵌入模型, 而非作为推理时的补救措施. 

NSA 与 MoBA 几乎同期发布(2025 年 2 月), 标志着稀疏注意力从"理论可行"进入"工业可用"阶段. 二者均采用了块级稀疏 + 硬件对齐内核的设计范式, 但在路由策略和压缩机制上选择了不同的技术路径. NSA 侧重于"可训练的压缩与选择", 通过三支路融合实现分层信息处理; MoBA 侧重于"动态块路由", 将 MoE 的门控机制直接迁移至注意力层, 强调与现有预训练模型的兼容性和渐进过渡能力. 

## 2. 工业价值与落地现状

稀疏与压缩注意力的工业价值在长上下文产品和大规模推理集群中已得到验证. 

| 应用场景 | 采用方案 | 解决的关键指标 | 部署现状 |
|:--------|:--------|:-------------|:--------|
| 百万级上下文对话(Moonshot Kimi) | MoBA | 1M token 前向计算 6.5x 加速, 单卡可承载更长序列 | 已接入商用模型, 支持文档阅读与多轮长对话 |
| 长链式思考推理(DeepSeek-R1) | NSA | 64K 序列解码 11.6x 加速, 训练阶段即原生稀疏 | 已用于预训练与 SFT, 兼容 GQA/MQA 架构 |
| 无限流式生成(边缘设备) | StreamingLLM | 固定长度 KV Cache, O(1) 显存增长 | 广泛集成于 vLLM、llama.cpp 等推理框架 |
| KV Cache 显存受限场景 | H2O | 基于累积注意力分数的动态驱逐, 保留 Heavy Hitters | 作为推理时优化插件存在 |

以 72B 参数模型、bf16 精度、80 层、64 头、head_dim=128 为例, 单 token 的 KV Cache 占用约 2.62 MB. 在 Batch Size=32、Sequence Length=128K 的并发场景下, KV Cache 总量达到 $2.62 \times 32 \times 128000 \approx 10.7$ TB. 即使采用 GQA(4 query 头共享 1 组 KV), 显存压力仍达 2.7 TB. 稀疏注意力通过将有效 KV 加载量减少 80%-95%, 将这一不可承载的负载压入单机 8xH100 的可管理范围. 

在长文本推理的 roofline 模型中, 解码阶段的算术强度 $I = \text{FLOPs} / \text{Bytes}$ 随序列长度增加而趋近于零. 系统完全处于 Memory Bound 区域, 吞吐量上限由 HBM 带宽决定, 而非 Tensor Core 算力. 稀疏注意力的核心工业价值在于**减少解码阶段从 HBM 读取的字节数**, 而非单纯减少 FLOPs. NSA 的 11.6x 解码加速和 MoBA 的 6.5x 前向加速, 本质上是将有效访存带宽利用率从随机稀疏的 <10% 提升至块级稠密计算的 >80%. 

预填充阶段与解码阶段的稀疏收益存在显著差异. 预填充阶段一次性处理整个 prompt, 计算密集度高, FlashAttention 已能充分利用算力. 稀疏注意力的主要收益集中在**解码阶段**——每一步仅生成一个 token, 但需读取全部历史 KV. MoBA 在解码时将历史 KV 的读取量从 $O(N)$ 降至 $O(k \cdot B)$, NSA 则通过三路融合将读取量降至窗口 + 压缩 + 选择三个固定预算之和. 这是稀疏注意力在长上下文推理中不可替代的根本原因. 

## 3. 物理直觉与核心机制

稀疏注意力的物理直觉源于两个观察：人类认知的非均匀注意力分配, 以及 GPU 内存层次结构的块状访问特性. 

### 3.1 认知类比：分层信息处理

人类阅读长文档时不会逐字维护对所有历史内容的等权重记忆. 认知过程呈现三个层次：

- **局部精度层**：对当前段落和邻近句子保持高精度记忆(对应滑动窗口注意力). 
- **全局概览层**：通过章节标题和摘要快速建立全文轮廓(对应 token 压缩注意力). 
- **关键检索层**：根据当前问题定向检索历史文档中的相关段落(对应 token 选择 / 块路由注意力). 

NSA 的三支路架构(压缩 + 选择 + 滑动窗口)直接映射了这一认知分层. MoBA 的块路由机制则类似于"根据问题关键词跳转到相关章节"的索引行为——query 与历史块的亲和度分数扮演了"目录索引"角色. 

### 3.2 硬件类比：块状数据搬运

GPU 的内存子系统是为稠密矩阵乘法设计的. Tensor Core 在处理连续的 $B \times B$ 分块时达到峰值效率, 而离散的单元素访问会导致严重的带宽浪费. 块级稀疏注意力的设计如同货运物流：将分散的单件快递(token-level 稀疏)重组为整车运输(block-level 稠密), 单次搬运的数据量从 2B(单个 bfloat16)跃升至 $B \times d \times 2$B(整个 key block), Cache Line 利用率从 1.5% 提升至接近 100%. 

MoBA 和 NSA 均将上下文划分为大小为 $B$ 或 $l$ 的连续块, 在块级别执行路由决策. 被选中的块在 SRAM 内部执行标准的稠密 FlashAttention 计算, 内核层面无需修改 attention 的数学形式, 仅需重排数据加载路径. 这保证了**稀疏决策与稠密计算的解耦**：路由逻辑决定加载哪些块, Tensor Core 以全速执行块内注意力. 

块级稀疏的另一深层优势在于与 GQA/MQA 架构的天然兼容性. 现代 LLM(Llama 3、Qwen 2.5 等)普遍采用 GQA, 4-8 个 query 头共享 1 组 KV. 在块级稀疏下, 同一 GQA 组的所有 query 头通常路由到相同或高度重叠的历史块集合, 内核可为整个组执行单次 KV 块加载, 多头复用同一份 SRAM 数据. NSA 的"组中心化加载机制"正是利用了这一特性, 将 HBM 读取次数压缩至与 GQA 组数成正比, 而非与头数成正比. 

## 4. 数学形式与算法对比

标准稠密注意力的输出为：

$$\text{Attn}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V \tag{1}$$
此式是稀疏注意力改进的数学基准, 其核心思想是通过 query 与 key 的相似度计算动态加权 value. 具体而言, 输入序列经投影矩阵 $W_Q$、$W_K$、$W_V$ 分别映射为 Query 矩阵 $Q$、Key 矩阵 $K$ 和 Value 矩阵 $V$, 三者形状均为 $(N \times d_k)$(其中 $N$ 为序列长度, $d_k$ 为头维度). $QK^T$ 计算所有 query-key 对的点积相似度, 生成 $N \times N$ 的注意力分数矩阵; 除以 $\sqrt{d_k}$ 是为了防止头维度较大时点积方差膨胀, 避免 softmax 进入梯度饱和区; Softmax 逐行将分数归一化为合法概率分布, 确保每个 query 对各 key 的关注权重非负且和为 1; 最后用该概率分布对 Value 向量加权求和, 得到每个位置的上下文聚合表示. 这一标准定义的复杂度为 $O(N^2 d_k)$, 当 $N$ 增长时计算量呈二次方膨胀, 这正是稀疏注意力试图突破的根本瓶颈. 

稀疏注意力的核心思想是放弃"每个 query 必须关注所有 key"的全局假设, 将标准注意力泛化为索引集合约束形式：

$$\text{SparseAttn}(q_t, K, V) = \text{Softmax}\left(\frac{q_t K[\mathcal{I}_t]^T}{\sqrt{d_k}}\right)V[\mathcal{I}_t] \tag{2}$$
这里 $q_t$ 表示第 $t$ 个位置的 query 向量, $\mathcal{I}_t \subseteq [t]$ 是该 query 被允许访问的 key 索引集合, 且满足 $|\mathcal{I}_t| \ll t$. $K[\mathcal{I}_t]$ 和 $V[\mathcal{I}_t]$ 分别表示由该索引集合选出的 Key 和 Value 子矩阵. 与式 (1) 相比, 式 (2) 将点积计算和加权求和都限制在子集 $\mathcal{I}_t$ 上：$q_t$ 只需与选中的 $|\mathcal{I}_t|$ 个 key 计算相似度, 而非全部 $t$ 个历史 key, 计算复杂度从 $O(t)$ 降至 $O(|\mathcal{I}_t|)$; Softmax 在缩减后的子集上完成归一化, 输出仍是合法的概率分布; 最终仅对被选中的 value 子集加权求和. 该公式是稀疏注意力的通用抽象——滑动窗口、块路由、压缩选择等不同方法的本质区别, 完全体现在索引集合 $\mathcal{I}_t$ 的构造策略上. 

MoBA 将历史序列划分为 $n$ 个连续块 $I_i$, 每个块包含 $B$ 个 token 的索引范围. 为了在不加载完整块的情况下判断哪些块与当前 query 相关, MoBA 先对每个块做均值池化压缩, 再计算 query 与块摘要的亲和度：

$$\bar{K}_i = \text{mean\_pool}(K[I_i]), \quad s_i = \langle q, \bar{K}_i \rangle \tag{3}$$
其中 $K[I_i]$ 表示第 $i$ 个块内的所有 key 向量集合, $\text{mean\_pool}(\cdot)$ 将其取平均得到块级摘要向量 $\bar{K}_i$. 这一步的动机是用单个向量代表整个块, 使路由决策的计算量从与块大小成正比的 $O(B \cdot d)$ 压缩为与维度成正比的 $O(d)$. 随后通过内积 $\langle q, \bar{K}_i \rangle$ 计算当前 query $q$ 与该块摘要的相似度 $s_i$, 作为该块是否包含相关信息的判别依据. 均值池化是一种无参压缩, 虽然会损失块内的细粒度结构信息, 但计算开销极轻; 较高的 $s_i$ 意味着该块可能包含与当前 query 语义相关的内容, 值得被选中并加载执行精细注意力. 

得到亲和度分数后, 门控机制通过 top-k 选择决定哪些块进入精细注意力计算：

$$g_i = \begin{cases} 1 & s_i \in \text{Topk}(\{s_j\}, k) \text{ 且 } \text{pos}(q) \ge i \cdot B \\
    0 & \text{otherwise} \end{cases} \tag{4}$$
这里 $g_i$ 是第 $i$ 个块的门控值(0 或 1), 决定是否选中该块. $s_i \in \text{Topk}(\{s_j\}, k)$ 表示仅保留与当前 query 最相似的 $k$ 个块, 这一步将 MoE 的门控路由机制直接迁移至注意力层, 是实现稀疏化的关键. 约束条件 $\text{pos}(q) \ge i \cdot B$ 则保证了因果性——当前 query 只能访问起始位置不大于当前位置的块, 禁止访问未来信息. 当 $g_i = 1$ 时, 该块的原始 KV 将被加载并执行标准 Attention; $g_i = 0$ 时该块被丢弃. 该门控将全局 $O(N)$ 的 KV 加载量压缩为 $O(k \cdot B)$, 是 MoBA 实现亚线性复杂度的核心机制. 

因果性通过禁止访问未来块实现, 当前块被强制选中($g_{\text{curr}} = 1$), 块内部再应用标准因果掩码. 最终, 当前 query 允许访问的完整索引集合由所有被选中块的索引并集构成：

$$\mathcal{I}_t = \bigcup_{g_i=1} I_i \tag{5}$$
其中 $I_i$ 表示第 $i$ 个块在原始序列中的连续索引范围. 并集操作将所有 $g_i = 1$ 的块内索引合并为一个统一的集合 $\mathcal{I}_t$, 即使这些块在原始序列中不连续. 该集合代入式 (2) 的稀疏注意力框架后, 即可在缩减后的子集上执行全精度 Softmax Attention. 这一设计的物理意义在于：MoBA 的稀疏决策发生在块级别以降低路由开销, 但块内仍保留原始 token 级精度, 避免均值池化带来的信息损失进入最终注意力计算. 

### 4.2 NSA：三支路门控融合

NSA 的设计动机源于人类认知的分层信息处理模式：对邻近内容保持高精度记忆, 对远处内容通过摘要建立全局轮廓, 并根据当前需求定向检索关键段落. 为此, NSA 对每路查询并行计算三个分支, 通过可学习门控 $g_t^c$ 动态加权融合：

$$o_t^* = \sum_{c \in \{cmp, slc, win\}} g_t^c \cdot \text{Attn}(q_t, \tilde{K}_t^c, \tilde{V}_t^c) \tag{6}$$
其中 $c \in \{cmp, slc, win\}$ 分别标识压缩(compression)、选择(selection)和滑动窗口(sliding window)三个分支, $g_t^c$ 是位置 $t$ 处经 Softmax 归一化的可学习门控权重, $\tilde{K}_t^c$ 和 $\tilde{V}_t^c$ 是各分支处理后的 Key/Value 表示. 压缩分支将 $l$ 个连续 key/value 经 MLP $\phi(\cdot)$ 聚合为单摘要 token, 以牺牲细粒度精度为代价换取全局概览能力, 实现 $O(N/d)$ 的序列压缩比; 选择分支复用压缩分支的注意力分数 $p_t^{cmp}$ 作为块重要性依据, 挑选 top-$n$ 个原始精度块执行精细注意力, 实现稀疏精确检索; 滑动窗口分支仅关注邻近 $w$ 个 token, 负责捕获局部精细依赖. 门控 $g_t^c$ 根据输入内容自适应调整三路贡献比例, 将全局压缩概览、稀疏精确选择和局部精细窗口三层信息融合为最终输出 $o_t^*$, 在降低计算量的同时维持了不同尺度的信息覆盖. 

| 维度 | MoBA | NSA |
|:-----|:-----|:-----|
| 路由粒度 | 块级别(block size $B$) | 块级别(compression block $l$, selection block $l'$) |
| 路由依据 | 块均值向量内积 | 压缩分支注意力分数引导 |
| 稀疏模式 | 动态 Top-k 块选择 | 压缩 + 选择 + 滑动窗口三路融合 |
| 计算复杂度 | $O(n \cdot B + n \cdot k \cdot B) = O(kN)$ | $O(N \cdot N/d + N \cdot n \cdot l' + N \cdot w) = O(N^2/d)$ |
| 原生训练 | 支持, 支持全注意力的渐进过渡 | 支持, 端到端可导 |
| 硬件优化 | 集成 FlashAttention 进行变长序列批处理 | Triton 编写专用内核, 组中心化加载 |
| 因果性处理 | 强制当前块 + 未来块屏蔽 | 滑动窗口天然局部, 选择/压缩只访问历史 |

MoBA 的数学本质是将 MoE 的门控路由从 FFN 层迁移至注意力层, 历史块即"专家". NSA 的数学本质则是**分层近似**：先用低秩压缩信息执行全量扫描, 再用高秩原始信息执行稀疏精算, 避免了对全序列 $N$ 个 token 的直接 $O(N^2)$ 计算. 

两种范式在信息论视角下构成互补. MoBA 的块均值池化是一种无参压缩, 信息损失较大但计算极轻; NSA 的 MLP 压缩是可学习的有参降维, 能保留更多语义特征但引入额外参数量和训练成本. MoBA 的路由决策完全基于 query 与块均值的相似度, 对分布外(OOD)查询的鲁棒性较弱; NSA 的三路门控融合提供了冗余通道, 即使选择分支失效, 滑动窗口和压缩分支仍能保证基础性能. 

## 5. 算力成本与数值对比

以单层单头、$d_k = 128$、batch size = 1 的解码步为例, 对比不同序列长度下的计算与访存特征. 

| 方法 | 序列长度 | 有效 KV 加载量 | 注意力 FLOPs | 访存字节数 | 稀疏率 |
|:-----|:--------|:-------------|:------------|:----------|:------|
| Full Attention | 32K | 32,768 | $2 \times 32K^2 \times 128 \approx 268$ M | 16.8 MB | 0% |
| Full Attention | 128K | 131,072 | $2 \times 128K^2 \times 128 \approx 4.3$ G | 67.1 MB | 0% |
| Full Attention | 1M | 1,048,576 | $2 \times 1M^2 \times 128 \approx 274$ G | 536.9 MB | 0% |
| MoBA ($B$=512, top-3) | 32K | $4 \times 512 = 2,048$ | $2 \times 2K^2 \times 128 \approx 1.05$ M | 1.05 MB | 93.75% |
| MoBA ($B$=2048, top-3) | 128K | $4 \times 2048 = 8,192$ | $2 \times 8K^2 \times 128 \approx 16.8$ M | 4.19 MB | 93.75% |
| NSA (cmp+slc+win) | 64K | ~1,024 (slc) + 2,047 (cmp) + 4,096 (win) | ~3.7 M | ~1.8 MB | ~95% |

MoBA 在 1M 序列长度下实现前向 6.5x 加速; NSA 在 64K 序列下实现解码 11.6x、前向 9.0x、反向 6.0x 加速. 二者的加速比差异源于优化目标的不同：MoBA 侧重于绝对计算量的削减, NSA 则通过 Triton 内核的组中心化加载和融合操作进一步消除了冗余内存传输. 

需特别注意：稀疏注意力的加速比随序列长度非线性增长. 在 8K 以下短序列中, 路由开销(块均值计算、top-k 排序、索引重排)和内核启动成本可能抵消稀疏收益, 甚至慢于全注意力. 以 MoBA 为例, 当序列长度 4K、块大小 512 时, 仅产生 8 个历史块, top-3 选择后的稀疏率仅 62.5%, 不足以摊平路由开销. 实际部署中应设置序列长度阈值(如 16K 或 32K), 低于阈值时回退至全注意力. 

预填充阶段的长 prompt 也需谨慎评估. 虽然预填充阶段计算密集, 但稀疏注意力要求先对完整 prompt 进行块划分和路由计算, 这一额外步骤可能延长首 token 延迟(Time To First Token, TTFT). NSA 的压缩分支通过 MLP 聚合 key, 其计算量与序列长度线性相关, 在极长 prompt(>256K)下可能构成新的瓶颈. 

## 6. 极简实现：块路由注意力骨架

以下代码展示 MoBA 风格块路由的核心逻辑, 不含生产级优化：

```python
import torch
import torch.nn.functional as F

def block_route_attention(q, k, v, block_size=512, top_k=2):
    """
    q: [1, 1, d]
    k, v: [1, seq_len, d], seq_len 已包含历史上下文
    返回: [1, 1, d]
    """
    seq_len = k.size(1)
    n_blocks = (seq_len + block_size - 1) // block_size
    d = q.size(-1)
    
    # 1. 将 KV 分块并计算每块均值
    k_blocks = [k[:, i*block_size:(i+1)*block_size, :] for i in range(n_blocks)]
    block_means = torch.stack([kb.mean(dim=1) for kb in k_blocks], dim=1)  # [1, n_blocks, d]
    
    # 2. 计算 query 与各块的亲和度, 屏蔽未来块
    pos = seq_len - 1
    curr_block_idx = pos // block_size
    scores = torch.matmul(q, block_means.transpose(-1, -2)) / (d ** 0.5)  # [1, 1, n_blocks]
    scores[:, :, curr_block_idx+1:] = float('-inf')  # 因果屏蔽
    
    # 3. Top-k 选择(强制包含当前块)
    _, top_indices = torch.topk(scores.squeeze(1), k=min(top_k, curr_block_idx+1), dim=-1)
    if curr_block_idx not in top_indices[0]:
        top_indices[0][-1] = curr_block_idx
    
    # 4. 拼接选中块的原始 KV, 执行标准注意力
    selected_k = torch.cat([k_blocks[i] for i in top_indices[0]], dim=1)
    selected_v = torch.cat([v[:, i*block_size:(i+1)*block_size, :] for i in top_indices[0]], dim=1)
    
    # 当前块内需应用因果掩码
    attn_mask = torch.ones(selected_k.size(1), device=q.device)
    selected_pos = pos - (curr_block_idx * block_size)
    attn_mask[selected_pos+1:] = 0
    
    scores = torch.matmul(q, selected_k.transpose(-1, -2)) / (d ** 0.5)
    scores = scores.masked_fill(attn_mask.unsqueeze(0).unsqueeze(0) == 0, float('-inf'))
    attn = F.softmax(scores, dim=-1)
    out = torch.matmul(attn, selected_v)
    return out
```

生产级实现需集成 FlashAttention 处理变长块拼接、使用 Triton 消除 Python 调度开销、并对 GQA 组共享 KV 进行单加载多复用优化. 

## 7. 边界条件与失效模式

稀疏注意力并非万能解, 其性能与稳定性在极端条件下存在明确边界. 

| 失效场景 | 物理根源 | 典型症状 | 缓解策略 |
|:--------|:--------|:--------|:--------|
| 短序列负收益 | 路由开销(top-k 排序、块均值计算)超过稀疏节省的计算量 | seq < 16K 时延迟高于全注意力 | 设置动态阈值, 短序列回退全注意力 |
| 块粒度过粗 | $B$ 过大导致被选块包含大量无关 token, 信息信噪比下降 | 长文本 PPL 上升, RULER 基准下降 | 消融实验确定最优块大小(MoBA 实验显示细粒度分段提升显著) |
| 块粒度过细 | $B$ 过小导致块数量激增, 路由计算和内存碎片化加剧 | 内核启动开销占比上升, 有效带宽下降 | 平衡块大小与 SRAM 容量, 通常 512-4096 为合理区间 |
| SFT 阶段梯度稀疏 | SFT 的损失掩码排除 prompt token, MoBA 稀疏模式导致梯度信号进一步稀释 | SFT 后模型性能退化 | 顶层(最后 1/4 层)切换为全注意力, 底层保持 MoBA |
| 均匀分布上下文 | 所有历史块与当前 query 的亲和度相近, top-k 选择退化为随机采样 | 信息丢失等效于粗粒度滑动窗口 | 引入可学习的压缩表示(NSA 方案)替代简单均值 |
| 路由崩塌 | 门控网络输出极度不均衡, 全部 query 涌向同一历史块 | 负载失衡, 部分 KV 块重复计算, 缓存利用率恶化 | 借鉴 MoE 的辅助负载均衡损失(aux loss)或容量因子约束 |
| 跨块依赖断裂 | 关键信息恰好位于块边界, 压缩/选择机制丢失边界连续性 | 命名实体识别、指代消解任务精度下降 | 引入块重叠(overlap stride, NSA 采用 $d < l$ 的重叠压缩) |
| 压缩分支信息损失 | MLP 压缩将 $l$ 个 token 降维为 1 个摘要 token, 高频细节湮灭 | 细粒度事实检索任务(如精确数字、日期)错误率上升 | 降低压缩比、增加选择分支的 top-n 数量、引入残差连接 |
| 门控网络过拟合 | 可学习的门控 $g_t^c$ 在训练数据分布上过度优化 | OOD 长文本上三路权重分配失衡, 某一主导分支 suppression | 门控输出加温度系数、训练时注入噪声、多任务联合训练 |

MoBA 论文中报告了一个关键发现：纯 MoBA 方案在 SFT 阶段的逐位置损失显著高于全注意力. 物理根源在于 SFT 的损失掩码仅计算 response token 的交叉熵, prompt token 不产生梯度. MoBA 的稀疏路由进一步减少了有效梯度路径, 形成双重稀疏惩罚. 解决方案是**分层混合策略**——将模型顶层替换为全注意力, 底层保持 MoBA 稀疏, 在 SFT 中恢复了与全注意力相当的性能. 

另一个深层风险是稀疏注意力的**评估偏差**. 主流长文本基准(RULER、Needle-in-a-Haystack)通常将关键信息集中放置在单一位置, 天然适合块选择机制定位. 但在真实文档中, 关键信息可能分散在多个不连续块, 且依赖跨块的长距离推理链路. 当前基准可能高估了稀疏注意力的实际能力, 社区亟需设计更能反映碎片化信息整合需求的评估协议. 

### 工业前沿：TransMLA 的 RoPE-NoPE 拆分与无痛 GQA 转 MLA

MLA(Multi-head Latent Attention)通过低秩压缩将 KV Cache 从 $O(d_{\text{head}})$ 降至 $O(d_c)$, 但原始实现需要从头训练. TransMLA 提出了一种**后训练转换**方案：将已有 GQA 模型的权重通过数学变换转换为 MLA 格式, 无需重新预训练即可获得接近原生 MLA 的推理效率. 

**核心洞察：RoPE 与 NoPE 的不可压缩性差异**. DeepSeek MLA 的 Key/Value 投影同时包含内容部分(NoPE, 无位置编码)和位置部分(RoPE, 旋转位置编码). RoPE 部分携带了精确的相对位置信息, 若将其与 NoPE 混合压缩, 会导致位置敏感性丧失. TransMLA 通过**正交矩阵拆分**将 GQA 的 KV 投影分解为两个正交子空间：

$$
W_{KV} = W_{KV}^{\text{NoPE}} \oplus W_{KV}^{\text{RoPE}} \tag{7}
$$
其中 $W_{KV}^{\text{NoPE}}$ 和 $W_{KV}^{\text{RoPE}}$ 通过正交投影矩阵 $P_{\text{NoPE}}$ 和 $P_{\text{RoPE}}$ 从原始 GQA 的 $W_{KV}$ 中提取. 正交性保证 $P_{\text{NoPE}} P_{\text{RoPE}}^T = 0$, 两个子空间的信息不互相干扰. 

**只压缩 NoPE, 保留 RoPE**. NoPE 部分仅携带内容语义信息, 天然适合低秩压缩. 对 $W_{KV}^{\text{NoPE}}$ 执行 SVD 分解并截断至秩 $r$：

$$
W_{KV}^{\text{NoPE}} \approx U_r \Sigma_r V_r^T \tag{8}
$$
其中 $U_r \in \mathbb{R}^{d \times r}$ 和 $V_r \in \mathbb{R}^{r \times d_{\text{head}}}$ 分别成为下投影和上投影矩阵, 与原生 MLA 的结构完全一致. RoPE 部分 $W_{KV}^{\text{RoPE}}$ 保持原始维度不做压缩, 因为位置编码的维度通常很小(如 $d_r = 64$), 压缩收益有限而精度损失显著. 

**联合分解压缩的权衡**. 若 GQA 的 group 数为 $g$, 每组共享 $n_h/g$ 个 query head 的 KV. TransMLA 的联合分解将所有组的 $W_{KV}^{\text{NoPE}}$ 堆叠为矩阵 $W_{\text{stacked}} \in \mathbb{R}^{g \times d \times d_{\text{head}}}$, 执行联合 SVD：

基于上述分析, 建立如下数学关系：
$$
W_{\text{stacked}}^{\text{NoPE}} \approx U_{\text{joint}} \Sigma_{\text{joint}} V_{\text{joint}}^T \tag{9}
$$
联合分解的优势在于跨组共享低秩子空间, 当不同组的注意力模式相似时, 可用更小的 $r$ 达到同等精度. 劣势是各组失去独立性, 若某组的注意力模式显著异于其他组, 联合压缩会引入系统性偏差. 

| 方法 | KV Cache 大小 | 是否需要重训练 | 相对精度损失 | 适用场景 |
|:-----|:-------------|:-------------|:-----------|:--------|
| GQA | $2 \cdot d_{\text{head}} \cdot n_h / g$ | 否 | 0% | 现有模型 |
| MLA (原生) | $2 \cdot d_c \cdot n_h$ | 是 | 0% | 新训练 |
| TransMLA | $2 \cdot (r \cdot n_h + d_r \cdot n_h / g)$ | 否 | $<1\%$ | 已有 GQA 模型转换 |

TransMLA 的工程价值在于：企业已有的大量 GQA 模型(如 Llama 3、Qwen 2.5)无需昂贵的重新预训练, 即可通过权重转换获得 MLA 级别的 KV Cache 压缩. 测试表明, 在 Llama-3-8B 上转换后, KV Cache 降至原 GQA 的 15%-20%, 推理吞吐量提升 2-3 倍, 下游任务精度损失 $<0.5\%$. 

## 8. 技术前瞻与演进方向

稀疏注意力仍处于快速迭代期, 以下方向代表社区共识度较高的演进路径：

**自适应块大小**

固定块大小对所有层、所有 head 是次优的. 浅层负责局部语法和词法, 适合细粒度块; 深层负责全局语义和推理, 适合粗粒度块. 未来架构可能为每层甚至每头独立学习最优块大小, 或引入层次化块结构(Hierarchical Blocking). 

**Softmax 稀释效应与 top-k 稀疏化**

标准注意力的 Softmax 归一化在长序列下存在隐性缺陷：随着序列长度 $L$ 增加, 分母的指数和 $\sum_{j=1}^L e^{s_{ij}}$ 增大, 导致每个 token 的注意力权重被"稀释". 当 $L$ 远超训练时的最大长度(如 10× 以上), 关键信号的权重可能降至与噪声同一量级, 模型丧失聚焦能力. top-k 稀疏注意力通过仅保留每 query 的 top-k 个注意力分数、将其余置零, 强制注意力集中在最相关的 $k$ 个位置上. 这种方法无需重新训练即可实现长度外推：对超过训练长度的序列, top-k 选择天然过滤掉远距离的弱相关 token, 避免 Softmax 稀释. 实验表明, 在 8K→64K 的外推场景下, top-k 稀疏注意力的困惑度(PPL)显著优于标准稠密注意力, 且 $k=128$ 即可覆盖 95% 以上的有效注意力质量. 

**与线性注意力的混合架构**

NSA 和 MoBA 仍基于 Softmax 注意力. 将线性注意力分支(如 Mamba 的状态空间压缩)作为第四路或替代压缩分支, 可在保持细粒度检索能力的同时进一步降低全局概览分支的成本. DeepSeek 的后续研究已在探索此类混合路线. 

**上下文长度从 1M 到 10M+**

MoBA 在 10M token 序列上验证了注意力计算时间缩减为 1/16. 随着上下文窗口竞赛从 128K 推向 1M+, 稀疏率需从当前的 90%-95% 进一步提升至 99% 以上. 这要求路由机制具备亚线性复杂度(如两级索引结构：先选章节, 再选段落), 而非当前的单层 top-k. 多级索引的数学形式可类比数据库的 B+ 树：第一层压缩分支对 10M token 进行 100x 压缩得到 100K 个摘要 token, 第二层选择分支在 100K 摘要中定位关键章节, 第三层才加载原始精度块. 每一层的路由计算量均远低于下一层, 整体复杂度可逼近 $O(N \log N)$ 或 $O(N)$. 

**稀疏注意力的训练基础设施**

现有分布式训练框架(Megatron-LM、FSDP)假设全稠密注意力. 稀疏注意力的变长 KV 块加载、动态负载均衡和梯度聚合需要全新的通信原语. NSA 的 Triton 内核已证明单卡效率可行, 但多卡环境下的稀疏 all-gather 和流水线并行仍是开放工程问题. 

**测试时计算的稀疏扩展**

o3 类模型依赖极长的内部 CoT 序列进行推理时计算扩展. CoT 序列具有强烈的局部重复和层次化结构(思考 -> 验证 -> 回溯), 天然适合稀疏注意力. 在 CoT 的验证阶段, 模型仅需回顾近期推理步骤和原始问题陈述, 无需关注中间推导的全部细节; 在回溯阶段, 模型需要定位历史决策点, 这恰好对应选择分支的"关键检索"能力. 将 NSA/MoBA 与 speculative decoding、lookahead decoding 结合, 可能是长思考链高效推理的下一个突破点. 

此外, 稀疏注意力为**多模态长序列**开辟了新的设计空间. 视频理解模型处理 1 小时视频(数万帧)时, 空间注意力(帧内 patch)与时间注意力(帧间关系)可分别应用不同的稀疏策略：空间维度保持稠密以维护图像细节, 时间维度采用 MoBA 式块路由以跳过冗余帧. 这种跨模态异构稀疏尚未被系统研究.

## 10. 为什么块级稀疏最终胜过 token 级稀疏

从纯数学角度看, 最理想的稀疏模式应该尽可能细, 最好直接精确到 token 级别：只保留最重要的几条边, 其余全部删掉.   
但从 GPU 执行角度看, 这种细粒度随机稀疏几乎是灾难. 

原因很简单：GPU 喜欢连续内存块, 不喜欢零散访存.   
token 级稀疏意味着：

- 访存模式高度随机
- 合并访存效果差
- grouped GEMM 难以形成高效块
- kernel launch 和调度开销无法摊薄

于是工业界最终越来越偏向块级稀疏(block-wise sparsity). 虽然它在理论上不如 token 级精细, 但它和现代硬件的物理结构更匹配. 块级稀疏的真正价值, 不是让稀疏决策更漂亮, 而是让稀疏决策之后依然能用稠密 kernel 高效执行. 

## 11. 稀疏注意力与 FlashAttention 是互补关系

有时会有人把稀疏注意力和 FlashAttention 看成对立路线：一个在减少连边, 一个在优化稠密算子. 实际上它们非常适合叠加. 

稀疏注意力解决的是“哪些块根本不用算”.   
FlashAttention 解决的是“那些确实要算的块, 如何高效地算”.   

这就是为什么 NSA 和 MoBA 这类现代方法通常不排斥 FlashAttention, 反而会把它当成块内执行后端. 块级稀疏负责选块, FlashAttention 负责把每个块内的 dense attention 做到 IO 最优. 

因此, 理解稀疏注意力时不要把它想成“替代 FlashAttention”, 而应该把它看成“为 FlashAttention 缩小要处理的有效工作集”. 

## 12. 训练时稀疏和推理时稀疏不是一回事

这也是一个非常关键的区分. 

### 推理时稀疏

推理时稀疏通常更偏后处理逻辑. 模型已经训练好了, 我们只是在推理时选择性保留部分 KV、部分窗口或部分块. 优点是接入成本低, 缺点是模型本身没有在这种稀疏模式下学过, 因此性能很容易打折. 

### 训练时原生稀疏

训练时原生稀疏则完全不同. 模型从一开始就在稀疏模式下学习路由与聚合, 注意力结构本身就是训练分布的一部分. 这样做更贵、更复杂, 但它换来的是：

- 更高的结构一致性
- 更少的推理时分布断层
- 更强的长上下文泛化

这也正是 NSA 一类方法的重要意义：它们不是“推理 patch”, 而是原生训练路线. 

## 13. 稀疏注意力的最大风险不是精度下降, 而是路由错判

很多人一提稀疏, 就只担心“信息丢了会不会精度下降”. 这当然重要, 但更本质的风险其实是：模型把错误的块当成了重要块. 

只要路由错判发生, 问题不是简单丢了一些次要连接, 而是：

- 真正关键的信息根本没被加载
- softmax 只在错误候选上归一化
- 后续层会在错误上下文上继续推理

这和稠密注意力里的“低权重但仍然存在”很不同. 稀疏路由一旦判错, 很多信息是被硬删除的. 因此稀疏注意力的核心不只是“稀疏率够不够高”, 而是“保留下来的候选是否真的覆盖了关键上下文”. 

## 14. 为什么说稀疏注意力更像检索系统

随着上下文变长, 稀疏注意力越来越像一个分层检索器, 而不只是一个近似 attention 算子. 

一个 query 面对十万、百万 token 历史时, 最合理的计算流程其实和搜索引擎很像：

1. 先通过便宜的摘要或索引做粗召回  
2. 再在少量候选块上做精排  
3. 最终在局部高精度上下文中执行真实聚合

MoBA、NSA 和多级块路由的很多设计, 本质上都在把注意力做成“可微检索系统”. 这一点在超长上下文时代会越来越明显. 

## 15. 稀疏注意力的部署判断标准

如果你在做系统选型, 最实用的判断标准不是“哪篇论文指标最好”, 而是下面几个现实问题：

1. 你的场景是训练还是推理  
2. 你关心的是 32K、128K 还是 1M+ 上下文  
3. 你是否能接受重新训练  
4. 你手上是通用 GPU 集群还是更定制化基础设施  
5. 你更怕精度损失, 还是更怕吞吐不够

通常可以粗略这样分：

- 只想低成本缓解推理压力：StreamingLLM / H2O
- 想做原生长上下文训练：NSA
- 想在已有模型上渐进接入长上下文：MoBA
- 想继续向千万级上下文探索：多级块索引与更强路由器

## 17. 本节小结

稀疏与压缩注意力的根本目标, 不是为了“看起来更省”, 而是为了在不完全放弃 Softmax 注意力表达能力的前提下, 把注意力从全连接暴力计算改造成更像检索系统的结构. 

块级稀疏之所以成为主流, 不是它理论最优, 而是它第一次把“稀疏决策”和“硬件友好执行”兼容起来.   
原生稀疏训练之所以重要, 不是因为推理 patch 不可用, 而是因为只有在训练阶段就学会稀疏, 模型才能真正把长上下文能力内化进参数.   

因此, 这一节真正建立的是一个判断框架：当上下文进一步拉长时, 注意力的未来未必是完全放弃 Softmax, 而更可能是越来越像一个可微、分层、硬件对齐的检索系统.
