# 2.4.5 · 新兴架构与混合模型

## 1. 演进脉络：单一架构的物理极限与混合范式的诞生

Transformer 的 $O(n^2)$ 注意力复杂度在长序列场景下构成硬性物理约束. 当上下文窗口从 4K 扩展至 128K,Attention 的 KV Cache 显存占用与计算量呈平方级膨胀,迫使工程团队投入大量精力在分块、稀疏化与压缩策略上. Mamba(Gu & Dao, 2023)以选择性状态空间模型(Selective SSM)将序列建模复杂度降至 $O(n)$,在 DNA 序列建模、音频处理等长序列任务上展示了与 Transformer 相当甚至更优的精度. RWKV(Peng et al., 2023)以线性 RNN 的形式复现了 Transformer 的并行训练特性,同时保持 $O(1)$ 的解码复杂度. 

线性架构的崛起并未终结 Transformer 的统治,反而暴露了一个更深层的事实：没有任何单一计算原语能够同时覆盖"局部精确模式匹配"、"长程依赖传播"与"恒定显存推理"三类物理需求. Attention 在短序列上的表达能力与训练稳定性经过七年工业验证,SSM 与线性 RNN 在长序列效率上具备不可比拟的理论优势. 混合架构(Hybrid Architecture)的核心洞见由此诞生——不再追问"哪种架构更好",而是将不同架构作为可组合的算力原语,依据任务特性在层级别或块级别进行编排. 

混合架构的演进呈现三条主线：

| 时间 | 里程碑 | 核心思想 | 解决的关键矛盾 |
|:----|:------|:--------|:------------|
| 2023.12 | Mamba | 选择性 SSM 替代全局 Attention | $O(n^2)$ 复杂度 → $O(n)$,但损失全局召回 |
| 2024.01 | Griffin (Hawk) | Gated Linear RNN + 局部 Attention | 线性 RNN 的隐状态瓶颈 + Attention 的局部精度 |
| 2024.03 | Jamba | Transformer 层与 Mamba 层交错堆叠 | SSM 的长程效率 + Attention 的短程精度 |
| 2024.05 | xLSTM | LSTM 记忆单元升级(sLSTM / mLSTM) | RNN 梯度消失与并行化训练 |
| 2024.02 | BitNet b1.58 | 三值权重量化 + 激活值缩放 | 推理成本与模型精度的帕累托边界 |
| 2024.12 | Zamba | 三阶段交错：Mamba → MLP → Attention | 不同层类型的互补激活模式 |

混合架构不是对 Transformer 的否定,而是对"单一计算范式假设"的超越. 当模型规模突破百亿参数、上下文长度突破百万 token,架构设计的核心问题从"选择最优模块"转向"编排模块的最优组合". 

## 2. 工业价值：混合架构的落地版图与性能基准

混合架构已从学术论文进入工业级模型的生产环境,其落地速度与范围超出多数预期. 

| 模型 | 机构 | 架构组合 | 参数规模 | 关键工业指标 |
|:-----|:-----|:--------|:--------|:------------|
| Jamba | AI21 Labs | Mamba-Transformer 交错 + MQA + MoE | 52B (256B 总参, 12B 激活) | 256K 上下文,单卡 A100 可推理 |
| RecurrentGemma | Google DeepMind | Griffin (gLSTM + 局部 Attention) | 2B / 9B | 推理吞吐量较同规模 Transformer 提升 2-3x |
| Zamba | Zyphra | Mamba-MLP-Attention 循环块 | 2.8B / 7B | 预训练速度提升 1.5x,下游任务优于同规模 Transformer |
| Samba | Microsoft Research | Mamba + 滑动窗口 Attention | 421M - 3.8B | 长序列建模 PPL 显著优于纯 Mamba |
| BitNet b1.58 | Microsoft | 1.58-bit 三值权重 | 3B 等效 | CPU 推理延迟降低 2-4x,能耗降低 70%+ |
| xLSTM-7B | NXAI / LIT AI | sLSTM + mLSTM 混合块 | 7B | 在 Long Range Arena 上超越 Mamba |
| Falcon-Mamba | TII | 纯 Mamba Decoder   | 7B | 推理时恒定显存,无 KV Cache |

Jamba 的工业意义最为深远. 其 52B 激活参数的 MoE 变体在保持 256K 上下文能力的同时,单次前向传播的激活参数量仅 12B. 这意味着企业级部署可在单张 A100 80GB 上运行 256K 上下文的文档分析任务,而同规格的稠密 Transformer(如 Llama 3 70B)在同等上下文长度下需要 8 卡以上的显存预算. AI21 Labs 公开的技术报告指出,Jamba 在"Needle in a Haystack"长文本召回测试中,256K 长度的准确率显著优于纯 Transformer 基线,这直接证明了 Mamba 层在长程信息保持上的物理优势. 

Google 的 RecurrentGemma 采取了另一条路线：彻底抛弃全局 Attention,以 Gated Linear RNN 承担序列建模主体,仅在局部窗口(如 2K)内保留 Attention 用于细粒度模式匹配. 这一设计在端侧部署(手机、嵌入式设备)上具备天然优势——推理阶段的恒定显存占用消除了 KV Cache 对内存容量的苛刻要求. Google 将其集成至 Gemma 开源家族,直接验证了混合架构在消费级硬件上的可行性. 

BitNet 的落地逻辑不同于上述模型. 它不改动架构拓扑,而是通过极端量化(权重三值化)重构了矩阵乘法的硬件映射. BitNet b1.58 在 3B 规模上的精度接近全精度 Transformer,但推理延迟在 ARM CPU 上降低 2-4 倍. Microsoft 将 BitNet 推理内核集成至 llama.cpp,意味着开源社区已有的量化工具链可直接迁移至三值量化范式. 这对于边缘计算、离线推理、低功耗 IoT 场景具有变革性意义. 

混合架构的工业价值可归纳为三个维度：

| 维度 | 纯 Transformer 瓶颈 | 混合架构解法 | 典型收益 |
|:-----|:------------------|:-----------|:--------|
| 长上下文 | KV Cache 线性增长,显存 $O(n)$ | Mamba/SSM 层承担长程建模,Attention 层压缩或稀疏化 | 256K 上下文单卡可跑 |
| 推理吞吐 | 解码阶段受限于显存带宽 | 线性 RNN 层 O(1) 状态更新,带宽需求锐减 | 吞吐量提升 2-5x |
| 端侧部署 | 大模型无法装入手机内存 | BitNet 量化 + 小型混合模型 | 7B 模型手机本地运行 |

## 3. 物理直觉：乐高积木的原子化编排

将混合架构理解为乐高积木的组合,每个"积木"是一种具有确定物理特性的计算原语. Transformer Attention 是精密光学显微镜——分辨率极高,能捕捉任意两个 token 之间的微观关联,但视场越大(序列越长),机械结构越笨重,成本越昂贵. Mamba SSM 是惯性导航系统——不依赖外部参照,以恒定速度持续积累位置信息,能耗极低,但对突变信号的响应存在延迟. 线性 RNN(Griffin)是机械飞轮——利用角动量平滑存储旋转能量,输出稳定且可预测,但无法精确复现瞬时脉冲. 

混合架构的设计哲学是：依据任务的数据流特征,在不同深度和不同位置插入不同类型的计算积木. 文本的前几层通常需要精确的局部模式匹配(词法、句法),Attention 积木在此处发挥最大价值. 中间层需要建立跨句、跨段的语义关联,Mamba 积木以线性复杂度承担长程信息流. 深层需要压缩与抽象,线性 RNN 或 BitNet 量化层以最低能耗完成最终表征提炼. 

Jamba 的 8:1 Mamba-Attention 层交错比例并非随意设定. 实验表明,当 Mamba 层占比低于 50% 时,长序列任务的 PPL 下降不显著; 当占比超过 90% 时,短序列的精确召回能力出现退化. 8:1 的比例处于帕累托前沿——以最小 Attention 开销换取最大长程保持能力. 这类似于机械工程中"齿轮比"的优化：不同转速区间匹配不同传动比,而非用单一齿轮覆盖全速域. 

BitNet 的量化策略则类似精密仪器的材料替代：将 16-bit 浮点运算的"钛合金零件"替换为 1.58-bit 三值的"碳纤维零件",结构拓扑不变,但整机重量与能耗大幅下降. 关键在于 BitNet 的激活值缩放因子(scale)以 FP16 保留,确保信号幅度不因权重粗粒度量化而崩溃. 

## 4. 数学推导：核心算子的形式化对比

### 4.1 Jamba 的层交错策略

Jamba 将 $L$ 层网络划分为重复的 "Attention-Mamba" 块. 设单层的隐藏状态为 $\mathbf{h} \in \mathbb{R}^{n \times d}$,其中 $n$ 为序列长度,$d$ 为隐藏维度. 

Attention 子层执行标准 Multi-Query Attention(MQA)：

$$\text{Attn}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right)\mathbf{V} \tag{1}$$
其中 $\mathbf{Q} = \mathbf{h}\mathbf{W}_Q$,$\mathbf{K} = \mathbf{h}\mathbf{W}_K$,$\mathbf{V} = \mathbf{h}\mathbf{W}_V$,MQA 使 $\mathbf{W}_K, \mathbf{W}_V$ 共享于所有头,将 KV Cache 压缩至 $2 \times n \times d_{kv}$. 

Mamba 子层执行选择性状态空间扫描：

$$\mathbf{h}' = \mathbf{h} + \text{SSM}(\text{SiLU}(\mathbf{h}\mathbf{W}_{in}) \odot \text{Conv1D}(\mathbf{h}\mathbf{W}_{x})) \tag{2}$$
选择性机制通过输入依赖的参数化实现：

$$\mathbf{B} = \mathbf{h}\mathbf{W}_B, \quad \mathbf{C} = \mathbf{h}\mathbf{W}_C, \quad \Delta = \text{softplus}(\mathbf{h}\mathbf{W}_\Delta + \mathbf{b}_\Delta) \tag{3}$$
离散化后的状态转移为：

$$\mathbf{\bar{A}} = \exp(\Delta \mathbf{A}), \quad \mathbf{\bar{B}} = \Delta \mathbf{B} \tag{4}$$
此式将连续时间状态空间参数 $(\mathbf{A}, \mathbf{B})$ 通过零阶保持离散化为步长 $\Delta$ 下的等效矩阵,$\mathbf{\bar{A}}$ 决定了历史状态的指数衰减率,$\mathbf{\bar{B}}$ 决定了新输入的缩放强度. 

$$\mathbf{h}_t = \mathbf{\bar{A}} \odot \mathbf{h}_{t-1} + \mathbf{\bar{B}} \odot \mathbf{x}_t, \quad \mathbf{y}_t = \mathbf{C} \mathbf{h}_t \tag{5}$$
式 (5) 描述了离散时间下的选择性状态递推：当前隐状态 $\mathbf{h}_t$ 由历史状态 $\mathbf{h}_{t-1}$ 经 $\mathbf{\bar{A}}$ 衰减后的残留与新输入 $\mathbf{x}_t$ 经 $\mathbf{\bar{B}}$ 放大后的贡献相加而成,输出 $\mathbf{y}_t$ 则通过 $\mathbf{C}$ 将状态投影到观测空间. 这一递推以 $O(1)$ 的常数显存完成长程信息积累. 

Jamba 的关键创新在于将上述两个子层按固定比例(如每 8 层 Mamba 后接 1 层 Attention + 1 层 Mamba)交错堆叠. Attention 层提供全局 token-to-token 的精确交互,Mamba 层以 $O(n)$ 扫描维持长程状态. 这种交错使全局交互的频率降低为原来的 $1/8$,但并未完全消除——与早期纯 SSM 模型(如早期 Mamba)相比,保留了在关键层进行"全局校准"的能力. 

### 4.2 Griffin 的 Gated Linear RNN

Griffin 的核心是 Gated Linear Recurrent Unit(GLRU),其状态更新抛弃了非线性激活,采用纯线性递推：

$$\mathbf{h}_t = \mathbf{r}_t \odot \mathbf{h}_{t-1} + (1 - \mathbf{r}_t) \odot (\mathbf{W}\mathbf{x}_t) \tag{6}$$
其中 $\mathbf{r}_t = \sigma(\mathbf{W}_r\mathbf{x}_t + \mathbf{b}_r)$ 为复位门(reset gate),$\sigma$ 为 sigmoid. 该递推的物理含义是：当前输入 $\mathbf{x}_t$ 以比例 $(1 - \mathbf{r}_t)$ 注入隐状态,历史状态以比例 $\mathbf{r}_t$ 衰减. 当 $\mathbf{r}_t \to \mathbf{1}$ 时,模型退化为纯记忆系统; 当 $\mathbf{r}_t \to \mathbf{0}$ 时,隐状态被当前输入完全重写. 

为保证训练稳定性,Griffin 对隐状态施加约束. 复位门 $\mathbf{r}_t$ 的取值被限制在 $(0, 1)$ 区间,避免了 LSTM 中遗忘门饱和导致的梯度消失. 更重要的是,线性递推的矩阵乘法可在序列维度上完全并行化：通过展开递推式,$\mathbf{h}_t$ 可表示为 $\mathbf{x}_1, \dots, \mathbf{x}_t$ 的加权和：

$$\mathbf{h}_t = \sum_{i=1}^{t} \left(\prod_{j=i+1}^{t} \mathbf{r}_j\right) \odot (1 - \mathbf{r}_i) \odot (\mathbf{W}\mathbf{x}_i) \tag{7}$$
该展开式允许在训练时通过前缀积(prefix product)在 GPU 上并行计算,在推理时恢复为 $O(1)$ 的递推更新. Griffin 的完整层定义结合局部 Attention(Local Attention)与 GLRU：

$$\mathbf{y} = \text{GLRU}(\mathbf{x}) + \text{LocalAttn}(\mathbf{x}) \tag{8}$$
局部 Attention 的窗口大小 $w$ 通常设为 512 或 1024,仅捕捉邻近 token 的细粒度依赖. GLRU 承担长程建模,局部 Attention 承担短程精度,二者相加构成残差连接. 

### 4.3 xLSTM 的指数门控记忆

xLSTM 扩展传统 LSTM,引入两种变体：sLSTM(scalar LSTM)与 mLSTM(matrix LSTM). 

sLSTM 取消遗忘门与输入门的独立控制,改为归一化门控：

$$\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \mathbf{z}_t \tag{9}$$
式 (9) 中,遗忘门 $\mathbf{f}_t$ 控制历史细胞状态 $\mathbf{c}_{t-1}$ 的保留比例,输入门 $\mathbf{i}_t$ 控制新输入 $\mathbf{z}_t$ 的注入比例,二者通过逐元素乘积实现对记忆内容的精细筛选. 

$$\mathbf{h}_t = \frac{\mathbf{c}_t}{\mathbf{n}_t}, \quad \mathbf{n}_t = \mathbf{f}_t \odot \mathbf{n}_{t-1} + \mathbf{i}_t \tag{10}$$
其中 $\mathbf{z}_t = \exp(\mathbf{W}_z \mathbf{x}_t)$ 引入指数激活,$\mathbf{n}_t$ 为归一化状态,消除传统 LSTM 的梯度消失路径. mLSTM 进一步将标量记忆扩展为矩阵记忆：

$$\mathbf{C}_t = \mathbf{C}_{t-1} + \mathbf{v}_t \mathbf{k}_t^T \tag{11}$$
此式将记忆从标量细胞状态升级为矩阵 $\mathbf{C}_t$,通过向量外积 $\mathbf{v}_t \mathbf{k}_t^T$ 逐时刻累加输入信息,使模型能够以类似线性注意力的形式存储和检索跨时间步的关联模式. 

$$\mathbf{h}_t = \mathbf{C}_t \mathbf{q}_t / (\mathbf{k}_t^T \mathbf{q}_t) \tag{12}$$
其中 $\mathbf{v}_t = \mathbf{W}_v \mathbf{x}_t$,$\mathbf{k}_t = \mathbf{W}_k \mathbf{x}_t$,$\mathbf{q}_t = \mathbf{W}_q \mathbf{x}_t$. mLSTM 的矩阵记忆更新规则等价于在线协方差估计,其输出可视为一种无 softmax 的线性注意力变体. 

### 4.4 BitNet b1.58 的量化算子

BitNet 的核心是将权重矩阵 $\mathbf{W}$ 量化为三值 $\{-1, 0, +1\}$. 量化函数定义为：

$$\text{round}_t(x) = \begin{cases} +1 & x > \Delta \\ 0 & |x| \leq \Delta \\ -1 & x < -\Delta \end{cases} \tag{13}$$
其中阈值 $\Delta$ 通常为 $0.5 \times \mathbb{E}(|\mathbf{W}|)$. 激活值保持 FP16 精度,但线性层的前向传播变为：

$$\mathbf{y} = \alpha \cdot (\mathbf{W}_t \mathbf{x}) \tag{14}$$
$\mathbf{W}_t$ 为三值矩阵,$\alpha = \frac{1}{nm} \|\mathbf{W}\|_1$ 为全精度权重的平均绝对值. 由于 $\mathbf{W}_t$ 的元素仅为 $\{-1, 0, +1\}$,矩阵乘法退化为加法与符号翻转,在专用硬件上可实现极高能效比. 

BitNet 的关键数学洞察是：大语言模型的权重分布呈近似拉普拉斯分布,大量权重接近零. 将接近零的权重强制置零(而非强制映射到 $\pm 1$)保留了模型的稀疏性,这是 BitNet b1.58 优于早期 1-bit 量化(仅 $\{-1, +1\}$)的核心原因. 

### 4.5 核心算子对比表

| 架构 | 核心算子 | 序列复杂度 | 状态维度 | 并行化策略 | 关键非线性 |
|:-----|:--------|:---------|:--------|:----------|:----------|
| Transformer Attention | Softmax(QK^T)V | $O(n^2)$ | $O(n)$ (KV Cache) | 完全并行 | Softmax |
| Mamba SSM | 选择性状态扫描 | $O(n)$ | $O(d)$ | 扫描并行 (Blelloch) | SiLU, Softplus |
| Griffin GLRU | 线性门控递推 | $O(n)$ | $O(d)$ | 前缀积并行 | Sigmoid |
| xLSTM mLSTM | 矩阵记忆外积更新 | $O(n)$ | $O(d^2)$ | 完全并行 | 无 (线性注意力) |
| BitNet Linear | 三值矩阵乘 + 缩放 | 与基架构相同 | 与基架构相同 | 完全并行 | 仅激活值保留非线性 |

## 5. 算力成本：训练与推理的物理开销全景

混合架构的成本优势不能仅从理论复杂度推断,必须结合具体硬件特性和实现细节进行数值走查. 

以处理 32K 长度序列、batch size = 1 的解码场景为例：

| 架构 | 参数量 | 训练 FLOPs/token | 推理 FLOPs/token | KV Cache 显存 | 峰值显存 (32K ctx) | 解码吞吐 (tokens/s, A100) |
|:-----|:-----:|:----------------|:----------------|:-------------|:------------------|:------------------------|
| Llama 3 8B (Dense Transformer) | 8B | $6 \times 8\text{B}$ = 48B | $2 \times 8\text{B}$ = 16B | $2 \times n_{layers} \times n_{heads} \times d_{head} \times n$ = 4GB | ~12 GB | ~45 |
| Mamba 2.8B | 2.8B | $6 \times 2.8\text{B}$ = 16.8B | $2 \times 2.8\text{B}$ = 5.6B | 0 (SSM 状态 $O(d)$) | ~3 GB | ~120 |
| Jamba 52B (MoE, 12B active) | 256B total | $6 \times 12\text{B}$ = 72B | $2 \times 12\text{B}$ = 24B | MQA 压缩后 ~1.5GB | ~18 GB | ~35 |
| DeepSeek-V3 (MoE, 37B active) | 671B total | $6 \times 37\text{B}$ = 222B | $2 \times 37\text{B}$ = 74B | MLA 压缩后 ~70KB/token | ~30 GB (128K) | ~20 (H800, EP64) |
| RecurrentGemma 9B | 9B | $6 \times 9\text{B}$ = 54B | $2 \times 9\text{B}$ = 18B | 局部 Attention 仅 ~200MB | ~8 GB | ~90 |
| BitNet 3B (等效 3B FP16) | 3B (1.58-bit) | 与 FP16 基线相近 (量化感知训练) | $2 \times 3\text{B}$ = 6B (但位宽降低) | 与基架构相同 | ~5 GB | ~80 (CPU 优势更大) |
| Zamba 7B | 7B | $6 \times 7\text{B}$ = 42B | $2 \times 7\text{B}$ = 14B | Mamba 层无 KV,Attention 层有 | ~9 GB | ~65 |

关键观察：

- **KV Cache 是长上下文推理的显存杀手**. Llama 3 8B 在 32K 上下文下的 KV Cache 占峰值显存的 33%. Mamba 与 Griffin 将序列状态压缩至 $O(d)$ 的固定向量,彻底消除该瓶颈. 
- **Jamba 的 MoE 设计使激活参数量与总参数量解耦**. 256B 总参数仅激活 12B,训练与推理成本按激活量计算. 但路由机制引入额外开销,实际吞吐量略低于同激活参数的稠密模型. 
- **DeepSeek-V3 将 MoE 的稀疏优势推向极致**. 671B 总参数仅激活 37B(激活率 5.5%),训练成本降至 250 GFLOPS/token,远低于同能力稠密模型(Qwen-72B 需 394 GFLOPS,LLaMA-405B 需 2448 GFLOPS). 配合 MLA 将 KV Cache 压缩至 70KB/token,128K 上下文下的峰值显存约 30 GB. KTransformers 推理引擎进一步使其可在约 $10,000 的消费级服务器上运行,达到 ~20 TPS,验证了 MoE 架构在本地化部署中的可行性. 
- **BitNet 的 FLOPs 优势不如显存带宽优势显著**. 三值矩阵乘法在 GPU Tensor Core 上无法充分利用 WMMA 指令(需 INT8 及以上),其收益主要体现在 CPU/边缘芯片的标量运算上. BitNet 在 ARM CPU 上的延迟降低幅度(2-4x)远大于在 A100 上的幅度(约 1.3x). 
- **Griffin 的局部 Attention 窗口大小直接影响显存**. 当窗口 $w = 512$ 时,KV Cache 降至全局 Attention 的 $w / n = 1/64$,几乎可以忽略. 

训练阶段的成本对比呈现不同格局：

| 架构 | 每 step 时间 (相对值) | 最大稳定 Batch Size | 长序列 (128K) 可行性 |
|:-----|:--------------------|:-------------------|:-------------------|
| Transformer 8B | 1.0x | 4 (per A100 80GB) | 需 TP/PP 切分,8卡起步 |
| Mamba 2.8B | 0.6x | 16 | 单卡可训 |
| Jamba 52B (MoE) | 1.2x | 2 (激活 12B,但路由通信开销) | 4 卡 EP 并行 |
| Griffin 9B | 0.9x | 8 | 单卡可训 |
| xLSTM 7B | 1.1x | 4 | 单卡可训 |

Mamba 与 Griffin 的训练加速源于 $O(n)$ 序列复杂度,在长序列上 batch size 可扩大 4 倍,抵消了部分参数效率劣势. Jamba 的 MoE 引入 All-to-All 通信,在专家并行(EP)配置不当成为训练瓶颈. 

## 6. 极简实现：混合层的伪代码映射

以下代码展示一个简化版 Griffin-Jamba 混合层的 PyTorch 实现,融合 Gated Linear RNN 与局部滑动窗口 Attention：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class HybridLayer(nn.Module):
    """
    Griffin-Jamba 混合层：Gated Linear RNN + 局部滑动窗口 Attention
    交替使用不同层类型可实现 Jamba 风格的架构
    """
    def __init__(self, d_model=1024, d_state=1024, window_size=512):
        super().__init__()
        self.d_model = d_model
        self.d_state = d_state
        self.window_size = window_size

        # Griffin GLRU 参数
        self.W_x = nn.Linear(d_model, d_state)
        self.W_r = nn.Linear(d_model, d_state)
        self.W_out = nn.Linear(d_state, d_model)

        # 局部 Attention 参数 (GQA 风格,单头 KV)
        self.n_heads = 8
        self.d_head = d_model // self.n_heads
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, self.d_head)   # MQA: 单头 K
        self.W_v = nn.Linear(d_model, self.d_head)   # MQA: 单头 V
        self.W_o = nn.Linear(d_model, d_model)

        self.norm1 = nn.RMSNorm(d_model)
        self.norm2 = nn.RMSNorm(d_model)

    def griffin_scan(self, x):
        """
        线性门控递推,训练时用前缀积并行,推理时用串行递推
        x: [batch, seq, d_model]
        return: [batch, seq, d_model]
        """
        B, T, _ = x.shape
        # 投影到状态空间
        z = self.W_x(x)           # [B, T, d_state]
        r = torch.sigmoid(self.W_r(x))  # [B, T, d_state], reset gate

        # 训练时：展开为加权和 (前缀积形式)
        # h_t = sum_{i=1..t} (prod_{j=i+1..t} r_j) * (1 - r_i) * z_i
        log_r = torch.log(r + 1e-6)
        cum_log_r = torch.cumsum(log_r, dim=1)  # [B, T, d_state]

        # 构造累积权重: log_weight[i,t] = sum_{j=i+1..t} log(r_j)
        cum_log_r_i = F.pad(cum_log_r, (0, 0, 1, 0))[:, :-1, :]  # [B, T, d_state]
        log_weights = cum_log_r.unsqueeze(1) - cum_log_r_i.unsqueeze(2)  # [B, T, T, d_state]
        weights = torch.exp(log_weights) * (1 - r).unsqueeze(1)   # [B, T, T, d_state]

        # 下三角掩码保证因果性
        mask = torch.tril(torch.ones(T, T, device=x.device)).unsqueeze(-1)
        weights = weights * mask

        # 加权和: [B, T, d_state]
        h = torch.einsum('btsd,bsd->btd', weights, z)
        return self.W_out(h)

    def local_attention(self, x):
        """
        局部滑动窗口 Attention,仅计算窗口内的 QK^T
        x: [batch, seq, d_model]
        """
        B, T, _ = x.shape
        q = self.W_q(x).view(B, T, self.n_heads, self.d_head).transpose(1, 2)
        k = self.W_k(x).unsqueeze(1).expand(-1, self.n_heads, -1, -1)
        v = self.W_v(x).unsqueeze(1).expand(-1, self.n_heads, -1, -1)

        # 滑动窗口掩码: 每个位置只看前 window_size 个位置
        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(self.d_head)
        window_mask = torch.full((T, T), float('-inf'), device=x.device)
        for i in range(T):
            start = max(0, i - self.window_size + 1)
            window_mask[i, start:i+1] = 0
        scores = scores + window_mask.unsqueeze(0).unsqueeze(0)

        attn = torch.softmax(scores, dim=-1)
        out = torch.matmul(attn, v).transpose(1, 2).contiguous().view(B, T, -1)
        return self.W_o(out)

    def forward(self, x):
        # Griffin GLRU 分支 (Pre-LN)
        residual = x
        x_norm = self.norm1(x)
        glru_out = self.griffin_scan(x_norm)
        x = residual + glru_out

        # 局部 Attention 分支 (Pre-LN)
        residual = x
        x_norm = self.norm2(x)
        attn_out = self.local_attention(x_norm)
        x = residual + attn_out

        return x
```

代码注释映射至数学公式：

- `r = torch.sigmoid(self.W_r(x))` 对应复位门 $\mathbf{r}_t = \sigma(\mathbf{W}_r\mathbf{x}_t)$. 
- `cum_log_r` 与 `log_weights` 对应前缀积展开式 $\prod_{j=i+1}^{t} \mathbf{r}_j$ 的对数域稳定计算. 
- `window_mask` 对应局部 Attention 的物理约束：每个 query 仅 attending 至前 `window_size` 个位置. 
- 双分支残差结构对应 Griffin 的 $\mathbf{y} = \text{GLRU}(\mathbf{x}) + \text{LocalAttn}(\mathbf{x})$. 

若将该层扩展为 Jamba 风格,只需在 `forward` 中加入路由判断：当 `layer_idx % 9 < 8` 时执行 Mamba 扫描,否则执行全局 MQA Attention. 

## 7. 边界条件：混合架构的兼容性陷阱与训练灾难

混合架构的收益并非免费,其组合复杂性引入了纯架构中不存在的失效模式. 

| 失效场景 | 物理根源 | 典型症状 | 缓解策略 |
|:--------|:--------|:--------|:--------|
| 层间分布漂移 | Mamba 层与 Attention 层的输出分布差异大,残差流不匹配 | Loss 在混合点剧烈震荡,深层梯度消失 | 引入 Layer-specific RMSNorm 缩放因子,逐层预热 |
| Mamba 状态爆炸 | 选择性 SSM 的 $\Delta$ 参数学习不当,导致 $\mathbf{\bar{A}}$ 特征值大于 1 | 长序列末端 token 的隐状态数值爆炸 | 对 $\Delta$ 施加 clipping,初始化 $\mathbf{A}$ 为负值保证衰减 |
| Griffin 递推衰减至零 | 复位门 $\mathbf{r}_t$ 趋于 1,隐状态不再接收新输入 | 模型对长距离信息丧失敏感度 | 对 $\mathbf{r}_t$ 施加最大衰减约束(如 $\mathbf{r}_t < 0.99$) |
| BitNet 量化崩溃 | 三值权重导致激活值分布严重偏移 | 训练 Loss 发散,评估指标归零 | 量化感知训练(QAT),保留 FP16 主干进行热启动 |
| MoE 路由负载失衡 | Jamba 的 MoE 层中 token 集中于少数专家 | 部分专家过拟合,其余专家零激活 | 辅助负载均衡损失 $\mathcal{L}_{\text{aux}} = \alpha \sum_{i} f_i \cdot P_i$ |
| Attention-SSM 注意力竞争 | 同一序列中 Attention 层关注短程,Mamba 层关注长程,二者信号冲突 | 下游任务性能低于单一架构基线 | 交替比例调优,引入门控融合机制替代硬切换 |

**层间分布漂移**是混合架构最隐蔽的陷阱. 纯 Transformer 的各层输出分布通过 Pre-LN 被严格锚定,但 Mamba 层的 SSM 扫描引入与时间步相关的幅度缩放. 当 Attention 层紧接着 Mamba 层时,输入到 Attention 的隐状态可能偏离训练分布,导致 QK^T 的数值范围异常,softmax 进入饱和区. Jamba 的解决方案是在每个混合过渡点插入额外的 RMSNorm 层,并对前 10% 的训练步数使用逐层学习率预热. 

**Mamba 状态爆炸**的数学根源在于离散化公式 $\mathbf{\bar{A}} = \exp(\Delta \mathbf{A})$. 若参数学习使 $\Delta$ 过大或 $\mathbf{A}$ 的实部为正,状态转移矩阵的特征值将超出单位圆. Mamba 原始实现通过对 $\mathbf{A}$ 使用负值初始化($\mathbf{A} = -\text{range}(1, d)$)缓解该问题,但在混合架构中,Attention 层的反向传播信号可能干扰 $\Delta$ 参数的稳定学习. 

**BitNet 量化崩溃**在混合架构中风险更高. 当 BitNet 层与全精度层交替存在时,量化误差的累积效应会被后续全精度层放大. Microsoft 的实验表明,BitNet 模型必须在训练初始阶段以 FP16 热启动 1000 步,再逐步切换至三值量化,否则 Loss 曲线呈现不可逆发散. 

**MoE 路由负载失衡**在 Jamba 中尤为关键. 由于 Mamba 层负责长程建模,MoE 层中的路由器可能将所有语义相关的 token 导向同一专家,导致专家专业化过度. DeepSeek-MoE 的共享专家设计与负载均衡损失在 Jamba 中需要重新调参,因为 Mamba 层的序列扫描改变了 token 的语义聚类特性. 

### 7.4 DeepSeek-V3.2：动态稀疏注意力(DSA)

前述 MLA 将优化焦点放在 KV Cache 的显存压缩上,而 DeepSeek-V3.2 引入的动态稀疏注意力(Dynamic Sparse Attention, DSA)则将优化方向拉回 attention 计算量本身. 其核心思想是：用一个轻量级的 lightning indexer 对历史 token 快速打分,再通过 top-k selector 只保留最重要的若干位置,让主 attention 仅在这些精选位置上执行全精度计算. 

Lightning indexer 为每个 query 位置维护一组独立的 indexer head,维度远低于主 attention head. 设 $h^I$ 为 indexer head 数,$d^I$ 为 indexer head 维度,第 $t$ 个位置对第 $i$ 个历史位置的 index score 为各 head 的加权和：

$$S^I_{t,i} = \sum_{j=1}^{h^I} \omega_{t,j}^I \cdot \text{ReLU}\left(q^I_{t,j} (k_i^I)^\top\right) \tag{15}$$

其中 $q^I_{t,j}$ 和 $k_i^I$ 分别为第 $j$ 个 indexer head 的 query 和 key,$\omega_{t,j}^I$ 为可学习的 head 加权系数. ReLU 激活将负相关分数截断为零,确保只有正向相关的 token 进入后续排序. 这一步的计算开销远低于主 attention：indexer head 的维度 $d^I \ll d_h$,且只需计算点积和 ReLU,无需 softmax 归一化. 

得到 index score 后,top-k selector 为每个 query 选出分数最高的 $k$ 个历史位置,构成选中集合 $\mathcal{S}_t$. 主 attention 仅在 $\mathcal{S}_t$ 上执行标准 softmax：

$$\alpha_{t,i} = \begin{cases} \displaystyle \frac{\exp\left(\frac{q_t k_i^\top}{\sqrt{d_h}}\right)}{\sum_{j \in \mathcal{S}_t} \exp\left(\frac{q_t k_j^\top}{\sqrt{d_h}}\right)}, & i \in \mathcal{S}_t \\ 0, & i \notin \mathcal{S}_t \end{cases} \tag{16}$$

与 MoBA 的块级路由不同,DSA 的稀疏粒度是单个 token 级别,精度更高; 与 NSA 的三路融合不同,DSA 只保留一条"精选全量"路径,结构更简单. 在 DeepSeek-V3.2 的实现中,DSA 直接实例化在 MLA 之上——indexer 的 query/key 从 MLA 的 latent 向量映射而来,无需额外的独立投影矩阵,参数增量极小. 

DSA 的物理意义在于将 attention 的计算瓶颈从"对所有历史 token 计算 softmax"转移为"对少量精选 token 计算 softmax". 当序列长度达到百万级时,即使 MLA 已将 KV Cache 压缩至极低,主 attention 的 $O(N^2)$ 计算量仍是瓶颈. DSA 通过索引器将这一复杂度降至 $O(N \cdot k)$,其中 $k$ 通常取数千量级,与序列长度无关. 

> 参考来源：[从 MHA、GQA 到 MLA、DSA、HCA/CSA：DeepSeek 文本大模型 Attention 架构演进](https://zhuanlan.zhihu.com/p/2033298038605751155)

## 7.5 工业前沿：DeepSeek-V4 的流形约束超连接与混合压缩注意力

DeepSeek 系列的注意力架构演进,是一场围绕计算量与显存占用的持续博弈. 按时间线梳理：

| 模型 | 时间 | Attention 架构 | 核心创新 | 解决矛盾 |
|:-----|:-----|:--------------|:--------|:--------|
| DeepSeek LLM 7B | 2023.11 | 标准 MHA | 每个 query head 独立 key/value head | 基础能力验证 |
| DeepSeek LLM 67B | 2024 | GQA | 多个 query head 共享 KV head | KV Cache 显存爆炸 |
| DeepSeek-V2/V3/R1 | 2024 | MLA | 缓存压缩后的 latent KV,矩阵吸收恢复 | 长上下文 KV 容量 |
| DeepSeek-V3.2 | 2025 | MLA + DSA | Lightning Indexer 筛选 Top-k 重要 token | 长上下文计算量 |
| DeepSeek-V4 | 2025 | mHC + CSA + HCA | 流形约束残差 + 分层压缩注意力 | 百万 token 上下文效率 |

从 MHA 到 GQA 是"减少 KV head 数量"的显存优化; 从 GQA 到 MLA 是"降维压缩 KV 表示"的结构性压缩; 从 MLA 到 DSA 是"稀疏选择重要 token"的计算优化; 从 DSA 到 CSA/HCA 是"先压缩再稀疏/全量"的分层策略. 每一步都不是对前一代的否定,而是在特定上下文长度和硬件约束下的最优解. 

DeepSeek-V4(2025)代表了这一演进路线的最新范式,其设计并非简单地在层级别交替不同算子,而是在**残差连接机制**、**注意力内核**与**KV 缓存管理**三个层面同时引入结构性创新. 以下从 mHC(流形约束超连接)、CSA(压缩稀疏注意力)与 HCA(重度压缩注意力)三个模块展开分析. 

### 7.5.1 流形约束超连接(mHC)

传统 Transformer 的残差连接为 $\mathbf{x}_{l+1} = \mathbf{x}_l + \mathcal{F}_l(\mathbf{x}_l)$,其中 $\mathcal{F}_l$ 是第 $l$ 层的变换(如 Attention 或 FFN). 当模型深度超过 100 层时,这种标量残差流的信号传播稳定性面临挑战. Hyper-Connections(HC)将残差流的宽度扩展 $n_{\text{hc}}$ 倍,引入三个线性映射：

$$
\mathbf{X}_{l+1} = \mathbf{B}^l \mathbf{X}_l + \mathbf{C}^l \mathcal{F}_l(\mathbf{A}^l \mathbf{X}_l) \tag{17}
$$
式 (15) 中,$\mathbf{X}_l \in \mathbb{R}^{n_{\text{hc}} \times d}$ 表示第 $l$ 层前的残差状态,其宽度从原始的 $d$ 维扩展为 $n_{\text{hc}} \times d$ 维. $\mathbf{A}^l$ 和 $\mathbf{C}^l$ 分别负责将扩展后的残差流投影到层变换的输入空间和从层变换输出空间映射回来,二者维度均为 $\mathbb{R}^{n_{\text{hc}} \times d}$. $\mathbf{B}^l \in \mathbb{R}^{n_{\text{hc}} \times n_{\text{hc}}}$ 是残差映射矩阵,实现跨通道的残差混合. 层变换 $\mathcal{F}_l$ 本身保持 $\mathbb{R}^d$ 的输入输出维度不变,因此实际进入 $\mathcal{F}_l$ 的输入是 $\mathbf{A}^l \mathbf{X}_l \in \mathbb{R}^d$,这意味着残差流的扩展不影响内部层的设计,却能在通道维度上实现更丰富的信息流动. 
标准 HC 的瓶颈在于深层堆叠时的数值不稳定. DeepSeek-V4 的核心创新是将 $\mathbf{B}^l$ 约束在**双随机矩阵流形**(Birkhoff 多面体)$\mathcal{M}$ 上：

$$
\mathbf{B}^l \in \mathcal{M} := \left\{ \mathbf{M} \in \mathbb{R}^{n \times n} \mid \mathbf{M}\mathbf{1}_n = \mathbf{1}_n,\ \mathbf{1}_n^T \mathbf{M} = \mathbf{1}_n^T,\ \mathbf{M} \geq 0 \right\} \tag{18}
$$
此式将上述直觉形式化,各项分别对应输入变换、非线性激活与输出生成. 
式 (16) 中,$\mathbf{1}_n$ 是 $n$ 维全 1 向量,$\mathcal{M}$ 定义为所有行和与列和均归一化为 1 且元素非负的双随机矩阵集合. 将 $\mathbf{B}^l$ 约束在该流形上带来两个关键性质：其一,双随机矩阵的谱范数天然有界,满足 $\|\mathbf{B}^l\|_2 \leq 1$,这使得残差变换成为非扩张映射,无论前向传播还是反向传播都更加稳定; 其二,双随机矩阵集合在矩阵乘法运算下保持封闭,因此当数十个 mHC 层连续堆叠时,整体状态转移矩阵仍不会偏离稳定区域. 
mHC 的三个映射参数由**动态分量**(输入依赖)和**静态分量**(输入无关)组合生成. 给定输入 $\mathbf{X}_l$,先展平并归一化：

$$
\hat{\mathbf{X}}_l = \text{RMSNorm}(\text{vec}(\mathbf{X}_l)) \in \mathbb{R}^{1 \times n_{\text{hc}} d} \tag{19}
$$
该操作将扩展后的残差张量展平为向量并施加 RMSNorm,得到固定尺度的归一化表示 $\hat{\mathbf{X}}_l$,为后续动态映射参数的生成提供稳定且与输入幅度无关的条件. 

然后生成未约束的原始参数：

$$
\tilde{\mathbf{A}}^l = \alpha_l^{\text{pre}} \cdot (\hat{\mathbf{X}}_l \mathbf{W}_l^{\text{pre}}) + \mathbf{S}_l^{\text{pre}} \tag{20}
$$
该关系式明确了系统的数学约束,可直接用于后续优化推导. 

推导过程如下：
$$
\tilde{\mathbf{B}}^l = \alpha_l^{\text{res}} \cdot \text{Mat}(\hat{\mathbf{X}}_l \mathbf{W}_l^{\text{res}}) + \mathbf{S}_l^{\text{res}} \tag{21}
$$
此式给出了形式化的数学定义,建立了输入与输出之间的定量关系. 

推导过程如下：
$$
\tilde{\mathbf{C}}^l = \alpha_l^{\text{post}} \cdot (\hat{\mathbf{X}}_l \mathbf{W}_l^{\text{post}}) + \mathbf{S}_l^{\text{post}} \tag{22}
$$
式 (17)-(19) 的设计动机是让残差连接具备输入自适应能力. $\mathbf{W}_l^{\text{pre}}$ 和 $\mathbf{W}_l^{\text{post}}$ 是 $\mathbb{R}^{n_{\text{hc}} d \times n_{\text{hc}}}$ 维的动态投影矩阵,$\mathbf{W}_l^{\text{res}}$ 则是 $\mathbb{R}^{n_{\text{hc}} d \times n_{\text{hc}}^2}$ 维的动态投影,用于生成残差映射. $\text{Mat}(\cdot)$ 操作将 $1 \times n_{\text{hc}}^2$ 的向量重塑为 $n_{\text{hc}} \times n_{\text{hc}}$ 的矩阵,以匹配 $\tilde{\mathbf{B}}^l$ 的维度. $\mathbf{S}_l^{\text{pre}}, \mathbf{S}_l^{\text{post}}, \mathbf{S}_l^{\text{res}}$ 为静态偏置项,在训练初期提供稳定的基线行为. 门控因子 $\alpha_l^{\text{pre}}, \alpha_l^{\text{res}}, \alpha_l^{\text{post}}$ 初始化为接近零的小值,使得模型早期主要依赖静态偏置(近似标准残差连接),随着训练深入,动态分量逐步激活,残差映射根据输入特征自适应调整. 
对未约束参数施加稳定性约束：

$$
\mathbf{A}^l = \sigma(\tilde{\mathbf{A}}^l), \quad \mathbf{C}^l = 2\sigma(\tilde{\mathbf{C}}^l) \tag{23}
$$
此式描述了变量之间的定量关系,其中每一项对应特定的物理操作. 
残差映射通过 Sinkhorn-Knopp 算法投影到 $\mathcal{M}$：

$$
\mathbf{M}^{(0)} = \exp(\tilde{\mathbf{B}}^l), \quad \mathbf{M}^{(t)} = \mathcal{T}_r(\mathcal{T}_c(\mathbf{M}^{(t-1)})) \tag{24}
$$
式 (20) 中 $\sigma(\cdot)$ 为 Sigmoid 函数,通过对 $\tilde{\mathbf{A}}^l$ 和 $\tilde{\mathbf{C}}^l$ 施加 Sigmoid 变换并配合缩放因子 2,确保输入/输出映射的输出严格落在非负有界区间,避免正负信号相互抵消. 式 (21) 是 Sinkhorn-Knopp 迭代的具体形式：先对原始矩阵取指数得到正值初值 $\mathbf{M}^{(0)}$,再通过 $\mathcal{T}_r$ 和 $\mathcal{T}_c$ 分别执行行归一化与列归一化的交替迭代,使矩阵收敛到双随机矩阵流形 $\mathcal{M}$. 实际实现中通常取 $t_{\max} = 20$ 步即可达到足够精度. 整个 mHC 机制相对于标准残差连接的端到端训练开销被控制在 6.7% 以内. 
### 7.5.2 压缩稀疏注意力(CSA)

当上下文长度达到百万 token 级别时,即使稀疏注意力(如 DeepSeek 的 NSA)的 KV Cache 仍过于庞大. CSA 的核心思想是**先压缩再稀疏**：每 $m$ 个原始 token 的 KV 条目压缩为 1 个条目,再对压缩后的条目应用 Top-k 稀疏选择. 

设输入隐状态 $\mathbf{H} \in \mathbb{R}^{n \times d}$,CSA 计算两组 KV 条目及其压缩权重：

$$
\mathbf{C}^a = \mathbf{H} \mathbf{W}^{aKV}, \quad \mathbf{C}^b = \mathbf{H} \mathbf{W}^{bKV} \tag{25}
$$
此关系式描述了变量之间的定量映射,为后续优化提供了数学基础. 

为了量化这一效应,定义：
$$
\mathbf{Z}^a = \mathbf{H} \mathbf{W}^{aZ}, \quad \mathbf{Z}^b = \mathbf{H} \mathbf{W}^{bZ} \tag{26}
$$
式 (22)-(23) 中,$\mathbf{W}^{aKV}, \mathbf{W}^{bKV}, \mathbf{W}^{aZ}, \mathbf{W}^{bZ}$ 均为 $\mathbb{R}^{d \times c}$ 维的可学习投影矩阵,$c$ 为注意力 head 维度. $\mathbf{C}^a$ 和 $\mathbf{C}^b$ 是两组候选 KV 表示,$\mathbf{Z}^a$ 和 $\mathbf{Z}^b$ 则是对应每组 KV 的压缩分数. 引入两组而非一组 KV 的核心工程考量是实现**重叠压缩**：相邻压缩块共享部分 KV 条目,从而有效缓解块边界处的信息截断问题,提升长文档的连贯性. 
每 $m$ 个 KV 条目通过加权 Softmax 压缩为一个条目：

$$
[\mathbf{S}^a_{mi:m(i+1)-1};\ \mathbf{S}^b_{m(i-1):mi-1}] = \text{Softmax}_{\text{row}}([\mathbf{Z}^a_{mi:m(i+1)-1} + \mathbf{B}^a;\ \mathbf{Z}^b_{m(i-1):mi-1} + \mathbf{B}^b]) \tag{27}
$$
上述表达式将物理直觉转化为精确数学语言,为后续算法实现与性能分析提供了理论基础. 

基于上述分析,建立如下数学关系：
$$
\mathbf{C}^{\text{Comp}}_i = \sum_{j=mi}^{m(i+1)-1} \mathbf{S}^a_j \odot \mathbf{C}^a_j + \sum_{j=m(i-1)}^{mi-1} \mathbf{S}^b_j \odot \mathbf{C}^b_j \tag{28}
$$
式 (24) 中,$\mathbf{B}^a, \mathbf{B}^b \in \mathbb{R}^{m \times c}$ 是可学习的位置偏置,用于为压缩块内的不同位置引入可区分的先验. Softmax 在行维度上跨 $2m$ 个元素进行归一化,同时覆盖当前块的 $m$ 个条目和前一相邻块的 $m$ 个条目,产生 $2m$ 个压缩权重. 式 (25) 利用这些权重对两组 KV 做 Hadamard 积(逐元素乘)后的加权求和. 重叠设计使每个压缩条目实际融合了 $2m$ 个原始 KV 的信息,却将序列长度压缩为原来的 $1/m$,在信息保留与存储效率之间取得了平衡. 
**Lightning Indexer** 用于在压缩条目中稀疏选择 Top-k. 对查询 token $t$,以低秩方式生成索引器查询：

$$
\mathbf{c}^Q_t = \mathbf{h}_t \mathbf{W}^{DQ}, \quad \mathbf{q}^I_t = \mathbf{c}^Q_t \mathbf{W}^{IUQ} \tag{29}
$$
上述表达式明确了系统的数学约束,其中每一项对应特定的物理操作. 
此式给出了精确的数学定义,为算法实现提供了理论基础. 

为建立定量关系,定义如下表达式：
$$
\mathbf{w}^I_t = \mathbf{h}_t \mathbf{W}_w, \quad I_{t,s} = \sum_{h=1}^{n^I_h} \omega^{I,h}_t \cdot \text{ReLU}\left(\mathbf{q}^{I,h}_t \cdot \mathbf{K}^{I\text{Comp}}_s\right) \tag{30}
$$
该关系式明确了系统的数学约束,可直接用于后续优化推导. 
式 (26)-(27) 的核心工程优化在于低秩索引设计. $\mathbf{h}_t \in \mathbb{R}^d$ 是查询 token 的隐状态,先通过下投影矩阵 $\mathbf{W}^{DQ} \in \mathbb{R}^{d \times d_c}$ 压缩为潜在向量 $\mathbf{c}^Q_t$,再由上投影矩阵 $\mathbf{W}^{IUQ} \in \mathbb{R}^{d_c \times c_I n^I_h}$ 生成索引器查询 $\mathbf{q}^I_t$. 索引器头权重矩阵 $\mathbf{W}_w \in \mathbb{R}^{d \times n^I_h}$ 负责生成多头组合系数 $\omega^{I,h}_t$. 索引器查询和主注意力查询共享同一个压缩潜在向量 $\mathbf{c}^Q_t$,避免了重复计算. 索引分数 $I_{t,s}$ 通过多头的加权和 ReLU 激活计算,只保留正相关条目,有效抑制噪声干扰,提升稀疏选择的精确度. $\mathbf{K}^{I\text{Comp}}$ 为压缩后的索引器键,与查询进行点积匹配以确定每个压缩条目的相关程度. 
Top-k 选择后,对选中的压缩条目执行 **Shared KV MQA**(共享 KV 的多查询注意力)：

$$
\mathbf{q}_t = \mathbf{c}^Q_t \mathbf{W}^{UQ}, \quad \mathbf{o}_{t,i} = \text{CoreAttn}(\mathbf{q}_{t,i},\ \mathbf{C}^{\text{SprsComp}}_t,\ \mathbf{C}^{\text{SprsComp}}_t) \tag{31}
$$
式 (28) 中,$\mathbf{W}^{UQ} \in \mathbb{R}^{d_c \times c n_h}$ 将共享的压缩潜在向量 $\mathbf{c}^Q_t$ 上投影为完整注意力查询 $\mathbf{q}_t$. $\mathbf{C}^{\text{SprsComp}}_t$ 表示针对查询 $t$ 经 Lightning Indexer 筛选后的 Top-k 压缩条目集合. 每个压缩条目同时充当 key 和 value(共享 KV),这进一步压缩了 KV Cache 的存储需求. 由于 $c \cdot n_h$ 的乘积通常很大,输出投影采用分组策略：将 $n_h$ 个输出分成 $g$ 组,每组先投影到较低维的中间表示,再拼接并投影到最终维度 $d$,以降低计算开销. 
### 7.5.3 重度压缩注意力(HCA)

HCA 以更高压缩率 $m'$($m' \gg m$)压缩 KV 缓存,但不使用稀疏选择——对所有压缩条目做全量注意力. 这种"全压缩 + 全注意力"的设计适用于对全局上下文有强依赖的层. 

HCA 的压缩过程比 CSA 更简单,不采用重叠压缩：

$$
\mathbf{C} = \mathbf{H} \mathbf{W}^{KV}, \quad \mathbf{Z} = \mathbf{H} \mathbf{W}^Z \tag{32}
$$
此式描述了变量之间的定量关系,其中每一项对应特定的物理操作. 

推导过程如下：
$$
\mathbf{S}_{m'i:m'(i+1)-1} = \text{Softmax}_{\text{row}}(\mathbf{Z}_{m'i:m'(i+1)-1} + \mathbf{B}) \tag{33}
$$
上述表达式将物理直觉转化为精确数学语言,为后续算法实现与性能分析提供了理论基础. 

基于上述分析,建立如下数学关系：
$$
\mathbf{C}^{\text{Comp}}_i = \sum_{j=m'i}^{m'(i+1)-1} \mathbf{S}_j \odot \mathbf{C}_j \tag{34}
$$
式 (29)-(31) 使用单一组 KV 表示,$\mathbf{W}^{KV}$ 和 $\mathbf{W}^Z$ 均为 $\mathbb{R}^{d \times c}$ 维的投影矩阵,分别生成候选 KV 值 $\mathbf{C}$ 和压缩分数 $\mathbf{Z}$. $\mathbf{B} \in \mathbb{R}^{m\' \times c}$ 是可学习的位置偏置,用于区分块内不同位置. Softmax 仅在当前块内的 $m\'$ 个元素上进行归一化,不涉及相邻块的重叠. 经过压缩后,序列长度从 $n$ 缩减为 $n/m\'$,注意力计算量相应降至 $O(n \cdot n/m\') = O(n^2/m\')$. 这种无重叠设计虽然牺牲了边界信息冗余,但获得了更简洁的计算路径和更高的压缩率,适合对全局上下文进行粗粒度聚合. 
对查询 token $t$ 执行全量 MQA：

$$
\mathbf{c}^Q_t = \mathbf{h}_t \mathbf{W}^{DQ}, \quad \mathbf{q}_t = \mathbf{c}^Q_t \mathbf{W}^{UQ} \tag{35}
$$
此式将上述直觉形式化,各项分别对应输入变换、非线性激活与输出生成. 

基于前述物理直觉,给出数学形式：
$$
\mathbf{o}_{t,i} = \text{CoreAttn}(\mathbf{q}_{t,i},\ \mathbf{C}^{\text{Comp}},\ \mathbf{C}^{\text{Comp}}) \tag{36}
$$
此式将上述约束形式化,各项分别对应输入变换、非线性激活与输出生成. 
此式给出了精确的数学定义,为算法实现提供了理论基础. 
式 (32)-(33) 的计算形式与 CSA 的注意力机制一致：查询 token 的隐状态 $\mathbf{h}_t$ 先经 $\mathbf{W}^{DQ}$ 下投影为压缩潜在向量 $\mathbf{c}^Q_t$,再由 $\mathbf{W}^{UQ}$ 上投影为完整查询 $\mathbf{q}_t$. 关键差异在于 attention 的 key 和 value 是**全部压缩条目** $\mathbf{C}^{\text{Comp}}$,而非 CSA 中的 Top-k 子集 $\mathbf{C}^{\text{SprsComp}}_t$. 这意味着 HCA 的 KV Cache 比 CSA 更小(因为 $m\' > m$),但注意力计算量更大(失去了稀疏选择的加速). CSA 与 HCA 在模型中**交替配置**：短程依赖层使用 CSA,通过稀疏选择保证局部精度; 长程聚合层使用 HCA,通过全量注意力保证全局召回. 
### 7.5.4 CSA/HCA 的工程实现要点

**滑动窗口 KV 与压缩 KV 的拼接**. CSA 和 HCA 都额外保留一小组最近 $n_{\text{win}}$ 个 token 的未压缩 KV 条目,与压缩/稀疏选择的 KV 拼接后一起做注意力. 这保证了对刚生成的 token 的精确局部依赖建模——压缩过程会损失细粒度位置信息,滑动窗口补偿了这一损失. 

**Query 和 KV 的额外 RMSNorm**. 在核心注意力操作之前,对每个查询头和压缩 KV 头执行额外的 RMSNorm,避免注意力 logits 爆炸. 这是 DeepSeek-V4 训练稳定性工程的关键细节：压缩后的 KV 值范围可能与未压缩 KV 不同,直接混合会导致 softmax 温度失控. 

**上下文并行(Contextual Parallelism)的压缩条目对齐**. 在序列并行场景下,压缩块可能跨越 rank 边界. DeepSeek-V4 采用两阶段通信：先交换边界 KV(每个 rank 向下一 rank 发送最后 $m$ 个未压缩 KV),再 All-Gather 压缩后的条目并通过 select-and-pad 重组. 这一机制保证因果性约束不被破坏——query token $t$ 只能看到满足 $s < \lfloor t/m \rfloor$ 的压缩块 $C_s$. 

| 组件 | 压缩率 | 稀疏选择 | 适用场景 | KV Cache 相对大小 |
|:-----|:------|:--------|:--------|:----------------|
| 标准 MHA | $1\times$ | 无 | 短序列($<32$K) | $100\%$ |
| CSA | $1/m$ | Top-k | 中长序列($32$K-$1$M) | $\approx 5\%$-$10\%$ |
| HCA | $1/m'$ | 无 | 全局上下文聚合层 | $\approx 2\%$-$5\%$ |
| SWA | 窗口截断 | 无 | 局部精确依赖 | $\approx 1\%$-$2\%$ |

**CSA/HCA 的失效模式**：

| 失效场景 | 物理根源 | 典型症状 | 缓解策略 |
|:--------|:--------|:--------|:--------|
| 压缩块边界信息丢失 | 重叠压缩窗口不足,块边缘 token 的依赖被截断 | 长文档的段落衔接处生成质量下降 | 增大重叠窗口(从 $m$ 扩展到 $2m$),保留更多边界 KV |
| Lightning Indexer 召回不足 | Top-k 选择遗漏关键压缩块 | 事实检索任务准确率下降 | 增大 $k$ 或使用多轮索引(coarse-to-fine) |
| HCA 全量注意力瓶颈 | 极长序列下 $n/m'$ 仍过大 | 推理延迟在 1M+ 上下文时激增 | 对 HCA 层引入分层压缩(二级压缩) |
| SWA 与压缩 KV 的数值不匹配 | 未压缩和压缩 KV 的分布差异 | 训练后期 Loss 小幅震荡 | 额外的层归一化 + 温度缩放 |

### 7.5.5 mHC + CSA/HCA 的协同设计哲学

DeepSeek-V4 的架构设计体现了一条工业级混合架构的新路径：**不在层级别混合不同算子类型**(如 Jamba 的 Mamba-Transformer 交替),而是在**同一层内对信息通路进行结构性重组**. mHC 重构残差连接的拓扑(从标量加法到矩阵混合),CSA/HCA 重构注意力对历史信息的访问模式(从逐 token 精确匹配到分层压缩召回). 

这种"层内混合"相比"层间混合"有两个工程优势：(1) 每层内部的计算模式更统一,便于 kernel 融合和流水线优化; (2) 残差流的扩展不破坏层接口,现有训练框架(如 DeepSpeed、Megatron)无需大幅修改即可适配. DeepSeek-V4 的训练基础设施报告显示,mHC 的融合 kernel 实现将端到端开销控制在 6.7%,而 CSA/HCA 在 1M token 上下文上的推理吞吐量较标准 Attention 提升超过 10 倍. 

## 8. 技术前瞻：下一代混合架构的演进方向

当前混合架构仍处于早期探索阶段,多条技术路线并行竞争,尚未收敛至主导范式. 

**动态架构选择(Dynamic Architecture Selection)** 是首要方向. 现有混合模型(Jamba、Zamba)采用固定层交错比例,无法根据输入序列的特性自适应调整. 下一代模型可能引入可学习的"架构路由器",依据序列长度、领域特征(代码 vs. 文学)或任务类型(生成 vs. 检索)动态选择当前层使用 Attention、SSM 还是线性 RNN. 这要求训练框架支持条件计算与动态图优化,对现有深度学习基础设施构成挑战. 

**测试时计算扩展与混合架构的协同** 构成第二条主线. OpenAI 的 o3 类模型展示了测试时计算(Test-time Compute)的价值——在推理阶段投入更多算力生成更长的 CoT 序列. 混合架构为此提供天然支持：Mamba 层以 $O(1)$ 状态更新承担 CoT 的主体流转,Attention 层仅在需要精确回溯或交叉引用时激活. Google 的 Titan 架构已在此方向探索,引入测试时记忆(test-time memory)机制,允许模型在推理过程中动态读写外部记忆槽. 

**硬件-架构协同量化** 将 BitNet 的思路推向极致. NVIDIA Blackwell 架构原生支持 FP4/FP6 精度,而混合架构中的不同层对量化敏感度不同. Attention 层的 QK^T 计算对数值精度要求极高(softmax 的指数运算放大误差),SSM 层的线性递推对精度相对不敏感. 未来混合模型可能采用层自适应精度：Attention 层保持 FP16,SSM 层降至 INT4,BitNet 层使用 1.58-bit,在硬件层面实现真正的异构计算. 

**状态空间模型的多模态扩展** 是工业界关注的焦点. Mamba 的 $O(n)$ 复杂度使其在视频(长时序帧序列)、音频(高采样率波形)和基因组(百万级碱基对)领域具备天然优势. 将 Mamba 层与 Vision Transformer 的局部 patch attention 混合,已在视频理解任务上显示出优于纯 ViT 的显存效率. Jamba 的架构设计原则可平移至多模态场景：文本分支使用 Transformer-Mamba 混合,视觉分支使用 CNN-SSM 混合,模态交互层使用轻量 Attention. 

**线性 RNN 的理论补全** 是学术界的攻坚点. Griffin 与 xLSTM 在工程上证明了线性递推的可行性,但其表达能力上界尚未被严格刻画. Transformer 的表达能力可通过 universal approximation theory 与上下文学习(in-context learning)理论进行分析,而线性 RNN 在何种条件下能够模拟 Attention 的 token-to-token 交互,仍是开放问题. 近期工作(如 Linear Attention 的核方法视角)开始建立这一理论桥梁,但距离完整的表达能力等价性证明仍有距离. 

| 方向 | 当前状态 | 关键瓶颈 | 预期突破时间 |
|:-----|:--------|:--------|:-----------|
| 动态架构路由 | 实验室原型 | 动态图优化开销,训练不稳定 | 2026-2027 |
| 测试时记忆扩展 | Titan 等初步探索 | 记忆槽的读写一致性与检索精度 | 2025-2026 |
| 层自适应精度 | BitNet + 混合架构结合 | 混合精度 kernel 的硬件生态 | 2025-2026 |
| 多模态 SSM 混合 | 视频/音频初步实验 | 跨模态状态对齐 | 2026-2028 |
| 线性 RNN 理论补全 | 核方法视角初步建立 | 表达能力严格等价性证明 | 2027+ |

混合架构的终极形态可能趋近于"神经网络操作系统"——不同类型的计算单元如同 CPU、GPU、NPU 的异构调度,由高层策略根据任务负载动态编排. 这要求架构设计从"设计一个更好的层"升级为"设计一个更好的调度器".
