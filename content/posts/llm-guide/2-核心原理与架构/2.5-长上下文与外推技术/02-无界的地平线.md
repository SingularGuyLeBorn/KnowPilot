在 Transformer 的世界里,**上下文长度 (Context Window)** 等同于智能体的“工作记忆”.早期模型受限于 $L=2048$ 或 $4096$ 的长度,如同金鱼般健忘.而通往 AGI 的关键一步,就是打破这层枷锁,实现近乎无限的长度外推 (Length Extrapolation).

很多人知道要改 rope_theta,要用 YaRN,但很少有人能从数学底层解释清楚：**为什么简单的线性内插会导致高频信息丢失？为什么非均匀缩放是必然的归宿？**

本文将剥离工程实现的迷雾,配合可视化概念图,从旋转位置编码 (RoPE) 的复数表示出发,一步步推导 YaRN 的“熵理论”,LongRope 的非均匀搜索,以及 DCA 的分块注意力机制.

---

## 1. 旋转的基石：RoPE 与频率的诅咒

一切长度外推算法的根基,都是对 **RoPE (Rotary Positional Embeddings)** 的数学修正.我们必须先严格定义它,才能理解为什么它不能直接“拉长”.

### 1.1 RoPE 的复数形式

将 Embedding 向量 $x \in \mathbb{R}^d$ 视为 $d/2$ 个复数对.对于第 $j$ 个复数平面 ($j \in [0, d/2)$),在位置 $m$ 处的旋转操作定义为：

$f(x_j, m) = x_j \cdot e^{i m \theta_j}$

其中 $\theta_j$ 是该维度的旋转频率,通常由基底 $b$ (base, e.g., 10000) 决定：

$\theta_j = b^{-2j/d}$

Attention 的核心在于 Query ($q$) 和 Key ($k$) 的内积.在 RoPE 下,相对位置 $(m-n)$ 自然浮现：

$\langle f(q, m), f(k, n) \rangle = \text{Re} \left( \sum_{j=0}^{d/2-1} q_j k_j^* e^{i (m-n) \theta_j} \right)$

### 1.2 外推的困境 (OOD 问题)与线性内插的崩溃

当测试长度 $L_{test}$ 超过训练长度 $L_{train}$ 时,相对位置 $m-n$ 会超出模型在训练期间见过的范围 $[0, L_{train}]$.这导致 Attention Score 出现剧烈的振荡,即 **OOD (Out-of-Distribution)** 问题.

最朴素的解决方案是 **线性内插 (Linear Interpolation, PI)** .引入比例因子 $s = L_{test} / L_{train}$,将位置索引“压缩”：

$f'(x, m) = x \cdot e^{i \frac{m}{s} \theta_j}$

但这带来了一个致命问题：**高频信息的灾难 (High-Frequency Information Loss)** .

**数学推导**：
原本相邻 token 的旋转角度差是 $\Delta = \theta_j$.
压缩后,角度差变为 $\Delta' = \theta_j / s$.

- **低频维度** ($\theta_j$ 很小): 旋转很慢,压缩后变得更慢,模型勉强能接受.

- **高频维度** ($\theta_j$ 很大): 向量旋转极快.当除以 $s$ 后,原本清晰的相位关系变得模糊.这就好比你将一张高分辨率图片强行缩小,**高频的纹理细节(Texture)会发生混叠 (Aliasing) 和丢失**.

## 2. YaRN: 熵理论与混合频率波段

**YaRN (Yet another RoPE for Nontraversable context)** 的核心洞察在于：**不同频率的维度,应该享受不同的待遇.** 不能“一刀切”地除以 $s$.

### 2.1 NTK 感知与波长定义

我们需要定义每个维度的“波长” $\lambda_j$.波长是指 RoPE 旋转 $2\pi$ (一圈) 所需的长度：

$\lambda_j = \frac{2\pi}{\theta_j} = 2\pi b^{2j/d}$

YaRN 将维度分为三类：

1. **高频维度 (High Frequency)** : $\lambda_j \ll L_{train}$.旋转极快.这里的相对位置信息本质上是“局部”的.**绝对不能**进行线性内插,否则会破坏局部精度.应保持**外推 (Extrapolation)** ,即不缩放.

2. **低频维度 (Low Frequency)** : $\lambda_j \gg L_{train}$.旋转极慢.这里的相对位置信息是“全局”的.必须进行**线性内插 (Interpolation)** ,否则模型无法理解从未见过的长距离.

3. **中频维度**: 过渡带.

### 2.2 坡度函数 (Ramp Function)

YaRN 定义了一个平滑函数 $\gamma(r)$ 来决定“多少成分做内插,多少成分做外推”.
定义比率 $r_j = \frac{L_{train}}{\lambda_j}$.

构造线性坡度函数 $\gamma_j$：

$\gamma_j = \begin{cases} 
0, & \text{if } r_j < \alpha \quad (\text{低频: 完全内插}) \\
1, & \text{if } r_j > \beta \quad (\text{高频: 完全外推}) \\
\frac{r_j - \alpha}{\beta - \alpha}, & \text{otherwise} \quad (\text{混合})
\end{cases}$

最终的频率修正为：

$\theta_j' = \theta_j \left( \frac{1-\gamma_j}{s} + \gamma_j \right)$

这意味着：**只压缩低频,放过高频.**

### 2.3 熵修正与温度系数 (The Temperature Trick)

这是 YaRN 最精彩但常被忽略的一步.
当我们对 RoPE 进行插值时,我们实际上把 Query 和 Key 向量拉得更近了.这导致 Attention Logits ($q \cdot k$) 的分布变得更加**平坦** (Uniform).
在信息论中,平坦的分布意味着 **熵 (Entropy)** 增加.模型变得“困惑”,注意力无法集中.

为了修正这种**熵增**,我们需要降低 Softmax 的温度,让分布重新变尖.
YaRN 修正 Attention 公式为：

$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d} \cdot \sqrt{t}}\right) V$

其中 $\sqrt{t}$ 是温度系数.YaRN 推导出 $\sqrt{t} \approx 0.1 \ln(s) + 1$.
**这一步修正了分布的“尖锐度”,保证了 PPL 不会因为长下文而退化.**

## 3. LongRope: 搜索出来的非均匀真理

YaRN 的 Ramp Function 是人工设计的直线.微软提出的 **LongRope** 提出了一个反直觉的问题：**为什么缩放因子必须是一条平滑的线？**

LongRope 认为：最优的缩放因子应该是**非均匀 (Non-uniform)** 的,且应该由**进化算法**搜索出来.

### 3.1 搜索空间的定义

LongRope 为每一个 RoPE 维度 $i \in [0, d/2)$ 分配了一个独立的缩放因子 $\lambda_i$.这产生了巨大的搜索空间.
为了让搜索可行,LongRope 引入了 **单调性约束 (Monotonicity Constraint)** ：

$\lambda_0 \le \lambda_1 \le \dots \le \lambda_{d/2-1}$

即：低频维度的缩放程度,永远不能小于高频维度.

### 3.2 进化搜索 (Evolutionary Search)

LongRope 不通过梯度下降训练模型,而是“进化”这些 $\lambda_i$.
目标：最小化长度 $L_{target}$ 上的 PPL.

1. **初始化**: 随机生成一组满足单调性的 $\Lambda$ 种群.

2. **变异**: 随机扰动,若违反单调性则拒绝.

3. **评估**: 在小规模验证集上计算 PPL.

4. **优胜劣汰**: 保留 Top-K.

### 3.3 短文退化与双策略推理

简单的非均匀缩放虽然解决了 2000k 长度的问题,但在短文 ($L < 2048$) 上往往效果变差.
LongRope 的解决方案非常工程化：**存两套参数**.

1. $\Lambda_{long}$: 搜索出的长文参数.2. $\Lambda_{short}$: 原始 RoPE 参数 (不缩放).

推理时,如果输入长度 $L < L_{threshold}$,使用 $\Lambda_{short}$; 否则使用 $\Lambda_{long}$.这保证了短文能力 **零降级**.

## 4. DCA: 分而治之的块级注意力

**DCA (Dual Chunk Attention)** 走了一条完全不同的路.它不折腾频率,而是直接修改 **Attention 的结构**.
面对 100k+ 的长度,DCA 的核心思想是：**局部精细,全局粗糙.**

### 4.1 块切分与双重身份

将输入序列 $X$ 切分为长度为 $C$ 的块 (Chunk).每个 Token 拥有双重位置身份：

1. **局部位置 (Token ID)** : 在当前块内的位置.

2. **全局位置 (Chunk ID)** : 所在的块是第几个块.

### 4.2 块内与块间注意力

DCA 将 Attention 拆分为两部分计算：

1. **块内注意力 (Intra-Chunk Attention)** :
对于同一个块内的 token,我们需要极高的精度.
这里使用**原始 RoPE**,不做任何缩放.

$\text{Pos} = \text{Token\_ID}_i - \text{Token\_ID}_j$

1. **块间注意力 (Inter-Chunk Attention)** :
对于很久以前的块,我们不需要知道它精确的位置是 10005 还是 10006,只需要知道“它在第 100 个块”.
这里 RoPE 作用于 **Chunk ID**.

$\text{Pos} = \text{Chunk\_ID}_i - \text{Chunk\_ID}_j$

**Concept**: Shows how the mechanism treats local and distant tokens differently.

### 4.3 数学形式化

DCA 的最终 Attention 是两者的融合：

$O_i = \underbrace{\sum_{j \in \text{SameChunk}} \alpha_{ij} v_j}_{\text{Intra: 局部不缩放}} + \underbrace{\sum_{j \notin \text{SameChunk}} \beta_{ij} v_j}_{\text{Inter: 全局Chunk相对位置}}$

通过这种方式,DCA 实际上将相对位置编码的频率谱分成了两段：

- **高频细节**由 Intra 处理(无需外推).

- **低频长程**由 Inter 处理(Chunk ID 很小,天然处于 RoPE 的训练范围内).

**数学本质**：DCA 巧妙地避免了将 $100,000$ 这样巨大的数字直接输入 RoPE,而是将其转化为了 $\text{Chunk\_ID} \approx 100,000/C$ 的小数字,从而在根源上消除了 OOD 问题.

## 5. 总结：通往无限的阶梯

从数学角度审视这三种算法的演进：

1. **YaRN**: **频率域的解析解**.利用 NTK 理论,手动划分高低频,分别进行内插和外推,并引入熵修正.它是**理论派**的巅峰.

2. **LongRope**: **频率域的数值解**.承认人工设计的局限性,利用进化算法在巨大的参数空间中搜索非均匀的最优缩放因子.它是**计算派**的胜利.

3. **DCA**: **空间域的结构解**.放弃在单一位置编码上死磕,通过分块机制,将长程依赖转化为粗粒度的相对位置.它是**架构派**的创新.

掌握了这三个算法,你就掌握了当前 LLM 打破长度诅咒的三种最核心范式.
