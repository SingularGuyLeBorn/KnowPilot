# Agentic 能力从哪里来: 基座大模型的训练过程

> 本文以 GLM-5 和 MiniMax M2 为例,拆解基座大模型获得 Agentic 能力的完整训练流程——从预训练、中训练到后训练,以及 Agentic 数据合成的标准流水线. 

---

## 1. 基模训练流程(以 GLM-5 为例)

### 1.1 第一阶段: Base Model Training

**预训练(Pre-Training)** : 
- 建立通用语言能力、广泛世界知识和基础推理能力
- 数据规模: 数 T token 级别的通用语料
- 目标: 让模型"读懂世界"

**中训练(Mid-Training)** : 
- 注入领域知识: 代码、工具使用、多轮对话
- 数据规模: 数千亿 token 的定向语料
- 目标: 让模型"具备基础"

### 1.2 第二阶段: Post-Training

**监督微调(SFT)** : 
- 学习指令遵循和对话格式
- 数据: 人工标注 + 模型生成的指令-响应对

**强化学习(RL)** : 

*Reasoning RL*: 
- 先强化推理能力
- 数据: 数学、逻辑、代码推理题目
- 奖励: 答案正确性 + 过程完整性

*Agentic RL*: 
- 让模型真正学会"执行任务"
- 数据: 工具调用、多步骤任务、环境交互
- 奖励: 任务完成率 + 执行效率

*General RL*: 
- 提升通用能力和安全性
- 数据: 多样化的指令和反馈

**On-Policy Cross-Stage Distillation**: 
- 防止一路训练一路遗忘
- 在不同阶段之间蒸馏知识,保持能力连续性

---

## 2. Agentic 数据合成

### 2.1 标准流水线

1. **种子任务收集**: 从真实场景收集初始任务
2. **任务扩展**: 用模型生成相似但多样的变体
3. **环境验证**: 在真实或可模拟环境中验证任务可执行
4. **轨迹标注**: 记录成功执行的完整轨迹
5. **质量筛选**: 过滤低质量或不可复现的轨迹

### 2.2 SWE 数据合成案例

**场景**: 修复用户注册接口的空用户名问题

1. **原始素材**: GitHub Issue 描述问题
2. **转化为可执行环境**: 搭建代码仓库、测试框架
3. **模型尝试修复**: 生成代码补丁
4. **验证**: 运行测试确认修复成功
5. **收集成功轨迹**: 作为训练数据

---

## 3. Agentic 能力的关键来源

| 能力 | 训练阶段 | 数据来源 |
|:-----|:--------|:--------|
| 工具使用 | Mid-Training + Agentic RL | 工具文档、API 调用日志 |
| 多步骤规划 | Agentic RL | 复杂任务拆解数据 |
| 环境感知 | Agentic RL | 沙箱交互轨迹 |
| 错误恢复 | Agentic RL | 失败-重试成功轨迹 |
| 指令遵循 | SFT + General RL | 多样化指令数据 |

---

## 4. 总结

Agentic 能力不是单一阶段训练的结果,而是**预训练 → 中训练 → SFT → Reasoning RL → Agentic RL → General RL** 多阶段累积的产物. 每个阶段都为模型增加新的能力维度,最终形成能够自主规划、执行、反思的 Agentic 系统.
