# 13.4 Agent 训练与进化

## 1. 演进脉络与技术坐标

传统 RLHF 训练模型生成"人类觉得好"的文本,但不直接训练模型"解决问题". Agentic RL 将强化学习的应用范围从"文本质量"扩展到"任务完成"——模型通过与环境交互、接收任务级别的奖励信号,学习如何更有效地使用工具和规划行动. DeepSeek-R1 和 OpenAI o1 系列是 Agentic RL 的典型代表,通过在推理阶段引入内部思考链展开,实现测试时计算扩展. 

与此同时,基座模型本身也在经历 Agentic 能力的注入. 从 GLM-5 的中训练阶段注入代码与工具使用数据,到后训练阶段的 Agentic SFT 与 RL,基座模型正在从"被动应答器"进化为"主动求解器".

## 3. 工业价值与落地现状

SWE-Bench 上 OpenHands + Claude 3.5 Sonnet 达到 53.3% 的解决率,这一跃升证明 Agent 架构不是锦上添花,而是决定 LLM 能否解决实际工程问题的关键变量. Agentic RL 的奖励函数通常包含任务奖励、格式奖励和效率奖励三个组件,多目标结构反映了 Agent 训练的核心矛盾: 模型不仅要"做对",还要"写得对"和"做得快". 

## 4. 技术前瞻

Agent 训练的未来方向包括: 强化学习训练规模化(从少量 SWE-Bench 任务扩展到数百万个真实软件工程任务)、环境重置的工程瓶颈(容器化编排的调度开销可能占据总训练时间的 30% 以上)、以及量化训练中的数值稳定性(FP8 精度下 Newton-Schulz 迭代的收敛域安全).
