# Tool-integrated Reasoning RL 综述

> 本文汇总 2025 年 Tool-integrated Reasoning(TIR)与 Agentic RL 领域的前沿工作,涵盖 Search-R1、ToRL、OTC、RAGEN、SimpleTIR、Memento、rStar2-Agent 等代表性方法. 

---

## 1. 背景: TIR 的兴起

Tool-integrated Reasoning(TIR)是指让大模型在推理过程中**主动调用外部工具**(搜索引擎、计算器、代码解释器、数据库等)来辅助解决问题. 

与传统的 Chain-of-Thought 不同,TIR 强调: 
- **动态工具选择**: 模型自主决定何时需要工具、选择什么工具
- **多轮交互**: 与工具进行多轮对话,逐步逼近答案
- **错误恢复**: 工具调用失败时,模型能调整策略重试

---

## 2. 代表性工作

### 2.1 Search-R1(UIUC, 2025.03)

**核心思想**: 训练 LLM 推理并利用搜索引擎进行强化学习. 

- 将搜索引擎作为外部工具集成到 RL 框架
- 模型学会在不确定时主动搜索,而非盲目猜测
- 奖励函数: 答案正确性 + 搜索效率

### 2.2 ToRL: Scaling Tool-Integrated RL(SJTU, 2025.03)

**核心思想**: 规模化工具集成强化学习. 

- 扩展 TIR 到更多工具类型和更复杂任务
- 提出工具调用标准化的训练流程
- 在数学推理和知识问答上取得显著提升

### 2.3 OTC: Optimal Tool Calls via RL(2025.03)

**核心思想**: 通过 RL 学习最优工具调用策略. 

- 将工具调用建模为 MDP(马尔可夫决策过程)
- 学习"何时调用、调用什么、如何解析结果"
- 最小化工具调用次数的同时最大化答案质量

### 2.4 RAGEN: Self-Evolution in LLM Agents(2025.05)

**核心思想**: 理解 LLM Agent 的自我进化机制. 

- 多轮 RL 训练使 Agent 逐步提升工具使用能力
- 分析自我进化的关键因素: 探索 vs 利用、记忆机制、策略更新

### 2.5 SimpleTIR(NTU, 2025.07)

**核心思想**: 端到端的多轮工具集成推理 RL. 

- 简化 TIR 的训练流程
- 统一的框架处理多种工具类型
- 降低 TIR 的训练门槛

### 2.6 Memento(UCL & Huawei, 2025.08)

**核心思想**: 不微调 LLM 的情况下提升 Agent 能力. 

- 冻结 LLM 参数,只训练工具选择和结果解析模块
- 大幅降低训练成本
- 适用于已有强大基座模型的场景

### 2.7 rStar2-Agent(MSRA, 2025.08)

**核心思想**: Agentic Reasoning 的技术报告. 

- 系统总结 Agentic Reasoning 的核心组件
- 提出标准化的评估基准
- 为后续研究提供参考框架

## 3. TIR 的关键技术挑战

| 挑战 | 解决方向 |
|:-----|:--------|
| 工具选择 | RL 学习最优工具调用策略 |
| 结果解析 | 结构化输出 + 错误恢复机制 |
| 多轮一致性 | 记忆模块 + 上下文管理 |
| 训练稳定性 | 课程学习 + 渐进式复杂度 |
| 评估困难 | 多维度评估(正确性、效率、鲁棒性)|

---

## 4. 未来方向

1. **工具自动发现**: 模型自主学习和使用新工具
2. **多 Agent 协作**: 多个 Agent 各自使用不同工具协作完成任务
3. **安全约束**: 防止工具调用的滥用和安全隐患
4. **实时工具**: 与实时数据源(股票、天气、新闻)集成


## 5. 2025 年 TIR 论文深度解读

### 5.1 Search-R1: 搜索引擎集成(UIUC, 2025.03)

**技术细节**: 
- 基于 PPO/GRPO 框架,算 loss 时对检索返回内容做 mask,只针对 LLM 产生的内容进行梯度计算
- 使用特殊 token 标记不同内容: `<search>...</search>`、`<information>...</information>`、`<answer>...</answer>`
- 奖励函数采用 rule-based rewards,只包含结果正确性奖励
- 实验基于 Qwen-2.5-3B/7B (Base/Instruct),PPO 作为默认优化方法

### 5.2 ToRL: 从 Base 模型直接训练(SJTU, 2025.03)

**核心创新**: 从 base 模型开始直接做 RL,不经过 SFT,更好地利用 base 模型的探索能力学习工具调用. 

- Rollout 过程: $s_k = r_1, c_1, o_1, ..., r_k, c_k, o_k$
  - $r_i$: 自然语言推理内容
  - $c_i$: LLM 生成的 code
  - $o_i$: 代码执行结果
- 工具调用最大发起数由超参数 $C$ 控制(实验中 $C=1$)
- 奖励设计: 答案正确为 1,错误为 -1,不可执行代码为 -0.5

### 5.3 OTC: 优化工具调用频率(2025.03)

**核心思想**: 通过 reward shaping 优化模型调用工具的频率,在不牺牲准确性的前提下降低训练和推理成本. 

- 在 reward function 上增加系数 $r_{\phi}^{tool} \in [0,1]$
- 代表工具调用次数 $m$ 与最优调用次数 $n$ 之间的关系
- 针对 PPO 和 GRPO 分别设计了不同的奖励系数计算方式
- 定义三个指标: EM(准确性)、TC(平均调用工具数)、TP(工具调用有效率)

### 5.4 RAGEN: 多轮 RL Agent 训练(2025.05)

**核心贡献**: 提出 StarPO (State-Thinking-Actions-Reward Policy Optimization) 和 RAGEN 模块化训练推理框架. 

**三项关键技术**: 
1. **StarPO-S**: 缓解多轮训练不稳定
   - 基于方差的轨迹过滤
   - Critic 基准测试
   - 解耦裁剪
2. **多样化 rollout**: 每个初始状态有多个响应,每轮交互执行多个动作
3. **细粒度奖励**: 仅鼓励动作形式的推理不能保证有效推理,需要推理感知的奖励信号

**关键发现**: 
- PPO 和 GRPO 进行轨迹级多轮 RL 训练时,后期会出现崩塌(echo trap)
- 对低信息量样本进行过滤可缓解不稳定
- 仅依赖稀疏结果 reward 可能是模型崩塌的原因之一

### 5.5 SimpleTIR: 端到端多轮 TIR RL(NTU, 2025.07)

**核心问题**: 在 base 模型上利用 Verifiable Reward 直接做多轮 TIR RL,训练过程会出现不稳定和熵崩塌. 

**原因分析**: 
- 外部工具返回内容与原始模型分布存在较大差异,导致分布偏移
- 多轮对话中的无效轮次(void turn)错误累积,形成破坏性反馈循环

**解决方案**: 在采样后过滤掉包含中间无效轮次的样本. 

**训练后能力**: Cross Validation、Progressive Reasoning、Error Correction Loop 等多样化推理能力. 

### 5.6 Memento: 不微调 LLM 的 Agent 微调(UCL, Huawei, 2025.08)

**核心思想**: 学习人类记忆机制,为 case-based reasoning(CBR)任务提出可强化学习的检索机制. 

**Memory-based MDP**: 
- $M_t = \{c_i\}_{i=1}^N$ 为 case bank(memory),存储历史交互中的 case $c_i = (s_i, a_i, r_i)$
- 给定当前状态 $s_t$,CBR agent 学习检索策略 $\mu$,选出最佳参考 case
- LLM 基于 case 作出决策 $a_t \sim p_{LLM}(\cdot|s_t, c_t)$

**检索策略学习**: 使用 soft Q-learning,通过 kernel-based estimation 近似学习 Q-function. 

**实验**: GAIA、DeepResearcher、SimpleQA、HLE 数据集上验证,学习轻量动态 RAG 机制比训练 LLM 泛化性更好. 

### 5.7 rStar2-Agent: MATH 问题的 Code 工具调用(MSRA, 2025.08)

**技术细节**: 
- 多轮交互: CoT → tool use → tool response → CoT → ...
- Tool call 格式: JSON 规范
- Tool 返回四种类型: 成功输出、无输出成功、执行错误、超时

**GRPO-RoC(Group Relative Policy Optimization with Resampling on Correct)** : 
- 过采样 2G 个样本,按答案正确性分为正样本和负样本
- 负样本: 均匀随机采样一半,保留失败模式多样性
- 正样本: 优先选择质量更高的轨迹(基于惩罚分数: 工具使用 + 格式合规)

**工程优化**: 
- 基于 VERL 和 SGLang 开发独立 code 响应环境,每 step 可同时处理 45k 工具调用
- 动态 KV 缓存分配: 实时监测每个 GPU 的"当前可用 KV 缓存容量",基于信息动态分配请求

**训练策略**: 
- 一阶段: 非推理场景的 CoT 训练,避免思维固化
- 二阶段: 三步渐进训练
  - 8K 长度简洁训练
  - 12K 长度扩展训练
  - 较难问题训练
