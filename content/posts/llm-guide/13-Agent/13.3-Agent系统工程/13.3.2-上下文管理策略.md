# Agent 框架中的上下文管理策略

> 本文系统解析 Agent 框架中的上下文工程，涵盖上下文卸载与检索、上下文摘要、上下文隔离(多智能体架构)和上下文缓存四大策略，为构建长程 Agent 系统提供可落地的工程参考。

---

## 1. 背景: 从"Chatbots that Respond"到"Agents that Act"

随着 Claude Code、OpenClaw 等 Agent Scaffold 的爆火，大模型领域的竞争焦点从通用能力转向 **Agent 落地** 和 **编程能力**。各大厂商的定位从"能响应的聊天机器人"转向"能执行任务的智能体"。

Agent 框架的核心挑战之一是**上下文管理**: 当 Agent 执行长程任务(如代码开发、数据分析、多步骤研究)时，上下文窗口很快会被历史对话、工具调用结果、中间产物填满。如何高效管理这些上下文，是 Agent 系统设计的核心问题。

---

## 2. 上下文卸载与检索(Context Offload & Retrieval)

### 2.1 核心思想

将超出上下文窗口的信息**转移到外部存储**(文件系统、数据库、向量数据库)，需要时按需检索。

### 2.2 实现方式

**文件系统存储**: 
- Agent 的中间产物(代码文件、分析报告、数据表格)直接写入文件系统
- 上下文窗口中只保留文件路径和摘要，而非完整内容
- 需要时通过 `read_file` 工具按需读取

**向量数据库检索**: 
- 将历史对话、知识文档编码为向量，存入向量数据库
- 当前任务需要历史信息时，用相似度检索召回相关内容
- 检索结果作为补充上下文注入当前对话

**分层存储架构**: 
```
热上下文(Hot Context): 当前对话窗口中的内容
温上下文(Warm Context): 最近几轮的工具调用结果
冷上下文(Cold Context): 向量数据库中的历史记录
归档上下文(Archived Context): 文件系统中的持久化产物
```

## 3. 上下文摘要(Context Summarization)

### 3.1 核心思想

当上下文接近窗口上限时，对早期对话进行**摘要压缩**，保留关键信息，丢弃细节。

### 3.2 摘要策略

**逐轮摘要**: 
- 每完成一个子任务，生成该子任务的摘要
- 后续对话只引用摘要，不再引用完整执行过程

**分层摘要**: 
- 第一层: 单轮对话摘要(几句话概括)
- 第二层: 多轮主题摘要(一个主题下的多轮对话概括)
- 第三层: 全局任务摘要(整个任务的目标、进展、关键决策)

**增量摘要**: 
- 新信息到来时，更新现有摘要而非重新生成
- 使用 diff 风格的增量更新，减少计算开销

### 3.3 摘要质量的关键

- **保留决策点**: 哪些选择导致了当前状态
- **保留约束条件**: 不可违反的规则和限制
- **保留未完成任务**: 还有哪些子任务待处理
- **丢弃执行细节**: 具体的命令输出、中间计算过程

## 4. 上下文隔离(多智能体架构)

### 4.1 核心思想

将复杂任务分解为多个子任务，每个子任务由**独立的智能体**处理，智能体之间通过**结构化消息**通信，而非共享完整上下文。

### 4.2 架构模式

**主从模式(Master-Slave)** : 
- 主智能体负责任务分解和协调
- 从智能体负责具体子任务执行
- 从智能体只接收与其子任务相关的上下文

**流水线模式(Pipeline)** : 
- 多个智能体按顺序处理数据
- 每个智能体的输出作为下一个智能体的输入
- 每个智能体只保留当前步骤所需的上下文

**协作模式(Collaboration)** : 
- 多个智能体并行工作，定期同步状态
- 每个智能体有独立的上下文窗口
- 同步时只交换关键状态和决策，而非完整历史

### 4.3 上下文隔离的收益

- **降低单智能体负载**: 每个智能体只需处理与其任务相关的上下文
- **减少噪声干扰**: 无关信息不会干扰当前任务的推理
- **提升并行度**: 多个智能体可同时工作，不受单一上下文窗口限制
- **增强可调试性**: 每个智能体的行为可独立追踪和调试

## 5. 上下文缓存(Context Caching)

### 5.1 核心思想

缓存重复使用的上下文片段，避免重复计算和传输。

### 5.2 缓存层级

**系统提示缓存**: 
- 系统提示(System Prompt)通常固定不变
- 缓存其 KV Cache，避免每次请求重新计算
- 适用于所有对话的通用指令和约束

**工具定义缓存**: 
- 工具调用的 JSON Schema 定义通常固定
- 缓存工具定义的 KV Cache，减少工具调用场景的开销

**历史对话缓存**: 
- 对于多轮对话，缓存前几轮的 KV Cache
- 新轮次只需计算新增 token 的 KV Cache

### 5.3 缓存失效策略

- **时间失效**: 缓存超过一定时间后失效(适用于动态变化的上下文)
- **版本失效**: 当系统提示或工具定义更新时，旧缓存失效
- **显式失效**: Agent 主动通知缓存失效(如完成任务后清理相关缓存)

---

## 6. 四大策略的协同

在实际 Agent 系统中，四种策略通常协同使用: 

```
1. 任务启动: 加载系统提示缓存 + 工具定义缓存
2. 任务执行: 
   - 热上下文窗口维护当前对话
   - 超出窗口时触发上下文摘要
   - 中间产物写入文件系统(上下文卸载)
   - 需要历史信息时检索向量数据库
3. 复杂任务: 
   - 分解为子任务，分配给多个智能体(上下文隔离)
   - 智能体间通过结构化消息同步状态
4. 任务完成: 
   - 归档文件系统产物
   - 更新全局任务摘要
   - 清理临时缓存
```

## 7. 工程实践建议

| 策略 | 适用场景 | 实现复杂度 | 关键工具 |
|:-----|:--------|:----------|:--------|
| 上下文卸载 | 长文档处理、代码开发 | 中 | 文件系统、向量数据库 |
| 上下文摘要 | 多轮对话、长程任务 | 低 | LLM 摘要生成 |
| 上下文隔离 | 复杂多步骤任务 | 高 | 多智能体框架 |
| 上下文缓存 | 高频重复查询 | 低 | KV Cache 复用 |
