# 13.2 Agent 认知架构

## 1. 演进脉络与技术坐标

早期 LLM 应用采用一次性的 prompt-response 范式，模型在单轮交互后即终止. ReAct 框架首次将推理轨迹(Reasoning Traces)与行动轨迹(Action Trajectories)交错展开，证明了在 Chain-of-Thought 之上叠加工具调用的可行性. 但 ReAct 的每一步推理与行动由同一个 LLM 实例完成，导致 planning 与 execution 的认知负荷相互干扰. 

Plan-and-Execute 模式对此进行解耦，将 Agent 拆分为 Planner 与 Executor. Reflection 模式通过引入独立的 Critic 模块修补上述缺陷. 2024 年以后，o1-like 推理模型将部分搜索过程内化为测试时计算扩展，改变了外部规划模块的角色定位. 

从控制论视角审视，Agent 的认知架构本质上是一个离散时间控制系统的决策核心. 经典控制理论中的反馈、稳定性与 Lyapunov 分析为 Agent 的收敛性和鲁棒性提供了数学语法，即使最终解法依赖深度学习而非传统控制算法.

## 3. 工业价值与落地现状

认知架构的设计选择直接决定 Agent 的任务成功率与输出可靠性. 从 ReAct 到 Reflection 的跃迁将 Agent 从"一次性推理器"转变为"迭代式求解器"，任务完成率从 12% 提升至 89%. o1-like 模型的测试时计算扩展进一步将复杂数学推理的准确率推至新高度，但其计算成本也呈指数级增长. 

## 4. 技术前瞻

当前认知架构的未解难题包括: 分层规划的理论基础缺失(人类认知存在战略目标、战术计划、肌肉记忆反射等多层抽象，而 LLM Agent 仅有粗暴二分的 Plan-and-Execute); Reflection 的元认知局限(Critic 与 Actor 共享同一模型权重，难以发现系统性盲区); 以及推理时计算扩展与外部控制器的协同设计.
