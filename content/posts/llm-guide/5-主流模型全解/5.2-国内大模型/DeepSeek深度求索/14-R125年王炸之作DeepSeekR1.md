## 1. 摘要

我们介绍了第一代推理模型，DeepSeek-R1-Zero 和 DeepSeek-R1。R1-Zero 是通过大规模强化学习 (RL) 训练的模型，且没有作为预处理步骤进行监督微调 (SFT)，展现出了显著的推理能力。通过强化学习，DeepSeek-R1-Zero 自然展现出许多强大且有趣的推理行为。

然而，它也面临着一些挑战，如可读性差和语言混合等问题。为了解决这些问题并进一步提升推理性能，我们提出了 DeepSeek-R1，该模型在强化学习前加入了多阶段训练和冷启动数据。

DeepSeek-R1 在推理任务上的表现可与 OpenAI-o1-1217 相媲美。为了支持研究社区，我们将 DeepSeek-R1-Zero、DeepSeek-R1 以及从 DeepSeek-R1 中提炼出的六个稠密模型(1.5B、7B、8B、14B、32B、70B)基于 Qwen 和 Llama 开源。

## 2. **1 引言**

近年来，大型语言模型(LLMs)经历了快速的迭代和演进(Anthropic, 2024; Google, 2024; OpenAI, 2024a)，逐渐缩小了与人工通用智能(AGI)之间的差距。

最近，后训练(post-training)作为完整训练流程的一个重要组成部分逐渐兴起。研究表明，它能够在推理任务上提高准确性，符合社会价值观，并适应用户偏好，同时相较于预训练，它所需的计算资源较少。

在推理能力方面，OpenAI 的 o1 系列模型(OpenAI, 2024b)率先通过增加“思维链”(Chain-of-Thought)推理过程的长度来引入推理时的扩展。这一方法在数学、编程和科学推理等各类推理任务上取得了显著的进展。然而，如何有效地进行测试时扩展依然是研究社区中的一个未解难题。

此前已有若干研究探索了不同的方案，包括基于过程的奖励模型(Lightman et al., 2023; Uesato et al., 2022; Wang et al., 2023)、强化学习(Kumar et al., 2024)，以及蒙特卡洛树搜索和束搜索等搜索算法(Feng et al., 2024; Trinh et al., 2024; Xin et al., 2024)。然而，这些方法都未能达到与 OpenAI o1 系列模型相媲美的通用推理表现。

在本文中，我们迈出了利用纯强化学习(RL)提高语言模型推理能力的第一步。我们的目标是探索大型语言模型在没有任何监督数据的情况下开发推理能力的潜力，重点关注它们通过纯 RL 过程的自我进化。

具体而言，我们以 DeepSeek-V3-Base 作为基础模型，采用 GRPO(Shao et al., 2024)作为 RL 框架，以提高模型在推理方面的表现。在训练过程中，DeepSeek-R1-Zero 自然展现出许多强大且有趣的推理行为。

在经过数千步的 RL 训练后，DeepSeek-R1-Zero 在推理基准测试上表现出了卓越的性能。例如，AIME 2024 上的 pass@1 得分从 15.6% 提升至 71.0%，并且通过多数投票，得分进一步提升至 86.7%，达到了 OpenAI-o1-0912 的水平。

然而，DeepSeek-R1-Zero 也面临一些挑战，如可读性差和语言混合等问题。为了解决这些问题并进一步提升推理性能，我们提出了 DeepSeek-R1，它结合了少量的冷启动数据和多阶段训练流程。

1)具体而言，我们首先收集成千上万的冷启动数据，以微调 DeepSeek-V3-Base 模型。

2)随后，我们像训练 DeepSeek-R1-Zero 那样进行推理导向的 RL。

3)当 RL 过程接近收敛时，我们通过拒绝采样在 RL Checkpoint上生成新的 SFT 数据，并结合来自 DeepSeek-V3 的监督数据，涉及写作、事实问答和自我认知等领域，然后重新训练 DeepSeek-V3-Base 模型。

4)经过新数据的微调后，Checkpoint再次进行强化学习，涵盖所有场景的提示。经过这些步骤，我们得到了一个名为 DeepSeek-R1 的Checkpoint，其推理能力已达到与 OpenAI-o1-1217 相当的水平。

我们进一步探索了从 DeepSeek-R1 到更小稠密模型的蒸馏过程。以 Qwen2.5 32B(Qwen, 2024b)为基础模型，直接从 DeepSeek-R1 进行蒸馏优于应用 RL 的方法。这表明，大型基础模型发现的推理模式对于提升推理能力至关重要。

我们将基于 Qwen 和 Llama(Dubey et al., 2024)系列的蒸馏模型开源。值得注意的是，我们蒸馏的 14B 模型在推理基准测试中远超当前开源的最先进模型 QwQ-32B-Preview(Qwen, 2024a)，而蒸馏的 32B 和 70B 模型在稠密模型中的推理基准上创下了新纪录。

### 2.1 **1.1. 贡献**

**后训练：在基础模型上进行大规模强化学习**

我们直接将强化学习(RL)应用于基础模型，而无需依赖监督微调(SFT)作为预处理步骤。这种方法使得模型能够通过“思维链”(Chain-of-Thought, CoT)探索解决复杂问题，从而发展出 DeepSeek-R1-Zero。

DeepSeek-R1-Zero 展现了自我验证、反思以及生成长篇思维链等能力，标志着研究领域的一个重要里程碑。值得注意的是，它是首个验证 LLM 推理能力可以仅通过 RL 激励，而不需要 SFT 的开源研究。这一突破为未来在该领域的发展奠定了基础。

我们提出了开发 DeepSeek-R1 的流程管道。该管道包含两个 RL 阶段，旨在发现更好的推理模式并与人类偏好对齐，以及两个 SFT 阶段，作为模型推理和非推理能力的种子。我们认为这一流程管道将有助于行业发展，创造更好的模型。

**蒸馏：更小的模型也能强大**

**我们展示了较大模型的推理模式可以被蒸馏到较小的模型中，从而获得比通过 RL 在小模型中发现的推理模式更好的性能。开源的 DeepSeek-R1 及其 API 将有助于研究社区未来蒸馏出更好的小型模型。**

**利用 DeepSeek-R1 生成的推理数据，我们微调了多个在研究界广泛使用的稠密模型。评估结果表明，这些蒸馏出的较小稠密模型在基准测试中表现非常出色。DeepSeek-R1-Distill-Qwen-7B 在 AIME 2024 上取得了 55.5% 的得分，超过了 QwQ-32B-Preview。此外，DeepSeek-R1-Distill-Qwen-32B 在 AIME 2024 上得分为 72.6%**

**在 MATH-500 上得分为 94.3%，在 LiveCodeBench 上得分为 57.2%。这些结果显著超过了先前的开源模型，并与 o1-mini 的表现相当。我们将基于 Qwen2.5 和 Llama3 系列的蒸馏模型(1.5B、7B、8B、14B、32B、70B)开源给社区。**

### 2.2 **1.2. 评估结果总结**

**推理任务**：(1)DeepSeek-R1 在 AIME 2024 上的 Pass@1 得分为 79.8%，略微超越 OpenAI-o1-1217。在 MATH-500 上，它取得了 97.3% 的惊人成绩，表现与 OpenAI-o1-1217 相当，并显著超越了其他模型。(2)在与编程相关的任务中，DeepSeek-R1 展现了专家级的能力，在 Codeforces 编程比赛中取得了 2,029 Elo 的评分，超越了 96.3% 的人类参赛者。在工程相关的任务中，DeepSeek-R1 的表现略优于 DeepSeek-V3，可能有助于开发者在实际任务中的应用。

**知识任务**：在 MMLU、MMLU-Pro 和 GPQA Diamond 等基准测试中，DeepSeek-R1 取得了卓越的成绩，显著超越了 DeepSeek-V3，分别在 MMLU 上得分 90.8%、MMLU-Pro 上得分 84.0%、在 GPQA Diamond 上得分 71.5%。尽管在这些基准测试中的表现略低于 OpenAI-o1-1217，但 DeepSeek-R1 超越了其他闭源模型，展现了其在教育任务中的竞争优势。在事实类基准测试 SimpleQA 上，DeepSeek-R1 超越了 DeepSeek-V3，展现了其处理基于事实问题的能力。在该基准测试中，OpenAI-o1 的表现也超越了 4o。

## 3. **2 方法**

### 3.1 **2.1. 概述**

以往的研究在提升模型性能时，往往依赖大量的监督数据。在本研究中，我们展示了即使不使用监督微调(SFT)作为冷启动，推理能力也可以通过大规模强化学习(RL)显著提升。此外，加入少量冷启动数据还可以进一步增强性能。

在接下来的部分，我们将介绍：(1)DeepSeek-R1-Zero，直接将 RL 应用到基础模型中，不使用任何 SFT 数据; (2)DeepSeek-R1，从一个通过数千个长思维链(CoT)示例微调的Checkpoint开始应用 RL; (3)将 DeepSeek-R1 的推理能力蒸馏到小型稠密模型中。

### 3.2 **2.2. DeepSeek-R1-Zero：基础模型上的强化学习**

强化学习在推理任务中已经展现出了显著的效果，这一点可以通过我们之前的研究(Shao et al., 2024; Wang et al., 2023)得到证明。然而，这些研究大多依赖于监督数据，而这些数据的收集往往非常耗时。

在本节中，我们探索了 LLM(大型语言模型)在没有任何监督数据的情况下，如何通过纯强化学习过程自我进化，从而发展出推理能力。我们首先简要概述了我们的强化学习算法，然后展示一些令人兴奋的结果，希望这些内容能为研究社区提供有价值的见解。

#### 3.2.2 **2.2.2. 奖励建模**

奖励是训练信号的来源，决定了强化学习(RL)优化的方向。为了训练 DeepSeek-R1-Zero，我们采用了一种基于规则的奖励系统，主要包括两种类型的奖励：

• 准确性奖励：准确性奖励模型评估回答是否正确。例如，对于具有确定性结果的数学问题，模型需要提供一个以指定格式(如放入框中)呈现的最终答案，从而使得可以通过可靠的基于规则的方式进行正确性验证。类似地，对于 LeetCode 问题，可以使用编译器基于预定义的测试用例生成反馈。

• 格式奖励：除了准确性奖励模型，我们还采用了格式奖励模型，要求模型将其思维过程放置在“”与“”标签之间。

我们在开发 DeepSeek-R1-Zero 时没有应用结果或过程神经奖励模型，因为我们发现神经奖励模型在大规模强化学习过程中可能会遭遇奖励劫持(reward hacking)，而且重新训练奖励模型需要额外的训练资源，并且会使整个训练流程更加复杂。

DeepSeek-R1-Zero 的缺点：尽管 DeepSeek-R1-Zero 展现了强大的推理能力，并自主发展出了意外且强大的推理行为，但它也面临一些问题。例如，DeepSeek-R1-Zero 在可读性差和语言混合等方面存在挑战。为了使推理过程更加易读，并将其与开放社区共享，我们探索了 DeepSeek-R1，一种利用 RL 与人类友好的冷启动数据的方法。

### 3.3 **2.3. DeepSeek-R1：冷启动的强化学习**

受到 DeepSeek-R1-Zero 取得的良好结果的启发，两个自然的问题出现了：1) 通过加入少量高质量数据作为冷启动，推理性能是否能够进一步提升或收敛加速？ 2) 我们如何训练一个既能生成清晰连贯的推理链(CoT)，又能展示强大通用能力的用户友好型模型？为了解决这些问题，我们设计了一个训练 DeepSeek-R1 的管道。该管道包含四个阶段，具体如下。

#### 3.3.1 **2.3.1. 冷启动**

与 DeepSeek-R1-Zero 不同，为了防止 RL 训练的初期阶段不稳定，DeepSeek-R1 构建并收集少量长推理链数据来对模型进行微调，作为初始的 RL 行为者。为了收集这些数据，我们探索了几种方法：使用少量示例提示生成长推理链、直接提示模型生成带有反思和验证的详细答案、收集 DeepSeek-R1-Zero 输出并以可读格式呈现，以及通过人工标注者后处理结果进行优化。

在本研究中，我们收集了成千上万的冷启动数据来对 DeepSeek-V3-Base 进行微调，作为 RL 训练的起始点。与 DeepSeek-R1-Zero 相比，冷启动数据的优势包括：

**可读性**：DeepSeek-R1-Zero 的一个关键限制是其内容通常不适合阅读。其回答可能会混合多种语言，或缺乏高亮格式来突出显示答案。相比之下，在为 DeepSeek-R1 创建冷启动数据时，我们设计了一个可读的模式，在每个回答的末尾包含总结，并过滤掉不适合阅读的回答。

#### 2.3.2 面向推理的强化学习

在使用冷启动数据对 DeepSeek-V3-Base 进行微调后，我们采用与 DeepSeek-R1-Zero 相同的大规模强化学习训练过程。

此阶段的重点是提升模型的推理能力，尤其是在推理密集型任务中，如编程、数学、科学和逻辑推理等，这些任务涉及到明确的问题和清晰的解决方案。

在训练过程中，我们观察到 CoT 经常出现语言混合，特别是在 RL 提示涉及多种语言时。为了缓解语言混合问题，我们在 RL 训练中引入了语言一致性奖励，这个奖励计算为 CoT 中目标语言词汇的比例。

尽管消融实验表明，进行这种对齐会稍微降低模型的性能，但这一奖励符合人类的偏好，使输出更具可读性。最终，我们通过直接将推理任务的准确性和语言一致性奖励相加，形成最终奖励。然后我们在经过微调的模型上应用强化学习(RL)训练，直到其在推理任务上收敛。

#### 2.3.3 **2.3.3. 拒绝采样和监督微调**

当面向推理的 RL 收敛时，我们利用所得的Checkpoint来收集下一轮的 SFT(监督微调)数据。与初始的冷启动数据主要关注推理不同，此阶段将包含来自其他领域的数据，以增强模型在写作、角色扮演和其他通用任务中的能力。具体来说，我们生成数据并对模型进行微调，如下所述。

**推理数据**：我们策划推理提示，并通过执行拒绝采样从上一步 RL 训练的Checkpoint生成推理轨迹。在之前的阶段，我们仅包括可以通过基于规则的奖励进行评估的数据。然而，在此阶段，我们通过将一些生成奖励模型的数据纳入数据集，来扩展数据集，通过将真实值和模型预测输入 DeepSeek-V3 进行判断。

此外，由于模型输出有时是混乱的并且难以阅读，我们过滤掉了混合语言的推理链、长段落和代码块。对于每个提示，我们会采样多个回应，并只保留正确的回应。总的来说，我们收集了约 60 万个与推理相关的训练样本。

**非推理数据**：对于非推理数据，如写作、事实问答、自我认知和翻译，我们采用 DeepSeek-V3 的管道，并重用部分 DeepSeek-V3 的 SFT 数据集。对于某些非推理任务，我们调用 DeepSeek-V3 来生成一个潜在的推理链，再通过提示回答问题。然而，对于一些简单的查询，如“你好”，我们不提供推理链作为回应。最终，我们收集了大约 20 万个与推理无关的训练样本。

我们使用上述策划的数据集(约 80 万个样本)对 DeepSeek-V3-Base 进行两轮微调。

#### 2.3.4 **2.3.4. 全场景强化学习**

为了进一步将模型与人类偏好对齐，我们实施了一个二级强化学习阶段，旨在提高模型的有用性和无害性，同时进一步完善其推理能力。

具体而言，我们使用奖励信号和多样化的提示分布来训练模型。对于推理数据，我们遵循 DeepSeek-R1-Zero 中的做法，使用基于规则的奖励来引导模型在数学、编程和逻辑推理领域的学习过程。

对于一般性数据，我们使用奖励模型来捕捉复杂和微妙场景中的人类偏好。我们在 DeepSeek-V3 的管道上构建，采用类似的偏好对和训练提示分布。

对于有用性，我们仅关注最终总结，确保评估强调响应对用户的实用性和相关性，同时尽量减少对底层推理过程的干扰。

对于无害性，我们评估模型的整个响应，包括推理过程和总结，以识别和缓解生成过程中可能出现的潜在风险、偏见或有害内容。

最终，奖励信号和多样化数据分布的结合使我们能够训练出一个在推理上表现优异，同时优先考虑有用性和无害性的模型。

### 2.4 **2.4. 蒸馏：赋予小模型推理能力**

为了使更高效的小模型具备像 DeepSeek-R1 这样的推理能力，我们直接对开源模型(如 Qwen(Qwen, 2024b)和 Llama(AI@Meta, 2024))进行微调，使用在 §2.3.3 中策划的 80 万个样本。

我们的研究结果表明，这种简单的蒸馏方法显著提升了小模型的推理能力。我们在这里使用的基础模型包括 Qwen2.5-Math-1.5B、Qwen2.5-Math-7B、Qwen2.5-14B、Qwen2.5-32B、Llama-3.1-8B 和 Llama-3.3-70B-Instruct。我们选择 Llama-3.3，因为它的推理能力略优于 Llama-3.1。

对于蒸馏模型，我们仅应用 SFT，并不包括 RL 阶段，尽管引入 RL 可以显著提升模型性能。我们的主要目标是展示蒸馏技术的有效性，而 RL 阶段的探索则留给更广泛的研究社区。
