# 第 5 章 · 主流模型全解

> **这不是一份参数速查表，而是一张追问"它们在后训练阶段做了什么不同的事"的技术解剖图。**
>
> 在这一章，我们不满足于知道"DeepSeek-V3 有 671B 参数"或者"GPT-4o 支持多模态"。我们要追问的是：为什么 DeepSeek 能在训练成本上做到 OpenAI 的十分之一？为什么 Claude 的 long context 稳定性在业界有口皆碑？为什么 Llama 3 的 8B 小模型能在某些任务上反超前一代的 70B 模型？

---

## 1. 群星闪耀时：为什么需要一张模型全景图

2022 年 11 月，ChatGPT 的发布像一颗超新星，点燃了整个大模型产业。两年半过去，这片星图已经膨胀到令人眼花缭乱的程度：OpenAI 的 GPT 系列、Anthropic 的 Claude 系列、Google 的 Gemini 与 Gemma、Meta 的 Llama 系列、Mistral AI 的 Mistral 与 Mixtral、xAI 的 Grok——这是国际阵营的头部玩家。国内战场上，阿里的通义千问(Qwen)、DeepSeek、智谱 AI 的 GLM、月之暗面的 Kimi、面壁智能的 MiniCPM、字节跳动的豆包、百度的文心一言、MiniMax 的 abab 系列，以及诸多垂直领域的专业模型，构成了一幅同样壮观的图景。

截至 2026 年初，Hugging Face 上的模型数量已突破 200 万。在这海量的模型中，真正具有技术影响力、生态号召力或商业落地价值的，不过数十个家族。但恰恰是这数十个家族，决定了整个行业的技术走向、开源生态的繁荣程度，以及千千万万工程师和研究者的日常选型。

**本章的核心目标**，就是为这些真正重要的模型家族建立一份"技术解剖档案"。我们不追求覆盖每一个发布过模型卡(model card)的 checkpoint，而是聚焦于那些在架构设计、训练方法、后处理策略或生态建设上做出实质性贡献的代表性模型。

## 2. 本章在全书中的定位

在进入具体模型之前，有必要明确本章与前四章的关系，以及它与后续章节的分工。

第 2 章教会你"发动机怎么造"——Self-Attention、RoPE、SwiGLU、RMSNorm，这些是所有现代大模型共享的底层原子。第 3 章和第 4 章教会你"怎么训练和调教"——预训练的数据配比、Tokenizer 设计、后训练的 SFT 与对齐策略。到了第 5 章，我们终于要面对一个实际问题：**当这些底层原理被不同的团队、用不同的工程决策组合起来时，会得到什么样的产物？**

换句话说，本章是**原理到产品的映射层**。我们会看到：
- 同样的 Decoder-only Transformer 架构，为什么 DeepSeek 选择 MLA(Multi-head Latent Attention)而 Llama 3 坚持使用 GQA？
- 同样的 RLHF 对齐框架，为什么 Anthropic 发展出了 Constitution AI 和 RL-CAI，而 OpenAI 转向了基于人类偏好的 DPO 变体？
- 同样的 MoE(Mixture of Experts)结构，为什么 Mistral 的 8×7B 与 DeepSeek-V3 的 256 位专家在路由策略和负载均衡上走向了完全不同的设计哲学？

本章之后，第 6 章将带你进入"怎么让模型跑得更快更便宜"的优化世界。届时你会发现，很多在第 5 章中读到的架构选择(如 GQA vs MHA、Dense vs MoE、长上下文策略)，直接决定了第 6 章中优化手段的设计空间。

## 3. 国内外模型全景图

### 3.1 国际阵营：闭源引领，开源跟进

国际大模型生态呈现出清晰的"双轨制"特征：闭源巨头掌握最前沿的能力边界，开源社区紧随其后，通过透明化和 democratization 推动技术普惠。

**闭源旗舰阵营**以 OpenAI、Anthropic 和 Google 为核心。OpenAI 的 GPT-4 系列(包括 GPT-4 Turbo、GPT-4o 及其多模态变体)长期被视为通用能力的标杆，其在复杂推理、代码生成和工具使用上的稳定性，至今仍是商业应用的首选。Anthropic 的 Claude 系列则以安全性、长上下文(Claude 3 系列已支持 200K token)和"有帮助且无害"(Helpful, Harmless, Honest, HHH)的对齐理念著称，Claude 3.5 Sonnet 在 2024 年中期一度在多个 benchmark 上追平甚至超越 GPT-4o。Google 的 Gemini 系列则代表了原生多模态架构的最高水平，Gemini 1.5 Pro 的 1M+ 上下文窗口和原生视频理解能力，展示了 Google 在基础设施和数据资源上的深厚积累。

**开源核心阵营**的旗手是 Meta 的 Llama 系列。Llama 2 在 2023 年中期的发布，标志着"开源模型首次达到商用级别"的历史性拐点; Llama 3(8B/70B/405B)则通过激进的 scaling——15T+ token 预训练、大幅扩展的词表、以及精心的后训练配比——将开源模型的能力边界推向了接近闭源旗舰的水平。围绕 Llama 形成的生态(Hugging Face 上的 fine-tune 变体、量化版本、领域适配模型)是当前开源社区最繁荣的星系。

除 Llama 外，Mistral AI 的 Mistral 7B 和 Mixtral 8×7B(Sparse MoE)以其高效的参数利用率和创新的架构设计，成为欧洲开源力量的代表。Google 的 Gemma 系列(Gemma 2B/7B/9B/27B)则展示了如何在学术友好型小模型上实现接近大模型的能力密度。xAI 的 Grok-1 虽以"开源"形式发布，但其后续的 Grok-1.5/2 已回归闭源，更多作为 Elon Musk 的 X 平台生态工具存在。

### 3.2 国内阵营：百花齐放，各有专攻

中国的大模型生态呈现出更为多元化的格局，不同玩家在模型架构、训练策略和应用落地上走出了差异化的技术路径。

**全能型选手**以阿里通义千问(Qwen)和 DeepSeek 为代表。Qwen 系列(从早期的 Qwen-7B/14B/72B 到 Qwen2、Qwen2.5，再到支持多模态的 Qwen-VL 和代码专用的 Qwen-Coder)以其出色的多语言能力(尤其在中英文混合场景)和开放的商用许可，成为国内开源模型的标杆之一。DeepSeek 则从 DeepSeek-V2 的 MLA 架构创新，到 DeepSeek-V3 的极致工程优化(仅用约 500 万美元训练成本达到接近 GPT-4o 的水平)，再到 DeepSeek-R1 的 reasoning 能力突破，走出了一条"以工程创新换成本优势"的独特道路。

**长上下文与 Agent  Specialist**以月之暗面的 Kimi 为代表。Kimi 从早期就以 200K 超长上下文窗口作为核心卖点，其基于滑动窗口注意力(Sliding Window Attention)和上下文压缩技术的工程实现，为长文档处理和多轮对话场景提供了独特的价值主张。Kimi 在 2024 年推出的 Kimi k1.5 进一步强化了 reasoning 和 agentic 能力。

**架构创新派**以智谱 AI 的 GLM(General Language Model)系列为代表。GLM 采用的自回归空白填充(Autoregressive Blank Infilling)架构，在预训练目标函数上与标准的 next-token prediction 有本质区别，这使其在文本理解和生成之间取得了不同的权衡。ChatGLM 系列作为国内最早开源的商用级对话模型之一，在教育、科研和小型企业场景中拥有广泛的用户基础。

**小模型极致派**以面壁智能的 MiniCPM 为代表。MiniCPM 系列(2B 参数级别)通过"知识密度"(capability density)的极致优化，在端侧部署和边缘计算场景中展现了惊人的性价比，其提出的"密度法则"(Density Law)对行业的小模型设计产生了深远影响。

**闭源商业巨头**包括百度的文心一言(ERNIE Bot)、字节跳动的豆包、腾讯的混元、MiniMax 的 abab 系列等。这些模型主要服务于各自的超级 App 生态，在 C 端用户触达上具有天然优势，但其底层技术细节公开程度较低，更多通过 API 能力和产品体验参与竞争。

## 4. 如何评估一个模型：四个核心维度

面对数十个模型家族、数百个具体 checkpoint，建立一套系统性的评估框架至关重要。我们提出**能力、效率、成本、生态**四个核心维度，作为贯穿本章的分析坐标系。

### 4.1 能力维度(Capability)

能力维度回答"这个模型有多聪明"的问题。但"聪明"是一个过于笼统的描述，我们需要将其拆解为可度量的子维度：

- **知识广度(Knowledge)** ：模型在预训练阶段接触了多少领域的信息，是否覆盖专业领域(法律、医学、金融)的知识。常用 benchmark：MMLU(大规模多任务语言理解)、C-Eval(中文综合评估)、CMMLU(中文多任务语言理解)。

- **推理深度(Reasoning)** ：模型在需要多步逻辑推导的任务上的表现。常用 benchmark：GSM8K(小学数学应用题，测试多步算术推理)、MATH(高中竞赛级数学题)、HumanEval / MBPP(代码生成与逻辑推理)。

- **指令遵循(Instruction Following)** ：模型能否准确理解并执行复杂、多约束的指令。常用 benchmark：IFEval(指令遵循评估)、MT-Bench(多轮对话能力)。

- **长上下文(Long Context)** ：模型在超长文本(10K–1M token)中保持注意力、进行信息检索和推理的能力。常用 benchmark："Needle in a Haystack"(大海捞针测试)、RULER、LV-Eval。

- **多模态(Multimodality)** ：模型处理文本、图像、音频、视频混合输入的能力。代表模型：GPT-4o、Gemini 1.5 Pro、Qwen-VL、Claude 3(vision)。

- **创造力与安全性(Creativity & Safety)** ：模型在开放式生成中的有用性(Helpfulness)和无害性(Harmlessness)。评估方式：人类偏好评估、红队测试(Red Teaming)、宪法 AI 一致性测试。

### 4.2 效率维度(Efficiency)

效率维度回答"这个模型跑起来有多快、有多省"的问题。在算力经济学日益成为核心约束的今天，效率与能力同等重要。

- **推理吞吐量(Throughput)** ：单位时间内能处理多少 token。受架构选择(GQA 比 MHA 快、MLA 比 GQA 更省显存)、序列并行策略、以及底层推理框架(vLLM、TensorRT-LLM、SGLang)的共同影响。

- **显存占用(Memory Footprint)** ：模型权重、KV Cache、激活值占用的显存总量。这决定了单张 GPU/单台机器能承载多大的 batch size 和上下文长度。量化(INT8/INT4/FP8)、KV Cache 压缩(如 H2O、SnapKV)、以及架构层面的 KV 共享(GQA、MLA)都是降低显存占用的关键手段。

- **首 token 延迟(Time to First Token, TTFT)** ：从用户发送请求到收到第一个输出 token 的延迟。这在交互式应用中(如聊天机器人)直接决定用户体验。

- **参数效率(Parameter Efficiency)** ：单位参数量能承载多少能力。MiniCPM 提出的"密度法则"正是对这一维度的极致追求——2B 参数做到接近 7B 模型的能力，意味着在端侧部署时的巨大优势。

### 4.3 成本维度(Cost)

成本维度回答"训练和使用这个模型要花多少钱"的问题。成本包括训练成本和推理成本两个层面。

- **训练成本(Training Cost)** ：预训练和后训练阶段消耗的总算力，通常以 GPU-hours 或美元计。DeepSeek-V3 的训练成本约为 557.6 万美元(使用 H800 GPU)，而同等规模的模型在传统训练流程下可能需要数千万甚至上亿美元。这种成本差异主要来自数据效率、架构优化(如 MoE 的稀疏激活)和训练策略(如 multi-token prediction)。

- **推理成本(Inference Cost)** ：每百万 token 的输入/输出定价。闭源 API 的定价策略差异巨大(GPT-4o 的输入约 $2.5/M token，Claude 3 Haiku 约 $0.25/M token，而 DeepSeek-V3 的 API 定价仅为 $0.14/M token)。开源模型则取决于自部署的硬件成本。

- **总拥有成本(TCO)** ：对于企业用户，还需要考虑模型微调的工程成本、运维成本、以及因模型能力不足导致的下游纠错成本。

### 4.4 生态维度(Ecosystem)

生态维度回答"这个模型能多好地融入我的工作流"的问题。一个模型的技术再先进，如果缺乏工具链支持、社区贡献和商用许可的清晰度，其落地价值也会大打折扣。

- **开源许可(License)** ：Apache 2.0(Llama 3 早期版本、Qwen 系列)是最宽松的，允许商业使用和修改; Llama 3.1+ 的定制许可对月活超过 7 亿的企业有额外限制; MIT(Mistral 部分模型)同样非常宽松; 而 GPT/Claude/Gemini 等闭源模型只能通过 API 访问。

- **工具链支持(Tooling)** ：是否有成熟的推理框架支持(vLLM、llama.cpp、Ollama)、是否有 Hugging Face Transformers 的官方集成、是否支持主流量化方案(GGUF、AWQ、GPTQ)。

- **社区活跃度(Community)** ：Hugging Face 上的下载量、微调变体数量、第三方评测覆盖度。Llama 3 的社区生态是当前最繁荣的，Qwen 在国内开发者中也有极高的采用率。

- **垂直适配(Vertical Adaptation)** ：是否有针对特定领域(代码、数学、法律、医疗)的官方或社区 fine-tune 版本。例如 CodeLlama、Qwen-Coder、DeepSeek-Coder 等代码专用模型，就是生态成熟度的体现。

## 5. 阅读指南：如何高效使用本章

### 5.1 本章结构导览

本章按"总—分—总"的结构组织：

- **5.1 模型对比与总结**：一张横向对比的全景图。如果你时间有限，只想快速了解"现在该用哪个模型"，读这一节就够了。我们提供了详细的对比表、评估基准解读、以及按场景(个人开发、企业应用、学术研究、端侧部署)的选型建议。

- **5.2 国内大模型深度解读**：对 Qwen、DeepSeek、GLM、Kimi、MiniCPM 等国内代表性模型进行逐一解剖。每一篇都遵循统一的"技术档案"模板：模型定位 → 技术档案卡 → 架构细节 → 训练数据与流程 → 性能评测 → 局限性与边界条件 → 演进脉络。

- **5.3 国外大模型深度解读**：对 GPT-4 系列、Claude 系列、Llama 系列、Gemma、Mistral 等国际模型进行同样的深度解剖。

- **5.4 大模型发展历程**：一条时间线，梳理从 GPT-1(2018)到 GPT-4o / Llama 3 / DeepSeek-V3(2024–2025)的关键技术里程碑，帮助你在历史脉络中理解"为什么今天的模型长这样"。

### 5.2 推荐的阅读路径

**路径 A：选型导向(工程师/产品经理)** 
1. 阅读本章总览(你正在读的这篇)→ 建立评估框架
2. 跳到 5.1 → 查看对比表和选型建议
3. 根据你的场景(API 调用 vs 本地部署、中文 vs 英文、通用 vs 垂直)，深入阅读 5.2 或 5.3 中对应模型的深度解读

**路径 B：研究导向(算法工程师/研究生)** 
1. 阅读本章总览 → 建立评估框架
2. 选择你最感兴趣的 1-2 个模型家族，完整阅读其在 5.2 或 5.3 中的深度解读
3. 特别关注每个模型解读中的"局限性与边界条件"和"演进脉络"——这两节最能启发新的研究思路
4. 阅读 5.4 → 将单点认知放入历史时间线中

**路径 C：全栈扫盲(学习者/转行者)** 
1. 确保你已经读完了第 2 章(核心原理与架构)和第 4 章(后训练)
2. 按顺序阅读本章全部内容
3. 每读完一个模型的深度解读，尝试回答："这个模型与上一个读的相比，在架构或训练策略上的核心差异是什么？"

## 6. 写在前面：关于时效性的诚实声明

大模型领域的技术迭代速度是惊人的。Llama 3 发布于 2024 年 4 月，DeepSeek-V3 发布于 2024 年 12 月，Qwen3 和 Gemini 2.0 的新版本在 2025 年持续更新。当你读到本章时，可能已经出现了更新的 checkpoint、更高的 benchmark 分数、或全新的架构变体。

**我们无法向你保证这里的每一个数字永远是最新的——那是骗子才敢说的大话。** 但我们能保证的是：
- 每一个模型的解读都基于其官方技术报告、论文或可信的第一手资料; 
- 每一个架构选择都被追问了"为什么"和"代价是什么"; 
- 每一个 benchmark 数字都附带了评估条件和版本信息，让你能判断它是否仍然有效; 
- 每一个模型的局限性和边界条件都被诚实地写出，而非被营销话术掩盖。

如果你发现某个模型的信息已经过时，或某个新发布的模型值得纳入本章，请参考知识库首页的"贡献与更新机制"进行反馈。本章是一个持续生长的有机体，而非一次性写就的教科书。
