基于这个评估集的人工注释者的偏好排名突显了我们的 700 亿参数的指令遵循模型在真实场景中相比同等规模的竞争模型表现出色。

为了开发出一个优秀的语言模型，我们认为创新、扩展和简化优化是非常重要的。我们在 Llama 3 项目中贯彻了这一设计理念，重点关注了四个关键因素：**模型架构、预训练数据、扩大预训练规模以及指导微调**。

## 2. 模型架构

我们在 Llama 3 中选择了一个相对标准的Encoder-Only的变压器架构。与 Llama 2 相比(Llama2 The total vocabulary size is 32k tokens)，我们进行了几项关键改进。**Llama 3 使用了一个词汇量为 128K 的分词器，更高效地编码语言，从而显著提高了模型的性能**。为了提高 Llama 3 模型的推理效率，我们在 8B 和 70B 大小的模型全系中都采用了**分组查询注意力(GQA)** ( Llama 2 只有34B以上的模型才使用GQA)

我们对长度为 **8,192 个标记**的序列进行了训练，并使用掩码确保自注意力不会跨越文档边界。

## 3. 训练数据

为了训练出最优秀的语言模型，筛选出大规模、高质量的训练数据至关重要。**Llama 3 在预训练数据上使用了超过 15T 个标记**，这些标记均来自公开可用的来源。我们的训练数据集比 **Llama 2 使用的2T数据集大七倍**，其中包含四倍的代码。为了为即将到来的多语言用例做准备，Llama 3 预训练数据集中超过 5% 的内容是高质量的非英语数据，涵盖了 30 多种语言(Llama2 28种)

![](./08-Llama2到Llama-3的架构演进-images/image_2.jpg)

Llama2 训练语言分布

为了确保 Llama 3 在最高质量的数据上进行训练，我们开发了一系列数据过滤流水线。这些流水线包括使用**启发式过滤器、NSFW 过滤器、语义去重方法和文本分类器**来预测数据质量(Llama 2论文未提及，详情可参考Yi大模型论文写的很详细)。发现以前的 Llama 2在识别高质量数据方面表现出惊人的优秀，因此**使用 Llama 2 生成了用于支持 Llama 3 的文本质量分类器的训练数据**。

我们还进行了大量实验，**评估了在最终预训练数据集中混合来自不同来源的数据的最佳方式**(Llama3技术报告未公布)。这些实验使我们能够选择一个数据组合，确保 Llama 3 在包括琐事问题、STEM、编码、历史知识等各种用例中表现良好。

## 4. 扩大预训练规模

对于一个 80 亿参数模型而言，最佳训练计算量的量级对应着大约 200B 个token，但我们发现即使在模型训练了两个数量级的数据后，模型性能仍然会持续改善。**我们的 80 亿和 700 亿参数模型在训练了多达 15T 个Token后仍然呈现对数线性改善**(Llama2 2T个Token)。**较大的模型可以通过更少的训练计算力达到与这些较小模型相当的性能**，但较小模型通常更受欢迎，因为它们在推理过程中更加高效。

## 5. Instruction微调

为了充分发挥我们预训练模型在聊天用例中的潜力，我们还对我们的指导微调方法进行了创新。我们的后训练方法是**受监督的微调(SFT)、拒绝采样、近端策略优化(PPO)和直接策略优化(DPO)的结合**(Llama2中也使用了受监督的微调(SFT)、拒绝采样、近端策略优化(PPO)方法，但是未提到直接策略优化(DPO)具体方法可参考 [DPO (PPO代替)解读与实践【大模型论文系列】](https://zhuanlan.zhihu.com/p/655421669))。

其中针对于拒绝采样是一种PPO优化方法。PPO的方法中仅会生成一个样本，在训练的每一步t中，样本是根据上一步的梯度更新后的更新模型策略的函数。而在拒绝抽样中，模型为给定提示探索了K个样本。在微调中，我们对模型的初始策略生成所有输出以收集新数据集，然后类似于SFT进行微调。

## 6. 总结

对比Llama2 到Llama 3的架构演进，发现有几点会提升大模型效果：

1. Llama 3 使用了一个词汇量为 128K 的分词器，更高效地编码语言，从而显著提高了模型的性能。2. Llama 3 在预训练数据上使用了超过 15T 个标记3. 为了确保 Llama 3 在最高质量的数据，使用**启发式过滤器、NSFW 过滤器、语义去重方法和文本分类器**
