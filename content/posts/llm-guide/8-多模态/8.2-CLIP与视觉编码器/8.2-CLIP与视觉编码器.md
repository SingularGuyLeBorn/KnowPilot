# 8.2 CLIP 与视觉Encoder 

## 1. 演进脉络与技术演进

2020 年前的计算机视觉被监督学习锁死在 ImageNet 的 1000 个类别中. AlexNet、VGG、ResNet 的迁移学习虽然强大, 但模型世界观被标注预算的上限决定. 一条关于 "金毛犬在草地上奔跑" 的 query, ResNet-50 最多回答 "狗"——它从未被训练过去理解开放语义. 分类任务的输出空间是一个预先定义的离散集合, 模型从未学习开放词汇的语义表达. 

更深层的问题在于, 传统视觉监督学习从未真正教会模型 "理解" 图像, 而只是教会了模型 "记忆" 人工定义的分类边界. 模型学到的特征表示围绕 1000 个分类决策面组织, 在判别 "哈士奇" 和 "狼" 时有效, 但在描述 "一只孤独的狼站在雪原上仰望星空" 时几乎空白. 

NLP 领域同期完成了范式跃迁. 2018 年 BERT 的 MLM 任务与 GPT 的自回归目标证明: **数据本身就是监督信号**. 文本的序列结构蕴含语法、语义和世界知识, 模型通过预测即可习得表示. 2019-2020 年 GPT 系列将这一思想推向极致: 千亿参数、海量互联网文本, 模型涌现出事实问答、推理、翻译、编程能力. 

这启发了视觉领域的核心追问: 图像是否存在类似的自监督结构？能否用海量无标注图像-文本对训练出开放词汇的视觉理解能力？

OpenAI 于 2021 年发布的 CLIP(Contrastive Language-Image Pre-training)给出了肯定答案. CLIP 摒弃逐像素标注, 利用互联网上 4 亿对图像-文本数据, 通过对比学习将两种模态映射到同一语义空间. 它在 ImageNet 上的 Zero-Shot 准确率追平了全量监督训练的 ResNet-50——后者消耗了 128 万张人工标注图片, 前者在训练时一张 ImageNet 标注图都未见过. 

CLIP 之后, 对比学习框架经历了三重关键演进: 

| 阶段 | 时间 | 核心突破 | 工程影响 |
|------|------|----------|----------|
| CLIP | 2021 | InfoNCE + 双塔对称Encoder  + 超大 batch | 奠定 VLM 视觉 backbone 范式 |
| OpenCLIP / LiT / FLAVA | 2022-2023 | 开源复现、锁定图像端微调文本、多任务预训练 | 降低复现门槛, 验证分阶段训练策略 |
| SigLIP / SigLIP 2 | 2023-2024 | Sigmoid 损失替代 Softmax, 支持非对称 batch | 成为 2024-2026 年 VLM 默认视觉Encoder  |
| 原生多模态 | 2024-至今 | GPT-4o、Gemini 的统一 token 化架构 | 视觉Encoder 或从独立模块走向原生融合 |

视觉 Transformer(ViT)的成熟为 CLIP 提供了关键的架构基础. 2020 年 Google 的 ViT 论文证明: 将图像切分为固定大小的 patch, 线性投影后输入标准 Transformer Encoder, 不依赖 CNN 的归纳偏置也能学习强大的视觉表示. CLIP 在图像侧同时实验了 ResNet 和 ViT, 结果 ViT 变体在大多数任务上优于 ResNet——这与整个视觉领域从 CNN 向 ViT 迁移的趋势一致. 

CLIP 并非视觉表示学习的唯一路径. DINOv2(Meta, 2023)通过自蒸馏在纯视觉数据上训练 ViT, 在密集预测任务上弥补了 CLIP 局部特征的短板. EVA-CLIP 则验证了 "MAE 自监督初始化 + 对比学习微调" 的分阶段策略优于端到端训练. 这些并行演进共同构成了现代多模态 AI 的视觉地基. 

## 2. 工业价值与应用场景

CLIP 及其改进版本已成为所有现代 VLM 的**事实标准视觉Encoder **. LLaVA、Qwen-VL、InternVL、DeepSeek-VL、Gemini、PaliGemma 的视觉模块无一例外基于 CLIP 或 SigLIP. 在 Encoder-Projector-Decoder 范公公式子中, 视觉Encoder 质量直接决定 VLM 的 "视力"——语言模型再强大, 接收不到准确的视觉信号也无法有效推理. 

CLIP 的工业落地覆盖三个核心场景: 

**Zero-Shot 开放词汇分类**
传统分类器被预定义类别集合锁死. CLIP 允许用户用任意自然语言描述作为类别, 计算图像与每个描述的相似度后取 argmax. 分类能力不再受训练时见过的类别限制, 只要能用语言描述的概念, CLIP 都可能识别. 

**跨模态检索**
将图库图像编码为向量存入向量数据库. 用户输入文本查询后, 文本Encoder 生成 query 向量, 通过最近邻搜索返回语义匹配的图像. CLIP 将图像和文本映射到同一语义空间, 使跨模态检索效果远超传统基于标签的方法. 

**多模态表征与 VLM backbone**
CLIP 训练出的联合嵌入空间成为后续研究的基石. 任何需要将不同模态映射到统一语义空间的工作, 几乎都以 CLIP 为起点. 在 VLM 中, CLIP 视觉Encoder 输出 patch-level 特征序列, 经 projection 层映射到 LLM 的 embedding 空间, 供语言模型进行自回归生成. 

| 应用场景 | 关键指标 | CLIP/SigLIP 的作用 |
|----------|----------|-------------------|
| VLM 视觉 backbone | 特征语义密度、分辨率、吞吐量 | 将像素转换为 LLM 可消费的语义向量 |
| 图像-文本检索 | Recall@K、检索延迟 | 统一嵌入空间支持跨模态最近邻搜索 |
| 内容审核 | 误报率、漏报率 | Zero-Shot 识别违规概念的开放描述 |
| 文生图条件控制 | 文本-图像对齐精度 | Stable Diffusion、DALL-E 的文本Encoder 来源 |
| 机器人视觉导航 | 实时性、指令跟随精度 | 将自然语言指令映射到视觉观测的语义空间 |

## 3. 物理直觉与核心机制

图像和文本是同一组语义实体在两种不同 "语法" 下的表达. 文本由离散 token 组成, 每个 token 携带明确语义指称; 图像由连续像素构成, 单个像素无意义, 语义在足够大的空间结构中涌现. 将二者对齐, 本质是异构空间的翻译问题. 

这种异构性带来了三个层面的对齐难题. 第一, 粒度不对齐: 文本的最小语义单位是 token, 图像的最小处理单位是像素, 但单个像素几乎没有语义, 视觉语义的基本单元通常是 "对象" 或 "区域". 第二, 结构不对齐: 文本具有序列结构, 词序编码语法和逻辑; 图像具有二维空间结构, 位置关系、遮挡关系、尺度关系构成视觉骨架. 第三, 语义密度不对齐: 文本中每个词都承载信息, 图像中大量像素可能是冗余的背景或重复纹理. 一个视觉概念可能需要很长的文本描述才能精确表达. 

CLIP 不追求逐词逐像素的细粒度对齐——这种标注昂贵且难以获取. 它利用互联网上天然存在的图像-文本对, 仅要求模型回答一个弱监督问题: **"在这 N 段文本中, 哪一段与这张图片匹配？"**

对比学习的物理直觉可类比为语言学习班的配对测试. Batch 中 $N$ 张图片与 $N$ 段文本一一对应. 对角线上的 $(x_i, y_i)$ 是正样本, 非对角线上的 $(x_i, y_j)$($j \neq i$)是负样本. 模型从未被告知 "图片的哪一部分对应文本的哪个词", 只被告诉 "这张图和这句话是一对". 

在表示空间中, CLIP 将所有图像和文本的 embedding 限制在 $d$ 维单位超球面上. 归一化后的点积等于余弦相似度, 模型只关心向量方向的相似性, 不关心模长. 如果没有归一化, 模型可能通过简单增大所有向量模长来人为提高相似度——归一化消除了这种 "作弊" 可能, 迫使模型真正学习有意义的方向对齐. 在球面上, 相似样本聚集成簇, 不相似样本彼此远离, 这种几何结构为下游最近邻检索提供了良好的空间基础. 

这种弱监督在 4 亿对数据规模下产生惊人的统计规律: 

- 描述中包含 "cat" 的文本, 几乎总是与包含猫的图片配对. 
- 描述中包含 "red" 的文本, 配对的图片中大概率存在红色物体. 
- 描述中包含 "running" 的文本, 对应图片大概率包含运动物体. 

"语义共现"的统计信号足够强, 模型自动推断出哪些视觉特征与哪些文本概念关联. 这类似于婴儿学语言: 不需要指着每个物体逐一教学, 通过日常生活中的自然观察即可建立词汇与世界的映射. 

对比学习将问题转化为 batch 内的多分类任务. 对于第 $i$ 张图片, $N$ 个候选文本构成 $N$ 个类别, 其中第 $i$ 类为正、其余 $N-1$ 类为负. 类别数等于 batch size, 每一步训练时都在变化, 模型习得的是开放集分类能力. 

## 4. 数学推导与公式对比

本节给出核心公式的精确定义, 完整推导与数值走查详见子文章 [01-CLIP与视觉Encoder ](./01-CLIP与视觉Encoder .md). 

CLIP 同时学习两个Encoder : 图像Encoder  $f_I: \mathcal{X} \to \mathbb{R}^d$ 和文本Encoder  $f_T: \mathcal{Y} \to \mathbb{R}^d$. 训练 batch 包含 $N$ 对匹配样本 $\{(x_i, y_i)\}_{i=1}^N$. 经 L2 归一化后, 余弦相似度退化为点积: 

$$
 \text{sim}(f_I(x_i), f_T(y_j)) = f_I(x_i)^\top f_T(y_j) \tag{1}
$$

其中 $f_I$ 为图像Encoder , 将输入图像 $x$ 映射到 $d$ 维嵌入空间; $f_T$ 为文本Encoder , 将输入文本 $y$ 映射到同一 $d$ 维嵌入空间. $x_i$ 和 $y_j$ 分别表示 batch 中第 $i$ 张图像和第 $j$ 段文本. 经 L2 归一化后, $f_I(x_i)$ 和 $f_T(y_j)$ 均为 $d$ 维单位向量, 消除了模长差异, 仅保留方向信息; 两者的点积 $f_I(x_i)^\top f_T(y_j)$ 等价于夹角的余弦值, 度量图像与文本的语义匹配程度, 值域 $[-1, 1]$, 正样本趋近于 $1$, 负样本趋近于 $-1$. 这一步将异构模态的语义对齐转化为统一空间中的几何距离度量. 

InfoNCE 损失(图像到文本方向)定义为: 

$$
 \mathcal{L}_{I \to T} = -\frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(\text{sim}(f_I(x_i), f_T(y_i)) / \tau)}{\sum_{j=1}^{N} \exp(\text{sim}(f_I(x_i), f_T(y_j)) / \tau)} \tag{2}
$$
其中 $\mathcal{L}_{I \to T}$ 为图像到文本方向的 InfoNCE 对比损失; $N$ 为训练 batch 中的图文对数量; $i$ 和 $j$ 分别为图像索引和文本候选索引. $\text{sim}(f_I(x_i), f_T(y_j))$ 表示图像 $x_i$ 与文本 $y_j$ 的余弦相似度, $\tau$ 为可学习的温度参数, 控制 softmax 分布的尖锐程度. 

分子 $\exp(\text{sim}(f_I(x_i), f_T(y_i)) / \tau)$ 对正样本对的相似度做温度缩放后的指数变换, 将相似度映射到正实数域并放大差异; $\tau$ 越小, 指数差异越悬殊. 分母 $\sum_{j=1}^{N} \exp(\text{sim}(f_I(x_i), f_T(y_j)) / \tau)$ 对第 $i$ 张图像与 batch 内所有文本的相似度取温度缩放后的指数和, 作为归一化常数, 将所有候选文本的竞争力纳入同一量纲. 分式整体将匹配度转换为 batch 内 $N$ 路分类的 softmax 概率, 把"多分类正确率"转化为可微分的损失目标. 对正样本概率取对数后, $[0,1]$ 的概率被映射到 $(-\infty, 0]$, 概率越接近 $1$, 损失越接近 $0$, 从而让正确分类获得最大似然奖励. 最后对所有图像取平均并取负号, 将最大化正样本概率转化为最小化损失函数, 以适配梯度下降优化器. 

CLIP 的最终损失是两个方向损失的均值: 

$$
 \mathcal{L}_{\text{CLIP}} = \frac{1}{2} \left( \mathcal{L}_{I \to T} + \mathcal{L}_{T \to I} \right) \tag{3}
$$

其中 $\mathcal{L}_{\text{CLIP}}$ 为 CLIP 的总体对比损失, 由图像→文本方向的 InfoNCE 损失 $\mathcal{L}_{I \to T}$ 和文本→图像方向的 InfoNCE 损失 $\mathcal{L}_{T \to I}$ 经对称平均系数 $\frac{1}{2}$ 加权得到. $\mathcal{L}_{I \to T}$ 以每张图像为查询、batch 内所有文本为候选, 迫使图像 embedding 靠近其匹配文本、远离其他文本; $\mathcal{L}_{T \to I}$ 则反向操作, 迫使文本 embedding 靠近其匹配图像、远离其他图像. 对两个方向取算术平均是为了消除单一方向可能引入的表示偏差, 使两个Encoder 在联合嵌入空间中双向等价. 

温度参数 $\tau$ 控制分布尖锐程度. $\tau \to 0^+$ 时 softmax 退化为 hard max, 只关注最难负样本; $\tau \to +\infty$ 时趋向均匀分布, 忽略负样本难度差异. CLIP 将 $\tau$ 设为可学习参数, 初始值约 0.07. 

SigLIP 将全局 softmax 替换为逐对 sigmoid: 

$$
 \mathcal{L}_{\text{SigLIP}} = -\sum_{i,j} \left[ y_{ij} \log \sigma\left(\frac{z_i^\top z_j}{\tau}\right) + (1 - y_{ij}) \log \left(1 - \sigma\left(\frac{z_i^\top z_j}{\tau}\right)\right) \right] \tag{4}
$$
其中 $\mathcal{L}_{\text{SigLIP}}$ 为 SigLIP 的逐对 sigmoid 二元交叉熵损失; $i$ 和 $j$ 分别遍历图像索引和文本索引, 支持非对称 batch($N_I$ 张图、$N_T$ 段文本). $y_{ij} \in \{0, 1\}$ 为标签, $y_{ij}=1$ 当且仅当图像 $i$ 与文本 $j$ 为匹配对(对角线), 否则为 $0$. $\sigma(x) = \frac{1}{1+e^{-x}}$ 为 sigmoid 函数, 将 logits 映射到 $(0,1)$ 概率; $z_i$ 和 $z_j$ 分别为图像 $i$ 和文本 $j$ 经 L2 归一化后的 $d$ 维 embedding; $\tau$ 为温度参数, 缩放 logits 的幅值. 

$z_i^\top z_j$ 度量图像与文本的语义相似度, 与 CLIP 相同, 但此处不做 softmax 全局归一化, 而是逐对独立判断. 温度缩放项 $\frac{z_i^\top z_j}{\tau}$ 控制 sigmoid 曲线的陡峭程度: $\tau$ 越小, 正负样本的决策边界越清晰. $\sigma\left(\frac{z_i^\top z_j}{\tau}\right)$ 将连续相似度转化为可解释的二元概率估计 $\hat{p}_{ij}$. 正样本项 $y_{ij} \log \hat{p}_{ij}$ 在 $y_{ij}=1$ 时鼓励 $\hat{p}_{ij}$ 趋近 $1$, 若正样本被误判则产生强负梯度; 负样本项 $(1-y_{ij}) \log(1-\hat{p}_{ij})$ 在 $y_{ij}=0$ 时鼓励 $\hat{p}_{ij}$ 趋近 $0$, 且每个负样本独立贡献损失, 不与其他负样本竞争归一化分母. 这种逐对独立的设计解除了 CLIP 对 softmax 全局归一化的依赖, 允许使用非对称 batch 和更大的 batch size. 
- $-\sum_{i,j}$: 对所有图像-文本对求和并取负号. 与 CLIP 的 softmax 不同, 此处每个 pair 的梯度独立计算, 无需 all-gather 全局归一化, 支持非对称 batch, 且对假阴性更鲁棒. 

关键差异在于: CLIP 的 softmax 要求全局归一化、对称 batch、图像数等于文本数; SigLIP 的 sigmoid 逐对独立计算, 支持非对称 batch, 消除分布式训练中的 all-gather 瓶颈, 对假阴性更鲁棒. 

| 特性 | CLIP (Softmax + CE) | SigLIP (Sigmoid + BCE) |
|------|---------------------|------------------------|
| 归一化 | 全局 softmax, 跨 batch 归一化 | 逐对 sigmoid, 无需全局归一化 |
| Batch 对称性 | 要求 $N_I = N_T$ | 支持非对称 batch |
| 分布式训练 | 需要 all-gather 通信 | 每个 pair 独立计算 |
| Batch size 敏感性 | 高 | 中 |
| 假阴性处理 | 严重干扰 softmax 归一化 | 影响相对较小 |
| 梯度特性 | 负样本通过分母耦合 | 正负样本梯度独立 |

## 5. 算力成本与数值走查

对比学习的计算效率来源于矩阵运算的复用. 一次前向传播生成 $N \times N$ 个相似度, 但只需要 $2N$ 个Encoder 前向传播($N$ 个图像 + $N$ 个文本). 相似度矩阵 $S = I \cdot T^\top$ 的计算复杂度为 $O(N^2 d)$, 当 $d \ll N$ 时, Encoder 前向是主要开销. 

有效 batch size 决定负样本丰富度. Batch size 为 $N$ 时, 每个查询面对 $N-1$ 个负样本, 总训练信号数量为 $N^2$ 对. 对比学习二次于 batch size: batch size 翻倍, 负样本数量变为四倍. CLIP 原始训练使用 256 张 V100, 总 batch size 达到 32768——没有足够的负样本, 对比学习失去 "对比" 意义. 

从梯度估计角度分析, InfoNCE 的梯度可看作基于负样本采样的蒙特卡洛估计. 更多负样本意味着对分母归一化项的更准确估计, 梯度方差更小, 训练更稳定. 当 $N=2$ 时, 每个查询仅有一个负样本, 若该负样本与查询极不相似, 模型几乎学不到有用梯度; 若碰巧很相似, 又会导致梯度不稳定. 当 $N=32768$ 时, 负样本覆盖从动物到风景、从抽象到具体的丰富语义空间, 容易的负样本提供稳定基础信号, 困难的负样本迫使模型学习细粒度区分. 

以 ViT-L/14 为例, 单张 $224 \times 224$ 图像被切分为 $(224/14)^2 = 256$ 个 patch. ViT-L 参数量约 307M, 文本Encoder 参数量约 122M, 总模型规模约 430M. 在 32768 batch size 下, 相似度矩阵存储需要 $32768^2 \times 4\text{ bytes} \approx 4.3\text{ GB}$(fp32), 这要求训练框架具备高效的内存管理和梯度Checkpoint策略. 

温度参数 $\tau$ 对数值行为的影响同样显著. 设 $\tau = 0.07$, 当正样本相似度为 $0.90$、负样本相似度为 $0.30$ 时, 温度缩放后的 logits 分别为 $12.86$ 和 $4.29$. softmax 分母中 $\exp(12.86) \approx 3.84 \times 10^5$ 主导, 正样本概率接近 $0.9998$, 损失贡献仅约 $0.0002$. 这说明当正样本已很好区分时, CLIP 的损失值极小, 模型主要依赖困难负样本提供梯度信号. SigLIP 的行为则不同: 每个负样本都有独立的二元交叉熵损失项, 即使 "容易" 的负样本也会贡献非零梯度, 训练信号更密集. 

| 组件 | CLIP ViT-B/32 | CLIP ViT-L/14 | SigLIP So400m/14 |
|------|--------------|---------------|------------------|
| 图像Encoder 参数量 | ~88M | ~307M | ~400M |
| 文本Encoder 参数量 | ~63M | ~122M | ~200M |
| 总参数量 | ~151M | ~430M | ~600M |
| 训练图像分辨率 | 224×224 | 224×224 / 336×336 | 224×224 |
| Patch 数量 | 49 | 256 / 576 | 256 |
| 训练数据量 | 4 亿对 | 4 亿对 | 数十亿对 |

## 6. 极简实现与源码映射

以下提供 CLIP 核心损失与模型骨架的极简 PyTorch 实现, 约 100 行的完整版本(含 ImageEncoder、TextEncoder、CLIPLoss、SigLIPLoss、CLIPModel 及公式对照表)见子文章 [01-CLIP与视觉Encoder ](./01-CLIP与视觉Encoder .md) 第 6 节. 

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class CLIPLoss(nn.Module):
    """
    CLIP 风格的对比损失 (InfoNCE)
    对应: L_CLIP = 0.5 * (L_{I->T} + L_{T->I})
    """
    def __init__(self, temperature=0.07, learnable_temp=True):
        super().__init__()
        if learnable_temp:
            self.logit_scale = nn.Parameter(torch.ones([]) * math.log(1.0 / temperature))
        else:
            self.register_buffer('logit_scale', torch.ones([]) * math.log(1.0 / temperature))

    def forward(self, image_features, text_features):
        N = image_features.shape[0]
        # S = I * T^T
        logits_per_image = image_features @ text_features.t()
        logits_per_text = logits_per_image.t()
        logit_scale = self.logit_scale.exp().clamp(max=100)
        logits_per_image = logits_per_image * logit_scale
        logits_per_text = logits_per_text * logit_scale
        labels = torch.arange(N, device=image_features.device)
        loss_i2t = F.cross_entropy(logits_per_image, labels)
        loss_t2i = F.cross_entropy(logits_per_text, labels)
        return (loss_i2t + loss_t2i) / 2


class SigLIPLoss(nn.Module):
    """
    SigLIP 风格的 sigmoid 二元交叉熵损失
    对应: L_SigLIP = -sum_{i,j} [y_{ij} log sigma(...) + (1-y_{ij}) log(1-sigma(...))]
    """
    def __init__(self, temperature=0.07, learnable_temp=True):
        super().__init__()
        if learnable_temp:
            self.logit_scale = nn.Parameter(torch.ones([]) * math.log(1.0 / temperature))
        else:
            self.register_buffer('logit_scale', torch.ones([]) * math.log(1.0 / temperature))

    def forward(self, image_features, text_features):
        N_I = image_features.shape[0]
        N_T = text_features.shape[0]
        logits = image_features @ text_features.t()
        logit_scale = self.logit_scale.exp().clamp(max=100)
        logits = logits * logit_scale
        labels = torch.eye(N_I, N_T, device=image_features.device)
        loss = F.binary_cross_entropy_with_logits(logits, labels, reduction='sum')
        return loss / (N_I * N_T)
```

核心源码映射要点: 

- `image_features @ text_features.t()` 对应相似度矩阵 $S = I \cdot T^\top$. 
- `F.normalize(x, dim=-1)` 对应 L2 归一化, 使点积等价于余弦相似度. 
- `F.cross_entropy(logits_per_image, labels)` 对应图像到文本方向的 softmax + CE. 
- `F.binary_cross_entropy_with_logits(logits, labels)` 对应 SigLIP 的逐对 sigmoid + BCE. 
- `self.logit_scale` 对应可学习的温度参数 $1/\tau$. 

## 7. 边界条件与失效模式

CLIP 与对比学习存在四个根本性失效边界, 工程部署时必须清醒认识: 

**假阴性(False Negatives)** 

Batch 内非对角线元素被强制标记为负样本, 但真实数据中语义相关的配对大量存在. 一张金毛犬图片与 "拉布拉多犬在海滩上奔跑" 的文本在语义上高度相关, 却被要求相似度接近 0. 在 CLIP 的 softmax 框架中, 假阴性参与分母竞争, 会严重干扰正样本学习; SigLIP 因 sigmoid 的逐对独立性, 影响相对较小. 根本原因是对比学习的监督信号是 "相对关系" 而非 "绝对语义". 

缓解策略: 扩大 batch size 从统计上降低 batch 内语义重复概率; 数据预处理阶段通过文本或图像特征去除过于相似的样本; 若能获得层次化标签(如 "狗" 是 "动物" 子类), 可设计层次化对比损失; 无负样本的替代方法如 DINO 自蒸馏也提供了另一条路径. 

**数据偏见**

CLIP 的训练数据来自互联网, 互联网不是中立的. 查询 "doctor" 时检索结果男性面孔比例畸高; "wedding" 的语义偏向西方婚礼. 对比学习的目标是让 "匹配的" 图像-文本对更近, 但 "匹配" 的定义本身已包含社会刻板印象. CLIP 不是在理解世界, 而是在复现训练数据中的统计关联. 

工程影响: 基于 CLIP 的搜索、推荐、审核系统会放大和传播这些偏见. 部署前必须进行偏见审计, 对特定维度实施投影去除(Projection-based Debiasing). 

**细粒度理解弱**

CLIP 被训练来匹配 "整张图片" 与 "整段文本", 损失函数只关心全局语义对齐. 图像Encoder 输出全局向量, 文本端输出句子级向量, 从未学习词与区域的对应关系. CLIP 擅长回答 "这张图是不是关于猫的", 但无法回答 "猫在图片的哪个位置"、"图片中有几只猫"、"猫的左耳上有没有缺口". 

这导致 CLIP 在目标检测、实例分割、视觉问答(涉及空间关系时)等任务上表现不佳. 缓解方向包括: 引入 ViT patch token 作为局部特征, 通过注意力机制与文本 token 交互; GLIP、Grounding DINO 在预训练阶段引入 phrase-region 对齐目标; SAM 提供通用图像分割能力, 与 CLIP 的语义能力互补; CLIP-DINOv2 混合方法结合 CLIP 的语义表示与 DINOv2 的空间特征. 

**分辨率限制**

CLIP 的视觉Encoder 通常在 $224 \times 224$ 或 $336 \times 336$ 分辨率下预训练. 当输入图像被 resize 到这些分辨率时, 小物体可能只剩数个像素, 细粒度纹理被模糊, 长宽比差异大的图像严重变形. ViT 的 patch size 固定, $1024 \times 1024$ 图像 resize 到 $224 \times 224$ 后每个 patch 对应原始区域约 $57 \times 57$ 像素, 大量细节被压缩进单个 patch. 

缓解策略: 预训练后使用更高分辨率图像对Encoder 进行微调; NaViT(Native Resolution ViT)支持任意分辨率和长宽比; 多尺度特征融合将不同分辨率图像分别编码后聚合; 将高分辨率图像切成多个 patch 分别编码, 同时保留全局缩略图编码, 通过注意力机制融合. 

## 8. 技术前瞻与演进

视觉Encoder 的演进围绕三个未解命题展开. 这些命题的答案将直接定义 2026 年之后多模态 AI 的技术路线: 

**命题一: 独立Encoder  vs 原生多模态架构**

当前主流 VLM 采用 "CLIP/SigLIP 视觉Encoder  + projection + LLM" 的缝合模式. GPT-4o、Gemini 被推测采用原生多模态架构——将图像也 token 化为离散或连续 token, 与文本 token 统一输入 Transformer. 原生架构的潜力在于跨模态交互发生在每一层而非仅在 projection 层, 不同模态通过统一的位置编码和注意力机制交互. 但训练难度极高, 需要海量、高质量、多模态交织的数据, 以及精心设计的采样策略来平衡不同数据源的比例. 2026 年的关键问题是: 独立Encoder 模式是否已触及天花板, 原生架构能否在工程上大规模复现并保持语言能力不退化？

**命题二: Patch-based ViT 是否是视觉编码的终极形态**

ViT 将图像切分为固定大小的 patch, 自注意力机制在 patch 序列上运算. 当分辨率和上下文窗口持续扩大, 序列长度平方级增长的计算成本成为瓶颈. 一张 $1024 \times 1024$ 图像在 patch size 为 14 时产生约 5400 个 patch token, 自注意力的 $O(n^2)$ 复杂度使计算量急剧膨胀. Mamba、RWKV 等线性复杂度架构在语言建模中展现出替代 Transformer 的潜力, 它们在视觉编码中的适用性仍在探索. 此外, 以 Sora 为代表的时空 patch 化方法将 patch 概念扩展到视频领域, 视频帧叠加时间维度后 token 数量再增数个量级, 时空 patch 的语义粒度与计算效率之间的张力尚未解决. 

**命题三: 对比学习是否会被生成式目标取代**

CLIP 和 SigLIP 基于判别式对比目标. 另一条路径是将图像生成建模为序列预测问题(如 DALL-E、Parti 的自回归图像 token 预测), 或用扩散模型的去噪目标学习视觉表示. 生成式目标的潜力在于与 LLM 技术栈的统一——两者都是 Next Token Prediction, 自回归视觉生成与语言模型的技术栈更加一致. 但当前生成式方法在视觉表示质量上仍略逊于对比学习, 且生成效率通常低于判别式方法. 未来的突破可能来自生成-判别联合训练框架, 或如 FLAVA 那样同时引入对比学习、masked modeling、captioning loss 的多任务混合目标. 

围绕这些命题, 近期值得关注的演进方向包括: 

| 方向 | 代表性工作 | 核心思想 |
|------|-----------|----------|
| 损失函数演进 | SigLIP 2 | 引入定位预训练目标与更大多语言数据 |
| 分辨率突破 | NaViT、CLIP-AnyRes | 支持任意分辨率与长宽比, 避免强制 resize |
| 局部特征增强 | CLIP-DINOv2 混合 | CLIP 语义特征 + DINOv2 空间特征互补 |
| 分阶段训练 | EVA-CLIP | MAE 自监督初始化 + 对比学习微调 |
| 数据Scaling | LAION-5B 及更大 | 验证视觉Encoder 的幂律 scaling 规律 |
