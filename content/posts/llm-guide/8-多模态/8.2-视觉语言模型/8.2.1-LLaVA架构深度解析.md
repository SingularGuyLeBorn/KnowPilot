# 8.2.1 · LLaVA 架构深度解析

## 1. 演进脉络: 从对比学习到指令微调

2021 年 CLIP 通过 4 亿图文对的对比学习, 验证了视觉与文本可共享同一嵌入空间. CLIP 的双塔结构输出单一相似度分数, 无法执行开放式文本生成. 这一"能判别、不能生成"的结构性缺陷, 催生了后续所有融合架构. 

2022 年 Flamingo 在冻结的 LLM 层间插入交叉注意力模块, 将视觉特征动态注入文本生成过程. 该方案要求修改 LLM 内部结构, 丧失了跨模型迁移能力. 2023 年初 BLIP-2 引入 Q-Former, 将 576 个视觉 patch token 压缩为 32 个可学习查询向量, 降低了 LLM 的序列长度负担, 但增加了 188M 额外参数和独立的预训练阶段. 

LLaVA(Large Language and Vision Assistant)于 2023 年 4 月发布, 其核心洞察重构了 VLM 的设计哲学: **大语言模型本身具备足够强的模式适应能力, 视觉-语言对齐不需要复杂的中间网络. ** LLaVA 在 CLIP ViT-L/14 与 Vicuna-7B 之间仅放置一个线性投影矩阵 $W \in \mathbb{R}^{4096 \times 768}$, 通过两阶段训练在 VQAv2、GQA 等基准上达到与 BLIP-2 相当的性能. 

LLaVA-1.5(2023.10)进行三处关键升级: Linear Projector 替换为两层 MLP; 引入 VQAv2、GQA、OKVQA 等学术 VQA 数据增强视觉推理; 图像分辨率从 224×224 提升至 336×336. LLaVA-1.5-13B 在 MMMU 和 MMBench 上的得分较 1.0 提升 10-15 个百分点. 

LLaVA-NeXT(又称 LLaVA-1.6, 2024.01)引入 AnyRes(Any Resolution)策略. 传统 VLM 将输入图像强制 resize 到固定正方形, 导致宽高比失真和细粒度信息丢失. AnyRes 将图像切分为多个局部图块配合全局缩略图, 各自独立编码后拼接为序列输入 LLM. 该策略使 LLaVA-NeXT 在文档 OCR、图表解析等高分辨率场景中取得质变. 

| 版本 | 发布时间 | 投影层 | 分辨率策略 | 阶段2数据量 | 核心贡献 |
|------|---------|--------|-----------|------------|---------|
| LLaVA-1.0 | 2023.04 | Linear | 224×224 固定 | 158K | 验证极简投影架构可行性 |
| LLaVA-1.5 | 2023.10 | MLP | 336×336 固定 | 665K | 学术数据增强、非线性投影映射 |
| LLaVA-NeXT | 2024.01 | MLP | AnyRes 任意分辨率 | 760K | 图像切分、多尺度视觉输入 |

## 2. 工业价值: 开源 VLM 的事实标准

LLaVA 系列产生了超出原始论文影响力的持续工业价值. Hugging Face 上基于 LLaVA 架构的衍生模型超过 50 个, 覆盖移动端、医疗、文档理解等垂直领域. 这些衍生工作大多只替换 LLM 骨干或视觉Encoder , 保留两阶段训练流程和投影层设计. 

训练成本的极端可控性是 LLaVA 引爆开源社区的直接原因. 复现 LLaVA-1.5-7B 需要 8 张 A100 运行约 30 小时, 总成本低于 1 万美元. Qwen-VL 的 Stage 1 使用 14 亿图文对, 训练成本高出两个数量级. LLaVA 的低门槛特性使全球数百个研究小组能够开展 VLM 研究, 直接推动了 2023-2024 年开源多模态模型的数量激增. 

商业系统广泛选择 LLaVA 作为视觉理解基线. Hugging Face Transformers、vLLM、TGI 等推理框架均提供 LLaVA 的原生支持; GGUF、AWQ、GPTQ 等量化方案拥有成熟社区实现. 在文档理解、工业质检、教育辅助场景中, 经过领域数据微调的 LLaVA 变体通常作为第一版可部署模型上线. 

| 应用场景 | 典型衍生模型 | LLaVA 架构角色 |
|---------|-------------|---------------|
| 端侧移动端 | MobileVLM, TinyLLaVA | 轻量化投影层 + 小 LLM |
| 医疗影像 | LLaVA-Med, RadVLM | 两阶段训练流程复用 |
| 文档 OCR | LLaVA-NeXT | AnyRes 高分辨率策略 |
| 具身智能 | LLaVA-Plus | 视觉-语言-动作接口 |

## 3. 物理直觉: 投影层作为语义转接头

LLaVA 投影层的物理角色可被精确类比为**光学显微镜的目镜系统**. 

光学显微镜的物镜收集样本原始光信号并形成中间实像. 该实像包含样本全部物理信息, 但其放大倍数和成像平面并不直接适配人眼. 目镜将中间实像进一步放大并调整到视网膜可接收的成像距离. 物镜与目镜之间是光学共轭关系: 目镜不需要"理解"样本是什么, 只需完成从"物镜成像平面"到"视网膜成像平面"的精确光学变换. 

CLIP ViT 对应物镜. 它将 336×336 像素的原始光场压缩为 576 个 1024 维的特征向量. 这些向量蕴含图像全部语义信息, 但其几何结构处于对比学习训练的视觉流形中, 与 LLM 的文本嵌入空间存在结构性错位. 

LLM 对应人眼及视觉皮层. 人眼只能接收特定波长范围和入射角度的光. LLM 的输入层只能接收特定维度(4096 维)和特定语义分布(语言嵌入空间)的向量. 

投影层对应目镜. Linear Projector 是单组目镜, 执行固定的线性光学变换, 将视觉流形中的点映射到语言嵌入空间的对应坐标. MLP Projector 相当于带有色差校正的多组目镜, 通过非线性激活修正两种"光学系统"之间的畸变. 

该类比揭示了 LLaVA 设计的深层物理逻辑: **投影层不需要具备"理解"能力, 它只需要成为两个预训练空间之间的保角映射. ** "猫"的图像特征经过投影后, 落在"cat"词嵌入附近的邻域即可. LLM 的深层 Transformer 会自行完成从"cat 词嵌入"到"猫是哺乳动物、会抓老鼠、在文化中象征独立"的全部语义展开. 

AnyRes 的物理直觉是**多焦距观察**. 面对宽幅风景照, 人类先看整体构图(全局缩略图), 再聚焦感兴趣的局部细节(局部图块). AnyRes 将原图切分为 $N \times M$ 网格, 每张局部图独立经过 ViT 编码和投影, 最终拼接为"全局概览 + 局部细节"的多尺度表示序列送入 LLM. 

## 4. 数学推导: 投影层、训练目标与 AnyRes

**投影层的矩阵变换**

设视觉Encoder 输出 $H_{\text{vis}} \in \mathbb{R}^{N \times d_{\text{vis}}}$, 其中 $N$ 为 patch token 数量, $d_{\text{vis}}$ 为视觉特征维度. 对于 CLIP ViT-L/14 处理 336×336 图像, patch 数量由图像边长与 patch size 的比值决定: 

$$
N = \frac{336}{14} \times \frac{336}{14} = 576, \quad d_{\text{vis}} = 1024 \tag{1}
$$
这里 $N$ 表示图像经 ViT patch 化后的总 token 数量, $336$ 是 ViT 输入图像的边长(像素), $14$ 是 ViT 的 patch size(每个 patch 的边长). 单条边(高或宽)上各产生 $336/14 = 24$ 个 patch, 二维网格总计 $24 \times 24 = 576$ 个不重叠 patch. $d_{\text{vis}} = 1024$ 是每个 patch token 的嵌入维度, 由 ViT-L 的模型宽度决定. 式 (1) 为后续投影层设计提供了精确的维度依据. 

LLaVA-1.0 的 Linear Projector 执行维度对齐变换: 

$$
H_{\text{proj}} = H_{\text{vis}} \cdot W^T, \quad W \in \mathbb{R}^{d_{\text{llm}} \times d_{\text{vis}}} \tag{2}
$$
$H_{\text{vis}}$ 是视觉Encoder 输出的 patch 特征矩阵, 形状为 $N \times d_{\text{vis}}$, 包含 $N$ 个处于对比学习视觉语义空间中的 patch token. $W$ 是线性投影矩阵, 负责将视觉维度对齐到 LLM 的输入维度; 其转置 $W^T$ 的形状为 $d_{\text{vis}} \times d_{\text{llm}}$, 使得矩阵乘法 $(N \times d_{\text{vis}}) \times (d_{\text{vis}} \times d_{\text{llm}})$ 恰好输出 $N \times d_{\text{llm}}$ 的结果. $H_{\text{proj}}$ 即为映射后的特征矩阵, 将 $N$ 个视觉 token 逐一变换到 LLM 可理解的语义坐标, 从而消除视觉空间与语言空间的维度不对齐. 

其中 $d_{\text{llm}} = 4096$(Vicuna-7B 的隐藏维度), $W^T \in \mathbb{R}^{1024 \times 4096}$. 输出 $H_{\text{proj}} \in \mathbb{R}^{576 \times 4096}$ 与 LLM 的词嵌入处于同一向量空间. 

Linear Projector 的隐含假设是视觉特征空间与语言嵌入空间之间的语义映射近似线性. 即若"猫的图像特征"减去"狗的图像特征"在视觉空间中的差向量, 应与"cat"词嵌入减去"dog"词嵌入在语言空间中的差向量近似成比例. 该假设在粗粒度概念对齐中成立, 但在细粒度视觉区分中失效. 

LLaVA-1.5 的 MLP Projector 引入非线性激活以放宽该假设: 

$$
h_{\text{proj}}^{(i)} = W_2 \cdot \text{GELU}(W_1 \cdot h_{\text{vis}}^{(i)} + b_1) + b_2 \tag{3}
$$
$h_{\text{vis}}^{(i)}$ 是第 $i$ 个 patch token 的原始视觉特征, 维度为 $d_{\text{vis}}$. $W_1$ 和 $b_1$ 构成第一层线性变换, 将 $d_{\text{vis}}$ 维视觉特征映射到 $d_{\text{hidden}}$ 维隐藏空间, 偏置 $b_1$ 为隐藏空间提供平移自由度. GELU 激活函数对隐藏层输出施加非线性变换, 打破线性投影的秩限制, 使视觉-语言映射关系能够弯曲、扭曲以适配非线性的语义流形结构. $W_2$ 和 $b_2$ 构成第二层线性变换, 将经 GELU 扭曲后的 $d_{\text{hidden}}$ 维表示精确投射回 $d_{\text{llm}}$ 维 LLM 输入空间. 最终输出的 $h_{\text{proj}}^{(i)}$ 是单个 patch token 的完整投影结果; 相对于 Linear Projector, MLP 增加了隐藏层非线性, 参数量约提升 5 倍, 但细粒度语义映射能力显著增强. 

其中 $W_1 \in \mathbb{R}^{d_{\text{hidden}} \times d_{\text{vis}}}$, $W_2 \in \mathbb{R}^{d_{\text{llm}} \times d_{\text{hidden}}}$. 典型配置取 $d_{\text{hidden}} = 4096$, MLP 总参数量为: 

$$
\text{Params}_{\text{mlp}} = 1024 \times 4096 + 4096 \times 4096 + 4096 + 4096 \approx 21\text{M} \tag{4}
$$
$\text{Params}_{\text{mlp}}$ 统计了 MLP Projector 的全部可训练参数. 其中 $1024 \times 4096$ 对应 $W_1$ 的权重规模, 负责将 1024 维视觉特征映射到 4096 维隐藏空间; $4096 \times 4096$ 对应 $W_2$ 的权重规模, 是参数量最大的一项, 将隐藏空间映射到 4096 维 LLM 输入空间; $4096 + 4096$ 是两个偏置向量 $b_1$ 和 $b_2$ 的合计, 虽仅占 8K 参数, 却为每层提供关键的平移自由度, 对优化收敛至关重要. 总参数量约 $4.2\text{M} + 16.8\text{M} + 8\text{K} \approx 21\text{M}$, 约为 Linear Projector($4096 \times 1024 = 4.2\text{M}$)的 5 倍, 但相对于 7B LLM 仅占 0.3%, 可忽略不计. 

该数值约为 Linear Projector($4096 \times 1024 = 4.2\text{M}$)的 5 倍, 但相对于 7B LLM 仍可忽略(占比 0.3%). 非线性激活允许 Projector 对视觉空间中弯曲的语义流形进行"拉直"变换, 再映射到语言空间的线性流形. 

**两阶段训练的目标函数**

Stage 1(投影层预训练)冻结视觉Encoder  $\theta_{\text{vis}}$ 和 LLM $\theta_{\text{llm}}$, 仅优化投影层参数 $\theta_{\text{proj}}$: 

$$
\mathcal{L}_{\text{stage1}} = -\sum_{t=1}^{T} \log P_{\text{LLM}}(w_t \mid w_{<t}, H_{\text{proj}}; \theta_{\text{proj}}) \tag{5}
$$
$\mathcal{L}_{\text{stage1}}$ 是标准的自回归语言建模负对数似然. $T$ 为目标文本序列的总长度, $w_t$ 是第 $t$ 个位置的文本 token, $w_{<t}$ 表示该位置之前的全部文本历史. $P_{\text{LLM}}(w_t \mid w_{<t}, H_{\text{proj}})$ 表示 LLM 在视觉条件 $H_{\text{proj}}$ 下预测下一个 token 的条件概率, 这是自回归语言模型的核心计算单元. 对正确 token 的预测概率取对数, 将概率乘法转化为对数加法, 数值稳定性更好, 同时使高概率对应低损失. 求和符号前的负号将"最大化正确序列概率"的 MLE 目标转化为最小化损失函数. 分号后的 $\theta_{\text{proj}}$ 明确标注仅投影层参数参与梯度更新, 从而保护 LLM 预训练阶段习得的数万亿 token 语言知识不被小规模图文数据的噪声梯度污染. 该阶段的数据为图文对 $(x_{\text{img}}, x_{\text{text}})$, 文本通常是图像的简短描述, 目标函数迫使 $\theta_{\text{proj}}$ 将视觉特征映射到 LLM 可解读的语义坐标. 

此阶段数据为图文对 $(x_{\text{img}}, x_{\text{text}})$, 文本通常是图像的简短描述. 目标函数迫使 $\theta_{\text{proj}}$ 将视觉特征映射到 LLM 可解读的语义坐标. 冻结 LLM 防止灾难性遗忘: LLM 在预训练阶段习得的数万亿 token 语言知识不应被 60 万条图文对的噪声梯度污染. 

Stage 2(视觉指令微调)解冻 LLM 和投影层, 联合优化 $\theta_{\text{llm}}$ 和 $\theta_{\text{proj}}$: 

$$
\mathcal{L}_{\text{stage2}} = -\sum_{t=1}^{T} \log P_{\text{LLM}}(w_t \mid w_{<t}, H_{\text{proj}}; \theta_{\text{llm}}, \theta_{\text{proj}}) \tag{6}
$$
如式 (5) 所示, Stage 2 的形式与 Stage 1 相同, 但关键区别在于 LLM 的参数 $\theta_{\text{llm}}$ 不再冻结. 这意味着模型具备调整自身内部表示以适应视觉条件的能力. $-\sum_{t=1}^{T} \log P_{\text{LLM}}(\cdots)$ 仍为标准自回归负对数似然, 其目的是让 LLM 学会在视觉条件 $H_{\text{proj}}$ 约束下进行开放式文本生成, 将预训练习得的纯文本推理能力泛化到视觉-语言联合语境. 此阶段数据为多轮对话格式的视觉指令 $(x_{\text{img}}, q, a)$, 包含开放式问答、推理和对话. Stage 2 的数据量(665K–760K)远小于 Stage 1, 但质量经过 GPT-4 生成或学术数据集的人工校验, 梯度信号更干净. 

**AnyRes 的图像切分策略**

设输入图像原始尺寸为 $(H, W)$, ViT 输入分辨率为 $S$(通常为 336). AnyRes 选择满足以下约束的整数网格 $(N, M)$: 

$$
N = \left\lceil \frac{H}{S} \right\rceil, \quad M = \left\lceil \frac{W}{S} \right\rceil, \quad N \times M \leq G_{\text{max}} \tag{7}
$$
$N$ 和 $M$ 分别是垂直方向和水平方向上的切分块数, $H$ 和 $W$ 为输入图像的原始高度与宽度(像素), $S$ 是 ViT 的输入分辨率边长. 原始高度与 ViT 输入边长的比值 $H/S$ 表示"需要多少个 $S$ 边长的块才能覆盖高度"; 向上取整 $\lceil H/S \rceil$ 防止因整数截断导致图像底部被裁剪, 例如 $H=500, S=336$ 时 $\lceil 1.49 \rceil = 2$, 即需要 2 行. 宽度方向的计算同理. 约束条件 $N \times M \leq G_{\text{max}}$ 控制总 grid 数不超过上限(LLaVA-NeXT 中 $G_{\text{max}}=6$), 防止极端宽高比图像产生过多局部图块, 避免视觉 token 序列长度爆炸导致 LLM 上下文溢出和推理延迟激增. 

$G_{\text{max}}$ 为最大允许 grid 数(LLaVA-NeXT 中通常取 6). 图像被 resize 到 $(N \cdot S, M \cdot S)$ 后切分为 $N \times M$ 个子图, 每个子图独立编码为 576 个视觉 token. 

全局缩略图额外生成: 将原图 resize 到 $S \times S$, 编码为 576 个 token. 最终输入 LLM 的视觉 token 总数为: 

推导过程如下: 
$$
L_{\text{vis}} = 576 + N \cdot M \cdot 576 = 576 \cdot (1 + N \cdot M) \tag{8}
$$
$L_{\text{vis}}$ 表示输入 LLM 的视觉 token 总长度. 其中 $576$ 来自全局缩略图——将原图 resize 到 $S \times S$ 后编码产生的 token 数, 保留图像的整体语义和全局上下文, 作为"先看全貌"的语义锚点. $N \cdot M \cdot 576$ 来自局部图块, 每个局部图块独立 resize 到 $S \times S$ 后各产生 576 个 token, 总计 $N \cdot M$ 个图块的 token 和, 构成"再看细节"的高分辨率信息源. 因子 $(1 + N \cdot M)$ 清晰表达了"1 份全局 + $N \cdot M$ 份局部"的多尺度结构. 式 (8) 量化了 AnyRes 引入的序列长度开销, 为显存预算和注意力计算复杂度提供准确的输入尺寸参考. 

对于 2×2 grid 配置: 

$$
L_{\text{vis}} = 576 \times 5 = 2880 \tag{9}
$$
如式 (8) 所示, 2×2 grid 下 $N \cdot M = 4$, 加上全局缩略图共 5 张独立编码图像. 每张 $336\times336$ 图像经 ViT-L/14 编码后产生 $24\times24=576$ 个 patch token, 将 1 个全局缩略图和 4 个局部图块分别编码后拼接, 总视觉 token 数为 $576 \times 5 = 2880$. 相对于单图固定分辨率的 576 个 token, AnyRes 在 2×2 grid 下将视觉序列扩展至 5 倍, 以换取细粒度空间信息的保留. 该数值为工程部署提供具体的序列长度参考, 用于 KV Cache 和显存预算的精确计算. 

该序列与文本 token 拼接后送入 LLM. AnyRes 的代价是序列长度的线性增长, 收益是细粒度视觉信息的保留和宽高比失真消除.

| LLaVA-1.5 7B | Stage 2 | 665K 指令 | 32 | 8×A100 | ~25h | ~58 GB |
| LLaVA-NeXT 7B | Stage 2 | 760K 指令 | 16 | 8×A100 | ~35h | ~65 GB |

Stage 1 只训练投影层时, AdamW 仅需存储约 4M-21M 参数的一阶和二阶动量. 8×A100(80GB)可在 batch size 128 下轻松容纳. Stage 2 解冻 7B LLM 后, 参数更新量膨胀至 70 亿, 优化器状态(FP32 一阶 + 二阶动量)占用约 56 GB: 

$$
\text{Memory}_{\text{optimizer}} = 7\text{B} \times 2 \times 4 \text{ bytes} = 56\text{ GB} \tag{10}
$$
此式描述了变量之间的定量关系, 其中每一项对应特定的物理操作. 
$\text{Memory}_{\text{optimizer}}$ 统计 AdamW 优化器状态的显存占用. $7\text{B}$ 是 Stage 2 解冻后参与梯度更新的可训练参数总量. 乘以 2 是因为 AdamW 需要同时存储一阶动量(梯度方向的指数移动平均)和二阶动量(梯度幅值的指数移动平均), 两者共同决定参数更新步长和方向. 再乘以 4 bytes 是因为每个动量状态以 FP32 精度存储, 确保数值稳定性; 使用 BF16/FP16 存储动量会导致优化器状态量化误差, 可能引发训练发散. 单卡优化器状态即占 56 GB, 加上 BF16 模型参数约 14 GB 和激活值, 单卡显存逼近 80 GB A100 的上限. 这说明 Stage 2 必须启用梯度Checkpoint等显存优化策略, 否则无法容纳完整训练状态. 

加上模型参数(BF16)14 GB 和激活值, 单卡显存逼近上限. LLaVA-1.5 和 LLaVA-NeXT 的 Stage 2 必须使用梯度Checkpoint(Gradient Checkpointing)才能将 7B 模型装入 8×A100. 

AnyRes 的引入显著增加了序列长度. 2×2 grid 下视觉 token 达 2880, 文本 token 假设 100, 总序列长度 2980. 标准 Transformer 的 $O(L^2 d)$ 注意力计算使 Stage 2 训练时长从 LLaVA-1.5 的 25 小时增加到 LLaVA-NeXT 的 35 小时, 增幅约 40%. 

数据规模的经济学: LLaVA-1.0 的 Stage 1 使用 CC3M、LAION 和 SBU 的过滤子集, 总计 595K. LLaVA-1.5 扩充至 1.2M, 新增 VQAv2、GQA、OKVQA 等学术数据集的图文对. Stage 2 数据从 158K 增长至 665K(1.5)和 760K(NeXT), 增量主要来自将学术 VQA 数据集改写为对话格式. 数据质量的投资回报率远高于数量的线性增长: MiniGPT-4 仅用 3.5K 高质量样本进行 Stage 2 微调, 生成了更连贯的描述文本. 

## 6. 极简实现: PyTorch 前向传播核心

以下代码展示 LLaVA-NeXT 的核心前向逻辑, 包含 MLP Projector 和 AnyRes 图像切分. 

```python
import torch
import torch.nn as nn
from transformers import CLIPVisionModel, LlamaForCausalLM

class LLaVAProjector(nn.Module):
    """LLaVA-1.5 MLP Projector: d_vis -> d_hidden -> d_llm"""
    def __init__(self, d_vis=1024, d_llm=4096):
        super().__init__()
        self.fc1 = nn.Linear(d_vis, d_llm, bias=True)
        self.act = nn.GELU()
        self.fc2 = nn.Linear(d_llm, d_llm, bias=True)

    def forward(self, x):
        # x: [B, N, d_vis] -> [B, N, d_llm]
        return self.fc2(self.act(self.fc1(x)))

class LLaVANeXT(nn.Module):
    def __init__(self, vision_name="openai/clip-vit-large-patch14-336",
                 llm_name="lmsys/vicuna-7b-v1.5"):
        super().__init__()
        self.vision = CLIPVisionModel.from_pretrained(vision_name)
        self.llm = LlamaForCausalLM.from_pretrained(llm_name)
        self.proj = LLaVAProjector(
            d_vis=self.vision.config.hidden_size,
            d_llm=self.llm.config.hidden_size,
        )
        # 冻结视觉Encoder , 与 LLaVA 原始策略一致
        for p in self.vision.parameters():
            p.requires_grad = False

    def encode_image(self, pixel_values):
        """pixel_values: [B, 3, 336, 336] -> [B, 576, 4096]"""
        # last_hidden_state: [B, 1+576, 1024]; 去掉 [CLS] token
        vis = self.vision(pixel_values).last_hidden_state[:, 1:, :]
        return self.proj(vis)

    def anyres_encode(self, image, grid=(2, 2)):
        """AnyRes: global thumbnail + local patches"""
        B, C, H, W = image.shape
        S = self.vision.config.image_size  # 336
        # Global thumbnail
        thumb = nn.functional.interpolate(
            image, (S, S), mode="bilinear", align_corners=False
        )
        tokens = [self.encode_image(thumb)]
        # Local patches
        patch_h, patch_w = H // grid[0], W // grid[1]
        for i in range(grid[0]):
            for j in range(grid[1]):
                patch = image[:, :, i*patch_h:(i+1)*patch_h,
                              j*patch_w:(j+1)*patch_w]
                patch = nn.functional.interpolate(
                    patch, (S, S), mode="bilinear", align_corners=False
                )
                tokens.append(self.encode_image(patch))
        # [B, 576*(1+grid[0]*grid[1]), 4096]
        return torch.cat(tokens, dim=1)

    def forward(self, images, input_ids, labels=None, grid=None):
        if grid is None:
            vis_tokens = self.encode_image(images)
        else:
            vis_tokens = self.anyres_encode(images, grid)
        text_embeds = self.llm.get_input_embeddings()(input_ids)
        inputs_embeds = torch.cat([vis_tokens, text_embeds], dim=1)

        B, N_vis = vis_tokens.shape[:2]
        vis_mask = torch.ones(B, N_vis, dtype=torch.long, device=images.device)
        attention_mask = torch.cat([vis_mask, torch.ones_like(input_ids)], dim=1)

        if labels is not None:
            vis_labels = torch.full(
                (B, N_vis), -100, dtype=labels.dtype, device=labels.device
            )
            full_labels = torch.cat([vis_labels, labels], dim=1)
        else:
            full_labels = None

        return self.llm(
            inputs_embeds=inputs_embeds,
            attention_mask=attention_mask,
            labels=full_labels,
        )
```

`anyres_encode` 实现了 LLaVA-NeXT 的多尺度编码. 全局缩略图保留整体语义, 局部图块保留高分辨率细节. `interpolate` 将每个 patch resize 到 336×336 再送入 ViT, 确保所有子图共享同一视觉Encoder 的预训练权重. 视觉 token 对应的 label 设为 -100, 使交叉熵损失只计算文本 token 的预测误差. 

## 7. 边界条件: 投影层瓶颈、幻觉与视觉遗忘

**投影层的信息瓶颈. ** Linear Projector 的秩上限为 $\min(d_{\text{vis}}, d_{\text{llm}}) = 1024$. 视觉特征空间中的信息经过投影后, 最多只能保留 1024 个线性无关维度. 该瓶颈在粗粒度 VQA 中不造成明显损害, 但在细粒度视觉区分任务(如医疗影像中的微小病灶识别、电路板缺陷检测)中系统性丢失判别信息. MLP Projector 通过隐藏层非线性扩展部分缓解该问题, 但其容量仍受限于约 21M 参数, 与 LLM 的 7B 参数相比仍是极端压缩. 

**视觉幻觉的耦合机制. ** LLaVA 的幻觉产生可分解为两个耦合因素. 第一, LLM 的文本先验分布 $P_{\text{prior}}(w_t)$ 在视觉证据不足时压制视觉条件分布 $P_{\text{vis}}(w_t \mid x_{\text{img}})$. LLM 预训练习得的"湖边常有野餐者"高频模式, 会在图像中人物轮廓不清晰时被错误激活. 第二, ViT 的 14×14 patch 化过程在空间上聚合局部信息, 无法区分"3"和"8"在模糊手写体中的细微差别, 编码阶段的歧义性通过投影层传递到 LLM, 导致采样偏差. 

**多轮对话中的视觉遗忘. ** LLaVA 在多轮对话后期对图像内容的引用准确率显著下降. 该现象并非上下文长度溢出, 而是注意力稀释的结果. 设首轮输入包含 576 个视觉 token 和 50 个文本 token. 第二轮追加 20 个 token 问题, 模型生成 100 个 token 回答. 第三轮时序列长度达 $576 + 50 + 20 + 100 + 20 \approx 766$. 自注意力计算中, 视觉 token 的 Key 向量与后期文本 Query 之间的内积被大量中间 token 稀释. 文本 token 之间的语言共现模式形成强注意力局部聚类, 视觉 token 被逐步边缘化. 三轮以上对话后, 模型对图像中物体位置的判断准确率下降 20-30%. 

**固定分辨率的形状扭曲. ** LLaVA-1.0 和 1.5 将输入图像强制 resize 到 224×224 或 336×336. 对于宽屏图表(1920×1080), 宽高比从 16:9 扭曲至 1:1, 文本行被纵向压缩、水平布局关系被破坏. AnyRes 通过切分策略部分解决该问题, 但 $G_{\text{max}}$ 上限(通常为 6)限制了超宽图像的处理能力. 当图像宽度超过高度的 6 倍时, AnyRes 仍须进行极端的横向压缩. 

## 8. 技术前瞻: UHD、视频扩展与 Agent 系统

**LLaVA-UHD 与超高分辨率. ** LLaVA-UHD(2024)将 AnyRes 的 $G_{\text{max}}$ 从 6 扩展至 36, 支持 1536×1536 乃至更高分辨率. 核心改进在于视觉 token 的位置编码重标定(positional rescaling), 使来自不同局部图块的 token 在 LLM 中保持正确的二维空间关系. UHD 在 DocVQA、InfoVQA 上的 F1 得分较 LLaVA-NeXT 提升 8-12 个百分点. 代价是视觉 token 数可达 $576 \times 37 = 21312$, 序列长度进入 20K 级别, 对推理延迟和显存占用构成严峻挑战. 

**视频扩展: LLaVA-Video. ** 将 LLaVA 从图像扩展到视频需处理时序维度. 均匀采样 $K$ 帧(如 8 帧), 每帧独立编码为 576 个 token, 总视觉 token 达 $K \times 576 = 4608$. LLaVA-Video 采用时序自适应采样: 对高动态片段保留更多帧, 对静态片段降采样. 该策略在 MSR-VTT 和 ActivityNet-QA 上的时序推理准确率较均匀采样基线提升 5-7%. 长视频(>5 分钟)的时序一致性仍是未解难题, 视觉 token 的指数级增长使 $O(L^2)$ 注意力计算不可持续. 

**与 Agent 系统的结合: LLaVA-Plus. ** 传统 LLaVA 只能被动回答关于输入图像的问题. LLaVA-Plus 将视觉理解与工具调用结合, 使模型主动调用外部视觉工具(SAM 分割、OCR 引擎、图像搜索引擎). 架构在 LLaVA-NeXT 基础上增加工具选择头(tool selection head), 预测需要调用的工具类型和参数. 该方向标志着 VLM 从"静态视觉问答"向"主动视觉推理 Agent"演进. 当前工具选择错误率约 8-12%, 在多工具串行调用场景中误差累积显著. 

| 方向 | 代表工作 | 核心改进 | 当前局限 |
|------|---------|---------|---------|
| 超高分辨率 | LLaVA-UHD | Grid 扩展至 36, 位置重标定 | 序列长度超 20K, 推理延迟高 |
| 视频理解 | LLaVA-Video | 帧级编码 + 时序自适应采样 | 长视频时序一致性差 |
| Agent 系统 | LLaVA-Plus | 视觉理解 + 外部工具调用 | 工具选择错误率约 8-12% |
