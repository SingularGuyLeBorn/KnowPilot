# 8.2.2 · Q-Former 与视觉 Token 压缩

## 1. 演进脉络: 从 MLP Adapter 到 Q-Former 再到 Perceiver Resampler

视觉语言模型的核心矛盾从未改变: 视觉Encoder 输出的 token 数量远超 LLM 的高效处理上限. ViT-L/14 在 336×336 分辨率下输出 576 个 patch token, 而自注意力的计算复杂度与序列长度呈平方关系. 当 LLM 上下文窗口仅为 2048 或 4096 时, 576 个视觉 token 直接吞噬近 30% 的可用上下文, 同时使 attention 的 $L^2$ 项膨胀约 40 倍. 

2023 年初, LLaVA 用单层线性投影矩阵 $W \in \mathbb{R}^{d_{\text{llm}} \times d_{\text{vis}}}$ 证明了模态对齐的门槛极低. Linear Projector 不减少 token 数量, 仅完成维度映射. 这种设计的优势在于零信息损失(所有 patch 原样进入 LLM)和极低的参数量(约 4M). 代价同样明显: LLM 的自注意力层必须直接处理数百个视觉 token, 推理延迟随分辨率提升呈二次增长. 

MLP Adapter 在 Linear Projector 的基础上增加了非线性隐藏层, 参数量扩展至约 20M. GELU 激活函数允许视觉-语言映射关系突破线性假设的约束. 然而 MLP Adapter 仍未解决序列长度问题——576 个视觉 token 依旧原封不动地涌入 LLM. 

BLIP-2 的 Q-Former 引入了一个根本性思路转变: 不再让 LLM 直面全部视觉 patch, 而是在视觉Encoder 和 LLM 之间插入一个可学习的瓶颈层(Bottleneck). Q-Former 使用 $N_q = 32$ 个随机初始化的 Query Embedding, 通过交叉注意力从 576 个视觉 patch 中聚合信息. 输出始终固定为 32 个 token, 与输入图像分辨率无关. 这种架构将 LLM 的序列长度从 $576 + T$ 压缩到 $32 + T$, 自注意力的 $L^2$ 项从约 40 万骤降至约 6,700. 

与 Q-Former 几乎同期, DeepMind 的 Flamingo 采用了功能等价的 Perceiver Resampler. 两者共享相同的数学内核——可学习 latents 通过交叉注意力压缩变长视觉特征——但在实现细节上存在分化: Q-Former 采用 BERT 风格的双向 Transformer, 支持图像-文本联合编码; Perceiver Resampler 采用更深的 latent Transformer 堆叠, 输出直接注入冻结的 LLM 输入层. 

| 维度 | Linear Projector | MLP Adapter | Q-Former | Perceiver Resampler |
|------|------------------|-------------|----------|---------------------|
| 压缩能力 | 无压缩 | 无压缩 | 576→32 | 576→64/32 |
| 核心算子 | 矩阵乘法 | MLP | 交叉注意力 + 自注意力 | 迭代交叉注意力 + 深层自注意力 |
| 是否修改 LLM 内部结构 | 否 | 否 | 否 | 否 |
| 模态交互位置 | 仅输入层 | 仅输入层 | 独立瓶颈模块 | 独立瓶颈模块 |
| 训练阶段复杂度 | 单阶段 | 单阶段 | 两阶段预训练 | 端到端训练 |
| 信息损失 | 零 | 零 | 中(选择性聚合) | 中(选择性聚合) |
| 引入参数量 | ~4M | ~20M | ~188M | ~100M+ |
| 与冻结 LLM 兼容性 | 通用 | 通用 | 通用 | 需适配 |

上述演进揭示了一条清晰的技术脉络: 当视觉Encoder 分辨率不断提升(224→336→448→896), 无压缩投影方案的算力开销已逼近不可接受阈值. Q-Former 和 Perceiver Resampler 用可控的信息损失换取了数量级的效率提升, 使高分辨率视觉理解在消费级 GPU 上成为可能. 

## 2. 工业价值: BLIP-2、InstructBLIP 与 Flamingo 的落地逻辑

BLIP-2(Salesforce, 2023)是第一个将 Q-Former 推向工业规模的系统. 其训练 pipeline 冻结了 EVA-CLIP ViT-g/14 图像Encoder 和 OPT/Flan-T5 大语言模型, 仅优化 Q-Former 的 188M 参数. 第一阶段在 129M 图文对上执行表示学习, 第二阶段在 5.5M 图文对上执行生成学习. BLIP-2 的零样本 VQA 准确率超过了同期需要端到端训练的 80B 参数模型, 证明了"冻结大模型 + 轻量桥接模块"范式的训练效率优势. 

InstructBLIP 在 BLIP-2 的基础上引入了任务感知查询(Task-Aware Queries). Q-Former 的输入除了可学习 query 外, 还拼接了文本指令的 embedding. 同一图像面对 "描述这张图片" 和 "图中的人穿什么颜色的衣服" 两个指令时, Q-Former 会提取不同的视觉特征子集. 这种设计使 InstructBLIP 在 ScienceQA、TextVQA 等需要精确视觉定位的任务上较 BLIP-2 提升 5-10 个百分点. 

Flamingo(DeepMind, 2022)代表了另一条工业路线. 其 Perceiver Resampler 输出 64 个视觉 token, 通过门控交叉注意力层(Gated Cross-Attention Dense Layers)每隔 4 层注入 Chinchilla LLM. Flamingo 的核心卖点是 few-shot 学习能力: 在 16 个多模态任务上, 仅需 4-32 个示例即可超越经过全量微调的专项模型. 这种能力来源于预训练阶段使用的交错图文数据(M3W 数据集, 43M 网页文档), 使模型学会了在上下文中动态绑定视觉概念与语言描述. 

| 模型 | 机构 | 视觉压缩模块 | LLM 骨干 | 冻结策略 | 核心工业指标 |
|------|------|-------------|---------|---------|-------------|
| BLIP-2 | Salesforce | Q-Former (32 queries) | OPT-6.7B / Flan-T5-XXL | 视觉Encoder  + LLM 全冻结 | 零样本 VQA 65.0% (VQAv2) |
| InstructBLIP | Salesforce | Q-Former (task-aware) | Vicuna-7B / 13B | 视觉Encoder 冻结, LLM 微调 | 指令遵循准确率提升 8.2% |
| Flamingo | DeepMind | Perceiver Resampler (64 latents) | Chinchilla-70B | 视觉Encoder  + LLM 冻结, 仅训练 gate 和 resampler | 16 任务 few-shot 平均超越全量微调 |
| MiniGPT-4 | KAUST | Q-Former (BLIP-2 复用) | Vicuna-13B | 视觉Encoder 冻结, LLM LoRA | 单卡 24GB 显存可部署 |

视觉 Token 压缩模块的工业价值不仅体现在训练成本上, 更体现在推理吞吐量上. 以 batch size=8、序列长度 512 的 Vicuna-7B 为例, 无压缩的 576 个视觉 token 使首 token 延迟(Time-to-First-Token, TTFT)达到 1,200ms; Q-Former 压缩至 32 个 token 后, TTFT 降至 180ms. 对于高并发在线服务, 这种 6.7× 的延迟缩减直接决定了产品的商业可行性. 

## 3. 物理直觉: Query Tokens 作为信息瓶颈

将 576 个视觉 patch 压缩为 32 个 query token, 本质是一个有损压缩过程. 理解 Q-Former 的物理直觉, 需要借助信息论中的"信息瓶颈"(Information Bottleneck)框架. 

原始图像经过 ViT 编码后, 576 个 patch token 构成了一个高维信息源 $X \in \mathbb{R}^{576 \times d}$. 这个信息源包含大量对语言任务冗余的数据: 背景天空的重复纹理、无关物体的边缘细节、光照变化的噪声. LLM 并不需要这些信息. Q-Former 的 32 个 query token 充当了压缩表示 $T \in \mathbb{R}^{32 \times d}$, 其训练目标是在最大化 $I(T; Y)$(压缩表示与文本答案的互信息)的同时最小化 $I(T; X)$(压缩表示与原始视觉特征的互信息). 

用一个精确的物理类比: 576 个 patch token 如同一份 576 页的现场庭审笔录, 记录了法庭上每一个咳嗽、每一声叹息、每一次椅子移动. LLM 需要的不是这份完整笔录, 而是一份 32 条的证据摘要. Q-Former 的 32 个 query token 就是 32 名专业书记员, 每位书记员负责一个特定领域: 书记员 A 只记录人物证词, 书记员 B 只记录物证编号, 书记员 C 只记录时间线……在训练过程中, 这些书记员逐渐学会了如何从 576 页原始笔录中快速定位并提炼自己负责的信息. 

这个类比同时解释了 Q-Former 的失效模式: 如果某类信息没有任何书记员负责(例如庭审笔录中的气味描述), 该类信息将永久丢失. 对应到视觉领域, 如果 32 个 query token 在预训练阶段从未学会关注图像中的数学公式, 那么下游的 OCR 和推理任务中, 公式信息就会被压缩过程过滤掉. 

另一个关键直觉是 query token 的"竞争性提取". 32 个 query 共享同一个注意力机制, 它们在训练过程中形成了一种劳动分工. softmax 操作使 attention 权重总和为 1, 当 query A 对某个 patch 赋予高权重时, query B 对该 patch 的可用权重自然降低. 这种竞争机制迫使每个 query 发展出差异化的"信息口味", 避免了 32 个 query 全部聚焦于图像中心物体的冗余情况. 

## 4. 数学推导: Q-Former 交叉注意力、BertLMHead 与 Perceiver 迭代注意力

### 4.1 Q-Former 的交叉注意力层

Q-Former 由 $L = 12$ 层双向 Transformer 组成. 每层包含两个注意力子层: 标准的自注意力(Self-Attention)和交叉注意力(Cross-Attention). 

设可学习 query embedding 为 $Q_0 \in \mathbb{R}^{N_q \times d}$, 视觉Encoder 输出为 $V_{\text{vis}} \in \mathbb{R}^{N_{\text{patch}} \times d_{\text{vis}}}$. Q-Former 首先通过一个线性投影将视觉特征对齐到 query 维度: 

$$
K_{\text{vis}} = V_{\text{vis}} W_K, \quad V_{\text{val}} = V_{\text{vis}} W_V \tag{1}
$$
$V_{\text{vis}}$ 是视觉Encoder 输出的原始 patch 特征矩阵, 形状为 $N_{\text{patch}} \times d_{\text{vis}}$. $W_K$ 和 $W_V$ 分别是 Key 投影矩阵和 Value 投影矩阵, 形状均为 $d_{\text{vis}} \times d$, 负责将视觉Encoder 的高维输出 $d_{\text{vis}}$ 映射到 Q-Former 的统一维度 $d$. $K_{\text{vis}}$ 生成注意力机制中的 Key 矩阵, 用于计算注意力权重("看哪里"); $V_{\text{val}}$ 生成 Value 矩阵, 用于聚合信息("取什么"). 两者使用独立投影矩阵, 允许模型分别学习"定位"和"提取"的最优变换. 两个投影同时执行, 将原始视觉特征一次性转换为注意力所需的 K/V 对, 为后续的交叉注意力计算准备输入. 

其中 $W_K, W_V \in \mathbb{R}^{d_{\text{vis}} \times d}$. 第 $l$ 层的交叉注意力计算为: 

$$
\text{CrossAttn}^{(l)}(Q^{(l)}, K_{\text{vis}}, V_{\text{val}}) = \text{softmax}\left(\frac{Q^{(l)} K_{\text{vis}}^T}{\sqrt{d}}\right) V_{\text{val}} \tag{2}
$$
$Q^{(l)}$ 是第 $l$ 层的 query embedding, 形状为 $N_q \times d$. $Q^{(l)} K_{\text{vis}}^T$ 计算 $N_q$ 个 query 与 $N_{\text{patch}}$ 个视觉 patch 之间的相似度矩阵, 形状为 $N_q \times N_{\text{patch}}$, 每个元素表示"第 $i$ 个 query 对第 $j$ 个 patch 的关注程度", 建立 query 与视觉空间之间的对应关系. 除以 $\sqrt{d}$ 是维度缩放(Scaled Dot-Product Attention): 当维度 $d$ 较大时, 点积的方差随 $d$ 线性增长, 导致 softmax 进入饱和区(梯度极小), 除以 $\sqrt{d}$ 将方差归一化到单位水平, 保持梯度流动性. softmax 对每行进行指数归一化, 得到注意力权重矩阵, 高权重对应 query 最关注的视觉区域. 最后用注意力权重对 Value 矩阵做加权求和, 输出形状 $N_q \times d$, 将 $N_{\text{patch}}$ 个视觉 patch 的信息聚合到 $N_q$ 个 query token 中, 实现视觉 token 压缩. 

$Q^{(l)}$ 在第 0 层初始化为可学习参数 $Q_0$, 在后续层中通过自注意力更新: 

$$
Q^{(l+1)} = \text{SelfAttn}^{(l)}(\text{CrossAttn}^{(l)}(Q^{(l)}, K_{\text{vis}}, V_{\text{val}})) \tag{3}
$$
如式 (2) 所示, 交叉注意力先让每个 query 从自身角度提取视觉信息, $N_q$ 个 query 各自关注图像的不同区域或语义层面, 形成初步的视觉摘要. 随后的自注意力让 $N_q$ 个 query 之间互相通信——例如, query A 发现"有猫", query B 发现"有沙发", 自注意力使它们交换信息, 形成"猫在沙发上"的组合语义, 建立全局依赖关系, 避免每个 query 独立提取信息导致的语义碎片化. $Q^{(l+1)}$ 经过"先提取、再交互"的完整层后被更新为更丰富的语义状态. 多层堆叠使 query 逐步从原始随机初始化演变为具有明确语义分工的可学习视觉摘要. 交叉注意力的输出维度始终为 $N_q \times d$, 与输入 patch 数量 $N_{\text{patch}}$ 无关, 这个维度不变性是 Q-Former 实现视觉 token 压缩的数学保证. 

### 4.2 两阶段预训练的目标函数

BLIP-2 的两阶段预训练策略是 Q-Former 区别于其他压缩方案的核心设计. 

**阶段一: 表示学习(Representation Learning)** . Q-Former 同时连接冻结的图像Encoder 和冻结的文本Encoder (来自 BLIP 的 BERT 风格文本Encoder ). 训练目标包含三个损失项: 

$$
\mathcal{L}_{\text{stage1}} = \mathcal{L}_{\text{ITC}} + \mathcal{L}_{\text{ITG}} + \mathcal{L}_{\text{ITM}} \tag{4}
$$
$\mathcal{L}_{\text{stage1}}$ 是 BLIP-2 阶段一(表示学习)的总损失, 由三项联合构成. $\mathcal{L}_{\text{ITC}}$(图像-文本对比损失)通过对比学习拉近匹配图文对的距离、推远不匹配对的距离, 学习全局语义对齐, 对应 InfoNCE 风格的跨模态对齐信号. $\mathcal{L}_{\text{ITG}}$(图像-文本生成损失)以 Q-Former 输出为条件自回归生成文本描述, 迫使 query token 承载生成所需的全部视觉信息, 对应细粒度的视觉到语言生成信号. $\mathcal{L}_{\text{ITM}}$(图像-文本匹配损失)通过二分类判断图文是否匹配, 学习更细粒度的对齐判别能力, 对应局部语义匹配信号. 三个损失从不同粒度(全局对比、序列生成、局部分类)约束 query token 的表示学习, 使 Q-Former 同时习得"图文是否相关"、"如何用文本描述图像"、"图文细节的匹配程度"三种能力, 避免单一目标的表示偏差. 

- 图像-文本对比损失(Image-Text Contrastive, ITC): 

$$
\mathcal{L}_{\text{ITC}} = -\frac{1}{2} \left[ \sum_{i} \log \frac{\exp(s(q_i, t_i) / \tau)}{\sum_j \exp(s(q_i, t_j) / \tau)} + \sum_{i} \log \frac{\exp(s(t_i, q_i) / \tau)}{\sum_j \exp(s(t_i, q_j) / \tau)} \right] \tag{5}
$$
$q_i$ 是 Q-Former 输出的 32 个 query 经平均池化后的图像表示向量, $t_i$ 是文本Encoder 对第 $i$ 段文本输出的 [CLS] token 表示, $s(\cdot, \cdot)$ 是余弦相似度函数, 输出 $[-1, 1]$ 的标量, $\tau$ 是温度参数. $s(q_i, t_i)$ 度量第 $i$ 对图文(正样本)的相似度, 值越大表示模型认为该对越匹配. 分子 $\exp(s(q_i, t_i) / \tau)$ 对正样本相似度做温度缩放后取指数, 将相似度映射到正实数域; 分母 $\sum_j \exp(s(q_i, t_j) / \tau)$ 是图像 $q_i$ 与 batch 内所有文本的指数相似度和, 作为 softmax 归一化常数引入所有负样本的竞争关系. 第一个求和项是图像→文本方向的 InfoNCE 损失, 迫使每张图像的表示靠近其匹配文本、远离其他文本; 第二个求和项是文本→图像方向的对称损失, 迫使每段文本的表示靠近其匹配图像、远离其他图像. 最后的 $-\frac{1}{2}[\cdots]$ 对两个方向的损失取平均并取负, 负号将最大化正样本概率转化为最小化损失, $\frac{1}{2}$ 保证双向对称. 该损失让 Q-Former 学到的图像表示与文本Encoder 的表示处于可对比的同一语义空间, 为阶段二的生成任务建立全局对齐基础. 

其中 $q_i$ 是 Q-Former 输出的 32 个 query 的平均池化向量, $t_i$ 是文本Encoder 的 [CLS] token, $s(\cdot)$ 为余弦相似度. ITC 强迫 query 聚合的图像表示与对应的文本表示在嵌入空间中接近. 

- 图像-文本生成损失(Image-Text Generation, ITG): 

Q-Former 输出接入一个 BERT 风格的文本Decoder  (BertLMHeadModel). 在因果注意力掩码约束下, 模型根据 query 提取的视觉信息自回归生成文本: 

$$
\mathcal{L}_{\text{ITG}} = -\sum_{t=1}^{T} \log P(w_t | w_{<t}, Q_{\text{output}}; \theta_{\text{Q-Former}}) \tag{6}
$$
$Q_{\text{output}}$ 是 Q-Former 最终输出的 32 个压缩视觉 token, $T$ 是目标文本序列长度, $w_t$ 是第 $t$ 个位置的文本 token, $w_{<t}$ 是该位置之前的所有文本历史. $P(w_t | w_{<t}, Q_{\text{output}})$ 表示 BERT 风格文本Decoder  以 Q-Former 输出的视觉 token 为条件, 预测下一个文本 token 的条件概率; 文本 token 只能 attend to query token, 不能直接访问原始视觉 patch. 对正确 token 的预测概率取对数后, 高概率对应低损失. 求和前的负号对所有位置取负对数似然和, 迫使 $Q_{\text{output}}$ 必须承载生成正确描述所需的全部视觉信息——如果关键细节未被 query 捕获, 文本Decoder  无法通过"偷看"原始 patch 来作弊, 只能产生错误预测, 损失值增大. 分号后的 $\theta_{\text{Q-Former}}$ 明确仅优化 Q-Former 参数, 在冻结视觉Encoder 和文本Encoder 的前提下, 让 Q-Former 学会提取对文本生成最有用的视觉摘要. 

ITG 的关键约束在于: 文本 token 只能 attend to query token, 不能直接 attend to 视觉 patch token. 这迫使 query token 必须承载生成文本所需的全部视觉信息——如果某个关键视觉细节未被 query 捕获, 文本Decoder  将无法通过直接偷看 patch token 来"作弊". 

- 图像-文本匹配损失(Image-Text Matching, ITM): 

ITM 是一个二分类任务, 判断给定的(图像, 文本)对是否匹配. Q-Former 输出与文本Encoder 输出拼接后输入分类头: 

$$
\mathcal{L}_{\text{ITM}} = -\left[ y \log \hat{y} + (1-y) \log(1-\hat{y}) \right] \tag{7}
$$
$y \in \{0, 1\}$ 是真实标签($y=1$ 表示图文匹配, $y=0$ 表示不匹配), $\hat{y} \in (0, 1)$ 是模型预测的概率. 正样本项 $y \log \hat{y}$ 在 $y=1$ 时鼓励预测概率趋近 1, 若模型将匹配对误判为不匹配会产生强惩罚; 负样本项 $(1-y) \log(1-\hat{y})$ 在 $y=0$ 时鼓励预测概率趋近 0. 负号将最大化似然转化为最小化损失. ITM 迫使 query token 学习细粒度的图文匹配判别能力——如式 (5) 所示的 ITC 只关心全局相似度, 而 ITM 通过分类头迫使 query token 捕捉足够细节来判断"这段文本是否真的描述了这张图像". 

ITM 强迫 query token 学习细粒度的图像-文本对齐信号, 而非仅仅依赖全局相似度. 

**阶段二: 生成学习(Bootstrapping Vision-to-Language)** . Q-Former 输出通过一个全连接层投影到 LLM 的嵌入维度, 直接输入冻结的 OPT 或 Flan-T5: 

$$
H_{\text{llm}} = Q_{\text{output}} W_{\text{proj}} \tag{8}
$$
$Q_{\text{output}}$ 是经过 12 层 Q-Former 处理后的 32 个压缩 query token, 形状为 $N_q \times d$, 每个 token 已聚合了图像的特定语义信息, 是"视觉摘要"的最终形态. $W_{\text{proj}}$ 是线性投影矩阵, 形状为 $d \times d_{\text{llm}}$, 负责将 Q-Former 的内部维度 $d$ 映射到目标 LLM 的嵌入维度(如 4096 for LLaMA). 矩阵乘法 $Q_{\text{output}} W_{\text{proj}}$ 将 32 个 query token 逐一映射到 LLM 输入空间, 完成从 Q-Former 语义空间到 LLM 词嵌入空间的最终对齐, 使冻结的 LLM 能够直接消费压缩后的视觉 token. 

其中 $W_{\text{proj}} \in \mathbb{R}^{d \times d_{\text{llm}}}$. 阶段二的损失函数为标准的语言建模损失: 

$$
\mathcal{L}_{\text{stage2}} = -\sum_{t=1}^{T} \log P_{\text{LLM}}(w_t | w_{<t}, H_{\text{llm}}; \theta_{\text{Q-Former}}, \theta_{\text{proj}}) \tag{9}
$$
$P_{\text{LLM}}(w_t | w_{<t}, H_{\text{llm}})$ 是冻结 LLM 在视觉前缀 $H_{\text{llm}}$ 条件下预测下一个文本 token 的概率, LLM 的内部参数不更新, 仅作为固定的语言生成引擎. $-\sum_{t=1}^{T} \log P_{\text{LLM}}(\cdots)$ 是标准的自回归负对数似然, 其目标是在不修改 LLM 内部结构的前提下, 仅通过优化 Q-Former 和投影层, 使 LLM 能够基于压缩视觉 token 生成与图像内容一致的文本. 分号后的 $\theta_{\text{Q-Former}}, \theta_{\text{proj}}$ 明确仅桥接模块参数参与优化, 验证"冻结大模型 + 轻量桥接"范式的有效性——LLM 的通用语言能力完全保留, 视觉理解能力通过 Q-Former 的"视觉方言翻译"注入. 

LLM 参数完全冻结, 仅优化 Q-Former 和投影层. 这种设计的工程含义是: Q-Former 必须在不修改 LLM 内部表示的前提下, 学会一种 LLM 能够理解的"视觉方言". 

### 4.3 Perceiver Resampler 的迭代注意力

Perceiver Resampler 的数学形式与 Q-Former 高度相似, 但增加了 latent Transformer 的深度. 设可学习 latents 为 $L_0 \in \mathbb{R}^{N_l \times d}$, Perceiver Resampler 执行以下迭代: 

$$
L^{(k+1)} = \text{Transformer}^{(k)}\left( \text{CrossAttn}(L^{(k)}, V_{\text{vis}}, V_{\text{vis}}) \right) \tag{10}
$$

$L^{(k)}$ 是 Perceiver Resampler 第 $k$ 次迭代后的 latent 表示, 初始为随机初始化的 $L_0$; $L^{(k+1)}$ 是第 $k+1$ 次迭代后的结果. $V_{\text{vis}}$ 是原始视觉Encoder 输出, 同时充当交叉注意力的 Key 和 Value 来源. latent 向量 $L^{(k)}$ 作为 Query, 与 $V_{\text{vis}}$ 执行交叉注意力, 从视觉 patch 中提取信息, 输出更新后的 latent 表示; 这让每个迭代步骤都能重新"阅读"原始视觉数据, 避免信息在深层传递中逐层衰减. 随后的 $\text{Transformer}^{(k)}$ 对交叉注意力输出执行完整的自注意力 + FFN 变换, 使 $N_l$ 个 latent 之间交换信息、提炼语义, 协调分工形成更结构化的压缩表示. Perceiver 通过多轮"提取-交互"迭代逐步精炼 latent 表示, 相对于 Q-Former 的单层处理, 更深的 latent Transformer 提供更丰富的非线性变换能力. 

与 Q-Former 的关键差异在于: Perceiver Resampler 的每层 latent Transformer 包含完整的自注意力 + 前馈网络, 且交叉注意力的 Key/Value 始终来自原始视觉特征(不随层更新). Q-Former 的交叉注意力 Key/Value 同样固定来自视觉Encoder , 但其自注意力发生在 query 之间, 且整体架构更贴近 BERT. 

在 Flamingo 中, Perceiver Resampler 输出的 latents 被拼接到 LLM 输入序列前端. LLM 的冻结参数无需任何修改即可接收这些压缩后的视觉 token. 门控机制(Gated Cross-Attention)通过可学习的 tanh 门控系数控制视觉信息注入的强度: 

$$
H^{(l)}_{\text{out}} = H^{(l)} + \tanh(\alpha^{(l)}) \cdot \text{CrossAttn}(H^{(l)}, V_{\text{latent}}, V_{\text{latent}}) \tag{11}
$$

$H^{(l)}$ 是第 $l$ 层 LLM 的原始隐藏状态(来自前一层自注意力 + FFN), $V_{\text{latent}}$ 是 Perceiver Resampler 输出的压缩视觉 token(64 个 latents). 交叉注意力以 $H^{(l)}$ 为 Query、$V_{\text{latent}}$ 为 K/V, 将视觉信息"注入"到 LLM 的文本表示中, 使文本 token 能够参考图像内容. $\alpha^{(l)}$ 是第 $l$ 层的可学习门控系数标量, 经 $\tanh$ 压缩到 $(-1, 1)$ 后缩放交叉注意力的输出: $\alpha^{(l)}$ 可正可负, 正值增强视觉注入, 负值抑制甚至反转视觉信号, 使不同层能自适应地调节视觉信息的贡献比例——浅层可能需要更多视觉上下文, 深层可能更依赖文本推理链. 残差连接 $H^{(l)} + \tanh(\alpha^{(l)}) \cdot \text{CrossAttn}(\cdots)$ 保留 LLM 的语言推理能力不被视觉信号淹没, 同时让视觉信息以可控增量参与文本生成. $\alpha^{(l)}$ 随层变化, 使深层 LLM 可以逐层调节视觉信息的注入比例. 

## 5. 算力成本: 参数量、计算量与显存占用

视觉 Token 压缩方案的选择直接影响训练成本和推理延迟. 以下数值基于 ViT-L/14($d_{\text{vis}}=1024$, 576 patches)和 LLaMA-2-7B($d_{\text{llm}}=4096$)的配置估算. 

### 5.1 投影层与压缩模块的参数量对比

| 模块 | 参数量公式 | 数值(百万) | 占总模型比例 |
|------|-----------|-------------|-------------|
| Linear Projector | $d_{\text{llm}} \times d_{\text{vis}}$ | 4.2 | 0.06% |
| MLP Adapter (hidden=4096) | $d_{\text{vis}} d_{\text{hidden}} + d_{\text{hidden}} d_{\text{llm}}$ | 21.0 | 0.30% |
| Q-Former ($L=12, N_q=32$) | $L \times (4d^2 N_q + 2d N_q N_{\text{patch}}) + d_{\text{llm}} d N_q$ | 188.0 | 2.68% |
| Perceiver Resampler ($L=6, N_l=64$) | $L \times (4d^2 N_l + 2d N_l N_{\text{patch}}) + d_{\text{llm}} d N_l$ | 98.0 | 1.40% |

Q-Former 的 188M 参数主要来自 12 层 Transformer 中的自注意力($4d^2$ 每 query)和交叉注意力($2d N_{\text{patch}}$ 每 query). 这个参数量远大于 Linear Projector, 但相对于 7B LLM 仍可忽略. 

### 5.2 单张图像前向传播的 FLOPs 对比

| 模块 | FLOPs 公式 | 数值(GFLOPs) |
|------|-----------|---------------|
| ViT-L/14 编码 | $2 N_{\text{patch}} d_{\text{vis}}^2 \times L_{\text{vit}}$ | ~45 |
| Linear Projector | $2 N_{\text{patch}} d_{\text{vis}} d_{\text{llm}}$ | 4.8 |
| Q-Former 交叉注意力 | $2 N_q N_{\text{patch}} d$ | 0.04 |
| Q-Former 自注意力 | $2 N_q^2 d \times L$ | 0.78 |
| LLM 自注意力(无压缩, 576+50 token) | $2 L^2 d_{\text{llm}} \times L_{\text{llm}}$ | ~1,280 |
| LLM 自注意力(Q-Former 压缩, 32+50 token) | $2 L^2 d_{\text{llm}} \times L_{\text{llm}}$ | ~22 |

关键结论: Q-Former 引入的额外计算(交叉注意力 + 自注意力约 0.82 GFLOPs)相对于 LLM 自注意力的节省(1,280→22 GFLOPs)完全可忽略. 压缩的核心价值不在于减少投影层计算, 而在于将 LLM 的二次注意力成本从 $O((N_{\text{patch}} + T)^2)$ 降至 $O((N_q + T)^2)$. 

### 5.3 推理阶段的显存占用

以 FP16 推理、batch size=1 为例: 

| 配置 | KV Cache 视觉部分(MB) | 总 KV Cache(MB) | 相对显存 |
|------|------------------------|-------------------|---------|
| 无压缩(576 token) | $2 \times 576 \times 4096 \times 32 \times 2 / 10^6 = 294.9$ | 约 850 | 100% |
| Q-Former(32 token) | $2 \times 32 \times 4096 \times 32 \times 2 / 10^6 = 16.8$ | 约 580 | 68% |
| Perceiver(64 token) | $2 \times 64 \times 4096 \times 32 \times 2 / 10^6 = 33.6$ | 约 600 | 71% |

KV Cache 的节省在长序列生成中被进一步放大. 当生成长度达到 2048 时, 无压缩方案的 KV Cache 总量接近 1.2GB, 而 Q-Former 方案维持在 0.9GB 以下. 对于多图像对话(4-8 张图同时输入), Q-Former 的压缩使视觉 KV Cache 从 1.2GB 降至 67MB, 这是支撑多图推理的工程基础. 

## 6. 极简实现: Q-Former 核心逻辑的 PyTorch 表达

以下代码实现 Q-Former 的核心计算逻辑, 包含可学习 query embedding、交叉注意力、自注意力层堆叠, 以及到 LLM 的投影. 代码严格对应第 4 节的数学公式. 

```python
import torch
import torch.nn as nn
import math

class QFormerLayer(nn.Module):
    """
    单层 Q-Former: 自注意力 + 交叉注意力 + FFN. 
    对应公式: Q^{(l+1)} = SelfAttn(CrossAttn(Q^{(l)}, K_vis, V_val))
    """
    def __init__(self, d_model=768, n_heads=12, n_queries=32):
        super().__init__()
        self.n_heads = n_heads
        self.d_head = d_model // n_heads
        
        # 自注意力: query 之间交换信息
        self.self_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        # 交叉注意力: query 提取视觉信息
        self.cross_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        # FFN
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_model * 4),
            nn.GELU(),
            nn.Linear(d_model * 4, d_model),
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)
    
    def forward(self, queries, vision_kv, self_attn_mask=None):
        # 交叉注意力: Q=queries, K=V=vision_kv
        # 对应 CrossAttn^{(l)}(Q^{(l)}, K_vis, V_val)
        cross_out, _ = self.cross_attn(
            queries, vision_kv, vision_kv, need_weights=False
        )
        queries = self.norm1(queries + cross_out)
        
        # 自注意力: query 之间交互
        # 对应 SelfAttn^{(l)}(...)
        self_out, _ = self.self_attn(
            queries, queries, queries, 
            attn_mask=self_attn_mask, need_weights=False
        )
        queries = self.norm2(queries + self_out)
        
        # FFN
        ffn_out = self.ffn(queries)
        queries = self.norm3(queries + ffn_out)
        return queries

class QFormer(nn.Module):
    """
    完整 Q-Former: N 层堆叠 + 输出投影至 LLM 维度. 
    对应 BLIP-2 两阶段预训练中的第一阶段输出模块. 
    """
    def __init__(
        self, 
        n_layers=12, 
        d_model=768, 
        n_heads=12, 
        n_queries=32,
        vision_dim=1024,
        llm_dim=4096,
    ):
        super().__init__()
        self.n_queries = n_queries
        self.d_model = d_model
        
        # 可学习 query embedding, 对应 Q_0 ∈ R^{N_q × d}
        self.query_embed = nn.Parameter(torch.randn(1, n_queries, d_model) * 0.02)
        
        # 视觉特征投影到 query 维度: K_vis = V_vis W_K, V_val = V_vis W_V
        # 此处简化为单一线性投影(原论文使用独立的 K/V 投影)
        self.vision_proj = nn.Linear(vision_dim, d_model)
        
        # L 层 Q-Former Transformer
        self.layers = nn.ModuleList([
            QFormerLayer(d_model, n_heads, n_queries) 
            for _ in range(n_layers)
        ])
        
        # 输出投影至 LLM 嵌入空间: H_llm = Q_output W_proj
        self.llm_proj = nn.Linear(d_model, llm_dim)
    
    def forward(self, vision_features):
        """
        Args:
            vision_features: [B, N_patch, vision_dim], ViT 输出的 patch 特征
        Returns:
            llm_inputs: [B, n_queries, llm_dim], 压缩后的视觉 token
        """
        B = vision_features.size(0)
        
        # 视觉特征投影到 query 维度
        vision_kv = self.vision_proj(vision_features)  # [B, N_patch, d_model]
        
        # 广播可学习 query 到 batch 维度
        queries = self.query_embed.expand(B, -1, -1)  # [B, N_q, d_model]
        
        # 因果掩码: query i 只能 attend to query j ≤ i(生成阶段需要)
        # 训练阶段可设为 None(双向注意力)
        causal_mask = torch.triu(
            torch.ones(self.n_queries, self.n_queries) * float('-inf'), 
            diagonal=1
        ).to(vision_features.device)
        
        # L 层迭代更新
        for layer in self.layers:
            queries = layer(queries, vision_kv, self_attn_mask=causal_mask)
        
        # 投影到 LLM 维度
        llm_inputs = self.llm_proj(queries)  # [B, N_q, llm_dim]
        return llm_inputs


# ========== 使用示例 ==========
if __name__ == "__main__":
    # 模拟 ViT-L/14 在 336×336 下的输出: 576 patches × 1024 dim
    dummy_vision = torch.randn(2, 576, 1024)
    
    qformer = QFormer(
        n_layers=12, 
        d_model=768, 
        n_heads=12, 
        n_queries=32,
        vision_dim=1024,
        llm_dim=4096,
    )
    
    compressed = qformer(dummy_vision)
    print(f"输入视觉特征: {dummy_vision.shape}")      # [2, 576, 1024]
    print(f"Q-Former 输出: {compressed.shape}")       # [2, 32, 4096]
    print(f"压缩比: {576 / 32:.1f}x")
    
    total_params = sum(p.numel() for p in qformer.parameters())
    print(f"总参数量: {total_params / 1e6:.1f}M")
```

代码与数学公式的映射关系: 

- `self.query_embed` 对应 $Q_0 \in \mathbb{R}^{N_q \times d}$, 随机初始化后在训练中学习. 
- `vision_proj` 对应视觉特征到 query 维度的对齐投影, 对应 $K_{\text{vis}} = V_{\text{vis}} W_K$ 的简化版本. 
- `QFormerLayer.cross_attn` 对应公式 $\text{CrossAttn}^{(l)}(Q^{(l)}, K_{\text{vis}}, V_{\text{val}})$. 
- `QFormerLayer.self_attn` 对应公式 $\text{SelfAttn}^{(l)}(\cdot)$, 使 32 个 query 之间交换信息. 
- `llm_proj` 对应阶段二的投影矩阵 $W_{\text{proj}}$, 将 Q-Former 输出映射到 LLM 的嵌入空间. 
- `causal_mask` 在生成任务中保证 query 的自回归性质; 在表示学习阶段(阶段一)可移除以使用双向注意力. 

## 7. 边界条件: 信息丢失、过拟合与兼容性

Q-Former 的压缩能力建立在"选择性信息丢弃"的前提上. 这种设计在带来效率增益的同时, 引入了若干必须警惕的边界条件. 

**信息丢失的结构性根源**. 576 个 patch 压缩为 32 个 query, 理论信息容量上限下降了 $576/32 = 18$ 倍. 如果输入图像包含空间上高度分散的多个关键区域(例如一张密集的表格, 文字分布在 20 个不同单元格中), 32 个 query 的注意力权重会被迫分散到大量 patch 上, 每个 query 的平均有效感受野扩大, 局部信噪比下降. 这种"注意力稀释"导致细粒度空间信息的模糊化. InstructBLIP 在 TextVQA 上的错误案例分析显示: 约 23% 的 failure case 源于 Q-Former 未能将足够的 attention 权重分配给图像边缘的小文本区域. 

**Query Token 的过拟合风险**. 32 个可学习 query embedding 在阶段一的 129M 图文对上训练, 其注意力模式高度依赖预训练数据的分布. 如果下游任务的视觉分布与预训练数据差异显著(例如医学 CT 影像、卫星遥感图、工业零件 X 光片), query token 学到的"通用视觉摘要"策略会失效. 具体表现为: query 的 attention map 在预训练数据(自然图像)上呈现清晰的物体聚焦模式, 但在域外数据上呈现随机的、低信噪比的分布. 这种过拟合无法通过阶段二的轻量微调完全修正, 因为 query embedding 的注意力偏好已在阶段一被深度固化. 

| 失效场景 | 物理根因 | 量化表现 | 缓解策略 |
|---------|---------|---------|---------|
| 小文本/密集表格识别 | 注意力稀释, query 感受野过大 | TextVQA 准确率下降 15-25% | 提高输入分辨率, 增加 query 数量至 64/128 |
| 域外视觉分布(医学/遥感) | Query 过拟合于自然图像统计特性 | Zero-shot 准确率接近随机 | 领域适配: 在目标域数据上继续阶段一训练 |
| 多图长序列对话 | 每图 32 token × N 图 = 视觉 token 累积 | 4 图后上下文溢出 | 动态 query 数量或层级压缩 |
| 高动态范围/极端光照 | ViT Encoder 本身特征退化 | Q-Former 输入质量下降 | 预处理归一化, 更换鲁棒视觉Encoder  |

**与不同视觉Encoder 的兼容性约束**. Q-Former 假设视觉Encoder 输出的是空间结构化的 patch 特征序列, 其中每个 token 对应图像的一个局部区域. 这个假设对标准 ViT(CLIP、EVA-CLIP、SigLIP)成立, 但对以下场景失效: 

- CNN Encoder (如 ResNet)输出的是空间特征图 $H \times W \times C$, 需要额外的展平和位置编码才能输入 Q-Former. 
- 基于全局池化的Encoder (如早期的 EfficientNet)输出单个向量, Q-Former 的交叉注意力退化为对一个 token 的加权复制, 丧失空间选择性. 
- 多尺度特征金字塔(如 FPN)输出多个分辨率的特征图, Q-Former 的单尺度交叉注意力无法直接处理, 需要引入可变形注意力或特征融合模块. 

**与 LLM 的维度绑定**. Q-Former 的 `llm_proj` 层将输出投影到特定 LLM 的嵌入维度(如 4096 for LLaMA, 5120 for GPT-3). 更换 LLM 时必须重新训练该投影层, 且 Q-Former 的 query embedding 也需要在新 LLM 上进行阶段二微调. 这种耦合使 BLIP-2 的 Q-Former 无法像 Linear Projector 那样"即插即用"地适配任意 LLM. 

## 8. 技术前瞻: 动态查询、MoE 与多尺度压缩

Q-Former 的固定 query 数量(32 或 64)是一个全局超参数, 与输入图像的内容复杂度无关. 一张纯色的空白页和一张拥挤的街景图获得完全相同的 32 个 query 预算, 造成了信息分配的严重失衡: 简单图像浪费了宝贵的 query 预算, 而复杂图像又因预算不足丢失关键细节. 

**动态查询数量(Dynamic Query Allocation)** . 最新的研究探索了根据图像内容自适应调整 query 数量的机制. MoVQ(2024)使用一个轻量的 CNN 前端估计图像的"信息密度图", 在高密度区域分配更多 query, 在低密度区域(如天空、墙面)分配更少. 数学上, 动态 query 的数量 $N_q(x)$ 成为输入图像 $x$ 的函数: 

$$
N_q(x) = \text{Base} + \alpha \cdot \text{Entropy}(\text{Grad}(x)) \tag{12}
$$

$N_q(x)$ 是针对输入图像 $x$ 动态分配的 query token 数量. $\text{Grad}(x)$ 计算图像的空间梯度, 边缘、纹理、文字区域产生高梯度, 平滑区域(天空、墙面)产生低梯度, 从而区分"信息丰富区域"和"信息稀疏区域". $\text{Entropy}(\text{Grad}(x))$ 对梯度幅值的空间分布计算信息熵: 若梯度集中在少数区域(如文档中的文字行), 熵较低; 若梯度均匀分布(如拥挤街景), 熵较高, 将"图像有多复杂"量化为单一标量. $\alpha$ 是缩放系数, 将熵值线性缩放后转化为额外的 query 配额, 使信息密度高的图像获得更多 query、信息稀疏的图像获得更少. $\text{Base}$ 是所有图像至少分配的固定配额, 确保即使极简图像也保留最低限度的表示能力, 同时为复杂图像预留弹性扩展空间. 最终 query 数量 = 保底配额 + 动态增量, 打破固定 query 数量的僵化分配, 使计算资源按需流向复杂图像. 

其中 $\text{Grad}(x)$ 是图像梯度幅值, $\text{Entropy}$ 度量其空间分布的离散程度. 街道场景可能获得 64 个 query, 而文档扫描可能仅需 16 个. 这种动态策略在保持平均序列长度不变的前提下, 将复杂图像的 VQA 准确率提升了 3-7%. 

**与 MoE 的结合**. Mixture of Experts(MoE)为 Q-Former 提供了另一条扩展路径. 传统 Q-Former 的 32 个 query 共享同一套注意力参数, 这迫使所有 query 学习通用的聚合策略. MoE-QFormer(2024)将 query 分组到不同的专家子网络中: 专家 A 专门处理文本密集型图像, 专家 B 专门处理自然场景, 专家 C 专门处理人脸/人物. 每个 query 通过一个门控网络 $G(q_i)$ 选择最相关的 $k$ 个专家: 

$$
q_i^{\text{out}} = \sum_{j=1}^{k} G(q_i)_j \cdot \text{Expert}_j(q_i, V_{\text{vis}}) \tag{13}
$$

$q_i^{\text{out}}$ 是第 $i$ 个 query 经 MoE 路由后的最终输出. $G(q_i)$ 是门控网络对 query $q_i$ 输出的路由权重向量, 长度等于专家总数; $G(q_i)_j$ 是第 $j$ 个专家对 query $q_i$ 的激活权重, 根据 query 的内容决定"这个 query 应该由哪个专家处理"——例如, 包含文本相关特征的 query 被路由到"文本专家", 包含人脸特征的 query 被路由到"人脸专家", 实现 query 级别的自适应计算. $\text{Expert}_j(q_i, V_{\text{vis}})$ 是被激活的第 $j$ 个专家子网络执行的独立交叉注意力计算, 其参数专门针对特定视觉模式训练, 用专家的专业化参数替代通用共享参数, 提升表示精度. 最后对 top-$k$ 个专家的输出做加权求和, 权重由门控网络分配, 使单个 query 能够融合多个专家的视角, 避免硬路由导致的信息割裂, 同时保持稀疏激活的计算效率(仅计算 $k$ 个专家而非全部). 

MoE 的引入使 Q-Former 在不增加推理计算量(仅激活部分专家)的前提下扩展了表达能力. 实验显示, 8 专家的 MoE-QFormer 在跨域 benchmark 上的泛化性能比基线提升 12%. 

**多尺度压缩(Hierarchical Token Compression)** . 单尺度的 Q-Former 在压缩高分辨率图像时面临两难: 增大 query 数量会抵消压缩的收益, 减少 query 数量会加剧信息丢失. 多尺度方案将视觉Encoder 不同层的特征同时输入压缩模块: 

- 低分辨率深层特征(高层语义, 如物体类别、场景类型)用少量 query 压缩. 
- 高分辨率浅层特征(低层细节, 如边缘、纹理、文字笔画)用较多 query 压缩. 
- 两个尺度的压缩输出拼接后输入 LLM. 

这种金字塔结构的压缩在文档理解任务中尤为有效: 高层语义 query 负责判断"这是一张发票", 低层细节 query 负责提取"金额: ¥3,240.00". 

| 技术方向 | 核心思想 | 优势 | 当前瓶颈 |
|---------|---------|------|---------|
| 动态 Query | 根据图像复杂度调整 $N_q$ | 信息预算按需分配 | 需要轻量前端估计复杂度, 引入额外延迟 |
| MoE-QFormer | 不同 query 路由到不同专家 | 跨域泛化能力强 | 专家负载均衡难, 训练稳定性差 |
| 多尺度压缩 | 同时使用多层 ViT 特征 | 兼顾语义与细节 | 不同尺度的特征对齐复杂 |
| 可学习 Token 丢弃 | 根据 attention 权重动态丢弃低权重 patch | 进一步压缩序列长度 | 丢弃策略不可微, 需强化学习或 Gumbel-Softmax |
| 与状态空间模型结合 | 用 Mamba 替代 Transformer 处理压缩后 token | 线性复杂度, 超长序列友好 | 视觉-语言对齐质量尚未经大规模验证 |

**与原生多模态架构的竞争**. Q-Former 代表了一种"桥接范式"——用轻量模块连接冻结的视觉Encoder 和 LLM. 与此同时, Chameleon、Gemini 等原生多模态模型正在证明: 取消桥接模块、直接从预训练阶段就统一处理视觉 token 和文本 token, 可能获得更高的模态融合质量. 对 Q-Former 而言, 其长期生存空间取决于一个核心问题: 在视觉Encoder 和 LLM 都持续增大(ViT 迈向 4B 参数, LLM 迈向 1T 参数)的背景下, 冻结组件 + 轻量桥接的策略是否仍具备算力经济性？当前共识是: 在 2025-2026 年的工业部署环境中, Q-Former 仍是性价比最高的选择; 但在 3-5 年的时间尺度上, 原生多模态预训练可能逐步侵蚀桥接范式的市场份额.
