# 8. 多模态

## 1. 章节定位与价值

人类对世界的理解从来不是单一模态的. 我们同时通过眼睛观察颜色与形状、通过耳朵聆听语调与节奏、通过皮肤感知温度与质地——这些来自不同感官的信息在大脑中融合, 构成了我们对世界的完整认知. 如果人工智能的目标是真正的通用智能(AGI), 那么它也必须具备类似的能力: **同时理解、推理和生成跨越多种模态的内容**. 

多模态大模型(Multimodal Large Models)正是朝着这个方向迈出的关键一步. 从早期的"图像分类器"到今天的"视觉语言模型"(Vision-Language Models, VLMs), 从简单的"文本转语音"到能够生成带音效的视频的"世界模型", 多模态技术正在以惊人的速度拓展 AI 的能力边界. 

本章的定位是知识库的"感官系统指南". 它将带领读者超越纯文本的局限, 探索如何让大模型具备"视觉"、"听觉"乃至"综合感知"的能力. 通过本章的学习, 你将理解: 

1. **多模态的核心架构原理**: 不同模态的信息如何在统一的表示空间中对齐和融合. 

2. **视觉理解的完整技术栈**: 从图像Encoder 到视觉问答, 从目标检测到文档理解. 

3. **视频与音频的建模挑战**: 时序建模、长视频理解、音视频同步等核心问题. 

4. **生成式多模态**: 从文本生成图像(Text-to-Image)到文本生成视频(Text-to-Video), 以及更复杂的跨模态生成任务. 

5. **多模态对齐技术**: 对比学习、融合嵌入、原生多模态架构等关键方法. 

本章的探索路径将从多模态的核心概念和通用架构出发, 逐步深入到视觉、音频、视频等具体领域, 并最终揭示连接不同模态的底层对齐技术.

## 2. 多模态的核心概念与通用架构

### 2.1 为什么多模态如此困难？

不同模态的数据具有根本性的差异: 

**表示形式的差异**
文本是离散的符号序列, 天然具有组合性和层次结构; 图像是连续的二维像素阵列, 局部相关性高但全局结构需要通过学习来捕捉; 音频是一维的时序信号, 同时包含时域和频域的信息; 视频则是图像序列与时序维度的叠加. 这些差异意味着, 无法直接将一个模态的原始数据输入到为另一个模态设计的模型中. 

**语义粒度的不匹配**
在文本中, 一个"词"通常对应一个相对明确的概念; 但在图像中, "一只猫"可能占据画面的一角, 也可能占据整个画面, 还可能与"沙发"、"窗户"等概念在像素层面紧密交织. 如何在不同模态之间建立"概念级别"的对应关系, 是多模态理解的核心挑战. 

**信息密度的差异**
一张 1024×1024 的 RGB 图像包含超过 300 万个像素值, 但其传达的语义信息可能只用一句话就能概括. 这意味着多模态模型需要具备强大的"信息压缩"能力——从海量的原始感知数据中提取出紧凑的语义表示. 

### 2.2 通用架构范式

尽管存在上述挑战, 多模态模型的发展已经形成了几种相对成熟的架构范式: 

**范式一: Encoder -投影-Decoder  (Encoder-Projector-Decoder)** 
这是当前最主流的多模态架构, 尤其适用于"以语言模型为核心、外挂其他模态理解能力"的场景. 其工作流程是: 

1. **模态专用Encoder **: 使用预训练的专用Encoder 处理各模态输入. 例如, 使用 ViT(Vision Transformer)处理图像, 使用 Whisper 或 wav2vec 2.0 处理音频. 

2. **投影/适配层**: 将各模态Encoder 的输出映射到语言模型的输入空间中. 这个投影层可以是简单的线性层(MLP Adapter)、也可以是更复杂的交叉注意力模块(如 Q-Former、Perceiver Resampler). 投影层通常需要在新构建的多模态数据上进行训练. 

3. **统一Decoder  **: 通常是一个预训练好的大语言模型(如 Llama、Qwen、InternLM), 它接收经过投影的各模态特征(以及文本 token), 生成统一的文本输出. 

这种范式的优点是实现相对简单、可以复用强大的预训练语言模型; 缺点是各模态Encoder 之间缺乏交互, 信息融合主要发生在最后的语言模型阶段. 

**范式二: 统一Encoder-Decoder(Unified Encoder-Decoder)** 
这种范式试图从架构层面就实现多模态的统一处理, 而不是依赖于后期的投影. 代表性的方法包括: 

- **将图像/音频 token 化**: 将非文本模态的内容也转化为离散的 token(如使用 VQ-VAE 将图像编码为视觉 token), 然后与文本 token 一起输入到统一的 Transformer 中. GPT-4o、Gemini 等先进模型就采用了类似的思想. 

- **原生多模态 Transformer**: 设计能够同时处理文本、图像、音频 token 的 Transformer 架构, 不同模态的 token 通过统一的位置编码和注意力机制进行交互. 这种方法的潜力最大, 但训练难度也最高——需要海量、高质量的多模态对齐数据. 

**范式三: 双塔/多塔对比(Dual/Multi-Tower Contrastive)** 
这种范式不追求生成能力, 而是专注于**跨模态的表示对齐**. 它使用两个或多个独立的Encoder 分别处理不同模态, 通过对比学习使语义匹配的跨模态样本在表示空间中距离相近、不匹配的样本距离远离. CLIP 和 SigLIP 是这一范式的典范.

## 3. 视觉理解与视觉语言模型

视觉是人类获取信息的最主要渠道, 因此视觉理解也是多模态研究最成熟、应用最广泛的领域. 

### 3.1 图像Encoder 的发展

视觉语言模型的质量在很大程度上取决于其视觉Encoder 的能力. 视觉Encoder 的核心任务是将原始像素转换为语义丰富的特征向量. 

**ViT(Vision Transformer)** 
ViT 将 Transformer 架构从 NLP 迁移到了计算机视觉. 它将图像切分为固定大小的 patches(如 16×16 像素), 将每个 patch 线性投影为一个向量, 然后在这些 patch embedding 上应用标准的 Transformer Encoder . ViT 的成功证明了 Transformer 的通用性——它不依赖于 CNN 特有的归纳偏置(如局部性和平移等变性), 纯粹通过数据和规模就能学习到强大的视觉表示. 

**EVA 系列与 InternViT**
为了提升视觉Encoder 的语义对齐能力, 研究者们开发了各种改进版本. EVA 系列通过"从 CLIP 蒸馏"的方式来初始化 ViT, 使其学到的视觉特征与语言模型的语义空间更加一致. InternViT 则通过大规模的多模态预训练, 进一步增强了视觉Encoder 对细粒度语义的理解能力. 

**SigLIP 视觉Encoder **
SigLIP 使用了一种改进的对比学习方法来训练视觉-语言对齐模型, 它在训练稳定性和小规模数据上的表现优于 CLIP. 许多最新的 VLMs(如 PaLI-3、Qwen2-VL)都采用了 SigLIP 作为其视觉Encoder . 

### 3.2 视觉语言模型的工作模式

视觉语言模型(VLM)通常支持多种任务模式: 

**图像描述(Image Captioning)** 
给定一张图片, 生成描述其内容的自然语言文本. 这是最基础的 VLM 任务, 也是评估模型视觉理解能力的基本 benchmark. 

**视觉问答(Visual Question Answering, VQA)** 
给定一张图片和一个关于图片内容的问题, 模型需要理解图像内容并回答该问题. VQA 要求模型具备细粒度的视觉定位和推理能力——不仅要"看到"图片, 还要理解问题所指向的特定区域或关系. 

**光学字符识别(OCR)与文档理解**
让模型读取图片中的文字. 这对于处理扫描文档、截图、海报、表单等场景至关重要. 现代 VLMs(如 GPT-4V、Qwen-VL、InternVL)的 OCR 能力已经达到了相当高的水平, 能够处理多栏排版、表格、公式等复杂文档结构. 

**视觉定位(Visual Grounding)** 
给定一个文本描述, 模型需要在图像中定位出对应的区域(通常以边界框或掩码的形式输出). 这是连接"语言理解"和"空间感知"的关键能力, 也是很多高级应用(如机器人指令跟随、图像编辑)的基础. 

**多图像/视频理解**
高级 VLMs 能够同时处理多张图像或视频帧, 理解它们之间的关系(如时间顺序、空间布局、因果关联). 这对于视频理解、视觉叙事、跨帧推理等任务至关重要. 

### 3.3 视觉生成

视觉理解是"从图像到文本", 而视觉生成则是"从文本到图像"(以及更复杂的条件生成). 

**扩散模型(Diffusion Models)** 
当前视觉生成的主流范式. 扩散模型通过逐步向图像添加噪声(前向过程), 然后训练一个神经网络来逆转这个过程(反向过程), 从而从纯噪声中生成清晰的图像. Stable Diffusion、DALL-E 3、Midjourney 等著名文生图模型都基于扩散模型. 核心改进方向包括: 
- **Latent Diffusion**: 在压缩的潜在空间(而非原始像素空间)中进行扩散, 大幅降低计算成本. 

- **条件控制**: 通过文本Encoder (如 CLIP)、ControlNet、IP-Adapter 等机制, 实现对生成内容的精细控制. 

- **效率优化**: Distillation(如 SDXL Turbo、LCM)将多步扩散压缩为单步或几步, 实现实时生成. 

**自回归视觉生成**
以 Google 的 Parti、OpenAI 的 DALL-E 为代表, 将图像生成建模为序列生成问题——将图像 token 化后, 使用自回归模型逐个预测 token. 这种方法与 LLM 的技术栈更加统一, 但在生成质量和效率上目前仍略逊于扩散模型.

## 4. 音频与视频

### 4.1 音频理解

音频模态包含丰富的信息: 语音的内容(说了什么)、说话人的身份(谁在说话)、情感状态(怎么说)、以及环境声音(背景是什么). 

**语音识别(ASR)** 
将语音转换为文本. Whisper(OpenAI)是当前最广泛使用的开源语音识别模型, 它通过大规模弱监督学习, 在多种语言和任务上取得了优异的表现. 更近期的模型(如 Qwen-Audio、SpeechGPT)进一步将语音识别与语言模型结合, 实现了端到端的"语音输入 → 语义理解 → 文本/语音输出". 

**音频事件检测与分类**
识别音频中的非语音事件(如音乐、掌声、汽车引擎声、婴儿哭声). 这对于视频内容理解、智能家居、安防监控等场景非常重要. 

**音乐理解与生成**
让模型理解音乐的结构、风格、情感, 甚至生成新的音乐片段. 这个方向正在快速发展, 但相比图像和文本, 音乐的理解和生成仍然面临独特的挑战(如时序结构的复杂性、多轨道的协调性). 

### 4.2 视频理解

视频是图像和音频的时序组合, 它引入了新的维度——**时间**. 

**时序建模的挑战**
视频中的信息分布在时间轴上, 模型需要理解: 
- **短期时序**: 动作的动态变化(如一个人"举起手"涉及多帧的连续变化). 

- **长期时序**: 事件的发展脉络(如一部电影的情节推进). 

- **时序关系**: 事件的先后顺序、因果关系、并发关系. 

**视频理解的技术路线**
- **基于帧采样的方法**: 将视频均匀采样为若干关键帧, 将每帧独立输入图像Encoder , 再将所有帧的特征拼接或聚合后送入语言模型. 这种方法简单高效, 但可能丢失细粒度的时序信息. 

- **基于视频Encoder 的方法**: 使用专门设计的视频Encoder (如 Video Swin Transformer、TimeSformer)直接对视频时空立方体进行建模, 捕捉帧间关系. 

- **原生多模态方法**: 将视频也 token 化为与文本统一的格式, 让 Transformer 自动学习时空关系. 这是目前最前沿的方向, 但训练成本极高. 

**视频生成**
Sora(OpenAI)在 2024 年初的发布, 标志着视频生成技术进入了一个新纪元. Sora 能够根据文本描述生成高质量、长时长、物理一致性较强的视频片段. 其技术细节虽然没有完全公开, 但据推测采用了基于 Diffusion Transformer(DiT)的架构, 将视频表示为时空 patches, 并在巨大的视频数据上进行了训练. 视频生成领域正在快速追赶图像生成的步伐, 核心挑战包括: 长视频的一致性保持、物理规律的准确模拟、以及生成效率的提升.

## 5. 多模态对齐技术

无论采用何种架构, 多模态模型的核心问题始终是一致的: **如何让来自不同模态的信息在同一个语义空间中对齐？** 对齐技术的质量直接决定了多模态模型的理解能力和泛化能力. 

### 5.1 对比学习对齐

对比学习(Contrastive Learning)是当前最主流的多模态对齐方法. 

**CLIP(Contrastive Language-Image Pre-training)** 
CLIP 是 OpenAI 在 2021 年发布的里程碑式工作, 它奠定了视觉-语言对齐的基本范式. CLIP 的训练目标非常简单: 给定一批(图像, 文本)配对, 让匹配的配对在表示空间中距离尽可能近, 不匹配的配对距离尽可能远. 具体来说: 
- 使用一个图像Encoder (ViT 或 ResNet)将图像编码为向量 $I$. 
- 使用一个文本Encoder (Transformer)将文本编码为向量 $T$. 
- 计算所有图像-文本配对之间的余弦相似度, 构成一个相似度矩阵. 
- 训练目标是在这个矩阵上同时优化图像→文本的分类损失和文本→图像的分类损失(对称交叉熵). 

CLIP 的惊人之处在于它的"涌现能力"——通过在大规模的互联网图文配对数据上进行这种简单的对比学习, 模型学会了将视觉概念与语言概念精确对齐, 从而可以零样本(Zero-Shot)地完成图像分类、检索等任务. 

**SigLIP**
SigLIP 对 CLIP 的训练目标进行了改进, 使用 Sigmoid 损失替代了 Softmax 损失. CLIP 的 Softmax 损失需要在整个 batch 上计算归一化, 对 batch size 非常敏感; SigLIP 的 Sigmoid 损失将每个(图像, 文本)配对视为独立的二元分类问题, 训练更加稳定, 在小 batch 上的表现也更好. 

### 5.2 融合嵌入与联合训练

对比学习实现了"粗粒度"的对齐——知道"这张图片和这段文字是匹配的". 但对于更复杂的任务(如 VQA、视觉定位), 需要"细粒度"的对齐——知道"图片中的这个区域对应文字中的这个词". 

**融合注意力机制**
在 Encoder-Projector-Decoder 架构中, 投影层(如 Q-Former)通常使用交叉注意力机制来实现细粒度对齐. Q-Former 使用一组可学习的查询 token(Query Tokens)与图像特征进行交叉注意力, 提取出与当前文本任务最相关的视觉信息. 这些查询 token 起到了"信息瓶颈"的作用, 迫使模型选择最重要的视觉特征. 

**指令微调对齐**
预训练阶段的对齐只能建立基础的跨模态关联. 要让模型能够遵循人类的复杂指令(如"描述这张图片左上角的物体"、"比较这两张图片的异同"), 还需要在构建好的多模态指令数据集上进行监督微调(SFT). 这些指令数据通常包含: (图像/视频, 指令, 回答)三元组, 其中回答可以是自由文本、边界框坐标、或者其他结构化输出. 

### 5.3 原生多模态预训练

最前沿的方向是**从头开始进行原生多模态预训练**, 而不是"先分别训练各模态Encoder , 再对齐". 这种范式的核心思想是: 将所有模态的数据都转化为统一的 token 序列, 然后使用标准的 Transformer 架构进行预训练. 模型在预训练过程中自然而然地学会了跨模态的表示和对齐. 

**数据准备**
原生多模态预训练需要海量、高质量、多模态交织的数据. 数据来源包括: 
- 带描述的图像和视频(如 LAION、InternVid)
- 带字幕的音频和视频(如 AudioSet、WavCaps)
- 交错的图文文档(如网页、PDF、电子书)
- 人工标注的多模态对话数据

**训练目标**
通常采用多任务混合训练: 在文本数据上使用 Next Token Prediction, 在图文数据上使用对比学习或 Captioning Loss, 在交错数据上使用统一的序列建模目标. 通过精心设计的采样策略(不同数据源的混合比例), 让模型在保持语言能力的同时, 逐步发展出多模态理解能力. 

**代表性模型**
GPT-4o、Gemini 系列被推测采用了类似的原生多模态架构, 它们展现出了真正"无缝"的多模态交互能力——能够同时处理文本、图像、音频输入, 并以任意模态组合输出结果. 这是多模态大模型的终极形态, 也是当前研究的最前沿.

> * **多模态输入层(四周放射流)**：不同的模态数据进入统一处理流程. **文本**经过 Tokenizer 转化为词嵌入向量; **图像**经由 ViT 分块后扁平化为图像 Token; **音频**信号被转换为梅尔频谱图(Mel-spectrogram)后编码为音频 Token; **视频**则通过时空 patch 采样法提取时序特征向量. 所有模态流均通过投影层或嵌入对齐技术转换为统一维度, 注入到中心的 Transformer. 
> * **核心处理层(中心 Transformer)**：基于自注意力机制(Self-Attention)对多模态序列进行深层交互推理, 使得异构模态之间能够在相同的语义特征空间下进行协同计算. 
> * **多模态输出层(底部输出)**：根据具体指令任务, 模型能够自回归地输出多种生成形态, 如**文本描述**(答复、翻译)、**生成图像**(文生图)、**合成语音**(文本转语音 TTS)及**生成视频**(文生视频等时空生成).
