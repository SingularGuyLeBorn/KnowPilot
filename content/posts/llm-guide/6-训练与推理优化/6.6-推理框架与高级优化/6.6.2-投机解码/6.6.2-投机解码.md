# 6.6.2 投机解码

## 1. 演进脉络与技术演进

自回归生成的数学结构将 $T$ 个 token 的联合概率分解为 $T$ 个条件概率的乘积：

$$P(x_1, x_2, \ldots, x_T) = \prod_{t=1}^{T} P(x_t \mid x_1, x_2, \ldots, x_{t-1}) \tag{1} $$

该分解式暴露了 LLM 推理的核心矛盾：位置 $t$ 的采样必须等待位置 $t-1$ 的结果. 每一步 forward pass 的计算复杂度约为 $O(N + L^2 \cdot d)$, 其中 $N$ 为参数量, $L$ 为序列长度, $d$ 为隐藏维度. 生成 $T$ 个 token 需要执行 $T$ 次严格串行的 forward pass. 在实际 profiling 中, 解码阶段占据端到端推理时间的 70%-90%. 

针对该瓶颈, 工程界此前的主流优化均沿"加速单步"方向推进. 

| 技术路线 | 优化对象 | 典型加速比 | 根本局限 |
|---------|---------|-----------|---------|
| KV Cache | 消除历史 token 的重复计算 | 10-100x(相对无缓存) | forward pass 次数不变 |
| 量化(INT4/INT8) | 降低单步计算量与内存带宽 | 1.5-3x | 串行结构未打破 |
| PagedAttention | 显存分配效率与 batch 调度 | 吞吐量 2-5x | 单序列延迟未改善 |
| 连续批处理 | GPU 利用率与吞吐率 | 吞吐率 3-10x | 单个请求等待时间不变 |
| FlashAttention | Attention 计算的 IO-Aware 优化 | 2-4x | 仅优化 $L^2$ 项系数 |

上述技术将单步延迟压缩至物理极限附近, 但未触及"$T$ 次串行步数"这一根本约束. 投机解码(Speculative Decoding)将优化方向从"加速每一步"切换为"减少步数"——通过小模型快速猜测未来 $k$ 个 token, 再由大模型一次性并行验证. DeepMind 于 2022 年发表的《Fast Inference from Transformers via Speculative Decoding》首次建立了 Draft-then-Verify 的完整数学框架, 并严格证明了输出分布等价性. 此后, 研究界沿"降低草稿成本"轴线持续演进：传统双模型方案(独立 Draft Model) $\to$ Medusa 单模型多解码头 $\to$ Lookahead / Prompt Lookup 无模型方案 $\to$ 训练阶段内建 Multi-Token Prediction(MTP)目标. 

## 2. 工业价值与应用场景

投机解码已被所有主流生产级推理框架采纳为标准配置. 

| 推理框架 | 支持版本 | 实现特点 | 典型配置 |
|---------|---------|---------|---------|
| vLLM | v0.3.0+ | 与 PagedAttention 深度融合, 动态内存页分配 | 独立 Draft Model, 自适应 $k$ |
| TensorRT-LLM | 全版本 | NVIDIA Tensor Core 深度优化验证阶段 kernel | 树形注意力并行验证 |
| SGLang | 全版本 | 结构化生成与投机解码联合优化 | Prompt Lookup + Draft Model 混合 |
| TGI (HuggingFace) | 全版本 | 生产级稳定性, 支持多种草稿策略 | 可插拔 Draft Model |
| Medusa | 独立框架 | 无需 Draft Model, 仅附加解码头 | 2-4 个额外 head, 微调训练 |

投机解码解决的硬指标是**单序列每 token 延迟(TPOT)** . 在高接受率场景($\alpha \approx 0.8-0.9$)下, TPOT 可降低 50%-70%, 直接对应用户体验层面的"响应速度翻倍". 该优化与量化、PagedAttention 正交叠加：目标模型量化为 INT8 降低权重带宽, 草稿模型可进一步量化为 INT4 以压缩显存; PagedAttention 管理验证阶段临时分配的 KV Cache 页, 接受后保留、拒绝后释放. 

投机解码的深层工业价值在于开启了"多 token 预测"(Multi-Token Prediction)范式. 传统训练目标仅优化 $P(x_t \mid x_{<t})$, 而投机解码的工程实践表明：若模型能同时预测未来多个 token, 推理吞吐量将获得质的飞跃. 该洞见催生了 Medusa 的多解码头架构、原生 MTP 训练目标(Moonlight、DeepSeek 系列), 推动 LLM 从"单步预测"走向"多步规划". 

## 3. 物理直觉与核心机制

标准自回归解码等价于一位极度严谨的作家：每写一字, 必须将前文全部重读一遍, 深思后方可落笔. 第 $n$ 字不确定, 第 $n+1$ 字绝不可动笔. 生成 5000 字需 5000 次"重读+思考", 且每次重读长度递增. 

投机解码引入"实习生-主编"协作结构. 实习生(Draft Model)凭直觉快速写出 $k$ 字草稿; 主编(Target Model)一次性浏览整段, 对每个字独立判断"此字在此处是否恰当". 主编的阅读是并行的——$k$ 个判断在同一次 forward pass 内完成. 

验证遵循"顺序审阅、首次错误即停止"原则. 假设草稿序列为：

```
位置 1: "深" (正确)
位置 2: "度" (正确)
位置 3: "学" (错误, 应为"习")
位置 4: "习" (碰巧对了, 但基于"学"推导)
```

位置 4 的"习"虽本身正确, 但其条件概率为 $P(x_4 \mid \text{"深度学"})$, 而非 $P(x_4 \mid \text{"深度习"})$. 两个条件分布完全不同. 因此位置 3 一旦出错, 位置 4 及之后的所有 token 必须全部作废. 这不是工程简化, 而是概率链式法则的数学必然. 

投机解码最精妙的性质在于**输出分布严格无损**. 主编对每一个被接受的 token 执行了"质量把关"：一个 token 被接受, 当且仅当它在目标模型的分布下是合理采样结果. 若草稿 token 与目标模型采样结果一致, 直接接受; 若不一致, 主编从修正分布 $p'(x) \propto \max(0, p(x) - q(x))$ 重新采样替代. 整个序列的联合分布与标准自回归采样完全一致——用户感知不到任何质量差异, 但速度提升 2-3 倍. 

修正分布的物理意义是"目标模型相对于草稿模型的正剩余信息". $p(v) > q(v)$ 意味着目标模型认为 $v$ 比草稿模型估计的更合理; $p(v) \leq q(v)$ 说明草稿模型未低估 $v$. 修正分布仅保留被低估的选项, 按低估程度 $(p(v) - q(v))$ 重新分配概率质量. 

## 4. 数学推导与公式对比

设目标模型分布为 $p(x \mid x_{<t})$, 草稿模型分布为 $q(x \mid x_{<t})$. 单次迭代包含三步. 

**Draft**：草稿模型自回归生成 $k$ 个候选 token：

$$\hat{x}_{t+i} \sim q(\cdot \mid x_{<t}, \hat{x}_t, \ldots, \hat{x}_{t+i-1}) \quad \text{for } i = 0, \ldots, k-1 \tag{2} $$
**Verify**：前缀与候选拼接后输入目标模型, 一次 forward pass 输出 $k$ 个位置的真实分布 $p_i(x)$. 

**Accept/Reject**：位置 $i$ 的接受率为

$$\alpha_i = \min\left(1, \frac{p_i(\hat{x}_{t+i})}{q_i(\hat{x}_{t+i})}\right) \tag{3} $$

验证从 $i=0$ 顺序执行. 首次拒绝发生在位置 $j$ 时, 从修正分布采样：

$$x_{t+j} \sim p'_j(x) = \frac{\max(0, p_j(x) - q_j(x))}{\sum_{x'} \max(0, p_j(x') - q_j(x'))} \tag{4} $$
**分布等价性**：对于任意输出 token $v$, 投机解码单步输出概率 $P_{\text{spec}}(v)$ 严格等于 $p(v)$. 证明分两项求和——"草稿即 $v$ 且被接受"与"拒绝后修正采样到 $v$"——两种情况($p(v) \geq q(v)$ 与 $p(v) < q(v)$)均收敛于 $p(v)$. 

**期望加速比**：设单步接受率为 $\alpha$, 草稿长度为 $k$, 期望接受数为

$$\mathbb{E}[A] = \sum_{i=1}^{k} \alpha^i = \alpha \cdot \frac{1 - \alpha^k}{1 - \alpha} \tag{5} $$

当 $\alpha = 0.7, k = 5$ 时, $\mathbb{E}[A] \approx 1.94$, 即平均每次验证产出约 2 个 token. 加速比理论上界由目标模型与草稿模型的计算量比决定：

$$\text{Speedup}_{\text{ideal}} \approx \frac{C_{\text{target}}}{C_{\text{draft}}} \tag{6} $$
完整数学证明、修正分布深入分析、最优草稿长度 $k^*$ 推导、Medusa 多解码头与树形注意力形式化, 参阅 [01-投机解码原理与应用](./01-投机解码原理与应用.md).

投机解码存在一个由接受率 $\alpha$ 与草稿长度 $k$ 共同决定的"死亡区域". 下表假设草稿模型速度为目标模型的 10 倍, 验证开销为 $0.5 + 0.1k$ 单位.

| 0.90 | 3 | 2.439 | 0.451 | **2.22x** |

## 6. 极简实现与源码映射

核心逻辑仅涉及三个状态机转换：Draft $\to$ Verify $\to$ Accept/Reject. 单次迭代的 PyTorch 骨架如下：

```python
def speculative_step(prefix, draft_model, target_model, k):
    # Draft: 草稿模型自回归生成 k 个候选
    draft_ids = prefix.clone()
    draft_tokens, draft_probs = [], []
    for _ in range(k):
        logits = draft_model(draft_ids).logits[:, -1, :]
        probs = F.softmax(logits, dim=-1)
        tok = torch.multinomial(probs, 1)
        draft_tokens.append(tok)
        draft_probs.append(probs.gather(-1, tok))
        draft_ids = torch.cat([draft_ids, tok], dim=1)

    # Verify: 目标模型一次性验证
    target_logits = target_model(draft_ids).logits
    L = prefix.shape[1]
    draft_logits = target_logits[:, L-1:L-1+k, :]
    target_probs = F.softmax(draft_logits, dim=-1)

    # Accept/Reject: 顺序验证, 首次拒绝即停止
    accepted = []
    for i in range(k):
        p_i = target_probs[0, i, draft_tokens[i]]
        q_i = draft_probs[i]
        alpha = min(1.0, (p_i / q_i).item())
        if random.random() < alpha:
            accepted.append(draft_tokens[i])
        else:
            # 修正分布采样替代 token
            # 完整实现含退化处理与全接受后额外采样
            break
    return accepted
```

`alpha = min(1, p_i / q_i)` 直接映射接受率公式; `clamp(p - q, min=0)` 映射修正分布. 全接受后的额外采样、Medusa 多解码头、树形注意力 CUDA Kernel 级实现, 参阅 [01-投机解码原理与应用](./01-投机解码原理与应用.md). 

## 7. 边界条件与失效模式

**分布兼容性陷阱**. 接受率与两个分布的总变差距离直接相关：$\mathbb{E}[\alpha] \approx 1 - D_{TV}(p, q)$. 草稿模型必须与目标模型属于同一模型族、在相似数据上训练. 跨域部署(如代码草稿模型为文学目标模型服务)会导致接受率骤降至 0.5 以下, 进入死亡区域. 不同任务对分布差异的敏感度差异显著：开放式创意写作接受率通常 0.5-0.6; 代码生成若草稿模型亦经代码训练, 接受率可达 0.8-0.9. 同一序列内的接受率随内容主题动态波动. 

**短序列 overhead 陷阱**. 投机解码存在固定开销：草稿模型 forward、验证 forward、控制流调度. 对于仅生成 5-10 个 token 的短序列(分类、简单问答), 固定开销超过收益. 工程经验法则：预期生成长度低于 20-30 个 token 时, 应回退至标准自回归解码. vLLM 等先进系统支持运行时自适应开关. 

**确定性任务的悬崖效应**. 低熵决策点(代码语法边界、数学公式括号、结构化数据字段)处, 目标模型分布高度集中(如 $P(:) = 0.99$). 草稿模型若稍有偏差(如 $q(:) = 0.3$), 接受率 $p/q$ 极小, 几乎必然触发拒绝. 修正采样仍极大概率采回目标分布的众数, 意味着浪费了一轮 Draft-Verify 循环只为确认一个本可直接生成的 token. 代码生成、JSON 结构化输出等场景频繁遭遇此问题. 

**显存开销：双模型的物理代价**. 传统方案需同时驻留目标模型与草稿模型. 70B FP16 目标模型约 140GB, 7B FP16 草稿模型额外 14GB, 加上 KV Cache 与激活值, 总显存需求可能超过 200GB. 缓解路径：草稿模型量化为 INT4/INT8; CPU offloading 草稿模型; 采用 Medusa 替代双模型方案(额外参数量 $<1\%$). 

**Batch Size $> 1$ 的复杂性爆炸**. 同一 batch 内不同序列接受长度不同(A 接受 4 个, B 接受 1 个, C 全拒), 导致序列长度错位, 需 padding 或 truncation 重新对齐. 树形注意力在 batch 场景下难以统一候选树形状. KV Cache 增长速度不一致与连续批处理交织, 内存管理复杂度急剧上升. 主流框架采用固定草稿长度、early stopping、独立调度器三种手段缓解, 但 batch 场景下的投机解码开销仍远高于单序列. 

### 工业前沿：EAGLE 系列——特征级草稿模型

传统投机解码依赖独立的草稿模型 $M_q$, 其瓶颈在于分布对齐困难：小模型的输出分布 $q$ 与大模型的 $p$ 存在系统性差距, 导致接受率受限. EAGLE 系列(Li et al., 2023-2024)提出了一条根本不同的路线：**不训练独立草稿模型, 而是利用目标模型自身的内部特征生成草稿**. 

**EAGLE-1 的核心洞察**：大模型最后一层 Transformer 输出的隐状态 $\mathbf{f}_t$ 已蕴含丰富的上下文语义信息. 直接用 $\mathbf{f}_t$ 自回归预测下一个隐状态 $\mathbf{f}_{t+1}$, 再通过原始 LLM 的 LM Head 生成 token, 其效果优于独立小模型(加速比 1.9 vs 1.5). 由于 token 采样是随机的, 不同采样结果会导致 $\mathbf{f}_{t+1}$ 分化, 因此草稿模型的输入为 $(\mathbf{f}_t, \mathbf{e}_{t+1})$, 其中 $\mathbf{e}_{t+1}$ 是已采样 token 的嵌入向量：

$$
\mathbf{f}_{t+1} = \text{DraftLM}(\mathbf{f}_t, \mathbf{e}_{t+1}), \quad \hat{x}_{t+1} = \text{LM\_Head}(\mathbf{f}_{t+1})
\tag{7} $$
该关系式明确了系统的数学约束, 可直接用于后续优化推导. 
草稿模型 DraftLM 是一个极轻量的 Transformer(通常 4-7 层), 在目标模型的特征空间上训练. 训练数据来自目标模型对真实文本的前向传播：收集每一层的 $(\mathbf{f}_t, x_{t+1}, \mathbf{f}_{t+1})$ 三元组, 最小化 $\|\mathbf{f}_{t+1} - \text{DraftLM}(\mathbf{f}_t, \mathbf{e}_{t+1})\|^2$. 由于 DraftLM 与目标模型共享 LM Head, 草稿 token 的分布天然更接近目标分布, 接受率显著提升. 

**EAGLE-2：树形草稿采样**. EAGLE-1 的草稿序列是线性链, 每个位置只预测一个 token. EAGLE-2 引入**动态草稿树**：在每个解码步骤, 草稿模型不仅预测最可能的下一个 token, 还生成多个候选分支, 形成树形结构. 目标模型通过**树形注意力**(Tree Attention)一次性并行验证整棵树的全部路径, 而非逐 token 验证. 

树形验证的关键优化是路径前缀共享：树中不同分支共享相同的祖先节点, 验证时这些共享前缀只需计算一次 attention. 实验表明, 在代码生成任务上, EAGLE-2 的接受率较 EAGLE-1 提升 15-20%, 端到端延迟降低 1.3-1.5 倍. 

**EAGLE-3：上下文自适应草稿**. 进一步观察到不同上下文类型的最优草稿策略不同：代码语法边界处应保守(短草稿), 开放式文本处应激进(长草稿、多分支). EAGLE-3 在 DraftLM 中引入轻量的上下文分类器, 根据当前隐状态动态调整草稿树的宽度和深度. 

| 方法 | 草稿来源 | 验证方式 | 相对加速 | 额外显存 |
|:-----|:--------|:--------|:--------|:--------|
| 传统投机解码 | 独立小模型 | 逐 token | 1.5-2.0× | 小模型完整权重 |
| Medusa | 多头线性投影 | 并行 | 1.8-2.5× | $<1\%$ 目标模型 |
| EAGLE-1 | 目标模型特征 | 逐 token | 1.9-2.2× | 4-7 层 DraftLM |
| EAGLE-2 | 目标模型特征 | 树形并行 | 2.5-3.5× | 4-7 层 DraftLM |
| EAGLE-3 | 自适应特征 | 自适应树 | 3.0-4.0× | 4-7 层 + 分类器 |

EAGLE 系列的工程启示是：**投机解码的瓶颈不是草稿模型的速度, 而是草稿与目标分布的对齐质量**. 利用目标模型自身特征空间生成草稿, 从根本上解决了分布偏移问题, 使投机解码从"外挂加速"进化为"内生能力". 

## 8. 技术前瞻与演进

投机解码当前属于**外挂式**优化：模型训练完成后, 通过额外机制加速推理. 研究界正沿三条路径将其内建化. 

**Multi-Token Prediction (MTP) 训练目标** 要求模型在训练阶段同时预测未来 $k$ 个 token. 损失函数从标准交叉熵扩展为：

$$\mathcal{L}_{\text{MTP}} = \mathcal{L}_{\text{next}} + \lambda \sum_{i=1}^{k} \mathcal{L}_i \tag{8} $$

隐藏状态被显式训练成包含多步未来信息, 推理时无需草稿模型或多解码头即可直接输出多个 token. Moonshot AI Moonlight、DeepSeek 系列已引入 MTP 目标. MTP 将"猜测能力"编码进模型参数, 从根本上消除了外挂机制的显存与调度开销. 

**自适应投机深度** 是当前系统的明显短板. 固定草稿长度 $k$ 无法适应生成内容的动态熵变：高确定性区域(代码语法边界)应关闭投机或取极小 $k$; 高熵区域(开放式写作)应取较大 $k$. 未来系统将在每个解码步骤根据当前上下文的分布熵 $H(p) = -\sum_x p(x) \log p(x)$ 动态调整 $k$. 熵低于阈值时令 $k=1$(标准解码), 熵高于阈值时令 $k=K_{\max}$(全速投机). 

**与 Test-Time Compute 的融合** 将放大投机解码的战略价值. Chain-of-Thought、Self-Consistency、MCTS 等推理时计算扩展技术将单次推理的 token 数量推高一个数量级. CoT 推理天然具有高重复性("Let's think step by step"、中间结论的固定模板), 为 Lookahead / Prompt Lookup 提供了丰富的 n-gram 匹配机会. 在推理成本持续膨胀的背景下, 投机解码从"可选优化"演变为"必要基础设施". 

**端到端多 token 架构** 代表更激进的演进方向. 层次化预测结构(先输出句子骨架, 再填充细节)、非自回归Decoder  (NAT)变体、以及基于扩散模型的序列生成, 均试图从根本上取消"一次一个 token"的架构假设. 投机解码及其变体是自回归范式内的最优解; 若未来架构彻底跳出自回归框架, 投机解码的历史使命将被取代. 但在可预见的周期内, 自回归仍主导生产环境, 投机解码的优化空间尚未耗尽.
