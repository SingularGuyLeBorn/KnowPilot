# DeepSeek-R1-Zero 原理剖析

>  **[返回 14.1-DeepSeek 家族总览](../../14.1-DeepSeek.md)**


> 本文聚焦 DeepSeek-R1-Zero 的核心机制: GRPO 算法、基于规则的奖励系统, 以及 RL 驱动的自我进化行为.

---

## 1 GRPO 的数理本质

### 1.1 从 PPO 到 GRPO: 价值模型的困境

Proximal Policy Optimization(PPO)是 LLM RL 阶段的事实标准算法. 其目标函数为:

$$
\mathcal{J}_{PPO}(\theta) = \mathbb{E}\left[ \min\left( \frac{\pi_\theta(a|s)}{\pi_{\theta_{old}}(a|s)} A^{GAE}, \text{clip}(\cdot) A^{GAE} \right) \right] - \beta \cdot \text{KL}(\pi_\theta \| \pi_{ref})
$$

其中 $A^{GAE}$ 是通过 Generalized Advantage Estimation 计算的, 依赖于一个独立训练的价值模型 $V_\phi(s)$:

$$
A^{GAE}_t = \sum_{l=0}^{\infty} (\gamma\lambda)^l \delta_{t+l}^{V}
$$

$$
\delta_t^V = r_t + \gamma V_\phi(s_{t+1}) - V_\phi(s_t)
$$

价值模型的训练目标是: 基于从序列开头到当前位置生成的 token, 预测从当前位置开始的预期累积奖励.

这个设计在长思维链(CoT)推理场景下遇到了根本性的困难. 当输出长度达到数万 token 时, 模型可能在生成过程中进行反思和修正 —— 最初生成的内容可能被后续完全否定. 在这种情况下, 基于部分响应预测最终奖励几乎是不可能的.

GRPO 的核心洞察是: 既然价值模型在长 CoT 上不可靠, 那就干脆不用它.

### 1.2 组内相对评分: 从绝对预测到相对比较

GRPO 的优势计算完全抛弃了价值模型, 改用同一问题下采样出的 $G$ 个输出之间的相对比较:

$$
A_i = \frac{r_i - \text{mean}(\{r_1, r_2, \cdots, r_G\})}{\text{std}(\{r_1, r_2, \cdots, r_G\})}
$$

这里 $r_i \in \{0, 1\}$ 是第 $i$ 个输出的规则奖励(正确或错误). 假设 $G=16$ 个输出中有 $k$ 个正确, 则正确输出的优势为正, 错误输出的优势为负.

这个设计的巧妙之处在于:

1. **不需要价值模型**: 省去了与策略模型同等规模的网络, 显存和计算开销减半.
2. **自动适应难度**: 对于简单问题(大部分输出都正确), 优势值的绝对值较小, 梯度更新温和; 对于困难问题(只有少数输出正确), 优势值的绝对值较大, 梯度更新强烈.
3. **与奖励信号完美匹配**: 由于奖励是二元的(0 或 1), 组内标准化天然地提供了有意义的相对信号.

### 1.3 裁剪比例的特殊处理

GRPO 的 clip ratio $\epsilon$ 在 R1-Zero 中被设为 10, 远高于 PPO 通常的 0.1-0.2.

$$
\text{clip}\left(\frac{\pi_\theta(o_i|q)}{\pi_{\theta_{old}}(o_i|q)}, 1-\epsilon, 1+\epsilon\right)
$$

当 $\epsilon=10$ 时, clip 区间变为 $[-9, 11]$. 这意味着概率比几乎不会被裁剪 —— 策略可以进行大幅度的更新.

这与传统 RL 的保守哲学形成鲜明对比. 通常, clip 机制是为了防止策略更新过大导致不稳定. 但在 R1-Zero 的场景中, 作者发现**不裁剪反而更好**:

- 低 clip ratio 会截断大量 token 的梯度, 降低模型性能
- 高 clip ratio 允许策略在发现有效推理模式时进行大幅度调整, 加速收敛

这实际上是一种「探索优先」的策略: 宁可训练不稳定, 也不要限制模型的探索空间.

> 译者注: $\epsilon=10$ 这个值初看起来很荒谬 —— 它几乎取消了裁剪机制. 但实验结果表明这是必要的. 原因可能是: 在长 CoT 场景中, 策略需要大幅度改变 token 分布来学习新的推理行为(如「wait」后的重新评估). 如果 clip ratio 太小, 这些行为改变的梯度会被截断, 模型就「学不会」反思. 这揭示了长 CoT RL 与传统短输出 RL 的根本差异: 前者需要更大的策略更新空间.

### 1.4 KL 散度的处理方式

GRPO 使用逆向 KL 散度的无偏估计:

$$
\mathbb{D}_{KL}(\pi_\theta \| \pi_{ref}) = \frac{\pi_{ref}(o_i|q)}{\pi_\theta(o_i|q)} - \log\frac{\pi_{ref}(o_i|q)}{\pi_\theta(o_i|q)} - 1
$$

与 PPO 的逐 token KL 惩罚不同, GRPO 将整个 KL 项作为损失的一部分. PPO 的逐 token 惩罚会隐式惩罚响应长度, 因为更长的响应累积了更多的 KL 惩罚. 而 GRPO 的方式不会惩罚长度, 允许模型自由地扩展思维链.

此外, 作者每 400 步将参考模型更新为最新策略, 而不是固定使用初始策略. 这允许策略在更大的空间内探索, 同时通过定期重置参考点来维持训练的稳定性.

## 2 基于规则的奖励系统

### 2.1 为什么不用神经奖励模型

DeepSeek-R1-Zero 明确拒绝了神经奖励模型(无论是结果型还是过程型), 原因有三:

1. **奖励黑客**: 在大规模 RL 中, 神经奖励模型容易被策略模型找到漏洞. 例如, 奖励模型可能对特定格式或关键词给予高分, 策略模型就会学会生成这些格式而不关心内容正确性.
2. **重训练成本**: 神经奖励模型需要随着策略模型的进化而持续重训练, 否则其判断会迅速过时. 这增加了额外的计算复杂性和训练流水线的耦合度.
3. **可靠性**: 对于数学和代码问题, 答案的正确性是二元的. 基于规则的验证器提供了完美的信号, 没有模糊地带.

### 2.2 准确性奖励的设计

**数学问题**: 模型需要在指定格式内(如 `\boxed{}`)提供最终答案. 答案可以是数字、表达式或方程. 使用 sympy 进行解析和表达式比较, 确保等价形式的答案都被接受.

**代码问题**: 使用编译器在预定义的隐藏测试用例上运行模型生成的代码. 通过所有测试用例得 1 分, 否则得 0 分.

**逻辑问题**: 多选题直接匹配选项, 定量推理问题匹配数值答案.

这种设计的覆盖范围是有限的: 它只能处理「可自动验证」的任务. 创意写作、开放式对话、情感分析等任务无法用这种奖励系统训练.

### 2.3 格式奖励的设计

格式奖励强制模型将推理过程放在 `<think>` 标签内, 最终答案放在 `<answer>` 标签内.

$$
\text{Reward}_{\text{format}} = \mathbb{1}[\text{格式正确}]
$$

这个设计有两个目的:

1. **可解释性**: 明确区分推理过程和最终答案, 便于分析和调试.
2. **可控性**: 在推理阶段, 可以独立控制思维链的长度和最终答案的生成.

值得注意的是, 格式奖励不约束推理过程的内容. 模型可以自由地探索任何推理策略, 只要最终答案正确且格式符合要求.

## 3 自我进化的动力学

### 3.1 响应长度的增长曲线

R1-Zero 在训练过程中展现出响应长度的稳定增长. 在 8200 步之前, 最大输出长度限制为 32768 token; 之后放宽至 65536 token.

响应长度的增长不是外部强制的, 而是 RL 优化的自然结果. 当模型发现「更多的思考时间 → 更高的正确率」时, 生成更长响应的策略就被正向选择了.

这与人类的学习过程惊人地相似: 面对困难问题时, 我们倾向于花更多时间思考. R1-Zero 通过 RL 自主地「发现」了这一策略.

### 3.2 反思行为的涌现

通过分析训练过程中代表性反思词汇的出现频率, 作者发现:

- 「wait」、「mistake」、「however」、「retry」、「verify」等词汇的频率从训练初期到末期增长了 5-7 倍
- 特别是「wait」一词, 在训练早期几乎不出现, 在 4000-7000 步偶尔出现, 在 8000 步之后出现显著的峰值

这表明反思行为不是一次性学会的, 而是分阶段涌现的:

1. **早期(0-4000 步)**: 模型学会基本的逐步推理, 但很少质疑自己的步骤.
2. **中期(4000-8000 步)**: 模型偶尔开始重新检查, 但频率不高.
3. **后期(8000 步后)**: 模型系统地使用反思策略, 「wait」成为常见词汇.

这种分阶段涌现与性能曲线的跃升相吻合: AIME 2024 的 pass@1 在 8200 步附近从约 50% 跃升至 77.9%.

> 译者注: 「顿悟时刻」的发现是这篇论文最引人入胜的部分. 它表明, 复杂的元认知行为(自我反思)可以从简单的奖励信号中自发涌现, 无需任何显式的程序设计. 这引发了一个深层的科学问题: LLM 的推理能力上限在哪里? 如果仅通过 RL 就能让模型学会反思, 那么通过更复杂的奖励结构(如过程奖励、好奇心驱动奖励), 模型是否还能学会更高级的认知策略(如假设检验、实验设计)? R1-Zero 的成功为这一问题提供了初步的肯定答案.

### 3.3 语言混合问题

由于 DeepSeek-V3-Base 主要在中文和英文数据上预训练, R1-Zero 在推理过程中会混合使用两种语言. 即使在英文查询下, 思维链中也可能出现中文词汇.

这揭示了预训练数据分布对 RL 行为的深远影响: RL 只是激活了预训练模型中已经存在的知识, 而这些知识的组织方式(包括语言混合)是由预训练数据决定的.

语言混合问题的解决方案(在 R1 中实现)是引入语言一致性奖励, 但这需要在推理能力和可读性之间做权衡.

## 4 R1-Zero 的局限性

### 4.1 可读性差

R1-Zero 的输出虽然推理正确, 但经常缺乏清晰的结构. 段落冗长、缺乏适当的换行和格式化, 使得人类阅读困难.

这是因为 RL 奖励完全不关心可读性 —— 只要答案正确, 任何格式的推理过程都获得相同的奖励.

### 4.2 通用任务性能弱

R1-Zero 的训练 narrowly focused on 可验证的推理任务. 对于写作、开放域 QA、创意生成等非推理任务, 模型性能有限.

这揭示了纯 RL 方法的根本局限: 没有可靠验证器的任务, 无法通过纯 RL 有效训练.

这也催生了后面的OPD系列,OPD很大一部分算法就是在没有显式奖励的环境下做模型训练的

### 4.3 训练不稳定性

虽然论文没有详细讨论, 但纯 RL 训练在大规模模型上存在固有的不稳定性. 奖励信号的稀疏性(大部分输出得 0 分)、策略更新的剧烈变化、以及长 CoT 的梯度传播困难, 都是潜在的挑战.

DeepSeek 通过以下工程手段来缓解:

- 定期更新参考模型(每 400 步)
- 使用较大的 clip ratio($\epsilon=10$) 避免梯度截断
- 使用 64×8 H800 GPU 的庞大计算资源来平滑训练动态

## 5 与 OpenAI o1 的方法论对比

| 维度     | DeepSeek-R1-Zero           | OpenAI o1(推测)               |
| -------- | -------------------------- | ----------------------------- |
| 训练信号 | 基于规则的奖励(可验证任务) | 未知(可能包含 PRM + 人类反馈) |
| SFT 阶段 | 完全跳过                   | 可能包含                      |
| 算法     | GRPO(无价值模型)           | 未知(可能为 PPO 变体)         |
| 开源     | 完整开源(权重 + 训练方法)  | 闭源                          |
| 推理行为 | 自发涌现反思、验证         | 被观察到类似行为              |
| 基础设施 | 基于 vLLM + DualPipe       | 未知                          |

从谱系上看, R1-Zero 代表了一条极端路线: 最小化人类干预, 最大化 RL 的自举能力. 它与 o1 的对比不仅仅是性能的对比, 更是两种哲学 —— 「人类引导 vs 机器自主」 —— 的实验.
