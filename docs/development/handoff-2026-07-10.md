## 3. 已确认的设计决策

来源：`docs/development/design-decisions.md`（已合并原 `swarm/` 三轮讨论）。

### 3.1 阻塞/非阻塞 `spawn_subagent`

- **阻塞式 `waitForResult=true`**：
  - 子 Agent 同步运行。
  - 结果直接作为 `spawn_subagent` 工具调用的结果返回给父 Agent LLM。
  - **不进**父会话异步任务结果队列。
  - 但这次调用作为运行记录进入左侧 Async 面板，标识为 `SubAgent`。
- **非阻塞式 `waitForResult=false`**：
  - 子 Agent 后台运行。
  - 子 Agent 完成后必须显式调用 `agent_report_back` 把结果发到父 Agent 的异步任务结果队列。
  - 父会话消费后，以**右侧 user 气泡**形式展示，带 `SubAgent` 标识。

### 3.2 队列组织

- 物理上三个队列：
  1. 用户发送队列（`userQueue`）
  2. 上级 Agent 发送队列（`superiorQueue`，后端 `AgentMessage` 表）
  3. 异步任务结果队列（`asyncResultQueue`）
- UI 上用户队列和上级队列合并展示，默认按时间排序，可拖拽调整。
- 异步任务结果队列优先级最高：只要存在未消费结果，优先消费它。

### 3.3 持久化

- `AgentMessage` 必须持久化（跨会话、跨前端生命周期）。
- `userQueue` 是否持久化待确认，倾向于新建 `SessionQueueItem` 表统一持久化用户和上级队列项。
- 异步任务结果 delivery 记录已有持久化。

### 3.4 推送 vs 轮询

- Phase 1 把 `pullAsyncQueue` 和 `pullAgentMessages` 改为“推优先、拉兜底”。
- 父/子会话有活跃 SSE 流时直接推送事件；无活跃流时 fallback 到拉取。
- 其他轮询（`session.listRunning`、`session.listChildren`、异步任务运行状态）Phase 1 维持轮询。

### 3.5 Pinned 延后消费

- 异步任务结果可以被用户 pinned。
- pinned 的项排在最后，取消 pin 后才被消费。
- Phase 1 实现。

## 6. 未来功能（需要在文档中描述）

### 6.1 Agent 自动开启新 Session

- Agent 判断当前会话交互轮数过多。
- Agent 自己写一个总结文档（比如 `content/sessions/[old-session-id]-summary.md`）。
- Agent 调用一个工具：
  - 结束当前 session（标记为 `archived` 或 `paused`）。
  - 启动同一个 Agent 的新 session。
  - 新 session 的第一条 user 消息设为总结内容。
- 如果用户当前正在看这个旧 session 页面，**不要自动切换**到新 session。
- 只在当前页面显示一条提示：「新 session 已创建：xxx」，用户手动点击跳转。

- 当前未实现。
