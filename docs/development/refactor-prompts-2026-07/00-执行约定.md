# 00 执行约定（所有工单共享，先读我）

> 本目录是 KnowPilot 重构工单提示词套件（2026-07-20）。每份 `NN-*.md` 是一个可直接粘贴给 AI 编码 agent 的完整工单。执行前先读本约定，再执行具体工单。

## 项目速览（给零上下文的实现者）

KnowPilot 是**单用户、本地优先**的智能知识管理 + AI Agent 平台。TypeScript pnpm monorepo：

- `apps/server/`：Express 5 + tRPC 11 + Prisma(SQLite)。所有实体 Service 收拢在 `src/services.ts`（单文件约定）；所有 tRPC 路由收拢在 `src/router.ts`；Agent 运行时在 `src/infra/`（loop/ 统一 ReAct 内核、agentStream SSE、sessionStreamHub、asyncJobOrchestrator 全局任务池、heartbeatEngine、swarm*、approvalGate、tools/native/* 工具域）。
- `apps/web/`：Next.js 16 + React 19。Chat 三层 store：`lib/useSessionMessages.ts` / `useStreamLifecycle.ts` / `useSessionComposeState.ts` + `useChat*` 域 hooks；通用组件收拢 `components/shared.tsx`；数据 hooks 收拢 `lib/hooks.ts`。
- `packages/shared/`：前后端共享 Zod schema + 常量（`LLM_MODEL_IDS`、`TIER_DEFAULT_TOOLS` 等）。
- **核心原则：本地 Markdown（`content/`）是数据唯一事实源，SQLite 只是查询/缓存层。** `pnpm db:sync` 双向同步；实体 CRUD 会写回 `content/` 文件。
- 测试：Vitest（server 在 `apps/server/src/__tests__/`，web 在 `apps/web/{lib,components}/__tests__/`）。lint：server/shared 是 `tsc --noEmit`，web 是 eslint。

## 铁律（违反即返工）

1. **禁止打补丁**：不变量必须收进 store/reducer/条件写/转移点。禁止新增 `setTimeout`/`queueMicrotask` 赌时序、`await refetch` 赌落库、散布在编排层的 `phase===xxx` 守卫。自检：删掉你新加的编排层代码，bug 还会复现吗？会 = 你打的是补丁，重新设计。
2. **禁止向后兼容包袱**：单用户未发布项目。改接口就改完全仓调用方；禁止兼容重载、双轨分支、deprecated 参数「先留着」；数据形态变化写一次性迁移脚本（`apps/server/src/scripts/`），执行完即删。
3. **单文件逻辑收拢**：Service 逻辑进 `services.ts`，路由进 `router.ts`，web 数据 hooks 进 `lib/hooks.ts`，通用 UI 进 `components/shared.tsx`。禁止新建同职能散文件（infra/ 下的新叶子模块除外，须无循环依赖）。
4. **中文注释/文档/commit 信息**；代码标识符用英文。commit 前缀 `fix:`/`feat:`/`refactor:`/`test:`/`docs:`。

## 工作流（每个工单）

1. 从最新 `master` 切工单指定分支：`git checkout -b <branch>`。
2. 环境：`pnpm install && pnpm --filter @knowpilot/server exec prisma generate`（若用新 worktree：`git worktree add tmp/worktrees/<branch> master`）。
3. **每个 bug：先写负向断言测试（旧实现下红）→ 改实现 → 变绿**。新功能配 Vitest 单测。能变异验证的（改条件/删行应变红）做一遍自查。
4. 一个子任务一个 commit，勤提交。
5. 合并前必过：`pnpm lint` 全绿 + `pnpm --filter @knowpilot/server test` 全绿（碰 web 的加 `pnpm --filter @knowpilot/web test`）。
6. `git checkout master && git merge --no-ff <branch>`，删分支。
7. 收尾：更新 `AGENTS.md`「当前状态与近期变更」（一段式沿用现有格式）；设计权衡追加到 `docs/development/design-decisions.md` 文末；在 `architecture-audit-2026-07-20.md` 总览表对应行标注「已修复@<commit>」（功能工单无对应行则跳过）。

## 红线

- 不碰 `feat/chat-kimi-composer` 分支与其工作区改动（用户 WIP）。
- 不提交 `tmp/`、`*.db`、`dev.db` 相关产物；schema 变更用 `pnpm --filter @knowpilot/server exec prisma db push` + 一次性迁移脚本，不手写 migrate 目录（除非工单明说）。
- 不引入新依赖（除非工单明确允许并说明理由）。
- 不做工单范围外的「顺手优化」。发现新问题 → 追加到 design-decisions.md 待办，不在本工单修。
- 遇到与工单描述不符的现状（代码已漂移），以代码现状为准重新评估，并在 commit/总结中如实说明偏差。
